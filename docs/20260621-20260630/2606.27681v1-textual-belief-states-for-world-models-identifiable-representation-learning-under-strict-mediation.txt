# Textual Belief States for World Models: Identifiable Representation Learning Under Strict Mediation 

Xiang Gao, Kaiwen Dong, Yuguang Yao, Padmaja Jonnalagedda, Kamalika Das 

Intuit AI Research 

{xiang_gao, kamalika_das}@intuit.com 

Abstract 

World models in partially observed environments rely on latent representations that summarize interaction history, but in many modern LLM-based architectures predictive performance fails to reflect representation quality due to history bypass, rendering the latent state unidentifiable. Strict latent state mediation, requiring predictions to depend only on the latent state and action, is a classical principle that resolves this, but enforcing it in text-based settings is an open challenge: textual latent states are discrete and non-differentiable, precluding variational training, and expressive LLM decoders readily ignore the bottleneck. We show how to make strict mediation work in the text domain. We formalize why it is necessary, showing that strict mediation makes representation quality empirically testable while history-leaky architectures break this connection. We then introduce textual latent states, which are discrete, interpretable, and variable-length, and factorized GRPO (fGRPO), a tree-structured reinforcement learning method that enforces strict mediation during training. Experiments on TextWorld and ScienceWorld show preserved one-step prediction accuracy alongside up to 57% gains in repre-sentation quality and 98% improvements in rollout performance, increasing with task complexity and horizon. 

1 Introduction 

World models learn the dynamics of an environment so that an agent can predict, plan, and reason without further interaction [Sutton, 1991, Ha and Schmidhuber, 2018]. In partially observed settings, where observations reveal only fragments of the underlying state, a world model must maintain a latent representation that summarizes the interaction history. This latent state is the object on which rollouts operate, the branching point for counterfactual queries, and the compressed summary that replaces an ever-growing history. A large body of work has focused on building better dynamics models [Hafner et al., 2019a,b, 2023, Schrittwieser et al., 2020], but a dynamics model is only as good as the representation it operates on. The central question is not only how to model dynamics, but how to know whether the learned representation is adequate. This question is difficult to answer for an architectural reason. In many existing world model architectures, the latent state coexists with other information that can substitute for it. Sequence models condition on the full observation history and maintain no explicit latent state [Yang et al., 2024, Li et al., 2025]. Prompting-based approaches treat the LLM context as an implicit state without learning a compressed representation [Hao et al., 2023, Wang et al., 2024]. Architectures that introduce a latent state but allow the decoder to also access the history may ignore it entirely: the model can achieve strong predictive performance while the representation carries no useful information [Xiang et al., 2025, Ge et al., 2024]. We call this the identifiability problem : when information can bypass the latent state, predictive accuracy does not constrain representation quality. We propose strict latent state mediation to address this: once a latent state is generated, all subsequent transitions and predictions depend only on that state and the action. This mirrors the POMDP belief-state factorization [Kaelbling et al., 1998,

Murphy, 2000]; classical world models with continuous latent states enforce it by construction [Hafner et al., 2019a,b]. However, recent LLM-based approaches frequently relax this constraint through large contexts or hybrid conditioning, enabling information to bypass the latent state. While this design improves predictive flexibility, it breaks the connection between predictive performance and representation quality. A key consequence, which motivates our approach, is that strict mediation makes representation quality empirically testable . If a strictly mediated model predicts accurately, the latent state must contain all task-relevant information; if it predicts poorly, the representation is insufficient. We formalize this principle and derive three consequences that motivate our approach. Under strict mediation, optimal predictive accuracy implies that the latent state is a sufficient statistic of history (Proposition 1). Under leaky architectures, optimal accuracy is compatible with an uninformative latent state (Proposition 2). Strict mediation also structurally aligns training and rollout conditions, so that the model never conditions on information during training that is unavailable during rollout, enabling bounded error propagation when dynamics are contractive, whereas history-based and leaky rollouts suffer geometric error growth (Proposition 3). The central technical challenge is enforcing strict mediation in text-based world models, where observations and actions are natural language. Textual latent states align with the LLM backbone, enable inspection, and provide an adaptive information bottleneck [Hafner et al., 2019a,b, 2025]. However, they are discrete and non-differentiable, precluding variational training, and expressive decoders make ELBO objectives prone to posterior collapse [Bowman et al., 2016, He et al., 2019]. We address this by treating latent state generation as a stochastic policy and introduce factorized GRPO (fGRPO) , a tree-structured extension that enforces strict mediation during training. We evaluate on TextWorld [Côté et al., 2018] and ScienceWorld [Wang et al., 2022]. Our approach preserves one-step prediction accuracy (Proposition 1), yields higher state-level F1 under matched prediction (Proposition 2), and improves rollout stability, with gains growing from 5% at horizon 1 to 80% at horizon 9 (Proposition 3). 

2 Problem Setup: Text Based POMDPs 

We are given an offline dataset of trajectories (o0, a 0, o 1, a 1, . . . , o T ) from a partially observable environment, where observations ot and actions at are textual. The true state is unobserved, and a single observation is generally insufficient to identify it, requiring aggregation over time. Our goal is to learn a model that predicts future observations using a latent belief state summarizing history. 

2.1 Latent Belief States and Predictive Sufficiency 

The interaction history at time t is ht = ( o0, a 0, o 1, a 1, . . . , a t−1, o t). Because the environment is partially observed, accurate prediction typically requires multi-step information. A latent belief state 

ˆst ∼ pθ (· | ht) is a learned summary of ht, represented as a text string produced by a language model. Its structure is learned and need not match predefined variables. We require predictive sufficiency .

Definition 2.1 (Predictive Sufficiency) . A latent state ˆst is sufficient if 

p(ot+1 | ht, a t) = p(ot+1 | ˆst, a t) for all ht, a t, i.e., ot+1 ⊥ ht | ˆst, a t. (1) A sufficient latent state retains all information from the past relevant for predicting the immediate next observation. This is weaker than full POMDP belief-state sufficiency, which requires sufficiency for all future observations [Kaelbling et al., 1998]; multi-step sufficiency follows when the transition model is also accurate, as the state at each step is re-derived via the learned dynamics. Whether this 2holds depends on both the objective and architectural constraints. Multiple sufficient statistics may exist for a given prediction problem; we do not require minimality. Strict mediation guarantees that the learned state is sufficient when prediction is optimal (Proposition 1), while our reward design (Section 4.4) encourages compactness by penalizing redundant and unnecessary facts. 

2.2 Architectural Considerations for Observation Prediction

3 Why Strict Mediation Is Necessary 

The architectural formulations in Section 2.2 differ only in what information is available at prediction time, but this distinction has a fundamental consequence: whether predictive performance constrains the learned representation. 

3.1 Sufficiency from Prediction 

Under strict mediation, all predictive information must pass through the latent state. This makes predictive sufficiency a direct consequence of predictive accuracy. 

Proposition 1 (Sufficiency under strict mediation) . In a strictly mediated model with pθ (ot+1 | ˆst, a t)

and ˆst ∼ pθ (· | ht), if pθ (ot+1 | ht, a t) = p(ot+1 | ht, a t) ∀ht, a t, then ˆst is a sufficient statistic: 

p(ot+1 | ht, a t) = p(ot+1 | ˆst, a t).

Proof sketch. Under strict mediation, the decoder cannot access ht. If two histories map to the same 

ˆst but require different predictions, the model cannot match both, contradicting optimality. This result is structurally simple, but important: it shows that under strict mediation, predictive accuracy becomes a diagnostic for representation quality. Formally, optimal prediction implies I(ot+1 ; ht | ˆst, a t) = 0 ;a full information-theoretic treatment and quantitative relaxation for suboptimal prediction are provided in Appendix A. 

3.2 The Leaky Bypass Problem 

This guarantee is specific to strict mediation. When the decoder can access the history, predictive accuracy places no constraint on the latent state.

3.3 Rollout Consistency and Stability 

Strict mediation also affects multi-step rollout. The key distinction is whether the information available at training matches rollout conditions. 

Proposition 3 (Training–inference consistency) . In strictly mediated models, rollout and training operate on the same inputs (ˆ st, a t), so one-step errors match. In leaky models, reliance on ht during training leads to degraded rollout performance. Beyond consistency, strict mediation changes how errors propagate. In stateless and leaky models, errors accumulate through an expanding history, leading to worst-case geometric growth. In contrast, strictly mediated models propagate error through a single latent state, enabling bounded error under contraction assumptions. The bounded-error conclusion requires LS < 1, a property of the learned dynamics, not the architecture; strict mediation ensures only the structural form of the recurrence, not the contraction property itself. 4 Enforcing Strict Mediation in the Text Domain 

The preceding analysis establishes why strict mediation matters. We now address the central challenge: enforcing it in the text domain, where latent states are discrete, non-differentiable, and must be trained without variational objectives. 

4.1 Textual Latent States 

We represent latent states as text rather than continuous vectors. While classical world models use continuous latents trained with variational objectives such as the ELBO [Hafner et al., 2019a,b], this is best suited to low-dimensional settings. In text-based environments, observations are free-form, horizons are long, and relevant information mixes entities, relations, and attributes, making textual states a better match. Textual latent states offer four advantages.(1) Alignment with LLMs: 

encoding, transition, and decoding are implemented as different conditioning modes of a single model, avoiding lossy modality conversions. (2) Adaptive bottleneck: capacity scales with content via variable-length representations, rather than a fixed-dimensional constraint. (3) Interpretability: 

states are directly inspectable and can be evaluated against ground-truth facts, enabling representation-centric evaluation. (4) Modality match: observations are already semantic text, so textual states preserve their structure. The main tradeoff is that textual states are discrete and non-differentiable, preventing standard reparameterization-based training. In addition, expressive decoders trained with ELBO-style objectives can ignore the latent state (posterior collapse) [Bowman et al., 2016, He et al., 2019]. Under strict mediation, this immediately harms prediction since there is no history bypass. Overcoming this training difficulty is the core technical problem this paper addresses. We therefore train latent state generation using reinforcement learning (Section 4.3). 

State schema. Strict mediation does not require a predefined latent-state vocabulary. In the most general form, the latent state may be any textual object passed between encoding, transition, and decoding. In this work, we instantiate latent states as JSON predicate–argument lists because they are easy to parse, inspect, and compare. For the main experiments, we further restrict predicates to a simulator-aligned vocabulary. This restriction is not required by the model, but enables more comprehensive state-level evaluation (Section 5.3). 

4.2 Model Architecture 

We implement strict mediation with a single pretrained autoregressive language model with parameters 

θ. The model is used in three prompt-defined stages with shared parameters: Encoding: ˆst ∼ pθ (· | ht), (3) Predicting: ˆst+1 ∼ pθ (· | ˆst, a t), (4) Decoding: ˆot+1 ∼ pθ (· | ˆst+1 , a t). (5) 4After ˆst is generated, the transition and decoding stages receive no access to ht. Thus, the latent state is the only channel from history to future prediction, both during training and inference. 

4.3 Training via Factorized GRPO 

Section 4.2 defines the information flow; we now turn to the key algorithmic challenge: training under strict mediation when the latent space is discrete text. Textual latent states are non-differentiable, which precludes gradient-based optimization through the latent space. We treat each generation stage (encoding, transition, and decoding) as a stochastic policy over text sequences and optimize for predictive accuracy using reinforcement learning. Standard GRPO [Shao et al., 2024] is incompatible with strict mediation: it operates on a single autoregressive continuation of a fixed prompt, so the transition and decoding steps would access the original history through the shared prompt context. We introduce factorized GRPO (fGRPO, Figure 1), which extends GRPO to tree-structured, multi-prompt rollouts. For a given history ht, we sample A candidate latent states, then B candidate next states per encoding, then C candidate observations per transition, forming a tree with ht as root and ˆo(a,b,c ) 

> t+1

as leaves. From a single tuple (ht, a t, o t+1 ), this yields 1 + A + AB GRPO groups (Figure 1d). Rewards are computed at the leaves by comparing ˆo(a,b,c ) 

> t+1

with ot+1 and propagated upward by averaging over children so each parent takes the mean reward of its immediate descendants; advantages are computed group-relative within each level. All groups are aggregated into a single batch to update the shared LLM parameters. 

4.4 Reward Design 

The reward signal combines predictive accuracy with structural regularization of the latent states. We summarize the key components below; full details are provided in Appendix F.2. Observation accuracy. The primary reward encourages latent states that support accurate next-observation prediction under strict mediation. Given a predicted observation ˆot+1 and ground truth ot+1 , we define Robs as the fact-level F1 score between the extracted fact sets of the two observations. This directly measures whether the model predicts the correct underlying environment facts, independent of surface form. Format validity. A binary indicator Rfmt (ˆ s) ∈ { 0, 1} checks whether the generated latent state satisfies required structural constraints. For a rollout branch, validity is enforced jointly across stages so all generated latent states must be valid; invalid states receive a penalty on the observation reward rather than an additive bonus, preventing the model from producing well-formed but semantically vacuous states. Auxiliary state rewards. We introduce three auxiliary rewards for valid states: (i) a support reward Rsup that encourages consistency between the predicted observation and the underlying state facts (both derived from text); (ii) a deduplication reward Rdedup that penalizes semantically redundant facts in the latent state; and (iii) an ablation reward Rabl that measures the fraction of facts in the latent state whose removal degrades prediction accuracy, encouraging compactness. The computation of these rewards does not rely on the access to hidden environment states. Combined reward. The final reward is:

5 Experimental Evaluation 

Our evaluation targets representation quality , rather than downstream control or planning performance. Under strict mediation, representation properties become empirically identifiable: if a latent state is sufficient, compact, and stable, these properties should manifest directly in prediction behavior. 

5.1 Datasets 

Our evaluation requires environments that are partially observed for the agent but fully instrumentable for the developer: the model sees only textual observations, while we access the complete Markov state for evaluation. We use two text-based benchmarks: T EXT WORLD [Côté et al., 2018] and SCIENCE WORLD [Wang et al., 2022]. Both define a fully specified symbolic state but render only 5a lossy natural-language observation at each step; a single observation is generally insufficient to identify the underlying state. We generate offline datasets by executing a diverse set of behavior policies (expert, ϵ-greedy, exploration-then-oracle, random walk, and failure modes) and logging trajectories of observations and actions. For T EXT WORLD , we generate 2, 726 trajectories ( 121 , 054 

steps) across 261 game configurations varying in world size ( 1–12 rooms), recipe length ( 1–5

ingredients), and skill requirements, with 24 random seeds per configuration. For S CIENCE WORLD ,we generate 600 trajectories ( 19 , 481 steps) across 30 task types. Average trajectory length is 45 steps for T EXT WORLD and 33 for S CIENCE WORLD . The main training set contains 2, 287 trajectories (102 , 920 steps) for TW and 540 trajectories ( 17 , 573 steps) for SW, with permanent holdout sets of 

439 TW trajectories and 60 SW trajectories reserved for final evaluation. All model training is strictly offline, without further environment interaction. Full details on data generation, complexity sweeps, noise injection, ground-truth logging, and holdout design are provided in Appendix D. 

5.2 Baselines 

We compare against three baselines corresponding to the architectural formulations in Section 2.2, isolating the effect of latent state mediation and representation choice. Stateless follows the history-based formulation, predicting the next observation directly from the full history and action via supervised fine-tuning. It serves as an upper bound on one-step predictive performance but does not learn an explicit representation. LeakyState follows the leaky formulation, introducing a textual latent state while allowing prediction to condition on both the state and the full history; it is trained using standard GRPO without enforcing strict mediation, so the model can rely on the history bypass as discussed in Proposition 2. StrictState-Emb follows the strict formulation but replaces textual latent states with continuous embeddings, as a fixed number of soft tokens prepended to the decoder input, trained via a variational objective LELBO = Eqϕ(st|ht)[log pθ (ot+1 |st, a t)] −

β KL( qϕ(st|ht)∥pθ (st|st−1, a t−1)) , mirroring classical world models such as RSSM and Dreamer. 

5.3 Evaluation Metrics 

Our evaluation targets representation quality rather than downstream control, using three metrics. 

Observation-level F1 (obs.F1 ) measures next-observation prediction quality: we extract structured facts (predicates and arguments) from both the predicted and ground-truth observations and compute F1 over the extracted fact sets (Appendix F.1), directly measuring whether the model predicts the correct environment facts rather than matching surface text. State-level F1 (state.F1 ) evaluates latent states by parsing them into structured predicates and computing F1 against the subset of ground-truth facts needed to render the next observation 1. This rendering-derived subset provides an exact sufficiency target without requiring irrelevant simulator variables (Appendix D). We additionally report the compression ratio (CR), defined as the ratio between the cumulative history length and the latent state length, measuring how aggressively the model compresses history into its representation. Finally, to evaluate temporal consistency, we measure state.F1 across multi-step rollouts at increasing prediction horizons H; degradation as H increases reflects the accumulation of prediction errors and allows comparison of rollout stability across models. We use greedy decoding during evaluation and report ±1 standard error over test examples as error bars 2.

6 Related Work 

Latent world models. Classical world models such as RSSM [Hafner et al., 2019b], Dreamer Hafner et al. [2019a, 2020, 2025], and MuZero [Schrittwieser et al., 2020] learn continuous latent states with explicit transition models, enforcing a Markov factorization in which predictions depend only on the current state and action. Our work shares this commitment to strict mediation but operates in the text domain, where latent states are discrete and variational training is poorly matched (Section 4.1). More recent approaches relax this factorization: DreamerV4 [Hafner et al., 2025] conditions on the full sequence of past latent embeddings, and PAN [Xiang et al., 2025] uses an autoregressive LLM backbone over the entire latent history. Our analysis in Section 3.2 formalizes the consequence: when the architecture permits history bypass, predictive accuracy does not constrain 8the representation. LLMs as world models. RAP [Hao et al., 2023] formulates reasoning as planning over textual states using a frozen LLM, enforcing strict mediation by construction but operating in fully observed MDPs without learning. Wang et al. [2024] evaluate LLMs on single-step state prediction given the full ground-truth state. Li et al. [2025] fine-tune LLMs on offline trajectories but condition on the full observation history without learning a compressed latent state. In all cases, either partial observability is absent, or no explicit latent representation is learned, or both. Our work addresses both. Since even a fine-tuned history-conditioned model fails to produce identifiable representations, the limitation is structural rather than a matter of model capacity or prompting strategy. Representation learning for sequential decision-making. The information bottleneck principle [Tishby et al., 2000] provides the theoretical foundation for compressed representations that retain task-relevant information. JEPA-style methods [LeCun et al., 2022] learn representations via embedded prediction rather than reconstruction. Our approach retains this signal but replaces variational training with reinforcement learning and enforces strict mediation to ensure identifiable sufficiency. Detailed comparison with prior work is in Table 5 in appendix. 

7 Conclusion 

We have shown that strict latent state mediation can be effectively enforced in text-based world models through factorized GRPO, yielding identifiable, interpretable representations whose quality is directly diagnosable from predictive performance. Confirmed empirically across compositional difficulty levels and prediction horizons, the resulting textual latent states compose entities, relations, and attributes as variable-length text, opening the door to LLM-grounded reward signals in settings where precise reward models are unavailable. The natural next steps are extending strict mediation to recurrent or hierarchical encoders that process history incrementally, and demonstrating that the representation-level gains established here translate into improved planning and task completion. 

Limitations. Our evaluation focuses on representation quality rather than downstream control or planning, and is limited to two synthetic, instrumented environments (TextWorld and ScienceWorld), which may not fully reflect real-world complexity. Two structural limitations also merit discussion. First, strict mediation places the full burden of history compression on a single-shot encoder with no incremental refinement, so any missed information is irrecoverable, since removing history bypass also removes the decoder’s ability to compensate for encoder errors. Second, while strict mediation and fGRPO are environment-agnostic, the auxiliary rewards that shape representations into interpretable, compact forms rely on structured fact extraction; extending these to less structured domains will require alternatives like learned similarity metrics or LLM-based evaluation. Broader Impacts . This work is primarily methodological and may improve the reliability and interpretability of AI systems. Potential risks include misuse of more capable generative models, though our study is limited to offline benchmarks and does not involve deployment.

Proof. Under strict mediation, the model’s prediction at each step depends only on (ˆ st, a t). During training, the model receives (ˆ st, a t) and produces pθ (ot+1 | ˆst, a t). During one-step rollout, the model receives the same input types and produces the same function. The information available to the model is identical in both regimes, so the one-step prediction error is the same. In multi-step rollout, the latent state ˆst+k is generated recursively by the learned transition function rather than encoded from the true history. The marginal distribution over latent states therefore shifts relative to training (covariate shift). However, under strict mediation, this shift is governed entirely by the transition function’s error propagation, formalized in Appendix B via the contraction analysis: when LS < 1,the state error remains bounded. Crucially, the form of the input does not change between training and rollout; the model always conditions on (ˆ st, a t) and never on ht. Therefore there is no structural mismatch of the kind that affects leaky architectures. Under a leaky architecture, the model’s training-time prediction is pθ (ot+1 | ˆst, h t, a t), which may depend on ht. During rollout, the true history is unavailable and must be replaced by the synthetic history of past predictions (ˆ ot+1 , . . . , ˆot+k). This introduces both covariate shift (the synthetic history has different statistics than the true history) and a structural input mismatch (the model was trained with access to true history but must operate without it). Any learned dependence on ht leads to degraded rollout performance that cannot be bounded by transition error alone. 

A.4 Information-Theoretic Restatement of Proposition 1 

Proposition 1 can be restated in information-theoretic terms. Under strict mediation, the encoding and decoding steps form a Markov chain: 

ht → ˆst → ot+1 (given at), (12) meaning that pθ (ot+1 | ˆst, a t, h t) = pθ (ot+1 | ˆst, a t). By the data processing inequality applied to this chain, 

I(ht; ot+1 | at) ≥ I(ˆ st; ot+1 | at), (13) since processing ht through the encoder into ˆst cannot increase information about ot+1 . Equality holds if and only if ˆst is a sufficient statistic of ht for ot+1 given at. Define the predictive information gap :

∆I = I(ot+1 ; ht | ˆst, a t). (14) This quantity measures how much information ht carries about ot+1 beyond what is captured in ˆst.By the chain rule of mutual information: 

I(ot+1 ; ht | at) = I(ot+1 ; ˆ st | at) + I(ot+1 ; ht | ˆst, a t), (15) so ∆I = I(ot+1 ; ht | at) − I(ot+1 ; ˆ st | at) ≥ 0 by the DPI. Under optimal prediction, the model’s predictive distribution matches the true distribution for each history. As shown in the proof of Proposition 1, this requires the encoder to map histories with different predictive distributions to different latent states. Consequently, ˆst captures all predictive information, implying ∆I = 0 .

Corollary 1 (Information-theoretic sufficiency) . Under strict mediation, if the model achieves optimal prediction, then I(ot+1 ; ht | ˆst, a t) = 0 : the latent state captures all information that the history contains about the next observation.

D Dataset Details 

Environments and partial observability. Both environments define a fully specified symbolic state (entities, attributes, relations, and task variables), but render only a lossy natural-language observation at each step. Observations may omit relevant variables, compress multiple state features into a single description, or vary in surface form without changing semantics. As a result, a single observation is generally insufficient to identify the underlying state, and accurate prediction requires integrating information across time. 

Controlling challenge factors. To comprehensively probe robustness and abstraction, we vary multiple factors during generation. In T EXT WORLD , we sweep game complexity through world size (number of rooms), task complexity (e.g., recipe length), and skill requirements (e.g., navigation vs. cooking/cutting/opening), and generate multiple instances via random seeds (yielding different layouts and object placements). In both T EXT WORLD and S CIENCE WORLD , we additionally inject controlled nuisance variation and noise into actions and/or text (e.g., exploration noise, invalid attempts, lexical perturbations), with step- and trajectory-level metadata recording action prove-nance and noise type. These knobs let us separately stress partial observability, long-horizon credit assignment, and sensitivity to semantically irrelevant perturbations. 

Ground-truth state logging. While the model never observes the ground-truth state, we log it for evaluation and diagnostics. At each timestep, we extract the full symbolic simulator state as a set of structured predicate–argument facts (e.g., at(entity, room) , in(object, container) ,

on(object, supporter) , object attributes such as open , closed , or food states such as raw ,

cooked ). These facts are obtained directly from the game engine’s internal state representation via instrumentation, using the same underlying rules that govern environment dynamics, and are not 15 inferred from observations or constructed manually. From this full post-action state, we construct a canonical subset of observation rendering facts , defined as the minimal set of facts required by the environment’s rendering logic to produce the natural-language observation. This subset is computed deterministically by following the game engine’s observation-generation rules (e.g., Inform7/TextWorld rendering), rather than by heuristic or learned extraction. Concretely, it includes (i) the player’s current location, (ii) all objects present in the current room, (iii) attributes of visible objects (such as food states or container states), (iv) contents of containers when they are open, (v) objects placed on supporters, and (vi) spatial relations and connectivity facts (e.g., exits, doors, and directions) needed to describe the layout. For actions such as look or navigation, this corresponds to the full set of facts required to render the room description; for local actions (e.g., take , open , examine ), it includes only the subset of post-action facts explicitly reflected in the resulting observation. Because both the full state and the observation rendering facts are derived directly from the game engine using its native rules, this construction provides an exact and unambiguous symbolic target aligned with the generated observation text. It therefore enables precise evaluation of predictive sufficiency: state.F1 measures whether the learned latent state captures exactly the information required to reproduce the next observation, independent of surface-form variation. In addition, access to the full logged state enables auxiliary diagnostics, such as verifying transition consistency across equivalent histories and distinguishing true state uncertainty from errors due to textual variation.

F.1 Fact Extraction from Observations 

All observation-level rewards and metrics operate on structured fact sets extracted from free-form observation text. Given an observation o, we define a deterministic mapping facts( o) that produces a set of predicate instances (e.g., in(apple, fridge) , open(fridge) ). The extractor operates purely on surface text and does not access hidden simulator states. For each environment, we use a rule-based parser aligned with the observation templates to recover entities, relations, and attributes explicitly stated or strongly implied by the text (e.g., location, containment, object state, and action effects). We additionally apply a small set of deterministic rules (e.g., fried(X) ⇒ cooked(X) )to capture logical implications. While our implementation uses environment-specific parsers, the framework does not depend on simulator internals. In domains without structured templates, facts( ·)

We enforce a lightweight structural constraint on the generated latent state. The state must be a JSON-encoded list of facts, where each fact is a dictionary with exactly two keys, 

"predicate" and "arguments" . The predicate must be a non-empty string, and the arguments must be a list of non-empty strings. No argument may exceed max_num_words_in_arg words (set to 8 for ScienceWorld and 3 for TextWorld cooking). Duplicate facts are disallowed, and duplicate arguments within a single fact are not permitted. The format reward Rfmt ∈ { 0, 1} indicates whether these constraints are satisfied.

These rewards also operate only on extracted facts from text. Support reward. The support reward Rsup measures whether facts stated in the predicted next observation ˆot+1 are represented in the predicted next latent state ˆst+1 . Cov(ˆ s, ˆo) = |facts (ˆ s) ∩ facts (ˆ o)||facts (ˆ o)| .

We define

This reward is not intended to define sufficiency of the latent state by itself: because the leave-one-out test is local to the sampled transition, a fact may be useful for future or counterfactual actions even if its removal does not affect the decoded observation for the current action. We therefore apply

Predictive sufficiency is enforced primarily by strict mediation and the observation reward over diverse action-conditioned transitions. In practice, Rabl biases the model against stale or redundant facts without aggressively pruning persistent state information needed for long-horizon rollout. All leave-one-out decoder evaluations are performed in parallel.

F.5 Parallelized factorized GRPO. 

We implement fGRPO using an asynchronous tree-rollout engine integrated with verl and vLLM. For each training example, the encoder, transition, and decoder stages are expanded as separate prompt calls, but sibling generations from the same parent node are submitted concurrently. Concretely, all A

encoder samples, all B transition samples under each encoded state, and all C decoder samples under each predicted next state are launched asynchronously, allowing the rollout tree to be generated in parallel rather than by a serial depth-first traversal. The resulting tree is then flattened into GRPO training samples, with the grouping identifier set to the parent node of each sibling set so that group-relative advantages are computed separately for encoder, transition, and decoder decisions. This design avoids relying on the standard verl mechanism that repeats prompts before generation. Instead, rollout multiplicity is controlled inside the world-model rollout itself, which is necessary because different nodes in the tree have different prompts and conditioning contexts. We also share request identifiers among siblings generated from the same parent prompt, enabling the rollout backend to batch compatible requests efficiently. For the ablation reward, decoder calls for the full state and all leave-one-fact-out states are likewise submitted concurrently; we skip this computation when it would be identical across decoder-only siblings and therefore cancel out in the GRPO advantage.
