## 摘要
智能导航系统需要一个基础导航模型，其观察策略可以在推理时从外部重新配置，因为指令跟随、物体搜索、目标跟踪和自动驾驶共享相同的感知-规划主干，但需要根本不同的策略来消耗视觉流。我们提出了Qwen-RobotNav，一个建立在Qwen-RobotNav之上的可扩展导航模型，通过一个具有两个互补维度的参数化接口来解决这个问题：选择导航行为的多个任务模式，以及控制视觉历史编码方式的可控观察参数（例如，token预算、每个相机的权重）。通过对所有参数进行训练时的随机化，Qwen-RobotNav对任何推理时配置都具有鲁棒性，且无需对Qwen-RobotNav主干进行任何架构修改。我们在1560万个样本上训练Qwen-RobotNav；与视觉-语言数据的联合训练防止了在仅轨迹训练中观察到的退化为反应式动作序列映射器的问题。参数化接口也使Qwen-RobotNav成为智能系统的自然构建块：对于长周期场景，上层规划器将目标分解为子任务，并在情节中动态切换Qwen-RobotNav的任务模式和上下文策略，通过重复调用同一模型来组合复杂行为。大量实验表明，Qwen-RobotNav在主要导航基准上取得了新的最先进结果。该模型在2B到8B参数之间表现出良好的扩展性，联合多任务训练开发了一个跨任务家族转移的共享空间规划基础，并在不同环境中对真实世界机器人展现出强大的零样本泛化能力。

## Abstract
Agentic navigation systems require a base navigation model whose observation strategy can be externally reconfigured at inference time, because instruction following, object search, target tracking, and autonomous driving share the same perception-planning backbone yet demand fundamentally different strategies for consuming the visual stream. We present Qwen-RobotNav, a scalable navigation model built on Qwen-RobotNav that addresses it through a parameterised interface with two complementary dimensions: multiple task modes that select the navigation behaviour, and controllable observation parameters (e.g., token budget, per-camera weights) that govern how visual history is encoded. With training-time randomization over all parameters, Qwen-RobotNav is robust to any inference-time configuration requiring zero architectural modification to the Qwen-RobotNav backbone. We train Qwen-RobotNav on 15.6M samples; co-training with vision-language data prevents the collapse into reactive action-sequence mappers observed in trajectory-only training. The parameterised interface also makes Qwen-RobotNav a natural building block for agentic systems: for long-horizon scenarios, an upper-level planner decomposes goals into sub-tasks and dynamically switches Qwen-RobotNav's task mode and context strategy mid-episode, composing complex behaviours from repeated calls to the same model. Extensive experiments show that Qwen-RobotNav sets new state-of-the-art results across major navigation benchmarks. The model exhibits favourable scaling from 2B to 8B parameters, with joint multi-task training developing a shared spatial-planning substrate that transfers across task families, and demonstrates strong zero-shot generalisation to real-world robots across diverse environments.

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **核心问题**：主流导航模型（如 NavFoM、ABot-N0）采用固定的观察上下文策略（如均匀子采样或滑动窗口），无法适应不同任务对视觉历史的不同需求。例如，指令跟随需要长期记忆以回顾远处地标，目标跟踪只需最近几帧，而物体搜索在探索阶段需要全局记忆、接近阶段需要局部关注。现有模型无法在推理时动态调整观察策略，更无法在情节内切换。
- **整体含义**：为解决上述问题，论文提出**将多任务导航的核心挑战从架构设计重新定义为观察上下文建模**，并设计一个**可参数化的接口**，使得上层智能体（或人类操作员）能够在推理时动态选择任务模式和观察配置，无需重新训练。这为构建**可重构的导航原语**奠定了基础，支持长周期复杂行为（如具身问答）的组合。

## 2. 论文提出的方法论：核心思想、关键技术细节、公式或算法流程

### 核心思想
- **统一导航模型**：构建于 Qwen3-VL 之上，将所有导航任务统一为**航点轨迹预测**问题，输出 K=8 个航点 (x, y, θ)。
- **任务自适应观察编码**：不再使用固定观察策略，而是提供一组可配置的参数，通过训练时随机化所有参数，使模型对推理时的任意配置具有鲁棒性。
- **智能导航系统集成**：将 Qwen-RobotNav 作为**反应式执行器**，上层规划器（如 Qwen3.6-Plus）分解长周期目标为子目标，并动态切换任务模式和观察配置，形成两级层次化系统。

### 关键技术细节

#### （1）模型架构
- **视觉编码器**：SigLIP-2 Vision Transformer，支持动态分辨率（2D-RoPE），通过 DeepStack 机制注入多层级视觉表示。
- **语言主干**：继承 Qwen3-VL 的 LLM，处理拼接后的视觉 token 和语言 token。
- **动作头**：4层 MLP，将 LLM 最终隐藏状态映射为 24 维输出（8 航点 × 3 自由度），训练使用 MSE 损失。

#### （2）任务自适应观察编码
- **参数控制轴**：
  - 视觉 token 预算 B（2048～4096）
  - 时间衰减 γ（1～3），控制对最近帧的偏重
  - 每个相机权重 w_c（可设置的默认值）
  - 帧采样模式 m（随机 / 最新）
- **token 分配算法**（Algorithm 1）：
  1. 根据帧采样模式选出 T′ 帧。
  2. 计算时间权重：ω_t = exp(γ·t/(T′-1))，γ=0 为均匀，γ=3 时最新帧权重为最旧帧的约 20 倍。
  3. 联合权重矩阵 W[t,c] = ω_t · w_c。
  4. 受约束分配：每个单元格至少获 b_min 个 token，剩余预算按权重比例分配，超出 b_max 的单元格释放超额并重新分配，直到稳定。
  5. 根据分配 token 数决定图像分辨率（保持原始宽高比）。
- **训练时随机化**：所有超参数（B, γ, w_c, b_min, b_max）每样本独立采样，帧采样模式以 50% 概率交替使用随机和最新。

#### （3）视点与时间标识
- 使用自然语言标签（如 “Time step 0 Front View <image>”）插入视觉 token 之间，无需架构修改。描述性名称（如“Front”）优于数值方位，因语言模型已内化语义。

#### （4）具身感知提示设计
- 通过系统提示中的自然语言前缀区分机器人（“imagine you are a robot”）与汽车（“imagine you are a car”），无需额外参数。

#### （5）训练策略
- **复合损失**：L = L_traj + λ L_VL，其中 L_traj 为航点 MSE，L_VL 为视觉-语言推理样本的下一 token 预测损失，λ=1.0。
- **联合训练**：85% 导航轨迹数据 + 15% 导航相关视觉-语言推理数据（共 1560 万样本），防止仅轨迹训练导致的感知退化。
- **优化**：AdamW（β1=0.9, β2=0.95, weight decay=1e-2），余弦学习率，线性预热 3% 步数，峰值学习率 2e-5（视觉和 LLM）/ 1e-4（动作头），梯度裁剪 1.0。

#### （6）智能导航系统
- **上层规划器**：调用 Qwen-RobotNav 时指定子目标指令 L_i、任务模式 τ_i（VLN/PointNav/ObjNav/Tracking）、观察配置 Φ_i。
- **导航工具接口**：除导航调用外，上层规划器还可使用辅助视觉工具（物体检测、场景理解、语义定位）。
- **轨迹证据与上下文压缩**：每次导航调用后生成紧凑证据记录（子目标、任务模式、进度、关键帧索引等），并维护一个证据笔记本，仅保留持久结论（搜索区域、候选位置、拒绝假设），实现两级记忆（单情节记忆 + 跨情节记忆）。

### 充分性评估
- **客观性**：对比的方法包括近年主流基础模型（NavFoM、ABot-N0、Uni-NaVid 等）以及针对特定任务的最优方法，且多数实验在两个规模（4B/8B）下报告，覆盖了本文核心贡献。
- **公平性**：在 ObjectNav 中特别注明 Qwen-RobotNav 仅使用 RGB 而多数基线使用深度/里程计，在 OVON 中注明使用单视图对比 ABot-N0 的全景视图；在自动驾驶中注明提供历史轨迹作为先验（表 8 中同时报告了无历史先验的版本以显示增益来源）。
- **潜在不足**：消融实验的样本量较小（500 个验证集片段），且仅在 VLN-CE R2R 上做消融，未在其他任务上验证趋势；EQA 中的步数比较可能受限于导航效率与正确性的权衡；AlpaSim 仅报告零样本结果且低于专用方法，未进行微调对比；部分 SPL 指标在长路径任务中较差（如 OVON 中 Qwen-RobotNav SPL 低于 ABot-N0 和 NavFoM，作者解释为“先到达再探索”的策略导致路径更长）。

## 6. 论文的主要结论与发现

1. **参数化观察编码的有效性**：训练时随机化所有超参数使得模型对任意推理配置具有鲁棒性，无需任务特定调参；token 预算 B 和衰减 γ 对 VLN 性能有显著影响，但最佳配置非单调（过大预算可能带来边际收益递减）。
2. **联合训练的必要性**：仅轨迹训练会导致模型退化为反应式动作映射器，添加 15% 视觉-语言数据可保持开放世界感知能力，并提升跨任务泛化。
3. **跨任务共享表示**：多任务联合训练形成了共享空间规划基础，从物体搜索到自动驾驶的知识可以相互迁移。
4. **规模扩展性**：从 4B 到 8B，多数任务性能提升，尤其长周期推理任务（RxR 提升 12% SR，EQA 提升 4-6 个点）。
5. **智能体导航系统优势**：利用 Qwen-RobotNav 的可配置接口，上层规划器可动态切换任务模式和观察配置，在 EQA 中以 77% 更少的导航步数实现 SOTA，说明更有效的探索。
6. **零样本真实世界泛化**：在从未见过的展览厅、公寓、咖啡店场景中成功导航，验证了模拟到现实的迁移能力。

## 7. 优点

- **方法创新性**：将导航问题重新定义为上下文建模问题，引入参数化接口和训练时随机化，为导航基础模型提供了新的设计范式。
- **技术简洁性**：所有视角和时间信息通过自然语言标签传递，无需修改架构；具身提示仅需修改 prompt，无需额外参数。
- **系统完整性**：从数据生成（含自动生成的 T2V 数据）、模型训练、消融分析、多基准评估，到真实部署形成完整闭环。
- **消融深度**：对数据规模、超参数（B、γ）、模型规模、联合训练效果均进行了系统消融，理论分析清晰。
- **公平性标注**：明确标注基线使用的额外传感（深度、里程计、全景）与自身的设置差异，讨论 SPL 较低的合理解释。
- **开放资源**：在 GitHub 上公开代码和模型权重（https://github.com/QwenLM/Qwen-RobotNav）。

## 8. 不足与局限

- **实验覆盖不均衡**：消融实验仅在一个任务（VLN-CE R2R）和少量样本（500 片段）上进行，未在跟踪、驾驶、物体搜索等任务上验证超参数敏感性与数据规模趋势。
- **SPL 指标较差**：在 OVON 和部分 VLN 数据上，SPL 低于 ABot-N0 和 NavFoM，作者归因于探索优先策略，但未提供定量分析（如路径冗余度 vs 成功率权衡曲线）。
- **EQA 中未消融规划器设计**：仅比较了 Qwen3.5-Plus 和 Qwen3.6-Plus，未对比不同规划策略（如是否使用记忆笔记本、是否使用辅助工具）的重要性。
- **自动驾驶泛化能力有限**：在 AlpaSim 零样本测试中，Qwen-RobotNav 的 Close Encounter Rate（22%）和 Off-Road Rate（34%/27%）明显高于专用方法，且 4B 与 8B 的 PDMS 几乎相同（79.5），说明模型在小规模闭环场景下可能不稳定性较高。
- **真实世界实验仅定性展示**：未提供定量指标（成功率、平均导航误差等），且仅展示了 3 个案例，可能缺乏统计显著性。
- **训练数据来源的领域偏差**：85% 轨迹数据来自模拟器（Matterport3D、HM3D、nuScenes、OpenScene），尽管使用了图像增强和 T2V 数据模拟真实感，但 domain gap 仍可能存在（作者也承认 sim-to-real 是主要瓶颈）。
- **计算资源开销**：2,816 H100 GPU 小时对于大多数团队而言昂贵，且未报告 4B 模型的训练成本。
