## 摘要
多模态大型语言模型（MLLM）在视觉推理方面表现出色，但依赖于基于文本的思维链（CoT），缺乏可解释的视觉中间步骤。现有方法使用不透明的标记或外部工具，缺少关键特性。我们提出Gen-VCoT，一个利用专家视觉模型生成RGB图像作为推理中间步骤的框架。它包含三个阶段：视觉定位（SAM分割）、几何推理（Marigold深度图）和语义推理（Qwen2-VL集成）。一个自适应路由器选择推理深度。评估显示，Gen-VCoT在空间（提升25%）和深度（提升50%）问题上表现更好，但可能损害简单的事实查询。在CLEVR上，文本CoT优于视觉中间步骤（91.2%对62.5%），表明任务依赖的最优表示。Gen-VCoT为可解释的多模态推理建立了新范式。

## Abstract
Multimodal large language models (MLLMs) excel at visual reasoning but rely on text-based chain-of-thought (CoT), lacking interpretable visual intermediates. Existing methods use opaque tokens or external tools, missing key properties. We propose Gen-VCoT, a framework using expert vision models to generate RGB images as reasoning intermediates. It has three stages: visual grounding (SAM segmentation), geometric reasoning (Marigold depth maps), and semantic reasoning (Qwen2-VL integration). An adaptive router selects reasoning depth. Evaluations show Gen-VCoT improves spatial (25% better) and depth (50% better) questions, but may hurt simple factual queries. Text CoT outperforms visual intermediates on CLEVR (91.2% vs 62.5%), showing task-dependent optimal representations. Gen-VCoT establishes a new paradigm for interpretable multimodal reasoning.

- **核心问题**：多模态大型语言模型（MLLMs）在视觉推理中主要依赖文本形式的思维链（Chain-of-Thought, CoT），但文本是间接的空间推理媒介，缺乏可解释、可人类解读的视觉中间表示。现有方法（如COVT使用连续视觉令牌但不可解释；Visual Sketchpad依赖外部工具且仅生成稀疏几何基元）无法同时满足可解释性、端到端可训练性和稠密视觉表示的需求。
- **研究动机**：受“生成即理解”范式（Vision Banana）启发，认为生成分割图或深度图的过程天然蕴含了场景理解能力，可作为视觉推理的中间步骤。
- **整体含义**：提出一种新范式——通过生成RGB图像（分割图、深度图）作为视觉思维链的中间表示，提升可解释性和空间推理准确性。

## 2. 方法论：核心思想、关键技术细节

- **核心思想**：将视觉推理分解为三个可解释的阶段，每个阶段生成可视化的RGB图像作为中间证据，最终由MLLM整合所有视觉信息给出答案。
- **关键技术细节**：
  - **阶段1：视觉定位（Where）**——使用Segment Anything Model (SAM) 生成实例分割图。采用8×8网格点提示，对每个点选取最高IoU的掩码，面积小于100像素的掩码被过滤，剩余掩码用高对比度颜色编码。
  - **阶段2：几何推理（How）**——使用Marigold（扩散模型）进行单目深度估计，以LCM变体（4步推理）集成5次预测，生成深度图，然后通过彩虹色映射（红色近，紫色远）生成伪彩色深度图。
  - **阶段3：语义推理（What）**——将原始图像、分割图和深度图作为多图像输入送入Qwen2-VL-7B-Instruct，由MLLM整合后输出答案。
  - **自适应推理路由器**：基于BERT-base（110M参数）的分类器，根据问题文本预测推理路径（仅分割、分割+深度、全流水线、无中间表示），训练目标为交叉熵损失加效率惩罚项（λ=0.1），在50个问题的验证集上达到85%准确率。
- **算法流程**（文字描述）：  
  1. 路由器根据问题选择路径；  
  2. 若路径需要分割，则SAM生成分割图并着色；  
  3. 若路径需要深度，则Marigold生成深度图并着色；  
  4. 将原始图像与生成的中间图像拼接（根据路径选择），送入Qwen2-VL得到答案。

## 3. 实验设计：数据集、基准、对比方法

- **数据集/场景**：
  - 三个合成场景（室内、街道、公园），每个场景8个问题，涵盖6个类别（物体识别、空间关系、深度感知、计数、属性、复杂推理），共24个问题-场景对。
  - CLEVR风格的合成场景：10个场景，每个场景2-4个彩色3D物体，共80个问题（存在性、计数、颜色查询、形状查询、空间推理）。
- **基准**：直接使用MLLM（Qwen2-VL）仅输入原始图像，无中间表示。
- **对比方法**：
  - Gen-VCoT全流水线
  - 无深度分支（仅分割）
  - 无分割分支（仅深度）
  - 文本CoT基线：提供结构化文本描述（颜色、形状、大小、材质、位置）代替视觉中间体。

## 4. 资源与算力

- 所有实验在单张 **NVIDIA RTX 3090（24GB VRAM）** 上运行。
- 模型加载顺序：SAM ViT-Large（1.3GB）、Marigold LCM（fp16, 3GB）、Qwen2-VL-7B（16.6GB），顺序加载以适配显存。
- 流水线耗时：预处理（SAM分割+Marigold深度估计）约8秒每张图像；推理时通过批量处理（对同一图像的多个问题）可将每问题时间降至约5.2秒（优化的批量处理19个问题共99.6秒）。

## 5. 实验数量与充分性

- **实验组数**：
  - 主实验：19个问题（复杂场景），对比全流水线与基线。
  - CLEVR实验：80个问题，对比Gen-VCoT、基线、文本CoT。
  - 消融实验：3个场景×8个问题×4种配置 = 96次评价，分析分割和深度的贡献。
  - 路由器训练：100个问题-路径对训练，50个问题验证。
- **充分性与公平性**：实验覆盖了多种问题类型和场景复杂度，且进行了三路对比（视觉中间体 vs 文本描述 vs 无中间体）。但存在明显偏差：所有场景均为合成场景，未使用真实世界基准（如GQA、CLEVR原始数据集），且问题数量较小（最多80个），可能不足以展示统计显著性。作者也承认这是限制。

## 6. 主要结论与发现

- **主结论**：Gen-VCoT在空间（提升25%）和深度（提升50%）问题上显著优于基线，但简单事实查询（存在性、计数、属性）上会因视觉中间体引入噪声而性能下降（CLEVR上Gen-VCoT 62.5% vs 基线85.0%）。
- **对比文本CoT**：文本CoT在CLEVR上达到91.2%，优于Gen-VCoT（62.5%）和基线（85.0%），表明对于事实查询，文本描述是更精确的信息通道。
- **自适应路由器的必要性**：任务依赖最优表示——空间推理用视觉中间体，事实查询用文本描述或跳过中间体。无路由器的统一应用会降低整体性能。
- **互补性**：分割图帮助物体识别和计数，深度图帮助空间排序和距离估计，两者互补，全流水线优于任何部分配置。

## 7. 优点

- **创新性**：首次系统性地使用专家模型生成的RGB图像作为视觉思维链的中间表示，同时满足可解释性、端到端、稠密表示、可解码四个关键属性。在视觉CoT分类中，Gen-VCoT是唯一同时满足所有条件的框架。
- **可解释性**：中间结果（分割图、深度图）可直接人类可视化，便于理解和调试。
- **自适应设计**：路由器根据问题复杂度动态选择推理深度，平衡效率与质量。
- **完整的评估**：进行了多场景、多问题类型、多种中间体组合的消融实验，并引入文本CoT对比，全面揭示了不同表示的优势与局限。
- **批量优化**：通过一次性加载模型并批量处理同一图像的多个问题，大幅降低单次推理开销（从16.8秒/问题降至5.2秒/问题）。

## 8. 不足与局限

- **实验范围有限**：仅使用合成场景，未在真实世界基准（如GQA、CLEVR原始数据集、MIRA）上评估，泛化性存疑。问题数量少（主实验19问题，CLEVR 80问题），统计可靠性不足。
- **依赖外部模型质量**：SAM和Marigold的误差会传播至最终答案，且未针对任务微调这些模型，可能限制领域特定性能。
- **计算开销**：预处理阶段约8秒/图像（分割+深度），虽然可通过批量分摊，但单张图像仍有一定延迟，不适用于实时场景。
- **路由器训练的局限性**：训练数据仅100个标注样本，规模小；路由器在模糊问题上表现不佳（验证集准确率85%），且未扩展到四模式（文本描述作为额外模式）。
- **CLEVR上的性能退化**：视觉中间体在简单查询上引入噪声，导致Gen-VCoT显著低于基线，表明该框架不适合所有任务。
- **未进行用户研究或鲁棒性分析**：未评估合成场景与真实场景的语义差距，也未测试对抗性条件或分布外场景。
