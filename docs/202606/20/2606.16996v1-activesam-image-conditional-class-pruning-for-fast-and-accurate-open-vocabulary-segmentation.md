## 摘要
Segment Anything Model 3（SAM 3）为概念提示分割提供了强大的冻结骨干网络，但将其直接应用于开放词汇语义分割（OVSS）效率低下：全分辨率解码通常在整个数据集词汇上运行，而每个图像仅包含一小部分活跃类别。我们提出ActiveSAM，一个无需训练、零样本推理框架，将SAM 3转变为主动词汇分割器。ActiveSAM首先规范化并扩展类别提示，然后从低分辨率存在预览中估计图像条件活跃集。仅保留的类别使用冻结的SAM 3解码器通过分桶提示复用进行全分辨率解码。预览阶段仅使用类别存在证据并跳过不必要的分割头计算，而最终阶段应用边际感知背景校准来抑制低置信度像素。ActiveSAM无需目标数据集训练、权重更新或先知类别存在标签。在八个OVSS基准测试中，ActiveSAM改进了无训练开放词汇语义分割的速度-精度权衡，平均比当前最先进的SegEarth-OV3高出约+1.4 mIoU，同时在大型词汇数据集上运行速度高达5.5倍。ActiveSAM在模拟真实世界分布偏移的图像损坏下也展现出最强的鲁棒性，使其非常适合部署在自动驾驶和具身AI等噪声输入领域。代码可在https://github.com/VILA-Lab/ActiveSAM获取。

## Abstract
Segment Anything Model 3 (SAM 3) provides a strong frozen backbone for concept-prompted segmentation, but applying it directly to open-vocabulary semantic segmentation (OVSS) is inefficient: full-resolution decoding is typically run over the entire dataset vocabulary, whereas each image contains only a small active subset of classes. We introduce ActiveSAM, a training-free, zero-shot inference framework that turns SAM 3 into an active-vocabulary segmenter. ActiveSAM first canonicalizes and expands class prompts, then estimates an image-conditioned active set from a low-resolution presence preview. Only the retained classes are decoded at full resolution, using bucketed prompt multiplexing with the frozen SAM 3 decoder. The preview stage uses only class-presence evidence and skips unnecessary segmentation-head computation, while the final stage applies margin-aware background calibration to suppress low-confidence pixels. ActiveSAM requires no target-dataset training, no weight updates, and no oracle class-presence labels. Across eight OVSS benchmarks, ActiveSAM improves the speed-accuracy tradeoff of training-free open-vocabulary semantic segmentation, outperforming the current state-of-the-art SegEarth-OV3 by approximately +1.4 mIoU on average while running up to 5.5x faster on large-vocabulary datasets. ActiveSAM also demonstrates the strongest robustness under image corruption that simulates real-world distribution shift, making it well-suited for deployment in noisy-input domains such as autonomous driving and embodied AI. Code is available at https://github.com/VILA-Lab/ActiveSAM.

## 2. 方法论：核心思想、关键技术细节

### 核心思想
利用SAM 3的冻结低分辨率存在头（presence head）作为早期路由信号：先对完整词汇进行低分辨率存在预览，筛选出图像相关的活跃类别子集，然后仅对这些活跃类别进行全分辨率解码，从而将全分辨率解码的计算量从全词汇规模缩小到活跃类别规模。

### 关键技术细节（四个模块）

1. **上下文提示扩展（Contextual Prompt Expansion, CPE）**
   - 对原始类别名称进行词汇规范化（如“windowpane” → “window pane”）。
   - 为每个类别构建增强提示，包含：
     - 规范化后的类别本身对应的提示令牌。
     - **语义邻居令牌**：在冻结文本嵌入空间中检索与当前类别最相似的 \( M_s = 2 \) 个其他类别名称，将其文本嵌入投影后作为额外令牌。
     - **WordNet上位词令牌**：取规范化类别名称的前 \( M_t = 2 \) 个名词上位词，编码并投影为令牌。
   - 所有令牌沿序列维度拼接，构成上下文提示 \( \bar{Z}_c \)，每个类别仅计算一次并缓存。

2. **预览驱动的类别选择（Preview-driven Class Selection）**
   - 对于词汇规模 \( V > 40 \) 的数据集，在低分辨率（672×672）上运行冻结的SAM 3存在头，计算每个类别的存在分数 \( q_c \)（sigmoid后的logit）。
   - 跳过分割头计算以节省时间。
   - 根据分位数 \( \beta = 0.80 \) 动态确定阈值 \( \tau(I) \)，选择分数不低于阈值的类别作为候选集 \( A_0(I) \)。
   - 若候选集大小超过上限 \( V_{gate}=40 \)，则保留分数最高的前40个类别，形成最终活跃集 \( A(I) \)。
   - 对于小词汇数据集（\( V \leq 40 \)），跳过预览阶段，直接处理完整词汇，避免不必要的剪枝。

3. **分桶全分辨率解码（Bucketed Full-Resolution Decoding）**
   - 在全分辨率（1008×1008）上运行一次图像编码器。
   - 将活跃类别按桶大小 \( K=32 \) 分组，每组内将上下文提示沿令牌维度拼接，与共享图像特征一起送入冻结的SAM 3接地解码器。
   - 解码器输出每个类别的实例分数图 \( S_c^{inst} \) 和语义分数图 \( S_c^{sem} \)，按逐像素最大值融合：\( S_c(p) = \max\{S_c^{inst}(p), S_c^{sem}(p)\} \)。
   - 初步预测为活跃类别的argmax：\( \hat{y}_0(p) = \arg\max_{c \in A(I)} S_c(p) \)。

## 6. 主要结论与发现

1. **性能最好**：ActiveSAM在8个基准上平均mIoU达65.3，超过SegEarth-OV3（63.9）约+1.4，超过SAM 3（57.5）约+7.8。
2. **速度大幅提升**：在大词汇数据集上比SegEarth-OV3快4.8~5.5倍（如ADE20K上1.92 vs 0.35 FPS，加速5.5倍）。
3. **鲁棒性强**：在图像损坏下mIoU保持稳定，平均领先SegEarth-OV3约1~2 mIoU，而CLIP基线严重退化。
4. **组件贡献明确**：CPE带来+1.0 mIoU提升（主要贡献者），MABC在含背景数据集上提升约+0.2~0.8 mIoU，类别剪枝在几乎不损失精度下实现加速。
5. **活动集高度稀疏**：例如COCO-Object中位活跃类别仅6个（词汇80个），ADE20K中位30个（词汇150个），证明大多数类别可在预览阶段被安全过滤。

## 7. 优点

- **创新性**：首次将SAM 3的存在预览作为早期路由信号，改变传统“先全解码再过滤”的计算范式，实现训练无关的加速。
- **实用性**：无需训练、无需更新权重、无需先知标签，可直接应用预训练SAM 3，易于部署。
- **全面性**：在8个基准上验证，覆盖多种场景，并专门考虑了背景类处理和分布偏移鲁棒性。
- **效率与精度权衡**：在提升精度的同时大幅降低计算量，实现了双赢。
- **模块化设计**：CPE、类别剪枝、MABC可独立移除或调整，便于未来扩展。
- **开源**：代码已公开，可复现。

## 8. 不足与局限

- **依赖低分辨率预览的召回率**：若物体极小或被严重遮挡，可能的低存在分数导致类别被过早丢弃，无法被全分辨率解码弥补，存在漏检风险。
- **CPE依赖英文词汇数据库**：WordNet和规范化规则基于英语，对专业领域（医学、工业）或非英语词汇可能效果不佳。
