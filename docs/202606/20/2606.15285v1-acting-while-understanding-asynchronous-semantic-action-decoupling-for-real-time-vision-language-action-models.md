## 摘要
视觉-语言-动作模型（VLAs）在机器人操作中展示了强大的任务理解和泛化能力，但全模型推理的高计算成本限制了其在低延迟、高频率闭环控制中的部署。我们提出了一种异步语义-动作解耦框架，该框架沿着现有VLA的内部语义-动作接口将语义理解与动作生成分离，无需重新设计视觉-语言骨干网络或引入外部规划器。低频理解模块异步更新可复用的语义条件，而高频动作模块则持续输出控制动作，无需重复调用完整模型。为了缓解过时语义与当前执行状态之间的时间不匹配，我们进一步引入了历史动作条件化和时间错位训练，这些方法提供了短时域执行上下文，并增强了在过时语义条件下的反馈控制鲁棒性。在LIBERO上使用π₀.₅和UniVLA的实验，以及使用UniVLA的真实机器人部署表明，所提出的框架实现了高达35.6 Hz的服务器端动作模块推理吞吐量，并提供了实现高频闭环控制的低侵入路径，而无需以控制频率运行完整的VLA推理。

## Abstract
Vision-Language-Action models (VLAs) have demonstrated strong task understanding and generalization in robotic manipulation, yet the high computational cost of full-model inference limits their deployment in low-latency, high-frequency closed-loop control. We propose an asynchronous semantic-action decoupling framework that separates semantic understanding from action generation along the internal semantic-action interface of existing VLAs, without redesigning the vision-language backbone or introducing an external planner. A low-frequency understanding module asynchronously updates reusable semantic conditions, while a high-frequency action module continuously outputs control actions without repeatedly invoking the full model. To mitigate the temporal mismatch between stale semantics and the current execution state, we further introduce historical action conditioning and time-misalignment training, which provide short-horizon execution context and improve feedback control robustness under stale semantic conditions. Experiments on LIBERO with $π_{0.5}$ and UniVLA, together with real-robot deployment using UniVLA, show that the proposed framework achieves up to 35.6 Hz server-side action-module inference throughput and offers a low-intrusion path to high-frequency closed-loop control without running full VLA inference at control rate.

## 2. 方法论

### 核心思想
- 将 VLA 分解为**低频理解模块**（负责处理视觉、语言）和**高频动作模块**（基于缓存语义、当前状态和历史动作生成动作）。
- 中间可复用语义条件 `z`：UniVLA 中为**最后一层隐藏状态**，π0.5 中为**KV-cache**。
- 动作模块在每个控制步重新规划动作块，而非执行完整模型推理。

### 关键技术细节
1. **异步语义刷新**：
   - 理解模块在时间 `τk` 计算 `z_{τk} = U_θ(o_{τk}, l, s̃_{τk})`，缓存后供后续动作模块使用。
   - 动作模块在时间 `t` 使用最新缓存 `z_{τk}`（`τk ≤ t`）生成动作 `Â_t = G_ϕ(z_{τk}, s_t)`。

2. **历史动作条件化**：
   - 提供长度为 `m` 的历史动作窗口 `h_t = [a_{t-m}, ..., a_{t-1}]`，作为动作模块的额外输入。UniVLA 用轻量投影层注入，π0.5 作为额外条件嵌入动作头。
   - 历史动作提供短时域局部进度信号，缓解过时语义带来的歧义。

3. **时间错位训练**：
   - 训练时，随机采样语义延迟 Δ 服从 Uniform{0,…, H-1}（H 为动作块长度），使用过时语义 `z_{t-Δ}` 训练动作模块，使其学会在过时条件下仍能稳定控制。

4. **推理时**：
   - 低频理解模块（~3-5 Hz）异步刷新语义缓存；高频动作模块（每控制步）使用最新缓存、当前状态、历史动作重新规划动作块，并通过**ACT 风格的指数加权时间积分**合并重叠预测，输出最终动作。

- **性能保持**：在 LIBERO 上，π0.5-Async-15k 平均成功率 98.1%（接近官方全量基线的 96.9%）；UniVLA-Async 的 LIBERO-Long 成功率 91.2%（同步为 92.0%），未明显退化。

- **真实机器人提升**：UniVLA-Async 在六个任务上平均成功率 93.3%（同步 78.3%），且加权完成时间从 17.4s 降为 13.1s。
- **消融关键**：朴素缓存复用（LIBERO-Long 成功率降至 85.2），完整方法恢复至 91.2，证明历史动作条件化和时间错位训练均不可或缺。

## 7. 优点

- **低侵入性**：无需重新设计 VLA 骨干或引入外部规划器，仅修改动作侧接口条件。适应性强，可跨不同 VLA 骨干（π0.5、UniVLA）应用。
- **核心创新**：明确区分语义理解与动作控制的异步时标差异，并针对过时语义提出双管齐下的鲁棒策略（历史动作上下文 + 训练时显式模拟延迟）。
- **实验充分**：模拟+真实机器人双验证，消融设计干净，定量与定性结果一致。
- **实用价值**：在实际部署中，异步解耦显著提升控制频率而不牺牲成功率，甚至因更及时的状态反馈而提升成功率。

## 8. 不足与局限

- **依赖中间条件形式**：框架要求 VLA 存在明确的可复用中间语义接口（如隐藏状态或 KV-cache），不适用于所有架构。
- **固定刷新调度**：当前语义刷新频率固定，无法自适应环境突变或不确定性突升，过时语义仍可能误导控制。
