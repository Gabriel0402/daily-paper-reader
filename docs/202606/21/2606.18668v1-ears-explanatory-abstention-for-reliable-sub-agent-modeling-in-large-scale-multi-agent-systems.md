## 摘要
在大型企业环境中，集中式多智能体系统（MAS）日益被采用，其中协调器将用户请求委托给轻量级、领域专业化的子智能体。虽然这种架构提高了模块化、可扩展性和成本效率，但其可靠性不仅依赖于准确的路由，还取决于子智能体根据能力约束校准其响应的能力。特别是，基于较小微调模型的子智能体往往难以进行此类校准，导致它们对模糊、不明确、路由错误或超出能力范围的请求过度回答，产生幻觉输出而非可操作的反馈。为应对这一挑战，我们提出了EARS（面向可靠子智能体建模的解释性弃权机制），这是一个面向生产的框架，将子智能体弃权重构为一种智能体间通信协议：子智能体不仅弃权，还将可操作的失败状态暴露给协调器。EARS使用一组校准的LLM-as-a-Judge模型策划人机交互数据，在子智能体失败模式分类下生成结构化的弃权标签和理由。这些数据用于微调子智能体，使其能够检测失败条件并返回理由，以供协调器级别的澄清、重新路由或回退。我们在一个支持企业商业智能工作流的大规模生产电商助手中评估了EARS。EARS将整体响应通过率从68.5%提高到78.9%，表明子智能体侧的解释性弃权提高了MAS的可靠性。

## Abstract
In large-scale enterprise settings, centralized multi-agent systems (MAS) are increasingly adopted, in which a coordinator delegates user requests to lightweight, domain-specialized sub-agents. While this architecture improves modularity, scalability, and cost efficiency, its reliability depends not only on accurate routing but also on sub-agents' ability to calibrate their responses to capability constraints. In particular, sub-agents built on smaller fine-tuned models often struggle with such calibration, leading them to over-answer ambiguous, underspecified, misrouted, or unsupported requests and produce hallucinated outputs instead of actionable feedback. To address this challenge, we present EARS (Explanatory Abstention for Reliable Sub-Agent Modeling), a production-oriented framework that reframes sub-agent abstention as an inter-agent communication protocol: a sub-agent does not merely abstain, but exposes an actionable failure state to the coordinator. EARS curates human-agent interaction data using an ensemble of calibrated LLM-as-a-Judge models, producing structured abstention labels and rationales under a taxonomy of sub-agent failure modes. These data are used to fine-tune sub-agents to detect failure conditions and return rationales for coordinator-level clarification, rerouting, or fallback. We evaluate EARS in a large-scale production e-commerce assistant supporting enterprise business intelligence workflows. EARS improves the overall response pass rate from 68.5% to 78.9%, demonstrating that sub-agent-side explanatory abstention improves MAS reliability.

## 1. 核心问题与整体含义（研究动机和背景）

- **背景**：在大规模企业级多智能体系统（MAS）中，普遍采用**集中式架构**，即由一个强大的协调器将用户请求路由到多个轻量级、领域专用的子智能体（sub-agent）。子智能体通常基于小型微调模型（如 Qwen3-32B-Instruct）以平衡性能与成本。
- **核心问题**：现有工作主要关注**协调器层的路由优化**（如 RIRS、MasRouter、成本感知编排），但**子智能体执行失败**以及由此引发的**信息沟通断裂**问题尚未被充分研究。子智能体由于缺乏对自身能力边界的校准，容易对模糊、不明确、路由错误或超出能力范围的请求**过度回答**，产生**幻觉输出**，而非给出可操作反馈。
- **动机**：近期 MAS 失败分类学研究（Cemri et al., 2025）表明，系统级失败多源于智能体间信息流断裂（如不寻求澄清、隐瞒信息、忽略其他智能体输入）。因此，子智能体的输出不仅是中间载荷，更是**下游恢复的通信信号**。将弃权（abstention）从面向用户的拒绝重构为面向协调器的**智能体间通信协议**，可提升 MAS 整体可靠性。

## 2. 方法论：核心思想与关键技术细节

EARS 框架包含两个阶段：

### 2.1 离线校准阶段
- **构建弃权分类体系（Taxonomy）**：基于失败模式将弃权分为四类：
  - **Ambiguous Query**：用户意图模糊不清。
  - **Insufficient Input**：意图明确但需要额外信息才能执行。
  - **Missing Capability**：请求功能不被子智能体支持。
  - **Misrouting**：请求应由其他子智能体处理。
- **人工标注种子集**：采用独立双重标注+仲裁流程，对 1202 条人机交互数据（600 条可解 + 602 条需弃权）进行标注，生成高质量种子集。
- **校准 LLM-as-a-Judge**：以种子集为基准，通过**人机协同迭代优化**法官模型的指令，使其在弃权分类上的准确率达到预设阈值（≥80%）。优化过程由第二个 LLM 分析失败案例并生成指南修订建议，避免过饱和自动优化。

### 2.2 在线数据飞轮阶段
- **大规模数据标注**：将校准后的法官模型应用于 13.8K 生产交互数据。为避免误标，采用**多模型集成策略**，同时使用 OpenAI GPT-5、Claude Sonnet 4.5、Gemini 2.5 Flash 及 o3 四个法官模型，仅当**全票一致**时接受标注（可调整精度-召回率权衡）。
- **有监督微调（SFT）**：利用标注后的训练集（含可解查询的 DSL 输出和需弃权查询的结构化响应）对子智能体进行**全参数微调**。每个训练样本包含用户查询、子智能体上下文以及预期响应（可解则输出 DSL；需弃权则输出弃权类别+理由）。训练完成后子智能体重新部署，收集新交互进入下一轮飞轮。

## 3. 实验设计

### 3.1 使用场景与数据集
- **场景**：大型电商企业的 AI 助手，支持商户关于店铺运营与管理的查询。采用**集中式协调器-子智能体架构**，协调器基于 Claude Sonnet 4.6，子智能体为**商业智能（BI）子智能体**（基于 Qwen3-32B-Instruct），负责回答数据驱动的性能指标分析查询，涵盖**客户细分**和**业务分析**两大能力。
- **数据集**：
  - 训练集：13.8K 从生产环境随机采样的交互（7.0K 客户细分 + 6.8K 业务分析查询），经自动标注得到 1,086 条弃权样本。
  - 评估集：4,000 条样本（1,524 客户细分 + 2,000 业务分析 + 476 弃权），以保证类别平衡。

### 3.2 基准（Baseline）
- **基线**：使用相同 Qwen3-32B-Instruct 模型，但**不包含弃权训练数据**的子智能体。即仅训练可解查询的 DSL 生成，无弃权能力。

### 3.3 对比方法与设置
- **LLM-as-a-Judge 模型对比**：比较 gpt-5、o3、claude-sonnet-4.5、gemini-2.5-flash 的弃权分类性能（准确率、精确率、召回率）及成本（相对 token 使用）。
- **集成策略对比**：单法官 vs. 4-法官全票共识。
- **数据来源消融**：人类标注数据 vs. EARS 自动标注数据 vs. 不同法官变体。
- **最终系统对比**：基线 vs. EARS（使用四法官全票共识标注数据微调）。

### 3.4 评估指标
- **弃权分类性能**：准确率、精确率、召回率。
- **成本**：以 gemini-2.5-flash 一次调用为 1，相对 token 消耗。
- **响应通过率（Pass Rate）**：分为两步：
  1. **语法有效性（Syntax）**：DSL 语法正确或弃权响应格式正确。
  2. **语义正确性（Semantic）**：由独立校准的 LLM-as-a-Judge（gpt-5）评估意图对齐、领域约束、逻辑正确性及弃权正确性。该法官经 1,050 条人工标注验证，Cohen’s κ=0.718，准确率 88.3%。
- **人工评估**：影子部署中随机采样 584 个产生弃权信号的会话，由领域专家对整体 MAS 响应质量进行二元评分（1-成功，0-失败），并评估子智能体弃权精确率。

## 6. 主要结论与发现

1. **EARS 显著提升端到端性能**：整体响应通过率从 68.5% 提升至 78.9%，相对提升 **15.2%**。
2. **弃权训练对不同任务影响不对称**：客户细分任务受益更大（语义通过率提升 33.6%），因其 23.8% 查询需弃权；业务分析查询弃权率 <2%，提升微小。
3. **多法官全票共识优于单法官**：4 法官共识在弃权分类上达到 93.1% 准确率，精确率 96.2%，显著高于单模型（最佳单模型 gemini-2.5-flash 为 90.4% 准确率）。在端到端通过率上，共识策略增益最明显（+4.5% 超过人类标注）。
4. **人类评估证实实践有效性**：在影子部署中，EARS 产生的弃权信号精确率 94.0%，端到端人类评分通过率 67.1%（基线在该子集仅 2.4%）。
5. **类别级改进显著**：尤其在“能力缺失”和“路由错误”场景，EARS 避免了基线因强行转化导致的误导性回复，并提供了可操作理由。

## 7. 优点

1. **问题定义创新**：将子智能体弃权重构为**智能体间通信协议**，而非简单的最终拒绝，填补了现有 MAS 可靠性研究重协调器轻子智能体的空白。
2. **可落地的生产级框架**：设计了完整的数据飞轮（人工种子 → 法官校准 → 大规模标注 → 微调 → 部署 → 新交互回收），适合持续迭代。
3. **多法官集成策略**：全票共识机制同时兼顾高精度（精确率 96.2%）和可调节阈值，避免低质量噪声影响训练。
4. **细粒度失败分类体系**：四类弃权标签（模糊、输入不足、能力缺失、路由错误）为协调器提供了明确的恢复信号，端到端评估表明该分类确实提升了后续回复质量。
5. **包含人类验证**：影子部署中独立评估了端到端响应质量和子智能体弃权精准率，增强了结论的可信度。
6. **消融实验设计清晰**：通过对比人类标注、不同法官、集成策略，揭示了噪声来源和共识的必要性。

## 8. 不足与局限

1. **领域覆盖单一**：仅验证了电商 BI 场景，金融、医疗、教育等领域的 abstention 模式可能不同，泛化能力未证明。
2. **依赖人工标注与校准**：法官校准需要人工种子集和迭代优化，成本高且可能引入主观偏差。论文自身也在 Limitations 中提出了对更自动化校准策略的期待。
3. **未探索协调器级优化**：EARS 只负责子智能体输出弃权信号，但协调器如何最佳利用这些信号（如冲突修复、回退策略）未被系统级优化。作者指出未来可结合 RL 训练协调器恢复策略。
4. **训练数据量有限**：弃权样本仅 1,086 条（训练集），且完全依赖法官全票共识产生，可能遗漏部分边缘情况。论文未讨论数据量对性能的边际效应。
5. **未比较其他弃权方法**：未与 Abstain-R1、LatentRefusal 等单智能体弃权方法直接对比（尽管这些方法面向不同设定，但可在 MAS 环境下适配）。
6. **计算开销**：全参数微调 32B 模型 + 四个 LLM-as-a-Judge 全票标注 + 人类校准，资源需求较高，中等规模团队可能面临门槛。论文未提供具体训练时长和总成本。
