## 摘要
现有的基于图的多智能体系统（MAS）设计主要通过优化预定义智能体、角色或群体的通信拓扑来改进协作。然而，由于每个节点仍然是闭集实体，这些方法难以泛化到需要未见能力组合的任务。我们提出SIGMA，一种技能关联图框架，将智能体构建为任务条件的可复用技能包。给定任务和技能库，SIGMA预测技能-智能体关联矩阵，从所选技能组合智能体节点嵌入，并在构建的智能体上解码通信拓扑。执行过程中，技能特定的邮箱将消息路由到相关分配的能力，使关联结构直接可操作。在六个推理和编码基准测试中，使用三个基础LLM，SIGMA达到了最佳平均性能，并分别比最强的非组合拓扑基线CARD提升了2.06、2.36和1.75分。它还对未见技能库表现出更强的鲁棒性，平均性能下降仅为0.96分。这些结果表明，在通信拓扑优化之外，组合式节点构建是多智能体设计的一个互补且重要的维度。代码可在 https://anonymous.4open.science/r/SIGMA-2338/ 获取。

## Abstract
Existing graph-based multi-agent system (MAS) designers mainly improve collaboration by optimizing communication topologies over predefined agents, roles, or groups. However, because each node remains a closed-set entity, these methods struggle to generalize to tasks that require unseen combinations of capabilities. We propose SIGMA, a skill-incidence graph framework that constructs agents as task-conditioned bundles of reusable skills. Given a task and a skill library, SIGMA predicts a skill-agent incidence matrix, composes agent node embeddings from selected skills, and decodes a communication topology over the constructed agents. During execution, skill-specific mailboxes route messages to the relevant assigned capabilities, making the incidence structure directly operational. Across six reasoning and coding benchmarks with three base LLMs, SIGMA achieves the best average performance and improves over CARD, the strongest non-compositional topology-based baseline, by 2.06, 2.36, and 1.75 points, respectively. It also shows stronger robustness to unseen skill libraries, with an average performance drop of only 0.96 points. These results suggest that compositional node construction is a complementary and important axis for multi-agent design beyond communication topology optimization. ## 1. 核心问题与整体含义（研究动机和背景）
- **现状与局限**：现有基于图的多智能体系统（MAS）设计主要通过优化预定义智能体（角色、群体）之间的通信拓扑来改进协作。但每个节点本质上是“闭集实体”，无法泛化到需要**未见能力组合**的任务——例如一个智能体槽无法根据任务不同而动态切换其核心能力（如工具使用、推理验证等）。

- **研究目标**：引入**技能组合（Compositionality）**作为新的设计维度，让智能体节点本身可被构建，而不仅仅是优化它们之间的通信。

## 2. 论文提出的方法论
### 核心思想
- 提出**SI-MAP协议**（Skill-Incidence Multi-Agent Protocol）：要求组合式LLM-MA设计满足四个特性——组合性（Compositionality）、可执行性（Executability）、拓扑兼容性（Topology compatibility）、可扩展性（Expandability）。
- 实例化为**SIGMA框架**：将可复用能力表示为技能卡片，预测任务条件化的**技能-智能体关联矩阵**，从所选技能组合节点嵌入，解码通信拓扑，并通过**技能特定邮箱**实现可执行的消息路由。

### 关键技术细节
- **技能卡片**：每个技能表示为 `s_m = (name, description, executable affordance, contract, source)`，由冻结编码器得到嵌入 `e_m`。
- **关联矩阵生成**：对每个技能 `s_m` 和智能体槽 `a_k`，计算 `α_{m,k} = F_θ([z_q, e_m, p_k, z_q ⊙ e_m])`，经由 sigmoid 得到软概率 `P_q[m,k]`，再通过稀疏投影得到硬矩阵 `B_q`（阈值+Top-k+重复修复）。
- **节点嵌入**：对每个槽，基于注意力加权组合技能嵌入：`u_{q,k} = Norm(p_k + z_q + Σ ω_{m,k} e_m)`。
- **拓扑解码**：结合节点亲和度 `r_{ij}` 与技能包兼容性 `c_{ij}`（跨智能体技能对互补性），加上弱工作流先验 `A_0`，得到边分数 `ℓ_{ij}`，阈值化为通信图。
- **邮箱执行**：每个关联对 `(m,k)` 拥有独立邮箱，消息通过cos相似度路由到最相关的技能邮箱，摘要后注入智能体提示。

### 算法流程（文字说明）
1. **离线训练**：用冻结编码器对任务查询和技能库编码；训练关联生成器 `F_θ` 和拓扑解码器 `D_{ϕ,ψ}`，使用确定性伪标签 `B_q^*` 作为监督，损失包括关联交叉熵、边重建损失（或锚拓扑）和稀疏惩罚。

- **评价**：实验覆盖不同模型尺度和任务类型，消融分离了各组件贡献，鲁棒性测试验证了泛化能力。匹配crew大小和上下文预算确保公平。不足：部分超参数（如 `k_s=3`）固定，未做系统调优；MMLU仅使用153题子集（虽与基线一致，但完整测试集更大）。

## 6. 论文的主要结论与发现
- SIGMA在所有三个基座上取得最高平均准确率（Qwen3-8B: 87.04, GPT-OSS-120B: 93.97, GPT-4o-mini: 88.80），分别比最强非组合基线CARD高出2.06、2.36、1.75分。
- 在未见技能库下，SIGMA性能下降最小（平均0.96分），远低于其他拓扑类方法（CARD 2.03, G-Designer 3.00, GPTSwarm 4.06）。
- 消融实验表明，关联生成、技能感知拓扑解码、邮箱路由、锚点先验均对性能有贡献；去除任一组件都会导致下降。
- 效率上，SIGMA用最少令牌达到最高准确率，相比GPTSwarm最高可减少96.7%令牌。
- 结论：组合式节点构建是补充通信拓扑优化的关键设计维度，有效多智能体设计应同时考虑“谁通信”和“能力如何组合”。

## 7. 优点
- **创新性**：首次提出技能关联图框架，将智能体构建从固定角色解放为可组合技能包，分离了“能力组合”与“拓扑生成”。
- **协议化设计**：SI-MAP协议清晰定义了四个必需属性，为后续研究提供规范。
- **可执行性**：技能邮箱机制使关联矩阵不仅用于节点构造，还用于消息路由，保证了关联的实际影响。
- **可扩展性**：新技能可以在测试时通过冻结编码器直接加入，无需重新训练拓扑解码器。
- **实验充分公平**：匹配crew大小、上下文预算，全面对比多种基线；消融和鲁棒性分析完善；每个基准均报告准确率与效率。
- **代码开源**：提供匿名代码仓库，促进可复现性。

## 8. 不足与局限
- **依赖技能库质量**：技能卡片的质量和覆盖范围直接影响结果，噪声卡片或缺失关键技能会导致错误。
- **伪标签偏差**：确定性伪标签和规范槽排序可能继承标记启发式的偏见。
- **固定crew大小**：当前假设固定智能体数量（K=5），尚未支持动态调整crew大小。
- **评估场景局限**：仅覆盖推理和编码基准，未涉及真实世界工具使用、更长时序、有副作用的任务。
- **可学习控制器较小**：虽然轻量高效，但可能限制复杂技能交互建模能力；全局最优分配未保证。
- **最终聚合问题**：案例分析显示，即使多数智能体给出正确推理，最终决策节点仍可能被少数有说服力的错误答案带偏（如MMLU案例E），需要更鲁棒的聚合策略。
- **未提及的计算细节**：虽然报告了GPU数量和训练时长，但未明确给出总训练时间（仅提到50 epochs），也未对比基线方法的训练成本。
