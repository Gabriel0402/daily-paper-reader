## 摘要
现代视觉-语言-动作模型通常难以泛化到新的设置（如改变相机视角或机器人形态），因为这些模型通常仅基于当前观测和语言指令进行条件处理。由于忽略了底层系统配置这一变量，这些模型隐含地假设了训练期间遇到的固定执行环境，导致任何新环境都需要进行数据密集型的微调。在本工作中，我们引入了情境世界建模框架，该框架将系统识别视为一个情境适应问题。ICWM使机器人策略能够从较短的历史自我生成且与任务无关的交互中自主推断出关键系统变量。与传统的利用示范来指定执行什么任务的情境学习不同，ICWM利用上下文窗口来理解系统如何运行。通过在任务执行前处理这些交互，模型隐式地捕获了当前系统的世界动力学，从而无需参数更新即可适应新配置。在仿真和真实世界机器人平台上的大量实验表明，ICWM在新型相机视角下显著优于标准VLA基线模型。

## Abstract
Modern Vision-Language-Action (VLA) models often fail to generalize to novel setups, such as altered camera viewpoints or robot morphologies, because they are typically conditioned only on current observations and language instructions. By ignoring the underlying system configuration as a variable, these models implicitly assume a fixed execution context encountered during training, necessitating data-intensive fine-tuning for any new environment. In this work, we introduce In-Context World Modeling (ICWM), a framework that treats system identification as an in-context adaptation problem. ICWM enables robot policies to autonomously infer essential system variables from a short history of self-generated, task-agnostic interactions. Unlike traditional In-Context Learning that uses demonstrations to specify what task to perform, ICWM leverages the context window to understand how the system operates. By processing these interactions before task execution, the model implicitly captures the world dynamics of the current system, enabling adaptation to novel configurations without parameter updates. Extensive experiments in simulation and on real-world robot platforms demonstrate that ICWM significantly outperforms standard VLA baselines on novel camera viewpoints.

## 2. 方法论：核心思想、关键技术细节

### 核心思想
- **In-Context World Modeling (ICWM)**：在任务执行前，机器人执行一小段任务无关的随机探索动作，记录对应的视觉-动作过渡元组（起始图像、动作、结束图像），将这些“自探交互片段”作为上下文拼接到任务查询之前，使VLA模型从上下文中隐式推断当前系统配置（ψ），进而产生正确的动作。

### 关键技术细节
- **系统配置的形式化**：将交互建模为部分可观测马尔可夫决策过程，状态分解为时间不变的系统配置ψ与时间变化的场景状态ξ。通过信息理论证明，交互上下文T = (o₀:ₜ, a₁:ₜ)比单一观测携带更多关于ψ的信息。
- **隐式系统推断函数Ψ(T)**：与VLA主干网络共享参数，利用Transformer处理上下文片段，提取动作-观测映射的因果结构。无需额外参数或专门的世界模型训练目标。
- **训练阶段**：在多样系统配置（如多种相机视角）的数据上训练，每个训练样本随机从同一配置的交互池中采样N个任务无关的交互片段作为前缀，模型需学习利用这些片段来预测任务动作，从而隐式学会世界建模。
- **测试阶段**：① **主动探测阶段**：机器人执行N次随机探测动作，记录过渡元组，构建交互上下文T；② **上下文执行阶段**：将T与当前观测和语言指令一起输入模型，单次前向传播中，Transformer先关注T生成配置感知的隐藏状态，再处理任务查询输出动作。

### 公式与算法流程（文字说明）
- 标准VLA策略：πθ(at | ot, l) → 忽略ψ
- 理想策略：πθ(at | ot, l, ψ) → 显式条件化

## 3. 实验设计

### 数据集/场景
- **仿真环境**：LIBERO benchmark（四个任务套件：Spatial, Object, Goal, Long），每个套件包含500个任务，共2000个训练轨迹。系统配置为相机方位角，训练使用8个视角，测试使用6个未见过的视角（OOD）。
- **真实机器人平台**：UR5e机械臂 + Robotiq 2F-85夹爪 + 12相机系统（6训练、6测试视角），四项操作任务（堆叠、放置、提升、抓取），每任务每新视角25次试验，共600次。

### Benchmark
- 跨视角泛化：在LIBERO上按照8个训练视角、6个测试视角进行标准评估；在真实机器人上同样进行训练-测试视角分离。
- 额外泛化测试：语义变化（干扰物、新台布纹理）；形态变化（末端加装刚性垫片、改变机械臂连杆长度）。

### 对比方法
- **内部控制比较**：
  - Multi-View BC (MV)：相同架构和多视角训练数据，但无上下文。
  - Explicit Configuration (EXP)：将真实相机角度作为文本输入加入MV。
- **上下文参考**：预训练的VLA模型（NORA, π-FAST, π0.5）在单一视角微调，作为外部参照，证明大规模预训练仍无法解决新颖视角泛化。

### 公平性保证
- 所有内部控制比较共享相同的多视角训练数据和架构，仅差异在于上下文的存在与否，从而隔离其贡献。
- 外部参考模型仅在单一视角微调，不作为直接基线，而是展示现有方法的局限性。

## 6. 主要结论与发现

1. **ICWM显著提升跨视角泛化**：在LIBERO上OOD成功率平均提升13.0%（相对MV基线），在真实机器人上提升71%（平均从17%到29%）。
2. **隐式系统识别优于显式配置**：ICWM比使用真实相机角度的EXP高出9.5%，证明交互上下文提供了更丰富的动力学信息。
3. **长时域任务收益最大**：在LIBERO-Long上ICWM提升幅度最大（+26.3% OOD），因上下文全局校准减少了累积误差。
4. **上下文组件不可或缺**：去除图像导致最大性能下降（-56.4%）；使用错误上下文比没有上下文更差，证明模型真正利用上下文进行推理。
5. **探测策略通用有效**：所有探测策略（随机、XY-only、Z-only、R-only）均优于无上下文基线，且不同策略在不同视角上各有优势。
6. **泛化到语义和形态变化**：ICWM在干扰物、新纹理、加长夹爪、变长连杆等多种OOD中均保持领先，且随差异增大优势更明显。
7. **隐式表示可识别**：t-SNE显示Ψ(T)在相同视角内聚类紧密，视角间分离清晰，说明模型学到与ψ结构化对应的表示。

## 7. 优点

- **方法创新**：将系统识别重定义为上下文适应问题，利用任务无关自我交互隐式推断动力学，无需参数更新、任务演示或额外模型参数，实用性强。
- **设计简洁有效**：不引入新模块（Ψ共享VLA参数），仅改变输入格式，与现有架构兼容。
- **实验全面严谨**：覆盖仿真、真实机器人、多种OOD条件（视角、语义、形态），消融实验设计精巧（包含反事实条件如错误上下文），论证充分。
- **理论支撑**：提供信息论证明，说明交互上下文比单帧观测携带更多系统配置信息。
- **实用性考虑**：分析推理延迟并提出KV缓存优化，考虑实际部署中重复使用上下文。

## 8. 不足与局限

- **实验覆盖有限**：
  - 视角泛化实验仅在LIBERO（仿真）和UR5e（真实）上进行，未涉及更复杂的操作任务（如灵巧手、移动操作）。
  - 语义变化仅限于干扰物和台布纹理，未评估物体置换、光照变化等。
  - 形态变化仅限于末端加长和连杆缩短，未测试不同机器人类型（如轮式、四足）。
- **识别困难视角**：仿真中135°视角对所有方法均表现很差，ICWM提升微弱，表明存在视角导致的感知瓶颈（如遮挡、视野丢失），该方法无法完全解决。
- **真实机器人探测阶段需要预定义工作空间**：虽然可以在机器人基坐标系中自动计算，但针对不同工作站仍需一次性配置，未实现完全自动化的通用探测。
- **推理延迟增加**：即使通过KV缓存优化，首次构建上下文仍需要额外时间（5-6秒探测+约0.2秒上下文处理），对于实时要求极高的场景可能构成挑战。
- **依赖自生成交互质量**：随机探测假设动作不会扰乱任务环境（避开物体），若探测动作造成意外碰撞或移动物体，可能导致上下文污染和后续失败。文中提到“确保探测工作空间避免接触任务相关物体”，但在实际复杂环境中可能难以保证。
- **未与在线适应方法比较**：没有与需要梯度更新的元学习、参数微调等在线适应方法对比，仅针对“无需参数更新”这一设定。因此不能判断ICWM是否优于那些方法。
- **模型规模固定**：使用3B参数级VLM，未探讨小模型是否也能从上下文世界建模中受益，或更大模型是否表现更好。
