## 摘要
视觉语言模型（VLM）智能体越来越多地部署在交互式游戏环境中。然而，针对VLM智能体的游戏基准通常报告每个（智能体，游戏）对的单次首次尝试分数，侧重于单智能体单人游戏，并且缺乏统一协议来公平地评估异质智能体类别（商业VLM、开源权重VLM和专用游戏策略）。我们通过OmniGameArena来填补这些空白，这是一个包含十二个新构建的虚幻引擎5游戏的实时基准，涵盖单人游戏（7个）、玩家对战（3个）和合作游戏（2个），具有统一的操作接口，以及改进动力学曲线（IDC），这是一种智能体反思框架，其中使用工具的反省型LLM自主地在多轮中优化有界技能提示。除了冷启动排行榜分数外，IDC还为每个（智能体，游戏）对揭示了两个额外的可观测指标：分数在反思轮次中的演变方式，以及所学技能在保留任务变体上的表现。我们报告了十二个VLM智能体在冷启动排行榜上的这些可观测指标，以及四个顶级智能体在IDC下的表现。

## Abstract
Vision-language model (VLM) agents are increasingly deployed in interactive game environments. Yet game benchmarks for VLM agents typically report a single first-attempt score per (agent, game) pair, focus on single-agent Solo play, and lack unified protocols for evaluating heterogeneous agent classes (commercial VLMs, open-weight VLMs, and specialized game policies) on the same footing. We address these gaps with OmniGameArena, a real-time benchmark of twelve newly built Unreal Engine 5 games spanning Solo (7), PvP (3), and Coop (2) with unified action interfaces, and the Improvement Dynamics Curve (IDC), an agentic-reflection harness in which a tool-using reflector LLM autonomously refines a bounded skill prompt across multiple rounds. Beyond cold-start leaderboard scores, IDC exposes two additional observables for each (agent, game) pair: how the score evolves across reflection rounds, and how the learned skill behaves on held-out task variants. We report these observables for twelve VLM agents on the cold-start leaderboard and four top agents under IDC.

    1.  **单次评分**：仅报告每个（智能体，游戏）对的单次首次尝试得分，忽略了智能体通过反复交互实现自我改进的动态过程。

    3.  **评估不统一**：缺乏统一的协议来公平地比较异质性智能体类别，例如商业VLM、开源VLM和专门训练的专用游戏策略。

- **整体含义**：现有基准无法全面衡量VLM智能体在真实部署环境中的关键能力，即其“自我改进”的动态过程以及在多种复杂互动模式下的泛化能力。

## **2. 论文提出的方法论：核心思想与关键技术细节**

论文提出了一个**双组件**系统来解决问题：

### **A. OmniGameArena：统一的游戏基准平台**

- **核心思想**：构建一个全新的、统一的、跨模式的游戏测试环境，以公平评估各类VLM智能体。
- **关键技术细节**：
    - **游戏构建**：在Unreal Engine 5 (UE5) 中新开发了12款游戏，涵盖 7个单人（Solo）、3个玩家对战（PvP）和2个合作（Coop）场景。
    - **避免数据泄漏**：所有游戏均为该基准测试专有构建，使用自定义资产和逻辑，有效降低了智能体从预训练数据中“记住”解决方案的风险（通过污染分析验证，0%被识别）。
    - **统一动作接口**：为所有智能体提供统一、标准化的键盘-鼠标或游戏手柄动作接口，确保在相同的环境条件下进行评估。
    - **标准化评估指标**：所有游戏的进度分数被统一归一化到 [0, 1] 的连续值上，便于跨任务比较。

### **B. Improvement Dynamics Curve (IDC)：改进动力学曲线框架**

- **核心思想**：在基准测试之上，引入一个智能体反思框架，让智能体通过多轮交互和自我反思来优化其行为策略，从而生成一条能反映其改进动态的“曲线”而非单个“点”。
- **关键技术细节（IDC流水线）**：
    1.  **经验获取模块**：智能体在当前的技能提示（skill prompt）指导下，执行 `K` 个回合（episodes），生成轨迹和回合分数。
    2.  **反思模块**：一个作为“反省者”的LLM（通常与玩家VLM相同）自主地分析和优化技能提示。它包含四个阶段：
        - **探索**：使用沙盒化只读工具（如读取文件、图片）自主检查本轮轨迹，而非执行固定脚本。
        - **诊断**：提交明确的失败模式列表，区分原因和解决方案。
        - **验证**：提出更新的技能提示，并由独立的LLM裁判（judge）进行验证，防止技能记忆地图内容或与诊断矛盾。可迭代最多5次。
        - **提炼**：最终确定新的技能提示，并可选地更新经验笔记本。
    3.  **持续模块**：跨轮次维护三个状态信息：
        - **经验笔记本**：由反省者写入的事实性日志（上限2000 tokens）。
        - **已验证技能**：当前轮次使用的技能提示（上限1200 tokens）和最佳技能缓存。
        - **改进曲线**：所有轮次的分数序列。
    4.  **最佳技能回滚**：当新一轮分数急剧下降时，自动回滚至历史最佳技能提示，防止灾难性的技能漂移。

- **产出**：经过R轮迭代，生成一个分数序列 [S0, S1, ..., SR]，即改进动态曲线。这比单一分数能揭示更多信息，如收敛速度和稳定性。

## **3. 实验设计**

- **数据集/场景**：
    - **基准测试**：OmniGameArena的12个UE5游戏，包括7个单人游戏（如障碍跑、生存射击）、3个PvP游戏（如1v1对决、攻防战）和2个合作游戏（如协同配送）。
    - **IDC实验**：专注于2个游戏进行深入分析：
        - **LastStand（单人）**：考验反应和避险能力。
        - **SharedFloor（合作）**：考验协同规划与分工能力。
    - **IDC迁移实验**：每个游戏额外设计了3个保留任务变体（如改变地图种子、修改掉落机制），以测试所学技能的泛化能力。
- **对比方法（智能体）**：共评估了12个VLM智能体，分为三类：
    - **商业VLM**：Claude Opus 4.7/4.6, Sonnet 4.6；GPT-5.5/5.4；Gemini 3.1 Pro/Flash-Lite Preview；Kimi K2.5。
    - **开源VLM**：Qwen3.5 (397B-A17B 和 122B-A10B)。
    - **专用游戏策略**：NitroGen (游戏手柄策略)，Open-P2P (键盘鼠标策略)。

- **评估协议**：
    - **冷启动排行榜**：所有智能体在12个游戏上运行，每个（智能体，游戏）对执行5个回合，使用**暂停决策质量（PDQ）** 模式（环境在推理时暂停）。
    - **延迟控制实验**：对部分模型进行**延迟控制实时（LCRT）** 评估，模拟真实环境中的推理延迟影响。
    - **IDC实验**：选择4个顶级智能体（Claude 4.6/4.7, GPT-5.5, Gemini 3.1 Pro），在2个目标游戏上运行10轮IDC（每轮5个回合），并将最佳技能迁移到3个变体任务上。

## **6. 论文的主要结论与发现**

- **冷启动排行榜**：
    - 无单一模型在所有任务中占主导地位，领导地位因游戏而异。
    - 更新的模型版本并不总是更好（如Claude Opus 4.6在某些任务上优于4.7）。
    - 开源VLM和专用游戏策略的商业化迁移能力很差，在多数任务上得分接近0。
- **PvP游戏**：
    - 多数游戏展现出清晰的“强者恒强”的等级体系，与Solo排行榜大致一致。
    - 但在像MidlineClash这样的任务中，出现了“非传递性”的对抗结果（如Kimi K2.5能稳定击败更强的Claude Opus 4.6），表明特定游戏需要特定战术。
- **合作游戏**：
    - 商业VLM与开源VLM之间的差距进一步扩大，开源模型在所有合作任务上得分为零。
    - 即使是表现最好的模型，合作效率也远未饱和，脑与脑之间的协调是一个未解决的关键能力缺口。
- **IDC实验**：
    - 所有顶级智能体都能通过IDC的多轮反思实现自我提升，但**峰值性能通常出现在曲线中部，而不是最后一轮**。这证明了最佳技能回滚机制的必要性。
    - **原始任务提升与变体迁移能力存在显著差异**。例如，Claude Opus 4.7在原始任务上改进巨大（+201%），但在所有变体任务上的迁移都呈负收益；而GPT-5.5原始增益较小（+130%），但成功在所有变体上实现了正迁移。这表明，**单轮排行榜分数完全无法揭示技能的风格和泛化性**。

## **7. 优点**

1.  **创新性基准**：OmniGameArena通过自建游戏有效解决了数据污染问题，并首次在一个平台上系统性地覆盖了Solo、PvP和Coop三种交互模式。
2.  **深度评估框架**：IDC的提出是论文的核心贡献。它将评估从静态的“点”扩展到动态的“曲线”，揭示了智能体改进的动力学过程，极大提升了评估的信息量。
3.  **统一性与公平性**：为不同类型的智能体（商业、开源、专用）设计了统一的接口和评估协议，使得跨类别比较成为可能且更具说服力。
4.  **洞察深刻**：实验设计巧妙，特别是原始任务提升与变体迁移能力的对比实验，深刻揭示了当前VLM智能体技能学习的本质：可能只是学到了一种“记忆”式的策略，而非真正的适应性和泛化能力。

## **8. 不足与局限**

1.  **IDC范围有限**：如作者所述，IDC实验仅覆盖了两个游戏和四个智能体，其结论（如峰值出现时机、转移能力差异）在更大范围的游戏和模型上是否普适，有待验证。
2.  **单技能格式**：IDC使用单一有界技能提示，而非一个不断增长的技能库。复杂的游戏可能需要更复杂的技能组织方式（如Voyager的做法）。
3.  **反省器与玩家一致**：反省者和玩家是同一个模型，可能会限制反省的客观性。使用一个更强或不同架构的模型作为反省者可能会带来更好的改进效果，但这一点未被测试。
4.  **计算资源信息缺失**：论文未报告运行这些实验所耗费的具体计算资源和时长，这对于其他研究者复现和评估成本构成障碍。
