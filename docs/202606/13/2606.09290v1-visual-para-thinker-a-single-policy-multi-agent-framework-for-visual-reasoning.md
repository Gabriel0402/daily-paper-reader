## 摘要
视觉推理需要整合分布在区域、属性和关系上的证据，这使得单链推理容易过早感知承诺并产生幻觉。我们提出Visual Para-Thinker++，一种单策略多智能体框架，其中共享的MLLM策略被实例化为角色条件化的主智能体、工作智能体和总结智能体。主智能体通过固定分配模式分解任务；工作智能体在上下文隔离下并行推理；总结智能体协调完整的工作智能体推理轨迹，而不是对最终标签进行多数投票。共享策略通过多智能体能力注入和角色解耦多智能体优化进行训练，为相应的token片段分配特定角色的奖励和优势，以减少协作角色之间的梯度冲突。原生推理引擎通过共享视觉前缀和KV缓存重用实现高效的多智能体展开。在V*、CountBench、RefCOCO系列和HallusionBench上，Visual Para-Thinker++一致优于单轨迹和推理时间并行基线，在易产生幻觉的视觉推理上尤其有显著提升。

## Abstract
Visual reasoning requires integrating evidence distributed across regions, attributes, and relations, making single-chain reasoning prone to early perceptual commitment and hallucination. We propose Visual Para-Thinker++, a single-policy multi-agent framework in which one shared MLLM policy is instantiated as role-conditioned Main, Worker, and Summary Agents. The Main Agent decomposes the task with fixed allocation patterns; Worker Agents reason in parallel under context isolation; and the Summary Agent reconciles full Worker reasoning traces rather than majority-voting on final labels. The shared policy is trained by Multi-Agent Capability Injection and Role-Decoupled Multi-Agent Optimization, which assign role-specific rewards and advantages to corresponding token segments to reduce gradient conflict among collaborative roles. A native inference engine enables efficient multi-agent rollout through shared visual prefix and KV cache reuse. Across V*, CountBench, the RefCOCO family, and HallusionBench, Visual Para-Thinker++ consistently outperforms single-trajectory and inference-time parallel baselines, with especially strong gains on hallucination-sensitive visual reasoning.

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **核心问题**：视觉推理需要跨区域、属性和关系整合证据，但传统的单链思维（single-chain reasoning）容易过早锁定一种感知解释（early perceptual commitment），并且延展推理链条往往只是加深而非纠偏，导致幻觉（hallucination）频发。现有测试时扩展策略（如长CoT、自一致性）在视觉领域收益有限，因为单链无法有效并行探索互补证据。
- **研究动机**：作者认为，与其让一个智能体的链条变长，不如将视觉推理组织为多角色协作——由不同角色（问题分解、局部证据收集、全局计数验证、中间结果调和）分别承担。为此提出 Visual Para-Thinker++，一种**单策略多智能体**框架。
- **整体含义**：通过共享一个多模态大语言模型（MLLM）策略，实例化为角色条件化的主智能体（Main Agent）、多个工作智能体（Worker Agents）和总结智能体（Summary Agent），实现并行推理与轨迹级调和，从而缓解幻觉、提升多维度视觉推理能力。

## 2. 论文提出的方法论：核心思想、关键技术细节

### 核心思想
- **单策略多智能体（Single-Policy Multi-Agent）**：所有智能体共享同一组参数 θ，仅通过角色 token（role tokens）和可见上下文区分角色身份。无需训练多个独立模型。
- **固定四路径协作协议**：主智能体 -> K 个工作智能体并行推理 -> 总结智能体轨迹调和。

### 关键技术细节
1. **角色定义**：
   - **主智能体（Main Agent）**：接收图像和问题，根据两种固定任务分配模式（块式 Block-based 或扫描顺序 Scan-order）将任务分解为 K 个子问题，分发给工作智能体。
   - **工作智能体（Worker Agents）**：在上下文隔离下并行执行各自子任务（彼此看不到对方的 token），收集局部证据。
   - **总结智能体（Summary Agent）**：读取所有工作智能体的完整推理轨迹，进行轨迹级证据调和，输出最终答案。

2. **两种固定分配模式**（默认 K=4）：
   - **块式分配**：将图像划分为四个不相交象限（或自适应分割），每个工作智能体负责一个区域，适用于细粒度感知、属性验证等。
   - **扫描顺序分配**：每个工作智能体保持全局视野但按不同扫描顺序（左→右、右→左、上→下、下→上）遍历图像，适用于计数和全局验证。

3. **两阶段训练优化**：
   - **第一阶段：多智能体能力注入（Multi-Agent Capability Injection, MAC）**：使用教师模型生成的轨迹进行角色感知的监督微调（SFT）。损失函数包含主、工作、总结三个部分的交叉熵，并通过上下文隔离掩码（context-isolation mask）确保工作智能体互不可见，而总结智能体视野完整。
   - **第二阶段：角色解耦多智能体优化（Role-Decoupled Multi-Agent Optimization, RDMAO）**：引入角色特定奖励和优势。
     - **工作智能体奖励** R_worker：基于交叉多数投票（cross-Worker majority-vote）对局部推理质量打分。
     - **整体结果奖励** R_out：基于总结智能体最终答案是否匹配 GT。
     - **角色解耦优势**：对每个奖励源分别进行组归一化得到 A_out 和 A_worker，然后按 token 段组合：主和总结 token 使用 A_out，工作 token 使用 A_out + λ·A_worker（λ=0.5）。这避免了梯度冲突，使工作智能体局部信号不污染主和总结梯度。
     - 优化使用 DAPO（组相对策略梯度 loss）。

4. **高效推理引擎**：
   - 基于 vLLM 设计，实现共享视觉前缀的 KV 缓存重用。主智能体预填一次后，通过 PagedAttention 的写时复制（Copy-on-Write）分叉出 K 个工作序列并行解码，最后合并 KV pages 供总结智能体使用，避免冗余预填，大幅提升吞吐量。
