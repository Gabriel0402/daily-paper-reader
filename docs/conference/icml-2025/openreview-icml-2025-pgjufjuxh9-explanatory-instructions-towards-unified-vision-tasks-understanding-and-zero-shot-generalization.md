tldr: 计算机视觉模型缺乏零样本任务泛化能力，归因于离散的术语化任务定义。本文提出解释性指令（Explanatory Instructions），用自然语言描述任务以替代术语，从而统一视觉任务理解。实验证明该方法在多种视觉任务（包括图像分割）上实现了零样本泛化，为视觉基础模型的通用性提供了新视角。

## Abstract
Computer Vision (CV) has yet to fully achieve the zero-shot task generalization observed in Natural Language Processing (NLP), despite following many of the milestones established in NLP, such as large transformer models, extensive pre-training, and the auto-regression paradigm, among others. In this paper, we rethink the reality that CV adopts discrete and terminological task definitions (e.g., "image segmentation"), and conjecture it is a key barrier that hampers zero-shot task generalization. Our hypothesis is that without truly understanding previously-seen tasks—due to these terminological definitions—deep models struggle to generalize to novel tasks. To verify this, we introduce Explanatory Instructions, which provide an intuitive way to define CV task objectives through detailed linguistic transformations from input images to outputs. We create a large-scale dataset comprising 12 million "image input $\to$ explanatory instruction $\to$ output" triplets, and train an auto-regressive-based vision-language model (AR-based VLM) that takes both images and explanatory instructions as input. By learning to follow these instructions, the AR-based VLM achieves instruction-level zero-shot capabilities for previously-seen tasks and demonstrates strong zero-shot generalization for unseen CV tasks.  Code and dataset will be open-sourced.

## 3. 实验设计

- **使用数据集**：
  - **DECVT**：1200万三元组，含术语型任务（AllWeather、NYU-Rain、D-HAZE、UIEB、LVIS、ADE20K等）和解释型任务（MagicBrush、SEED-Data-Edit、HQ-Edit、InstructPix2Pix等及搜索图像对）。
  - **评估benchmark**：MultiGen-20M（可控生成）、Places（inpainting/outpainting）、NYU-Depth V2、ADE20K、Rain100L、SOTS、UDC(T-OLED)、LOLv2等。
- **对比方法**：
  - 可控生成：ControlNet、T2I-Adapter、OmniGen、PixWizard、Lumina-mGPT。
  - 密集预测/低层：DepthAnything、Marigold、Mask DINO、Mask2Former、Bae et al.、InvPT、AirNet、PromptIR、Unified-IO、Painter、InstructCV、InstructDiffusion、PixWizard。
  - 图像编辑：InstructPix2Pix、MagicBrush、PnP、Null-Text Inv.、Emu Edit、UltraEdit等。
- **实验设置**：
  - **指令级零样本**：用全部DECVT训练，测试时使用未出现在训练集中的新解释性指令（如“将水元素变成火元素”），验证模型对新指令的理解能力。
  - **任务级零样本**：从训练集中排除某些术语型任务（如Canny-to-Image、Depth-to-Image、HED-to-Image等），评估模型对这些未见任务的泛化能力。

## 4. 资源与算力

- **全量训练**（Sec 4.1）：在64块A100 GPU上训练，每GPU batch size=8，共2个epoch（约47k迭代），总计**5,400 GPU小时**。图像分辨率448×448。
- **任务级零样本快速验证**（Sec 4.2）：在8块A100 GPU上训练，batch size=128，2个epoch（约46k迭代），总计**1,340 GPU小时**。图像分辨率448×448。
- **初始化模型**：Lumina-mGPT-7B-768-Omni（全量）或Lumina-mGPT-7B-768（快速验证）。

## 5. 实验数量与充分性

- **实验组数**：
  - 定量实验（表1）：4种生成任务（Canny-to-Image、HED-to-Image、Inpainting、Outpainting），对比6+方法。
  - 表2：7种密集预测/低层任务（深度估计、分割、表面法线、去雨、去雾、UDC恢复、低光增强），对比10+方法。
  - 附录B：图像编辑（MagicBrush、Emu Edit测试集），对比8种方法。
  - 附录C：大量定性示例（指令级与任务级零样本，涉及去雪、去模糊、边界检测等）。
- **消融实验**：论文主要聚焦于验证解释性指令的零样本能力，未做典型消融（如不同数据比例、不同指令格式等）。但通过排除任务（Sec 4.2）验证了任务级泛化，可视为一种消融。
- **充分性**：实验覆盖了多种视觉任务类型（生成、预测、恢复），且同时验证了指令级和任务级零样本。但缺少与术语指令训练的对比（如仅用术语指令训练模型能否零样本？）；此外，任务级零样本只在少数生成任务（Canny-to-Image等）上成功，范围有限。整体上实验设计合理，但客观性因模型性能仍落后于专用方法而受限。

## 6. 主要结论与发现

- **解释性指令可以有效替代术语化任务定义**：模型通过学习自然语言描述的图像变换，能够理解任务真实目标，实现指令级零样本（即从未见过的指令也能正确执行）。
- **首次在CV中展示任务级零样本泛化**：在Canny-to-Image、Depth-to-Image、HED-to-Image、Inpainting、Outpainting等未见任务上，仅通过解释性指令即可完成生成，无需任何任务微调。
- **相比术语指令的优势**：术语指令（如“Semantic segmentation”）导致模型只能处理固定任务，而解释性指令的连续语义空间使任务边界模糊，从而促进泛化（通过BERT特征可视化验证）。
- **局限**：任务级零样本泛化目前仅对生成任务有效，反向任务（Image-to-Canny、Image-to-Depth）失败；模型定量结果仍弱于专用模型和部分通用模型。

## 7. 优点

- **创新性强**：首次系统性地用自然语言详细描述视觉任务变换（而非任务名称）并验证零样本泛化，为视觉任务统一提供了新范式。
- **数据集规模大、覆盖广**：DECVT包含12M三元组，覆盖术语型和解释型两类，双向变换描述增加了任务多样性。
- **实验设计清晰**：明确区分指令级和任务级零样本，通过排除特定任务来严格验证任务泛化能力，具有说服力。
- **方法简洁有效**：采用vanilla token-based VLM，未引入复杂结构，凸显了解释性指令本身的价值。

## 8. 不足与局限

- **任务级零样本范围有限**：仅在可控生成（Canny/HED/Depth-to-Image）及一些恢复任务上成功，对反向任务（图像到边缘/深度）失败。作者分析可能是因为图像tokenizer与文本模态未充分对齐。
- **性能不如专用模型**：定量指标（FID、PSNR等）低于ControlNet、PixWizard等，说明零样本泛化以性能为代价。
- **数据噪声与质量**：GPT-4o生成的解释性指令存在不完整或偏差，影响模型稳定性和保真度。
- **模型稳定性不足**：通过语言控制增强强度（如低光增强）效果不稳定；部分指令导致图像出现变形。
- **仅限图像对**：未探索视频、3D等复杂数据，且GPT-4o对视频描述错误率高，限制了扩展性。
- **缺乏消融实验**：未对比不同数据比例、不同指令详细程度对泛化的影响，未对比仅用术语指令训练的结果。
- **计算资源消耗大**：7B模型仍需数千GPU小时，可能限制可重复性。
