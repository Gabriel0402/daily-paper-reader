## 摘要
视觉-语言-动作（VLA）模型有望实现通用机器人操纵，但通常作为短时界策略进行训练和部署，假设最新的观测足以进行动作推理。这一假设在非马尔可夫长时界任务中失效，因为任务相关的证据可能被遮挡或仅出现在轨迹早期，且杂乱环境和干扰项使得细粒度视觉定位变得脆弱。我们提出了 CodeGraphVLP，这是一个分层框架，通过将持久的语义图状态与可执行的基于代码的规划器以及进度引导的视觉语言提示相结合，实现了可靠的长时界操纵。语义图在部分可观测性下维护任务相关的实体和关系。合成的规划器在语义图上执行，以进行高效的进度检查，并输出子任务指令及子任务相关的对象。我们利用这些输出构建抑制杂乱的观测，使 VLA 执行器专注于关键证据。在真实世界的非马尔可夫任务中，CodeGraphVLP 相比强大的 VLA 基线和支持历史记录的变体提高了任务完成率，同时与 VLM 在环规划相比显著降低了规划延迟。我们还进行了广泛的消融研究，以确认各组件的贡献。

## Abstract
Vision-Language-Action (VLA) models promise generalist robot manipulation, but are typically trained and deployed as short-horizon policies that assume the latest observation is sufficient for action reasoning. This assumption breaks in non-Markovian long-horizon tasks, where task-relevant evidence can be occluded or appear only earlier in the trajectory, and where clutter and distractors make fine-grained visual grounding brittle. We present CodeGraphVLP, a hierarchical framework that enables reliable long-horizon manipulation by combining a persistent semantic-graph state with an executable code-based planner and progress-guided visual-language prompting. The semantic-graph maintains task-relevant entities and relations under partial observability. The synthesized planner executes over this semantic-graph to perform efficient progress checks and outputs a subtask instruction together with subtask-relevant objects. We use these outputs to construct clutter-suppressed observations that focus the VLA executor on critical evidence. On real-world non-Markovian tasks, CodeGraphVLP improves task completion over strong VLA baselines and history-enabled variants while substantially lowering planning latency compared to VLM-in-the-loop planning. We also conduct extensive ablation studies to confirm the contributions of each component.

### 1. 核心问题与整体含义
*   **研究背景**：现有的视觉-语言-动作（VLA）模型（如 RT-2, OpenVLA）大多基于**马尔可夫假设**，即认为当前的单帧观察足以决定下一步动作。
*   **核心痛点**：在真实的长程（Long-horizon）任务中，该假设会失效。原因包括：
    *   **非马尔可夫依赖**：正确的动作往往取决于过去发生的事情（例如：物体被遮挡后需记住其位置，或需记住已完成的步骤）。
    *   **环境干扰（Clutter）**：背景中的无关物体会干扰 VLA 模型的视觉定位，导致动作错误。
    *   **规划延迟**：使用大视觉语言模型（VLM）进行实时在线规划会导致极高的推理延迟，难以满足闭环控制需求。
*   **研究动机**：提出一种既能维持持久记忆、又能高效处理复杂逻辑、且对环境干扰鲁棒的层次化 VLA 框架。

### 2. 方法论
CodeGraphVLP 结合了三种核心技术：
*   **持久化语义图状态 (Persistent Semantic-Graph State)**：
    *   系统维护一个图 $G_t = (V_t, E_t)$，节点代表物体（含类别、掩码、属性），边代表空间关系（如“在...之内”、“在...之上”）。
    *   利用实例分割（YOLOE）和 VLM 进行初始化，并使用视频物体跟踪器（Cutie）在运行中实时更新图状态，从而在物体被遮挡时仍能保留其信息。
*   **代码即规划器 (Code-as-Planner)**：
    *   利用 LLM（如 GPT-4o/5）根据任务指令一次性生成一段 Python 代码。
    *   该代码作为高层规划器，通过查询语义图（而非原始图像）来检查任务进度，并输出当前的子任务指令及相关物体列表。这种方式避免了重复调用昂贵的 VLM。
*   **进度引导的去干扰提示 (Clutter-Free Visual-Language Prompting)**：
    *   **视觉层面**：根据规划器输出的相关物体，利用语义图掩码将图像中无关区域涂黑（Masking），只给底层 VLA 提供“干净”的观察。
    *   **语言层面**：将长程指令分解为简短的子任务指令（如“拿起黑杯子”），降低 VLA 的理解难度。

### 3. 实验设计
*   **实验场景**：真实世界的桌面机器人操纵（使用 UR10e 机械臂）。
*   **Benchmark 任务**：
    1.  **Pick-and-Place Twice**：要求将方块移到另一个盘子再移回来（测试对历史进度的记忆）。
    2.  **Place-and-Stack**：将方块放入杯中（方块随后被遮挡）并叠放另一个杯子（测试对遮挡物体的推理）。
    3.  **Swap Cups**：利用中转位交换两个杯子的位置（测试复杂逻辑规划）。
*   **对比方法**：
    *   主流 VLA 基线：$\pi_0$, $\pi_0$ FAST, $\pi_0.5$, Gr00T N1.5。
    *   历史增强变体：增加多帧历史输入的 Gr00T N1.5。
    *   消融实验：对比了有无语义图、有无代码规划器、有无视觉去干扰提示的效果。

### 4. 资源与算力
*   **算力设备**：使用了 **4 块 NVIDIA A6000 GPU**。
*   **训练细节**：
    *   使用 LoRA（低秩自适应）技术对 VLA 模型进行微调。
    *   训练迭代次数为 50,000 次。
    *   Batch size 为 128，学习率为 $1 \times 10^{-5}$。
*   **推理性能**：推理在单块 GPU 上运行，代码规划器的延迟极低（约 0.3s），远优于 VLM 在线规划（约 3s）。

### 5. 实验数量与充分性
*   **实验规模**：收集了 400 段高质量的人类演示视频作为训练数据。
*   **充分性评价**：
    *   实验设计非常具有针对性，专门挑选了传统 VLA 难以处理的非马尔可夫场景。
    *   对比了当前最先进的多个开源 VLA 模型，确保了基线的强度。
    *   消融实验完整，分别验证了语义图、代码规划和视觉过滤的独立贡献，逻辑严密且公平。

### 6. 主要结论与发现
*   **性能大幅提升**：CodeGraphVLP 在所有任务上的成功率均显著高于基线，平均成功率达到 **81.7%**，而最强基线仅为 56.7%。
*   **规划效率极高**：通过将 LLM 推理转化为代码执行，规划延迟降低了约 **90%**，使闭环控制更加流畅。
*   **视觉过滤的必要性**：实验证明，仅仅提供子任务指令是不够的，通过掩码去除视觉干扰项能将成功率从 40% 提升至 85%。

### 7. 优点
*   **架构优雅**：巧妙地将符号化推理（代码/图）与神经网络执行（VLA）结合，取长补短。
*   **解决长程痛点**：显式地处理了遮挡和历史依赖问题，这是目前端到端 VLA 模型的通用弱点。
*   **实用性强**：低延迟特性使其非常适合部署在真实机器人硬件上。

### 8. 不足与局限
*   **感知依赖性**：系统的可靠性高度依赖于前端的物体检测和跟踪算法。如果初始物体识别出错，后续的代码执行将“步步皆错”。
*   **代码生成的鲁棒性**：虽然 LLM 生成代码能力很强，但仍存在生成错误逻辑或不可执行代码的风险，文中缺乏对代码自动纠错机制的深入讨论。
*   **场景泛化限制**：目前实验集中在桌面操纵，对于更复杂的开放世界环境（如物体种类极多、动态变化剧烈）的适应性尚待验证。
