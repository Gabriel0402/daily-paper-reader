## 摘要
语言模型（LLMs）已成为具身控制的强大推理引擎。特别是，上下文学习（ICL）使现成的纯文本 LLM 能够在无需任何任务特定训练的情况下预测机器人动作，同时保留其泛化能力。将 ICL 应用于双臂操作仍具挑战性，因为高维联合动作空间和紧密的手臂间协调约束会迅速使标准上下文窗口过载。为解决这一问题，我们引入了 BiCICLe（双臂协调上下文学习），这是首个使标准 LLM 能够在无需微调的情况下执行少样本双臂操作的框架。BiCICLe 将双臂控制建模为多智能体主从问题，将动作空间解耦为顺序的、有条件的单臂预测。这自然地扩展到了“手臂辩论”（Arms' Debate）——一个迭代优化过程，并引入了第三个 LLM 作为裁判（LLM-as-Judge）来评估并选择最合理的协调轨迹。在 TWIN 基准测试的 13 个任务中进行的评估显示，BiCICLe 实现了高达 71.1% 的平均成功率，比表现最好的免训练基准高出 6.7 个百分点，并超过了大多数监督学习方法。我们进一步展示了在全新任务上强大的少样本泛化能力。

## Abstract
Language Models (LLMs) have emerged as powerful reasoning engines for embodied control. In particular, In-Context Learning (ICL) enables off-the-shelf, text-only LLMs to predict robot actions without any task-specific training while preserving their generalization capabilities. Applying ICL to bimanual manipulation remains challenging, as the high-dimensional joint action space and tight inter-arm coordination constraints rapidly overwhelm standard context windows. To address this, we introduce BiCICLe (Bimanual Coordinated In-Context Learning), the first framework that enables standard LLMs to perform few-shot bimanual manipulation without fine-tuning. BiCICLe frames bimanual control as a multi-agent leader-follower problem, decoupling the action space into sequential, conditioned single-arm predictions. This naturally extends to Arms' Debate, an iterative refinement process, and to the introduction of a third LLM-as-Judge to evaluate and select the most plausible coordinated trajectories. Evaluated on 13 tasks from the TWIN benchmark, BiCICLe achieves up to 71.1% average success rate, outperforming the best training-free baseline by 6.7 percentage points and surpassing most supervised methods. We further demonstrate strong few-shot generalization on novel tasks.

### 1. 核心问题与整体含义（研究动机和背景）
*   **核心问题**：如何利用大语言模型（LLM）的上下文学习（In-Context Learning, ICL）能力，在**无需任何任务特定训练或微调**的情况下，实现复杂的双臂机器人协调操作。
*   **研究背景**：双臂操作比单臂控制困难得多，其联合动作空间随维度指数增长（双臂通常为 14 自由度），且手臂间存在极强的时空协调约束。现有的 ICL 方法在单臂任务上表现良好，但在处理双臂任务时，由于动作序列过长、复杂度过高，容易导致 LLM 预测崩溃或手臂动作不协调。

### 2. 方法论：BiCICLe 框架
BiCICLe（Bimanual Coordinated In-Context Learning）的核心思想是将双臂控制解耦为多智能体协作问题。
*   **主从解耦（Leader-Follower Decomposition）**：
    *   **Leader 预测**：将双臂演示数据剥离为单臂格式，Leader 智能体首先根据场景观测预测其完整的动作轨迹（$Z^7$ 空间）。
    *   **Follower 预测**：Follower 智能体在接收到场景观测的同时，还将 Leader 的预测计划作为额外的上下文输入，从而在降低预测维度的同时实现显式的动作对齐。
*   **手臂辩论（Arms’ Debate）**：一种迭代优化策略。Leader 根据 Follower 的初步计划重新规划，Follower 再根据优化后的 Leader 计划进行最终调整。这种“状态无关”的迭代避免了长对话历史导致的上下文漂移。
*   **Best-of-N 与 LLM 裁判（LLM-as-Judge）**：利用 LLM 生成的随机性，并行生成 5 组候选轨迹，由第三个 LLM 充当裁判，根据碰撞风险、目标匹配度、抓取逻辑和工作空间限制进行评分，选出最优方案。
*   **表示技术**：
    *   **空间体素化**：将工作空间划分为 $100^3$ 的体素网格，将连续坐标映射为整数索引。
    *   **动作离散化**：将 6D 位姿和夹持器状态转化为 7 元组整数序列。
    *   **文本序列化**：将物体位置和动作序列转化为 LLM 可理解的文本提示词。

### 3. 实验设计
*   **数据集与场景**：主要在 **TWIN 基准测试**（基于 RLBench 的双臂扩展版）上进行，包含 13 个具有挑战性的双臂操作任务（如：抬箱子、递交物品、扫地、拉绳子等）。
*   **对比方法**：
    *   **免训练基准**：RoboPrompt（单智能体/独立双智能体）、KAT（基于视觉特征点）、VLM-LF（基于视觉语言模型）。
    *   **监督学习方法**（作为上限参考）：ACT、PerAct 2、$\pi_0$-keypose、AnyBimanual、3DFA（当前 SOTA）。
*   **泛化性测试**：设计了两个全新的任务（Close Jar, Take Item Out of Box），用于测试模型在完全未见过的分布外的表现。
*   **真机实验**：在双 Franka Panda 机器人平台上验证了从模拟到现实（Sim-to-Real）的迁移能力。

### 4. 资源与算力
*   **模型后端**：主要实验使用了 **GPT-5-mini**（通过 OpenAI API 调用）。为了验证框架的通用性，还使用了开源模型 **Qwen 2.5 7B** 和 **Qwen 2.5-VL 7B**。
*   **算力说明**：由于该方法是基于 ICL 的免训练方法，**文中未提及任何模型训练所需的 GPU 算力或时长**。
*   **推理延迟**：文中详细记录了推理时间。BiCICLe 基础版的每回合中值耗时约为 125.3 秒，而加入“手臂辩论”后增加至 278.2 秒。

### 5. 实验数量与充分性
*   **实验规模**：每个任务在 3 个随机种子下进行了 100 次评估，总计进行了数千次模拟实验。
*   **消融实验**：非常充分。涵盖了领导臂分配（左 vs 右）、协作模式（对话 vs 辩论）、观测表示（仅位置 vs 位置+旋转）、点云提取算法等多个维度的对比。
*   **客观性**：通过引入开源模型（Qwen）进行对比，证明了方法不依赖于特定闭源模型；通过与多种监督学习方法的对比，客观展示了 ICL 方法在性能与效率之间的权衡。

### 6. 主要结论与发现
*   **性能卓越**：BiCICLe + Best-of-N 达到了 **71.1%** 的平均成功率，比最强的免训练基准高出 6.7%，并超越了 PerAct 2 和 $\pi_0$-keypose 等监督学习方法。
*   **解耦的必要性**：实验证明，单智能体（SA）在对称同步任务中表现尚可，但在非对称任务中表现极差；而 BiCICLe 通过主从解耦兼顾了低维度预测和高协调性。
*   **泛化优势**：在面对新任务时，BiCICLe 的表现（54.5%）远超仅用少量样本微调的监督模型 3DFA（10.0%），证明了 ICL 在长尾任务中的潜力。

### 7. 优点
*   **架构创新**：首次将多智能体主从博弈思想引入机器人 ICL 领域，有效解决了双臂协调的维度灾难。
*   **无需训练**：极大地降低了双臂机器人部署的门槛，仅需少量演示即可适配新任务。
*   **鲁棒的优化机制**：Arms’ Debate 和 LLM-as-Judge 策略在不改变底层模型的情况下，显著抑制了 LLM 生成的随机错误。

### 8. 不足与局限
*   **精度限制**：受限于 $100^3$ 的体素化分辨率和离散的旋转分桶，对于极细微的操作（如抓取细绳、滑动笔记本）精度不足。
*   **推理成本**：多次调用 LLM API 导致推理延迟较高，且 Token 消耗量大（Best-of-N 模式下每回合约 18 万个 Token），难以实现实时闭环控制。
*   **感知依赖**：实验依赖于模拟器提供的地面真值分割掩码，在真实世界中对 3D 感知系统的噪声较为敏感。
*   **长程任务挑战**：对于步骤极多的复杂任务，LLM 的上下文窗口限制了演示样本的数量，可能影响推理质量。
