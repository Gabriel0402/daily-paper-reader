## 摘要
超高分辨率（UHR）遥感影像兼具千米级的空间背景与仅占据数个像素的关键信息。这种巨大的空间尺度导致视觉 token 数量呈平方级爆炸，从而阻碍了对小目标信息的提取。以往的方法包括直接下采样、密集切片或全局 top-k 剪枝，但这些方法要么损失了与查询相关的图像细节，要么带来了不可控的计算量。本文提出 UHR-BAT，一种在严格上下文预算约束下高效选择视觉 token 的、查询驱动且区域保持的 token 压缩框架。具体而言，我们利用文本引导的多尺度视觉 token 重要性估计，有效应对了实现精确且低成本特征提取的挑战。此外，通过引入按区域保留与融合的策略，我们进一步减少了视觉 token 的冗余，从而进一步降低计算预算。实验结果表明，UHR-BAT 在多个基准测试中均达到了当前最先进的性能。代码将发布于 https://github.com/Yunkaidang/UHR。

## Abstract
Ultra-high-resolution (UHR) remote sensing imagery couples kilometer-scale context with query-critical evidence that may occupy only a few pixels. Such vast spatial scale leads to a quadratic explosion of visual tokens and hinders the extraction of information from small objects. Previous works utilize direct downsampling, dense tiling, or global top-k pruning, which either compromise query-critical image details or incur unpredictable compute. In this paper, we propose UHR-BAT, a query-guided and region-faithful token compression framework to efficiently select visual tokens under a strict context budget. Specifically, we leverage text-guided, multi-scale importance estimation for visual tokens, effectively tackling the challenge of achieving precise yet low-cost feature extraction. Furthermore, by introducing region-wise preserve and merge strategies, we mitigate visual token redundancy, further driving down the computational budget. Experimental results show that UHR-BAT achieves state-of-the-art performance across various benchmarks. Code will be available at https://github.com/Yunkaidang/UHR.

## 一、研究背景与核心问题

- **研究动机**  
  超高分辨率（Ultra-High-Resolution, UHR）遥感影像可同时包含千米级全局背景和仅占若干像素的小目标（如车辆、船只），这导致：
  - 图像视觉 token 数呈平方级增加；
  - 导致 MLLM（多模态大语言模型）注意力计算和上下文处理成本骤升；
  - 直接下采样、切片（tiling）或全局 top-K 剪枝等策略会损失小目标信息或计算不可控。

- **核心问题**  
  如何在**有限上下文预算下**高效压缩视觉 token，使模型既保持全局空间一致性，又不丢失细粒度语义特征，是 UHR 场景下多模态理解的关键挑战。

- **研究目标**
  - 保持 **任务相关性（query-aware）**：基于文本问题选择与之相关的视觉区域；
  - 保持 **区域完整性（region-faithful）**：避免稀疏目标被压缩或误删；
  - 实现在固定 token 上下文预算下的高精度理解。

## 二、方法论与技术框架

### 1. 整体框架：UHR-BAT (Budget-Aware Token Compression)
UHR-BAT 是一个结合**查询引导压缩**与**区域保持合并**的多尺度视觉 token 选择机制，具体分为三大模块：

1. **查询感知的多尺度 token 重要性估计**
   - 利用跨模态注意力矩阵（来自 Vision-Language 接口）计算每个视觉 token 对文本查询的相关系数：
     \[
     a_i = \frac{1}{M}\sum_{j=1}^M A_{j,i}
     \]
   - 该得分表示视觉 token 对文本关键区域的响应强度。

2. **多尺度一致性与位置编码**
   - 将图像缩放为多个尺度 \({I^{(s)}}\)，低分辨率捕捉全局上下文，高分辨率保留局部细节；
   - 引入「尺度特定位置嵌入」与跨尺度对齐：
     \[
     h_i^{(s)} = e_i^{(s)} + p_i^{(s)} + q^{(s)}
     \]
   - 通过双线性插值将 anchor 尺度的重要性分数对齐至其他尺度，实现跨尺度一致选择。

3. **区域保持与合并策略（Region-wise Preserve-and-Merge, RPM）**
   - 先通过 **K-means 特征坐标聚类** 或 **SAM 图像分割** 将 token 划分为 R 个语义区域；
   - 在每个区域内：
     - 保留比区域平均重要性高的 token；
     - 对冗余 token 进行平均池化合并；
   - 最后按区域重要性排序并截断至预算上限：
     \[
     \bar{E}^{(s)} = C(E^{(s)}; B_s, \{a_i^{(s)}\})
     \]
   - 保证每个区域至少贡献一个高价值 token，从而在压缩中保留结构完整性。
