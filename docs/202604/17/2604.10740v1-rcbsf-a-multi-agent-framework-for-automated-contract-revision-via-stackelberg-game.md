## 摘要
尽管大型语言模型（LLMs）在法律人工智能领域被广泛采用，但其在合同自动修订中的应用仍受到虚构安全性和缺乏严格行为约束的限制。为了解决这些问题，我们提出了风险约束双层 Stackelberg 框架（Risk-Constrained Bilevel Stackelberg Framework，RCBSF），将修订任务形式化为一个非合作的 Stackelberg 博弈。RCBSF 建立了一个层级的领导者-跟随者结构，其中全局规约代理（Global Prescriptive Agent，GPA）为由受约束修订代理（Constrained Revision Agent，CRA）和局部验证代理（Local Verification Agent，LVA）组成的跟随系统分配风险预算，以迭代优化输出。我们提供理论证明表明，该双层结构的优化可收敛至一个均衡点，相较于无引导配置，能获得严格更高的效用。通过统一基准的实证验证表明，RCBSF 实现了当前最先进的性能，以平均风险解决率（Risk Resolution Rate，RRR）84.21% 超越迭代基线，并提高了 token 使用效率。我们的代码可在 https://github.com/xjiacs/RCBSF 获取。

## Abstract
Despite the widespread adoption of Large Language Models (LLMs) in Legal AI, their utility for automated contract revision remains impeded by hallucinated safety and a lack of rigorous behavioral constraints. To address these limitations, we propose the Risk-Constrained Bilevel Stackelberg Framework (RCBSF), which formulates revision as a non-cooperative Stackelberg game. RCBSF establishes a hierarchical Leader Follower structure where a Global Prescriptive Agent (GPA) imposes risk budgets upon a follower system constituted by a Constrained Revision Agent (CRA) and a Local Verification Agent (LVA) to iteratively optimize output. We provide theoretical guarantees that this bilevel formulation converges to an equilibrium yielding strictly superior utility over unguided configurations. Empirical validation on a unified benchmark demonstrates that RCBSF achieves state-of-the-art performance, surpassing iterative baselines with an average Risk Resolution Rate (RRR) of 84.21\% while enhancing token efficiency. Our code is available at https://github.com/xjiacs/RCBSF .

## 2. 方法论：RCBSF 框架与核心机制

### 2.1 基本思想
- 将合同修订建模为**非合作 Stackelberg 博弈**（Leader-Follower Game），在此过程中：
  - **全局规约代理（GPA，Leader）**：设定风险预算、分配约束条件；
  - **受约束修订代理（CRA，Follower）**：执行修订任务；
  - **局部验证代理（LVA，Follower 内部子模块）**：对修订结果进行局部审计与风险检测。

- 结构上形成双层优化（bilevel optimization）：外层为战略层、内层为执行层，通过多轮“修订–审计–修订”交互实现风险最小化。

### 2.2 关键技术细节

- **5维风险超平面表示**  
  GPA 通过函数 Ψ 将合同文本映射为风险特征空间，构建包括以下五维要素的风险张量：  
  `风险向量 h = {Category, Location, Evidence, Issue, Suggestion}`  
  → 对每个风险点提供类别、位置、证据、问题、修改建议。

- **双层优化目标**
  - 上层（Leader）：最大化风险消解效用  
    \[
    J_L(x, h) = \sum_k w_k \xi(x, r_k) - \lambda \cdot \Theta_{budget}(h)
    \]  
    其中 \(\Theta_{budget}\) 控制审计的资源成本。
  - 下层（Follower）：生成满足约束的修订文本  
    \[
    x^*(h) = \arg\max_{x'} \left[\sum_t \log P_\theta(x'_t|x'_{<t},x,h) - \gamma D_{KL}(\pi_{audit}(x')||\pi_{target}(h))\right]
    \]

- **迭代更新机制**  
  在每轮修订中，LVA 根据残余风险计算梯度信号，引导 CRA 更新文本。  
  形如：
  \[
  x_{t+1} = x_t + \eta \nabla_x F_{fusion}(x_t, Q_{outer}, Q_{inner})
  \]
  保证修订的连续收敛至局部平稳点。

- **理论保证**  
  作者证明：
  - **定理3.1**：Stackelberg 均衡下的效用严格优于无引导的 Nash 均衡。  
  - **定理3.2**：内层迭代过程收敛至梯度范数小于 ε 的固定点。

### 2.3 创新点
- 将风险控制纳入生成过程，通过**风险预算约束与博弈均衡机制**确保生成的合同同时合法、精简且无幻觉。
