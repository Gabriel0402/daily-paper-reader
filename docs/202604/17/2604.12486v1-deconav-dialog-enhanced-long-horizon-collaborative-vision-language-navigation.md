## 摘要
长时序协作视觉语言导航（VLN）对于多机器人系统完成单个智能体无法实现的复杂任务至关重要。CoNavBench 率先引入了首个协作长时序 VLN 基准，包括接力式多机器人任务、协作分类法，以及基于图的生成与评估机制，用于模拟共享环境中的任务交接与汇合。然而，现有的基准和评估往往未严格在共享世界时间线上同步双机器人的执行，并且通常依赖静态协调策略，无法在出现新的跨智能体证据时进行自适应调整。我们提出了 DeCoNav（Dialog enhanced Long-Horizon Collaborative Vision-Language Navigation），一种去中心化框架，将事件触发式对话与动态任务分配和重规划相结合，实现实时、自适应的协作。在 DeCoNav 中，机器人在无中心控制器的情况下通过对话交换紧凑的语义状态。当出现新的证据、不确定性或冲突等信息性事件时，会触发对话以动态重新分配子目标并在同步执行下进行重规划。我们在 DeCoNavBench 上实现了该系统，涵盖 176 个 HM3D 场景中的 1,213 个任务。DeCoNav 的双机器人成功率（BSR）提升了 69.2%，验证了基于对话驱动的动态重分配规划在多机器人协作中的有效性。

## Abstract
Long-horizon collaborative vision-language navigation (VLN) is critical for multi-robot systems to accomplish complex tasks beyond the capability of a single agent. CoNavBench takes a first step by introducing the first collaborative long-horizon VLN benchmark with relay-style multi-robot tasks, a collaboration taxonomy, along with graph-grounded generation and evaluation to model handoffs and rendezvous in shared environments. However, existing benchmarks and evaluations often do not enforce strictly synchronized dual-robot rollout on a shared world timeline, and they typically rely on static coordination policies that cannot adapt when new cross-agent evidence emerges. We present Dialog enhanced Long-Horizon Collaborative Vision-Language Navigation (DeCoNav), a decentralized framework that couples event-triggered dialogue with dynamic task allocation and replanning for real-time, adaptive coordination. In DeCoNav, robots exchange compact semantic states via dialogue without a central controller. When informative events such as new evidence, uncertainty, or conflicts arise, dialogue is triggered to dynamically reassign subgoals and replan under synchronized execution. Implemented in DeCoNavBench with 1,213 tasks across 176 HM3D scenes, DeCoNav improves the both-success rate (BSR) by 69.2%, demonstrating the effectiveness of dialogue-driven, dynamically reallocated planning for multi-robot collaboration.

## 一、研究背景与核心问题

- **研究动机**：  
  长时序视觉语言导航（VLN）要求智能体在复杂环境中依据自然语言指令执行多阶段路径规划与策略推理，这在现实场景（如室内探索、协作搬运）中极具挑战。  
  对于**多机器人协作任务**，需求进一步复杂化——机器人需要：
  - 同步理解共享的语言目标；
  - 在不完全可观测的环境中协同计划；
  - 动态应对环境变动和通信延迟。

- **现有问题**：
  1. 当前多机器人 VLN 基准（如 CoNavBench）尚未严格要求双机器人在**同一时间轴**上同步执行；
  2. 多数方法使用静态或预设的任务分工策略，缺乏**在线自适应协调**能力；
  3. 评估缺少语义一致的场景验证机制，使得任务可靠性存在噪声。

- **核心研究目标**：  
  作者提出一个可以结合**事件触发式对话**与**动态子任务重规划**的去中心化协作框架，使机器人能在运行中根据“新证据或冲突”进行**语义层面交流与决策更新**，并保持时间同步。

## 二、方法论：DeCoNav 框架

### 1. 核心思想
- **去中心化协同架构**：  
  无中心控制器，每个机器人独立感知、决策，仅通过语义状态包进行轻量通信。
- **事件触发式对话机制**：  
  当出现状态变化（例如新目标发现、冲突、停滞）时，触发对话，重新分配任务子目标。
- **实时同步执行（Shared Clock）**：  
  所有机器人在共享时钟下同步行动，确保交互评估的因果一致性。

### 2. 模块组成
DeCoNav 包含三大功能模块：

1. **Semantic Visual Bus (SVB)** — *语义视觉总线*  
   - 作为通信骨干，交换语义包而非图像数据；  
   - 每个语义包包含房间类型、关键物体描述、任务阶段状态、时间戳；  
   - 保持带宽低但保证协调信息完整。

2. **Event-driven Dialogue Replanning (EDR)** — *事件驱动的对话重规划模块*  
   - 负责检测新事件、筛选有效触发、更新局部策略上下文；  
   - 每当事件触发，机器人在对话后更新指令、重排子目标优先级，实现**在线任务重新分配**。

3. **Synchronous Parallel Execution (SPE)** — *同步并行执行机制*  
   - 每一步中两个机器人同时执行推理与动作提交；  
   - 保证评估中的严格同步与执行一致性；  
   - 在真实双机部署中通过 **语义消息时效约束 (freshness constraint)** 确保同步稳定。

### 3. 相关公式（文字描述）
- 系统的共享语义总线状态定义为：  
  \( B_t = \{S_t^1, S_t^2, A_t, R_t, D_t\} \)，其中：
  - \(S_t^r\)：第 r 个机器人的语义状态；
  - \(A_t\)：关键锚点与物体证据；
  - \(R_t\)：当前角色与中继任务分配；
  - \(D_t\)：历史对话事件记录。  
  每一步都会更新 \(B_t\)，用于后续的对话与 replanning。

## 三、实验设计

### 1. 数据集与平台
- **基准平台**：DeCoNavBench（作者扩展自 CoNavBench）
- **核心创新**：引入 **ROVE**（Room-and-Object Verified Episode Generation）数据生成与验证流程，确保语义与可见性正确。
  - **RTSA**：房间类型语义对齐模块  
  - **TriGate**：三重验证（语义可见性、一致性、可识别性）

### 2. 任务设定
- 双机器人接力式任务  
  ```
  机器人1: start → pickup → handoff  
  机器人2: start → handoff → delivery
  ```
  两者同时执行并通过事件对话调整策略。

### 3. 数据规模与特征
- **任务数**：1,213  
- **场景数**：176 个 HM3D 场景  
- **平均路径长度**：每机器人约 20.6 米  
- **图像总量**：超 205 万帧 RGB-D 图像  
- **每步动作空间**：{前进0.25 m, 左转15°, 右转15°, 停止}

### 4. 对比与模型设置
- **对比对象**：
  - CoNavBench 基线模型；
  - DeCoNav 启用/禁用各模块的消融实验。
- **主要评估指标**：
  - SR（单机器人成功率）
  - BSR（双机器人同时成功率）
  - SPL（路径效率）
  - ISR（中间子任务成功率）
  - NE（导航误差）
