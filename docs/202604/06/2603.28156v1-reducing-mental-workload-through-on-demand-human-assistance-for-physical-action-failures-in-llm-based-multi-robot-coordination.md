## 摘要
基于大语言模型（LLM）的多机器人协作引起了越来越多的关注，因为 LLM 能够通过分解任务和生成高层规划，将自然语言指令直接转化为机器人动作方案。然而，从物理执行失败中恢复仍然具有挑战性，且任务往往因重复执行相同的失败动作而陷入停滞。尽管已有研究提出了利用混合现实进行机器人远程操作的框架，但针对多机器人环境中物理失败的远程错误解决尝试仍较少。在本研究中，我们提出了 REPAIR（Robot Execution with Planned And Interactive Recovery），这是一种将远程错误解决集成到基于 LLM 的多机器人规划中的人机协同框架。在该方法中，机器人自主执行任务；然而，当发生不可恢复的失败时，LLM 会向操作员请求协助，通过远程干预实现任务的连续性。在真实环境中使用多机器人垃圾清理任务进行的评估证实，与全自主方法相比，REPAIR 显著提高了任务进度（在时限内清理的物品数量）。此外，对于易于收集的物品，它实现了与全远程控制相当的任务进度。结果还表明，操作员的心理负荷在体力需求和努力程度方面可能存在差异。项目网站为 https://emergentsystemlabstudent.github.io/REPAIR/。

## Abstract
Multi-robot coordination based on large language models (LLMs) has attracted growing attention, since LLMs enable the direct translation of natural language instructions into robot action plans by decomposing tasks and generating high-level plans. However, recovering from physical execution failures remains difficult, and tasks often stagnate due to the repetition of the same unsuccessful actions. While frameworks for remote robot operation using Mixed Reality were proposed, there have been few attempts to implement remote error resolution specifically for physical failures in multi-robot environments. In this study, we propose REPAIR (Robot Execution with Planned And Interactive Recovery), a human-in-the-loop framework that integrates remote error resolution into LLM-based multi-robot planning. In this method, robots execute tasks autonomously; however, when an irrecoverable failure occurs, the LLM requests assistance from an operator, enabling task continuity through remote intervention. Evaluations using a multi-robot trash collection task in a real-world environment confirmed that REPAIR significantly improves task progress (the number of items cleared within a time limit) compared to fully autonomous methods. Furthermore, for easily collectable items, it achieved task progress equivalent to full remote control. The results also suggested that the mental workload on the operator may differ in terms of physical demand and effort. The project website is https://emergentsystemlabstudent.github.io/REPAIR/.

#### 1. 核心问题与研究动机
*   **核心问题**：虽然大语言模型（LLM）在多机器人任务分解和高层规划方面表现出色，但在面对**物理执行失败**（如抓取落空、目标丢失）时，机器人往往会陷入无效的重复动作循环，难以自主恢复。
*   **研究动机**：
    *   **全自主系统（Auto）**：在复杂物理环境下鲁棒性不足，一旦发生感知或操作错误，任务容易停滞。
    *   **全远程操作（Teleop）**：随着机器人数量增加，操作员的心理和体力负担呈指数级增长，不可持续。
    *   **解决方案**：提出一种“按需干预”的半自主框架，仅在机器人无法自我修复时请求人类协助，以平衡任务成功率与人类工作负荷。

#### 2. 方法论（REPAIR 框架）
REPAIR（Robot Execution with Planned And Interactive Recovery）的核心思想是将远程错误修复（Remote Error Resolution）作为一种特殊的“机器人技能”集成到 LLM 规划中。
*   **任务分解与分配**：LLM 接收自然语言指令（如“清理房间”），根据机器人能力（移动机器人 Kachaka 和移动协作机器人 Fetch）将其分解为子任务序列。
*   **动作规划与执行**：LLM 为每个机器人生成具体的动作技能（如 `navigation`, `pick`, `place`）。
*   **按需协助机制（核心）**：
    *   当物理动作（如抓取）失败且无法通过简单重试解决时，LLM 调用 `help` 技能。
    *   LLM 生成描述当前困境的请求消息发送给操作员。
*   **人机交互闭环**：
    *   操作员通过 VR/MR 界面接管机器人，完成关键物理操作。
    *   操作员向 LLM 提供反馈（告知已完成的操作）。
    *   LLM 根据反馈更新环境状态认知，重新生成后续规划并恢复自主运行。

#### 3. 实验设计
*   **实验场景**：真实室内环境（包含餐厅、客厅、工作区等 4 个区域）。
*   **实验任务**：多机器人协作垃圾清理。Fetch 负责抓取，Kachaka 负责运输。
*   **物体难度分级**：
    *   **Level 1（易）**：形状规则、易抓取的瓶子。
    *   **Level 2（难）**：易倾倒的空瓶、形状不规则的纸屑（易导致感知/抓取失败）。
*   **对比方法（Benchmark）**：
    1.  **Teleop**：全远程手动控制。
    2.  **Auto**：全自主 LLM 规划（失败时仅尝试自主重规划）。
    3.  **REPAIR（本文方法）**：LLM 规划 + 按需人工辅助。
*   **评估指标**：任务进度（规定时间内清理的物品数）和 NASA-TLX 心理负荷量表。

#### 5. 实验数量与充分性
*   **实验规模**：招募了 **12 名受试者**（8 男 4 女，年龄 20-30 岁）。
*   **实验设计**：采用**被试内设计（Within-subject design）**，即每位受试者均需依次操作三种系统（Teleop, REPAIR, Auto），以消除个体差异影响。
*   **统计分析**：使用了 Friedman 检验和带有 Holm 校正的 Wilcoxon 符号秩检验。
*   **充分性评价**：实验设计较为严谨，包含了真实环境测试和主客观双重指标。虽然样本量（12人）在人机交互研究中属于标准规模，但对于多机器人协作的复杂性而言，覆盖的故障类型仍有扩展空间。

#### 6. 主要结论与发现
*   **任务进度**：REPAIR 的表现显著优于全自主（Auto）方法。在处理简单物体（Level 1）时，REPAIR 的效率接近全手动控制（Teleop）。
*   **心理负荷**：
    *   在**体力需求（Physical Demand）**和**努力程度（Effort）**子项上，REPAIR 表现出低于 Teleop 的趋势（尽管在 Holm 校正后的两两比较中显著性不明显，但 Friedman 检验显示整体差异显著）。
    *   全自主方法（Auto）虽然人类负担最低，但任务进度几乎为零（无法从物理失败中恢复）。
*   **系统鲁棒性**：通过引入人工干预，系统能够跨越 LLM 无法处理的“物理鸿沟”，保证了任务的连续性。

#### 7. 优点与亮点
*   **实用性强**：针对 LLM 在机器人领域落地的痛点（物理执行不可靠）提出了切实可行的“人机共生”方案。
*   **闭环反馈**：不仅是简单的接管，还实现了“人类操作 -> 文本反馈 -> LLM 更新规划”的逻辑闭环。
*   **真实部署**：在真实物理环境和真实机器人上进行了验证，而非仅停留在仿真阶段。

#### 8. 不足与局限
*   **UI 复杂性**：实验中不同机器人使用不同控制设备（Quest 3 和手机），增加了操作员的认知切换成本。
*   **干预时机单一**：目前仅在物理动作失败时请求协助，未探讨在任务分解或规划阶段进行人工干预的潜力。
*   **判定误差风险**：未深入探讨机器人“误报”失败（明明成功却求助）或“漏报”失败（陷入循环却不求助）对系统性能的影响。
*   **MR 影响未分离**：实验未能严格区分心理负荷的降低是来自于“按需干预模式”还是“MR 界面本身”的优越性。
