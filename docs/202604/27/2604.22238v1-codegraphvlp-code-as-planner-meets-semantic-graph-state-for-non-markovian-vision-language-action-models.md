## 摘要
视觉-语言-动作（VLA）模型有望实现通用机器人操控，但通常作为短时界策略进行训练和部署，假设最新的观测足以进行动作推理。这一假设在非马尔可夫长时界任务中失效，因为任务相关的证据可能被遮挡或仅出现在轨迹早期，且杂乱环境和干扰项使得细粒度视觉定位变得脆弱。我们提出了 CodeGraphVLP，这是一个分层框架，通过将持久的语义图状态与可执行的基于代码的规划器以及进度引导的视觉语言提示相结合，实现了可靠的长时界操控。语义图在部分可观测性下维护任务相关的实体和关系。合成的规划器在语义图上执行，以进行高效的进度检查，并输出子任务指令及子任务相关对象。我们利用这些输出构建抑制杂乱的观测，使 VLA 执行器专注于关键证据。在真实世界的非马尔可夫任务中，CodeGraphVLP 相比强大的 VLA 基线和支持历史记录的变体提高了任务完成率，同时与 VLM 在环规划相比显著降低了规划延迟。我们还进行了广泛的消融研究，以确认各组件的贡献。

## Abstract
Vision-Language-Action (VLA) models promise generalist robot manipulation, but are typically trained and deployed as short-horizon policies that assume the latest observation is sufficient for action reasoning. This assumption breaks in non-Markovian long-horizon tasks, where task-relevant evidence can be occluded or appear only earlier in the trajectory, and where clutter and distractors make fine-grained visual grounding brittle. We present CodeGraphVLP, a hierarchical framework that enables reliable long-horizon manipulation by combining a persistent semantic-graph state with an executable code-based planner and progress-guided visual-language prompting. The semantic-graph maintains task-relevant entities and relations under partial observability. The synthesized planner executes over this semantic-graph to perform efficient progress checks and outputs a subtask instruction together with subtask-relevant objects. We use these outputs to construct clutter-suppressed observations that focus the VLA executor on critical evidence. On real-world non-Markovian tasks, CodeGraphVLP improves task completion over strong VLA baselines and history-enabled variants while substantially lowering planning latency compared to VLM-in-the-loop planning. We also conduct extensive ablation studies to confirm the contributions of each component.

### 1. 核心问题与整体含义（研究动机和背景）
现有的视觉-语言-动作（VLA）模型（如 OpenVLA, $\pi_0$ 等）大多遵循**马尔可夫假设**，即认为当前的单帧观察足以决定下一步动作。然而，在真实世界的长程（Long-horizon）任务中，这一假设经常失效：
*   **非马尔可夫依赖**：关键信息可能在之前的步骤中出现（如物体被遮挡或移出视野），机器人需要记忆历史状态。
*   **环境干扰**：杂乱的背景和无关物体会干扰 VLA 的视觉定位，导致动作执行失败。
*   **效率瓶颈**：现有的分层系统（VLM 规划器 + VLA 执行器）通常需要频繁调用大模型进行在线重规划，导致高延迟，难以满足实时控制需求。

### 2. 方法论：核心思想与关键技术
CodeGraphVLP 提出了一种分层架构，核心思想是**“用结构化图表征记忆，用代码实现高效规划”**。

*   **持久化语义图状态（Semantic-Graph State）**：
    *   系统维护一个图 $G_t = (V_t, E_t)$，节点 $V$ 代表任务相关实体（含掩码、属性），边 $E$ 代表空间关系（如 `in`, `on`, `near`）。
    *   通过多视图关联和几何规则感应关系，并在执行过程中利用跟踪器（如 Cutie）和分割模型（YOLOE）动态更新图状态，从而在部分可观测下保持对环境的理解。
*   **代码即规划器（Code-as-Planner）**：
    *   在任务开始时，利用 LLM（如 GPT-4o/5）根据初始环境和任务指令生成一段 Python 代码。
    *   该代码作为高层规划器，通过查询语义图来检查任务进度（利用 `graph.task_memory` 记录已完成子任务），并输出当前的**子任务指令**和**相关物体列表**。
*   **进度引导的视觉-语言提示（Visual-Language Prompting）**：
    *   **去干扰视觉提示**：根据规划器输出的相关物体，对原始图像进行掩码处理，仅保留关键物体（其余部分黑屏），强制 VLA 关注核心证据。
    *   **子任务语言提示**：将复杂的长程指令分解为简短的指令，降低 VLA 的推理负担。

### 3. 实验设计
*   **场景与任务**：在真实 UR10e 机器人平台上设计了三个具有挑战性的非马尔可夫任务：
    1.  **Pick-and-Place Twice**：将方块移到另一盘子再移回（初始和终态视觉相似，需记忆进度）。
    2.  **Place-and-Stack**：将方块放入杯中并用另一杯子堆叠（方块被遮挡后需记忆其位置）。
    3.  **Swap Cups**：交换两个杯子的位置（涉及中间缓冲区和多步逻辑）。
*   **对比基线**：
    *   **原生 VLA**：$\pi_0$, $\pi_0$ FAST, $\pi_{0.5}$, Gr00T N1.5。
    *   **历史增强型**：Gr00T N1.5 + Multi-frame（输入多帧历史图像）。
    *   **消融变体**：不带去干扰视觉提示的 CodeGraphVLP、基于 VLM 在线重规划的变体。

### 4. 资源与算力
*   **训练算力**：使用了 **4 张 NVIDIA A6000 GPU**。
*   **训练细节**：使用 LoRA 微调，迭代 50K 次，Batch Size 为 128，学习率为 $1 \times 10^{-5}$。

*   **规划器生成**：使用 GPT-5 进行一次性代码合成。

### 5. 实验数量与充分性
*   **实验规模**：收集了共 400 场人类演示数据进行微调。每个任务进行了多轮真实机器人测试。
*   **充分性**：实验涵盖了成功率对比、中间步骤完成率分析、规划延迟测试以及针对关键组件（代码规划器、视觉掩码）的消融研究。
*   **客观性**：对比了当前最前沿的开源 VLA 模型，并量化了延迟（秒/步），实验设计较为公平。

### 6. 主要结论与发现
*   **性能领先**：CodeGraphVLP 在所有任务上的成功率均大幅领先基线（平均成功率 81.7%，而最强基线仅为 56.7%）。
*   **高效规划**：相比于每步调用 VLM 进行规划（延迟约 3 秒），基于代码的规划器延迟仅为 **0.328 秒**，效率提升近 10 倍。
*   **视觉引导至关重要**：消融实验显示，去除“去干扰视觉提示”后，成功率从 85% 骤降至 40%，证明了减少视觉噪声对 VLA 执行的重要性。

### 7. 优点
*   **架构优雅**：巧妙结合了符号化推理（代码/图）与神经网络执行（VLA），兼顾了逻辑严密性和动作泛化性。
*   **低延迟**：通过一次性代码合成避免了昂贵的在线大模型推理，更符合工业部署需求。
*   **鲁棒性强**：通过语义图和视觉掩码有效解决了长程任务中的遮挡和干扰问题。

### 8. 不足与局限
*   **依赖底层感知**：系统的可靠性高度依赖于初始的物体分割和属性识别，如果 VLM 在初始化阶段识别错误，后续代码执行将失效。
*   **代码合成风险**：虽然 LLM 生成代码能力很强，但在极端复杂逻辑下可能存在 Bug，且文中未深入探讨代码自动纠错机制。
*   **泛化范围**：实验主要集中在桌面操控任务，对于更广阔的开放场景（如移动机器人、复杂家庭环境）的适用性有待进一步验证。
