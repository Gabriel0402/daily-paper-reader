## 摘要
桌面交互流提供了交织用户任务的连续且隐私敏感的记录。在设备端将这些流转换为按任务组织的个人日志面临两个主要挑战：详尽的视觉语言模型（VLM）处理会耗尽本地资源，而全局流处理会导致跨任务的上下文污染。我们提出了 FOCAL（Filtered On-device Continuous Activity Logging），这是一个采用统一“过滤-规划-记录”架构的隐私优先多智能体系统。它级联了一个用于噪声抑制的轻量级过滤智能体（Filter Agent）、一个仅限文本的大脑智能体（Brain Agent）用于任务归因、一个记录智能体（Record Agent）用于选择性视觉推理，以及一个任务隔离的记忆智能体（Memory Agent）用于上下文连贯的摘要。在 DesktopBench（包含 420 个复杂会话中的 2,572 张截图）上的实验表明，与基线相比，FOCAL 将总 Token 消耗降低了 60.4%，VLM 调用次数减少了 72.3%，同时将关键信息召回率（KIR）从 0.38 提升至 0.61。至关重要的是，在 A→B→A 任务中断情况下，FOCAL 保持了 0.81 的任务准确率（Task Acc）和 0.80 的 KIR，而基线的任务准确率则崩溃至 0.03。FOCAL 开创了将无指令桌面流高效、端侧地摘要为多视角个人日志的先河。

## Abstract
Desktop interaction streams provide a continuous, privacy-sensitive record of interleaved user tasks. Transforming these streams into task-organized personal logs on-device faces two main challenges: exhaustive Vision-Language Model (VLM) processing strains local resources, and global stream processing causes cross-task context pollution. We present FOCAL (Filtered On-device Continuous Activity Logging), a privacy-first multi-agent system utilizing a unified filter-plan-log architecture. It cascades a lightweight Filter Agent for noise suppression, a text-only Brain Agent for task attribution, a Record Agent for selective visual reasoning, and a task-isolated Memory Agent for context-coherent summarization. Experiments on DesktopBench (comprising 2,572 screenshots across 420 complex sessions) show FOCAL reduces total token consumption by 60.4% and VLM call count by 72.3% versus a baseline, while boosting Key Information Recall (KIR) from 0.38 to 0.61. Crucially, under $A{\to}B{\to}A$ task interruptions, FOCAL maintains Task Acc 0.81 and KIR 0.80, whereas the baseline collapses to Task Acc 0.03. FOCAL pioneers the efficient, on-device summarization of instruction-free desktop streams into multi-perspective personal logs.

*   **研究动机**：将这些原始流转化为按任务组织的日志有助于提高生产力和知识检索。
*   **核心挑战**：
    1.  **端侧资源受限**：桌面截图包含大量隐私，必须在本地处理，但详尽调用视觉语言模型（VLM）会严重消耗本地 CPU/GPU 资源，干扰正常工作。
    2.  **跨任务上下文污染**：用户任务经常交织（如 A→B→A），传统的全局流处理会将不同任务的描述混在一起，导致总结逻辑混乱。
*   **目标**：在无用户指令、无预设任务边界的情况下，实现高效、隐私、任务隔离的端侧桌面活动总结。

### 2. 方法论：核心思想与技术细节
FOCAL 提出了一个**统一的“过滤-规划-记录”（Filter-Plan-Log）架构**，通过多智能体协作将任务感知控制前置于昂贵的视觉推理。

*   **核心思想**：先利用轻量级元数据（如应用名、窗口标题）进行任务归因和采样决策，再对关键帧进行 VLM 推理。
*   **五大智能体流程**：
    1.  **Filter Agent（过滤）**：前端模块，剔除无意义的桌面噪声，获取有效的动作元数据和截图。
    2.  **Brain Agent（大脑/规划）**：**关键创新点**。仅基于文本元数据（App, Title）对整个会话进行一次性规划，决定每个动作属于哪个任务（Task ID），并判断是否需要进行视觉采样（即是否调用 VLM）。
    3.  **Record Agent（记录）**：仅针对被采样的动作调用 VLM，结合当前任务的局部记忆生成语义描述。
    4.  **Memory Agent（记忆）**：管理**任务隔离的存储空间**。它确保每个任务的观察结果只进入该任务专属的上下文，防止跨任务污染。
    5.  **Summary Agent（总结）**：在会话结束时，将各隔离存储中的证据整合为结构化的多视角总结。
