## 摘要
尽管视觉-语言-动作（VLA）模型在短时程任务中表现强劲，但在长时程操纵任务中却存在系统性失败。我们研究表明，在当前的反应式执行设置下，仅靠延长上下文长度无法解决这一问题；相反，失败源于执行循环中反复出现的三个缺陷：记忆间隙、验证间隙和恢复间隙。我们提出了 HELM，这是一个模型无关的框架，通过三个组件解决这些缺陷：一个通过 CLIP 索引关键帧检索关键任务历史的情景记忆模块（EMM）；一个根据观测、动作、子目标和记忆条件上下文在执行前预测动作失败的学习型状态验证器（SV）；以及一个执行回滚和重新规划的控制框架（HC）。SV 是核心的学习贡献：它始终优于基于规则的可行性检查和集成不确定性基准，且其有效性关键取决于对情景记忆的访问。在 LIBERO-LONG 基准测试中，HELM 将 OpenVLA 的任务成功率提高了 23.1 个百分点（从 58.4% 提升至 81.5%），而将上下文窗口扩展到 H=32 仅带来 5.4 个百分点的提升，且同等预算的 LoRA 适配仍比 HELM 低 12.2 个百分点。HELM 还提高了 CALVIN 上的长时程性能，并显著增强了受控扰动下的恢复成功率。消融实验和机制分析分离了各组件的贡献，我们还发布了 LIBERO-Recovery，作为一种用于评估长时程操纵中故障恢复的扰动注入协议。

## Abstract
Vision-Language-Action (VLA) models fail systematically on long-horizon manipulation tasks despite strong short-horizon performance. We show that this failure is not resolved by extending context length alone in the current reactive execution setting; instead, it stems from three recurring execution-loop deficiencies: the memory gap, the verification gap, and the recovery gap. We present HELM, a model-agnostic framework that addresses these deficiencies with three components: an Episodic Memory Module (EMM) that retrieves key task history via CLIP-indexed keyframes, a learned State Verifier (SV) that predicts action failure before execution from observation, action, subgoal, and memory-conditioned context, and a Harness Controller (HC) that performs rollback and replanning. The SV is the core learning contribution: it consistently outperforms rule-based feasibility checks and ensemble uncertainty baselines, and its effectiveness depends critically on access to episodic memory. On LIBERO-LONG, HELM improves task success rate by 23.1 percentage points over OpenVLA (58.4% to 81.5%), while extending the context window to H=32 yields only a 5.4-point gain and same-budget LoRA adaptation remains 12.2 points below HELM. HELM also improves long-horizon performance on CALVIN and substantially boosts recovery success under controlled perturbations. Ablations and mechanism analyses isolate the contribution of each component, and we release LIBERO-Recovery as a perturbation-injection protocol for evaluating failure recovery in long-horizon manipulation.

## 论文详细总结（自动生成）

### 1. 核心问题与整体含义
*   **研究背景**：视觉-语言-动作（VLA）模型在短程任务中表现出色，但在长程操纵任务（Long-horizon tasks）中性能急剧下降。
*   **核心发现**：研究表明，长程任务的失败并非单纯因为上下文窗口（Context Window）不够长。在当前的反应式执行模式下，即使增加上下文长度，模型仍面临三大结构性缺陷：
    1.  **记忆间隙（Memory Gap）**：固定窗口会丢弃已完成子目标的证据，导致模型重复执行已完成的动作。
    2.  **验证间隙（Verification Gap）**：模型反应式地提出动作，缺乏执行前的可行性检查，导致错误动作静默执行并累积。
    3.  **恢复间隙（Recovery Gap）**：一旦发生错误，模型无法检测并继续在受损状态下运行，引发连锁失败。
*   **研究动机**：开发一个模型无关的框架，通过增强记忆、验证和恢复机制来弥补上述缺陷。

### 2. 方法论
HELM 框架通过三个核心组件对冻结的 VLA 模型进行“外挂式”增强：
*   **情节记忆模块 (EMM)**：
    *   **核心思想**：建立一个基于 CLIP 索引的关键帧存储库。
    *   **技术细节**：在子目标完成、检测到失败或每隔 20 步时，将当前观测的 CLIP 嵌入作为键（Key），状态信息作为值（Value）存入。检索时，通过余弦相似度提取最相关的 Top-k 历史帧，并将其序列化为文本注入 VLA 的语言输入。
*   **状态验证器 (SV)**：
    *   **核心思想**：在动作执行前预测其失败概率。
    *   **技术细节**：一个轻量级的 3 层 MLP。输入包括当前观测、拟执行动作、当前子目标以及**检索到的记忆上下文**。SV 的关键在于“记忆增强”，实验证明没有记忆辅助的验证器效果会大幅下降。
*   **控制框架 (HC)**：
    *   **核心思想**：管理任务进度并执行恢复策略。
    *   **流程**：检索记忆 -> 提出动作 -> SV 验证。若 $p_{fail}$ 超过阈值，则触发回滚（Rollback）至最近的成功检查点，或进行重规划（HELM-Fwd 变体）。

### 3. 实验设计
*   **数据集/场景**：
    1.  **LIBERO-LONG**：包含 10 个任务，平均 5.8 个子目标。
    2.  **CALVIN (ABC → D)**：评估在未见环境中的长程链条完成能力。
    3.  **LIBERO-Recovery (新提出)**：在子目标边界注入随机扰动（如物体位移、夹爪状态翻转），专门测试恢复能力。
*   **对比方法 (Baselines)**：
    *   基础模型：OpenVLA (H=8) 和 Octo。
    *   变体：长上下文版本 (H=32, 64)、Oracle 记忆（上限测试）、基于规则的验证器、集成模型不确定性（Ensemble）、同等数据预算下的 LoRA 微调、以及 Reflexion（自我反思）方法。

### 4. 资源与算力
*   **硬件使用**：单张 **NVIDIA A100 80GB** GPU。
*   **训练时长**：状态验证器（SV）的训练需要约 5 万步的 Rollout 数据，训练时间约为 **2 小时**。
*   **推理开销**：每步推理增加约 **12ms** 的延迟，相比原始 VLA 模型仅增加约 15% 的计算负担。

### 5. 实验数量与充分性
*   **实验规模**：在 LIBERO-LONG 上进行了 500 个回合的评估，在 CALVIN 上进行了 1000 条链条的测试。
*   **充分性**：
    *   **消融实验**：详细拆解了 EMM、SV 和回滚机制的贡献，证明了 EMM 是最大的单一贡献项。
    *   **公平性**：通过 LoRA (50K) 基准证明，将同样的数据预算用于训练 SV 比直接微调 VLA 主干网络更有效。
    *   **客观性**：引入了跨模型验证（OpenVLA 和 Octo），证明了框架的通用性。

### 6. 主要结论与发现
*   **性能飞跃**：HELM 将 OpenVLA 在 LIBERO-LONG 上的成功率从 **58.4% 提升至 81.5%**（+23.1 pp）。
*   **上下文局限性**：单纯将上下文窗口扩大到 H=32 仅能提升 5.4 pp，证实了长程失败的结构性原因。
*   **验证器的优越性**：学习型 SV 优于基于规则的检查和集成不确定性方法，且其性能高度依赖于记忆上下文（移除记忆后性能下降 6.1 pp）。
*   **恢复能力**：在注入扰动的测试中，HELM 的恢复成功率（RSR）远超基准模型。

### 7. 优点
*   **模型无关性**：作为一个外挂框架，无需重新训练庞大的 VLA 模型，易于迁移。
*   **闭环鲁棒性**：通过“验证-回滚”机制，将传统的反应式开环执行转变为具有自我纠错能力的闭环过程。
*   **高效性**：SV 模块极轻量，适合实时机器人控制。
*   **评估贡献**：提出的 LIBERO-Recovery 协议填补了长程操纵中故障恢复评估的空白。

### 8. 不足与局限
*   **数据依赖性**：SV 的训练仍需要约 5 万步的环境特定数据，限制了其在完全陌生环境下的零样本（Zero-shot）部署能力。
*   **回滚的现实挑战**：在模拟器中回滚状态很容易，但在真实物理世界中，某些动作（如打破杯子）是不可逆的，此时回滚机制失效，需依赖效果略差的前向恢复（HELM-Fwd）。
*   **Sim-to-Real 风险**：实验均在仿真环境中完成，真实世界的视觉噪声和物理不确定性可能对 CLIP 检索和 SV 的准确性提出更高挑战。
