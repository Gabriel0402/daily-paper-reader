## 摘要
在未知环境中协调多机器人系统（MRS）进行搜索，对于需要超越几何探索的语义推理任务而言具有极大挑战性。经典的协调策略依赖于边界覆盖或信息增益，无法融入高层任务意图，例如搜索与特定房间类型相关的物体。我们提出了“语义区域图推理”（SAGR），这是一个分层框架，通过环境的结构化语义拓扑抽象，使大语言模型（LLM）能够协调多机器人探索和语义搜索。SAGR 从语义占据地图中增量构建语义区域图，将房间实例、连通性、边界可用性以及机器人状态编码为紧凑的任务相关表示，供 LLM 进行推理。LLM 根据空间结构和任务上下文执行高层语义房间分配，而确定性边界规划和局部导航则负责处理分配房间内的几何执行。在 Habitat-Matterport3D 数据集上进行的 100 个场景实验表明，SAGR 在保持与最先进探索方法竞争力的同时，持续提高了语义目标搜索效率，在大型环境中提升高达 18.8%。这些结果突显了结构化语义抽象作为基于 LLM 的推理与复杂室内环境下多机器人协调之间有效接口的价值。

## Abstract
Coordinating multi-robot systems (MRS) to search in unknown environments is particularly challenging for tasks that require semantic reasoning beyond geometric exploration. Classical coordination strategies rely on frontier coverage or information gain and cannot incorporate high-level task intent, such as searching for objects associated with specific room types. We propose \textit{Semantic Area Graph Reasoning} (SAGR), a hierarchical framework that enables Large Language Models (LLMs) to coordinate multi-robot exploration and semantic search through a structured semantic-topological abstraction of the environment. SAGR incrementally constructs a semantic area graph from a semantic occupancy map, encoding room instances, connectivity, frontier availability, and robot states into a compact task-relevant representation for LLM reasoning. The LLM performs high-level semantic room assignment based on spatial structure and task context, while deterministic frontier planning and local navigation handle geometric execution within assigned rooms. Experiments on the Habitat-Matterport3D dataset across 100 scenarios show that SAGR remains competitive with state-of-the-art exploration methods while consistently improving semantic target search efficiency, with up to 18.8\% in large environments. These results highlight the value of structured semantic abstractions as an effective interface between LLM-based reasoning and multi-robot coordination in complex indoor environments.

### 1. 核心问题与整体含义（研究动机和背景）
在未知的室内环境中，协调多个机器人进行搜索通常依赖于几何信息（如探索未知的边界）。然而，当任务涉及“去卧室找一件夹克”这类包含高层语义指令时，传统的几何协调策略（如边界覆盖或信息增益）无法利用房间类型与物体之间的语义关联。
现有的基于大语言模型（LLM）的机器人规划方法虽然具备语义推理能力，但往往直接处理高维的视觉图像或原始占据地图，导致 Prompt（提示词）过长、计算成本高且难以满足实时性要求。因此，论文的核心问题是如何**在保持实时性的前提下，将 LLM 的高层语义推理能力与多机器人的几何探索效率相结合**。

### 2. 方法论：核心思想与技术细节
SAGR 采用了一种分层架构，将复杂的语义搜索任务分解为三个层面：
*   **语义区域图构建（核心创新）：** 系统从增量构建的语义占据地图中提取“房间实例”。每个房间作为一个节点，存储房间类型（如厨房、卧室）、唯一ID、包含的前沿（Frontier）统计数据、当前机器人位置以及空间连通性（边）。这种抽象将密集的地图压缩为紧凑的拓扑图，极大地减小了 LLM 的输入规模。
*   **语义推理层（LLM 规划）：** LLM 接收序列化的语义区域图、任务说明（如目标房间类型）和先前的计划摘要。LLM 的任务是进行高层决策：将不同的机器人分配到最有可能包含目标的房间实例中，同时兼顾探索的覆盖范围。
*   **前沿选择与局部执行层：** 
    *   一旦 LLM 完成房间分配，底层算法使用**匈牙利算法（Hungarian Algorithm）**在分配的房间内进行具体的前沿点分配。
    *   使用 **TSP（旅行商问题）求解器**优化机器人在房间内的访问顺序。
    *   局部执行层负责避障导航和传感器数据采集，并不断更新地图。

### 3. 实验设计
*   **数据集与场景：** 使用 **Habitat-Matterport3D (HM3D)** 数据集，包含真实的室内公寓布局。实验涵盖了 10 个不同的公寓布局，每个布局生成 10 个随机场景（初始位姿和目标位置随机），共计 **100 个实验场景**。
*   **环境规模：** 分为小型（2台机器人）、中型（3台机器人）和大型（4台机器人）环境。
*   **Benchmark（对比方法）：**
    1.  **Hungarian Frontier Assignment：** 经典的基于欧几里得距离的前沿分配方法。
    2.  **RACER：** 最先进的多机器人协作探索方法，基于分层网格和 CVRP 建模。
    3.  **AEP + DVC：** 结合了信息增益采样（AEP）和动态 Voronoi 分区（DVC）的策略。

### 4. 资源与算力
*   **硬件支持：** 论文提到本地部署的开源模型（如 Qwen2.5-7B, Llama-3.1-8B）运行在配有 **4 张 NVIDIA A5000 GPU** 的服务器上。
*   **推理效率：** 使用语义区域图后，每次协调查询的 Token 数少于 500 个。
*   **耗时：** 云端模型（如 GPT-4o）的推理时间约为 **2.5 秒/次**，本地模型的推理延迟稍高（约 5-6 秒）。协调周期 $T_{coord}$ 设置为 50 步。

### 5. 实验数量与充分性
*   **实验规模：** 进行了 100 组完整场景测试，涵盖了从纯探索到语义搜索的不同任务。
*   **消融实验：** 
    *   针对上下文信息（邻居关系、计划摘要、目标房间类型）进行了消融，验证了各组件对性能的贡献。
    *   对比了多种 LLM 底座（GPT-4o, Claude-Haiku, Gemini, Qwen2.5, Llama-3.1, DeepSeek-R1），评估了模型能力对搜索效率的影响。
*   **评价：** 实验设计较为充分，通过不同规模的环境和多种基准方法的对比，客观地展示了该方法在语义任务上的优势。

### 6. 主要结论与发现
*   **语义搜索效率显著提升：** 在大型环境中，SAGR 相比于 AEP+DVC 和 Hungarian 方法，语义搜索的完成时间分别缩短了 **18.8%** 和 **19.2%**。
*   **探索能力保持竞争力：** 在不含语义目标的纯探索任务中，SAGR 的表现与专门优化的几何方法（如 AEP+DVC）相当，证明了其高层抽象并未牺牲底层的探索效率。
*   **鲁棒性：** 尽管 LLM 具有随机性，但实验表明其在多次运行中的方差与确定性算法相当，表现出良好的协调稳定性。

### 7. 优点
*   **高效的抽象接口：** 语义区域图成功地将高维环境信息压缩为 LLM 可理解的符号化拓扑结构，解决了 LLM 处理原始图像数据时的计算瓶颈。
*   **解耦设计：** 将高层语义推理与底层几何规划解耦，既发挥了 LLM 的常识推理能力，又保留了传统导航算法的可靠性。
*   **即插即用：** 框架不依赖于特定的 LLM，可以根据算力需求灵活切换云端或本地模型。

### 8. 不足与局限
*   **感知依赖：** 系统的表现高度依赖于上游感知模块对房间类型识别的准确性。如果语义标签错误，LLM 的推理将误入歧途。
*   **动态环境挑战：** 目前主要在静态室内环境验证，未充分讨论在包含移动障碍物或动态变化场景下的表现。
*   **实时性瓶颈：** 虽然已大幅优化，但 2.5 秒以上的 LLM 推理延迟在极高动态的 MRS 任务中仍可能成为瓶颈。
*   **仿真与现实差距：** 实验全部在 Habitat 仿真器中完成，尚未在真实的物理机器人集群上进行实地验证。
