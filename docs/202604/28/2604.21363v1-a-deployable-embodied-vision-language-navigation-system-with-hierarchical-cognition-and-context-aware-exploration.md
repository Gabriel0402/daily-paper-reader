## 摘要
弥合具身智能与嵌入式部署之间的差距仍然是智能机器人系统中的一个关键挑战，其中感知、推理和规划必须在计算、内存、能量和实时执行的严格限制下运行。在视觉语言导航（VLN）中，现有方法通常在强大的推理能力与在真实世界平台上的高效部署之间面临根本性的权衡。在本文中，我们提出了一种可部署的具身 VLN 系统，该系统在真实世界机器人平台上实现了高效率和鲁棒的高层推理。为了实现这一目标，我们将系统解耦为三个异步模块：用于持续环境感知的实时感知模块、用于空间语义聚合的记忆集成模块，以及用于高层决策的推理模块。我们增量式地构建了一个认知记忆图来编码场景信息，并将其进一步分解为子图，以便利用视觉语言模型（VLM）进行推理。为了进一步提高导航效率和准确性，我们还利用认知记忆图将探索问题建模为上下文感知的加权旅行维修员问题（WTRP），从而最小化视点的加权等待时间。在仿真和真实世界机器人平台上的广泛实验表明，与现有的 VLN 方法相比，该系统提高了导航成功率和效率，同时在资源受限的硬件上保持了实时性能。

## Abstract
Bridging the gap between embodied intelligence and embedded deployment remains a key challenge in intelligent robotic systems, where perception, reasoning, and planning must operate under strict constraints on computation, memory, energy, and real-time execution. In vision-language navigation (VLN), existing approaches often face a fundamental trade-off between strong reasoning capabilities and efficient deployment on real-world platforms. In this paper, we present a deployable embodied VLN system that achieves both high efficiency and robust high-level reasoning on real-world robotic platforms. To achieve this, we decouple the system into three asynchronous modules: a real-time perception module for continuous environment sensing, a memory integration module for spatial-semantic aggregation, and a reasoning module for high-level decision making. We incrementally construct a cognitive memory graph to encode scene information, which is further decomposed into subgraphs to enable reasoning with a vision-language model (VLM). To further improve navigation efficiency and accuracy, we also leverage the cognitive memory graph to formulate the exploration problem as a context-aware Weighted Traveling Repairman Problem (WTRP), which minimizes the weighted waiting time of viewpoints. Extensive experiments in both simulation and real-world robotic platforms demonstrate improved navigation success and efficiency over existing VLN approaches, while maintaining real-time performance on resource-constrained hardware.

#### 1. 核心问题与整体含义
该论文针对**具身视觉语言导航（VLN）**在真实机器人平台上部署的瓶颈问题展开研究。核心矛盾在于：强大的高层推理（如调用大语言模型 LLM 或视觉语言模型 VLM）通常伴随着高计算延迟，而机器人底层的感知与避障需要极高的实时响应频率。此外，现有的零样本（Zero-shot）导航方法多采用贪婪探索策略，导致在复杂环境中的搜索效率低下。

#### 2. 方法论：HiCo-Nav 系统架构
论文提出了一种**异步解耦的三层分层认知架构**，旨在平衡实时性与深度推理能力：
*   **实时感知层（Layer 1）**：运行频率约 20Hz，负责 SLAM 定位、避障、局部路径规划及物体检测（YOLO-World），确保机器人对动态环境的即时反应。
*   **记忆构建层（Layer 2）**：增量构建**认知记忆图（CMG）**。CMG 是一个包含视觉锚点（关键帧）和物体节点的二分图。通过**整数线性规划（ILP）**优化锚点选择，在保证语义覆盖的前提下最大限度压缩内存占用。
*   **深度推理层（Layer 3）**：运行频率约 1-2Hz。系统将 CMG 分解为多个子图，并根据 CLIP 相似度进行优先级排序，异步调用 VLM（如 Qwen3-Omni）进行高层逻辑推理和目标验证。
*   **上下文感知探索（WTRP 建模）**：将探索问题建模为**加权旅行维修员问题（Weighted Traveling Repairman Problem）**。该方法综合考虑了结构语义传播（物体间的空间关联）、边界外视觉证据以及运动成本，通过全局优化而非贪婪选择来最小化发现目标的期望时间。

#### 6. 主要结论与发现
*   **性能领先**：HiCo-Nav 在所有测试数据集上均优于现有的零样本 VLN 方法，尤其在处理复杂文本描述（TextNav）时优势明显。

*   **全局探索优于贪婪探索**：WTRP 建模显著减少了机器人在不同区域间的无效往返，提升了长程导航的效率。

#### 7. 优点（亮点）
*   **系统工程创新**：成功将高延迟的 VLM 集成到资源受限的嵌入式平台，实现了“高层慢思考”与“底层快反应”的结合。
*   **记忆管理高效**：引入 ILP 优化记忆图，解决了长期导航中语义地图冗余的问题。
*   **探索逻辑严密**：将导航搜索转化为经典的数学优化问题（WTRP），为具身智能的路径规划提供了更科学的理论框架。

#### 8. 不足与局限
*   **维度局限**：系统目前基于 2D 探索图，无法有效处理**跨楼层**导航任务（失败案例分析显示跨楼层是主要失败原因之一）。
*   **感知鲁棒性**：在四足机器人剧烈运动时，图像模糊会导致小物体检测失败。
*   **环境依赖**：语义传播依赖于物体间的共存先验，在物体极度稀疏或布局完全随机的极端环境下，性能可能会受限。
