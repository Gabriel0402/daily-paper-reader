## 摘要
视觉-语言-动作（VLA）模型尽管在短程任务中表现强劲，但在长程操纵任务中却存在系统性失败。我们表明，在当前的反应式执行设置下，仅靠扩展上下文长度无法解决这一问题；相反，它源于执行循环中反复出现的三个缺陷：记忆差距、验证差距和恢复差距。我们提出了 HELM，这是一个与模型无关的框架，通过三个组件解决这些缺陷：一个通过 CLIP 索引关键帧检索关键任务历史的情节记忆模块（EMM）；一个根据观测、动作、子目标和记忆条件上下文在执行前预测动作失败的学习型状态验证器（SV）；以及一个执行回滚和重新规划的治理控制器（HC）。SV 是核心学习贡献：它始终优于基于规则的可行性检查和集成不确定性基准，且其有效性关键取决于对情节记忆的访问。在 LIBERO-LONG 数据集上，HELM 将任务成功率比 OpenVLA 提高了 23.1 个百分点（从 58.4% 提升至 81.5%），而将上下文窗口扩展到 H=32 仅带来 5.4 个百分点的提升，且同等预算的 LoRA 适配仍比 HELM 低 12.2 个百分点。HELM 还提高了 CALVIN 上的长程性能，并显著提升了受控扰动下的恢复成功率。消融实验和机制分析分离了各组件的贡献，我们发布了 LIBERO-Recovery 作为评估长程操纵中故障恢复的扰动注入协议。

## Abstract
Vision-Language-Action (VLA) models fail systematically on long-horizon manipulation tasks despite strong short-horizon performance. We show that this failure is not resolved by extending context length alone in the current reactive execution setting; instead, it stems from three recurring execution-loop deficiencies: the memory gap, the verification gap, and the recovery gap. We present HELM, a model-agnostic framework that addresses these deficiencies with three components: an Episodic Memory Module (EMM) that retrieves key task history via CLIP-indexed keyframes, a learned State Verifier (SV) that predicts action failure before execution from observation, action, subgoal, and memory-conditioned context, and a Harness Controller (HC) that performs rollback and replanning. The SV is the core learning contribution: it consistently outperforms rule-based feasibility checks and ensemble uncertainty baselines, and its effectiveness depends critically on access to episodic memory. On LIBERO-LONG, HELM improves task success rate by 23.1 percentage points over OpenVLA (58.4% to 81.5%), while extending the context window to H=32 yields only a 5.4-point gain and same-budget LoRA adaptation remains 12.2 points below HELM. HELM also improves long-horizon performance on CALVIN and substantially boosts recovery success under controlled perturbations. Ablations and mechanism analyses isolate the contribution of each component, and we release LIBERO-Recovery as a perturbation-injection protocol for evaluating failure recovery in long-horizon manipulation.

### 1. 核心问题与研究动机
*   **核心问题**：现有的视觉-语言-动作（VLA）模型（如 OpenVLA）在短程任务中表现优异，但在长程操纵任务（Long-horizon tasks）中性能大幅下降。
*   **研究背景**：作者发现，单纯增加上下文窗口长度（Context Length）并不能解决长程任务的失败。这种失败源于当前“反应式执行”模式下的三个结构性缺陷：
    1.  **记忆差距 (Memory Gap)**：固定窗口会丢弃已完成子目标的证据，导致模型重复执行已完成的动作。
    2.  **验证差距 (Verification Gap)**：模型直接执行动作而缺乏预执行的可行性检查，导致错误累积。
    3.  **恢复差距 (Recovery Gap)**：一旦发生错误，模型无法检测并回滚，导致连锁失败。

### 2. 方法论：HELM 框架
HELM 是一个模型无关的增强框架，由三个核心组件组成：
*   **情节记忆模块 (EMM)**：
    *   维护一个基于 CLIP 索引的关键帧存储库。
    *   在子目标完成、检测到失败或每隔 20 步时写入关键帧。
    *   通过余弦相似度检索最相关的历史上下文，并将其序列化为文本注入 VLA 的语言输入中。
*   **状态验证器 (SV) —— 核心贡献**：
    *   一个轻量级的 MLP 模型，在动作执行前预测失败概率 $p_{fail}$。
    *   **关键创新**：SV 的输入不仅包含当前观测和动作，还包含从 EMM 检索到的记忆上下文。实验证明，没有记忆辅助的验证器效果会大打折扣。
*   **治理控制器 (HC)**：
    *   管理子目标堆栈。
    *   当 SV 预测失败概率超过阈值时，触发回滚（Rollback）或重规划（Replan）。
    *   提供“前向恢复”变体（HELM-Fwd），适用于动作不可逆的场景。

### 6. 主要结论与发现
*   **结构性差距**：将上下文从 8 扩展到 64 仅提升了极小的成功率，证明长程任务的瓶颈不在于窗口大小，而在于执行逻辑。
*   **HELM 的提升**：在 LIBERO-LONG 上，HELM 将 OpenVLA 的成功率从 **58.4% 提升至 81.5%**（+23.1 pp）。
*   **验证器的有效性**：学习到的 SV 优于基于规则的检查和高成本的集成模型。
*   **模型无关性**：HELM 在 Octo 模型上也实现了显著提升（+21.6 pp），证明了其通用性。

### 7. 优点与亮点
*   **即插即用**：不需要重新训练庞大的 VLA 骨干模型，通过外挂模块实现性能飞跃。
*   **闭环控制**：引入了预执行验证和回滚机制，使 VLA 从“开环反应”转向“闭环监控”。
*   **新评估协议**：提出的 LIBERO-Recovery 为机器人领域研究故障恢复提供了标准化的工具。
*   **高效性**：SV 模块极其轻量，推理延迟低，适合实时部署。

### 8. 不足与局限
*   **数据依赖**：SV 的训练仍需要特定环境的 5 万步 rollout 数据，这限制了其在完全陌生环境下的零样本（Zero-shot）部署能力。
*   **回滚假设**：回滚机制假设动作是可逆的（如放回原处），但在某些现实场景（如液体倾倒、物体破碎）中无法实现。
*   **仿真与现实差距**：实验全部在仿真环境（LIBERO, CALVIN）中完成，尚未在真实机器人上验证视觉噪声和物理不确定性对 SV 准确性的影响。
*   **子目标分解**：依赖于 VLA 自身的子目标分解能力，若初始分解错误，后续恢复也会受限。
