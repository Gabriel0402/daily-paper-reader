## 摘要
当前的具身智能系统在开放世界环境中仍面临高层次推理与低层次物理执行之间的显著差距。尽管视觉-语言-动作（Vision-Language-Action, VLA）模型在感知和直觉响应方面表现出色，但其开环特性限制了长时程任务的性能。结合系统2认知机制的智能体虽能改进规划能力，但通常运行在具有预定义工具包的封闭沙盒中，对真实系统的控制能力有限。OpenClaw 提供了具有完整系统权限的本地运行时环境，但缺乏支持长时间、多机器人执行所需的具身控制架构。因此，我们提出了 ABot-Claw，即 OpenClaw 的具身扩展，集成了以下特性：1）具备能力驱动调度的统一具身接口，用于异构机器人协同；2）以视觉为中心的跨具身多模态记忆，用于持续的上下文保持与具身检索；3）基于评论者（critic）的闭环反馈机制，结合通用奖励模型用于在线进展评估、局部修正与重新规划。通过跨越 OpenClaw 层、共享服务层和机器人具身层的解耦架构，ABot-Claw 实现了真实世界交互，闭合了从自然语言意图到物理行动的环路，并支持在开放、动态环境中逐步自进化的机器人智能体。

## Abstract
Current embodied intelligent systems still face a substantial gap between high-level reasoning and low-level physical execution in open-world environments. Although Vision-Language-Action (VLA) models provide strong perception and intuitive responses, their open-loop nature limits long-horizon performance. Agents incorporating System 2 cognitive mechanisms improve planning, but usually operate in closed sandboxes with predefined toolkits and limited real-system control. OpenClaw provides a localized runtime with full system privileges, but lacks the embodied control architecture required for long-duration, multi-robot execution. We therefore propose ABot-Claw, an embodied extension of OpenClaw that integrates: 1) a unified embodiment interface with capability-driven scheduling for heterogeneous robot coordination; 2) a visual-centric cross-embodiment multimodal memory for persistent context retention and grounded retrieval; and 3) a critic-based closed-loop feedback mechanism with a generalist reward model for online progress evaluation, local correction, and replanning. With a decoupled architecture spanning the OpenClaw layer, shared service layer, and robot embodiment layer, ABot-Claw enables real-world interaction, closes the loop from natural language intent to physical action, and supports progressively self-evolving robotic agents in open, dynamic environments.

## 二、方法论与系统设计

### 1. 总体架构思想  
- ABot-Claw 是对 OpenClaw 框架的**具身化扩展**，以实现持久、自主、协作的机器人智能体。  
- 核心思想：通过 **统一接口、多模态记忆、闭环反馈机制** 打造具身智能的“三位一体”基础设施。

### 2. 三大关键组件  

#### （1）统一具身接口与异构机器人调度（Heterogeneous Embodiment Integration）
- **目标**：实现不同机器人（机械臂、双足人形、四足机器人）间的共同调度与任务协作。
- **技术要点**：
  - 基于 ROS 的**统一接口映射机制**，不同硬件通过技能（skills）层抽象，如 *navigate*, *manipulate*, *inspect*。
  - 运行时由中央代理负责任务分配，考虑 **能力、位置、负载、优先级** 推理，实现跨具身的动态任务路由。
  - 支持并行与串行协作：例如移动机器人运输物体→机械臂执行精密操作。
  
#### （2）视觉中心的跨具身多模态记忆（Visual-Centric Multimodal Memory）
- **功能**：提供机器人长期、共享环境记忆，实现跨机器人感知与语义一致性。
- **存储类型**：
  - *图像语义记忆*（高维视觉嵌入）
  - *关键帧记忆*（稀疏轨迹快照）
  - *对象中心记忆*（物体类别、位姿、检测来源）
  - *空间锚点记忆*（如“厨房”、“走廊”等语义地标）
- **检索方式**：
  - *语义检索*：基于视觉语言编码器的向量匹配；
  - *结构化检索*：按对象类别、时间窗口、坐标约束过滤；
  - *导航结果标准化输出*：所有检索结果均返回可执行的3D位姿。
- **共享机制**：所有机器人共享同一记忆空间，避免重复探索，支持长期跨时空环境认知。

#### （3）基于评论者的闭环反馈机制（Closed-loop Feedback with Generalist Critic）
- **目标**：解决执行过程的非确定性与鲁棒性不足问题。
- **机制**：
  - 使用**通用奖励模型（Generalist Reward Model）**评估当前状态与目标达成度；
  - 输出实时进度评分，用于确定三种决策：
    - 任务完成；
    - 局部修正（重试、调整姿态、更新视角）；
    - 重新规划（replanning）。
- **自学习闭环**：执行日志与反馈信号被持续记录，用于未来决策优化。

### 3. 工程层级设计（工程三层架构）
- **OpenClaw层**：任务解析、调度与代码生成（自动生成可执行 Python 脚本）。
- **共享服务层**：集中部署感知、记忆与评估模块（服务化设计，支持模型快速替换与集中管理 GPU）。
- **机器人层**：ROS接口驱动真实执行，封装硬件差异，实现跨平台通用性。
