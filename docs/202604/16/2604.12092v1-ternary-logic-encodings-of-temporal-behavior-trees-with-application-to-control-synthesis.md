## 摘要
行为树（Behavior Trees，BTs）为设计者提供了一种直观的图形化界面，用于构建自主系统的长时间规划。为了确保其正确性与安全性，严格的形式化模型与验证技术至关重要。时序行为树（Temporal Behavior Trees，TBTs）通过利用现有的时序逻辑形式化方法来描述和验证BTs的执行过程，提供了一种有前景的途径。然而，目前的分析局限于离线的事后分析与轨迹修复。本文提出了一种基于三值信号时序逻辑（Signal Temporal Logic, STL）的TBTs重构方法，使其适用于控制综合。三值逻辑引入了第三个真值——“未知”（Unknown），形式化地捕捉了轨迹尚未完全满足或违反规范的情形。我们提出了基于混合整数线性编码的部分轨迹STL与TBTs三值逻辑建模方法，使得对线性动力系统的控制策略可以通过混合整数优化以构造性正确的方式获得。我们通过解决最优控制问题展示了该框架的实用性。

## Abstract
Behavior Trees (BTs) provide designers an intuitive graphical interface to construct long-horizon plans for autonomous systems. To ensure their correctness and safety, rigorous formal models and verification techniques are essential. Temporal BTs (TBTs) offer a promising approach by leveraging existing temporal logic formalisms to specify and verify the executions of BTs. However, this analysis is currently limited to offline post hoc analysis and trace repair. In this paper, we reformulate TBTs using a ternary-valued Signal Temporal Logic (STL) amenable for control synthesis. Ternary logic introduces a third truth value \textit{Unknown}, formally capturing cases where a trajectory has neither fully satisfied or dissatisfied a specification. We propose mixed-integer linear encodings for partial trajectory STL and TBTs over ternary logic allowing for correct-by-construction control strategies for linear dynamical systems via mixed-integer optimization. We demonstrate the utility of our framework by solving optimal control problems.

- **总体目标**：  
  本文提出一种在 **三值信号时序逻辑 (ternary Signal Temporal Logic, STL)** 框架下重新定义 TBT 的方法，使其既能表示任务序列与分支逻辑，又能实现混合整数优化求解，从而为线性系统生成满足任务规范的最优控制策略。

## 二、方法论：核心思想与技术细节

- **核心思想**：  
  以三值逻辑（Kleene’s strong logic, K₃）形式化行为树节点的三种状态：
  - **True（成功）**
  - **False（失败）**
  - **Unknown（运行中或未确定）**

  该三值逻辑可自然表达行为树运行时的“Running”状态，从而统一 STL 与 BT 的语义。

- **主要技术方法**：
  1. **三值 STL 语义扩展**：  
     将 STL 的布尔语义扩展到三值逻辑域，使轨迹部分可判定，即在未完成整个时序区间时仍可得到“未知”状态。
  2. **混合整数线性编码（MIL Encoding）**：  
     将每个逻辑状态映射为整数值：  
     - True → +1  
     - Unknown → 0  
     - False → −1  
     通过新增的二进制变量保证唯一激活，并结合 “large‑M” 技术将逻辑约束转化为线性不等式。
  3. **时序行为树算子编码**：  
     - **Sequence (→)**：通过存在量词的序列联接表示任务依次成功的关系。  
     - **Selector (?)**：逻辑上是 Sequence 的对偶，表示并行或故障回退机制。  
     两者的整数编码形式通过嵌套的最值（min/max）操作实现时序短路（short‑circuit）特征。
  4. **混合整数二次规划（MIQP）求解**：  
     将控制系统模型（线性离散系统）与 TBT 逻辑约束结合形成 MIQP 问题，包括：
     \[
     \min_{\{u_t\}} \sum_t u_t^T R u_t
     \]
     subject to 动力学方程与逻辑约束 \(x,t |= φ\)，保证轨迹满足 TBT 规范。

- **逻辑与复杂度分析**：  
  - STL 编码的复杂度为 \(O(T^2)\)。  
  - Sequence/Selector 操作由于多层嵌套，复杂度增至 \(O(T^{k+1})\)，其中 \(k\) 是子公式数。  
  - 尽管复杂度提升，三值逻辑扩展使任何部分轨迹均可判定，避免不可决问题。
