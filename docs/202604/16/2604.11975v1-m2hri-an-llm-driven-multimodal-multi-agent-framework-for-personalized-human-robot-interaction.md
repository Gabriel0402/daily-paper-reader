## 摘要
多机器人系统在家庭和医院等社会环境中具有巨大的潜力，但现有的多机器人研究往往将机器人视为功能上相同的个体，忽略了机器人个性如何影响用户感知，以及当这种个性存在时，协调机制如何塑造多机器人行为。为此，我们提出了 M2HRI——一个基于大型语言模型的多模态多智能体框架，为每个机器人赋予独特的个性与长期记忆，并设计了基于这些差异的协调机制。在一个控制用户研究（n = 105）中，我们在多智能体人机交互（HRI）场景中发现：由大型语言模型驱动的个性特征具有显著可区分性并提升了互动质量；长期记忆改进了个性化与偏好意识；集中式协调机制显著减少了重叠，同时提升了整体交互质量。总体而言，这些结果表明，智能体的个体差异与结构化协调对于实现连贯且社会适宜的多智能体人机交互至关重要。项目网站与代码可访问：https://project-m2hri.github.io/。

## Abstract
Multi-robot systems hold significant promise for social environments such as homes and hospitals, yet existing multi-robot works treat robots as functionally identical, overlooking how robots individual identity shape user perception and how coordination shapes multi-robot behavior when such individuality is present. To address this, we introduce M2HRI, a multimodal multi-agent framework built on large language models that equips each robot with distinct personality and long-term memory, alongside a coordination mechanism conditioned on these differences. In a controlled user study (n = 105) in a multi-agent human-robot interaction (HRI) scenario, we find that LLM-driven personality traits are significantly distinguishable and enhance interaction quality, long-term memory improves personalization and preference awareness, and centralized coordination significantly reduces overlap while improving overall interaction quality. Together, these results demonstrate that both agent individuality and structured coordination are essential for coherent and socially appropriate multi-agent HRI. Project website and code are available at https://project-m2hri.github.io/.

## 二、方法论与框架设计

### 2.1 核心思想  
M2HRI 的核心理念是：  
> “在多机器人协作中，同时维持**个体独立性**与**群体协调性**。”

通过 LLM 为每个机器人赋人格、整合多模态感知与长期记忆，并使用中央协调代理管理交互流程。

### 2.2 框架总体结构  

如论文 Fig. 2 所示，M2HRI 包含以下五大模块：

1. **Perception（感知）**  
   - 使用 Vision-Language Model 处理语音与视觉输入。  
   - 将语义结果映射为文本化场景描述，用于下游推理。  

2. **Memory（记忆）**  
   - 分为三层结构：  
     - 工作记忆（Working memory）——短期对话上下文；  
     - 语义记忆（Semantic memory）——持久的世界知识与用户信息；  
     - 情景记忆（Episodic memory）——过往交互经历与偏好记录；  
   - 使用 LLM 控制的记忆检索器执行存取与更新。

3. **Personality（人格）**  
   - 基于五大人格模型（Big Five：O、C、E、A、N），每个维度取值 1–5。  
   - 将人格向量映射为自然语言描述注入系统提示（prompt），影响 LLM 输出风格。  

4. **Interaction Planner（交互规划）**  
   - LLM 结合感知结果、记忆、人格与动作库生成交互策略。  
   - 输出结构化动作序列（如「手势→发声→点头」）。

5. **Execution（执行）**  
   - 由 NAO 机器人执行计划行为，实现语言与动作的同步表达。

### 2.3 多智能体协调机制  
- **集中式协调（Central Coordinator）**：  
  - 各机器人并行形成本地观察；  
  - 协调器使用 LLM 综合语境与人格特征，为每个机器人计算响应得分 \( r_{i,t} \in [0,1] \)；  
  - 得分高于阈值的机器人轮流发言，避免重叠。

### 2.4 系统实现细节  
- 平台：2 台 NAO 机器人；  
- 感知：RGB 摄像头 + 麦克风 → 外部工作站；  
- 模型：  
  - **LLM：GPT‑4o‑mini**（通过 AutoGen 实现）；  
  - **VLM：支持语音与视觉融合**；  
  - **记忆数据库：ChromaDB + text‑embedding‑3‑large**；  
- 通信：ZMQ 进行消息同步，NaoQi 控制执行。

## 三、实验设计

### 3.1 实验类型与目标
设定三个研究问题（RQ）：
1. **RQ1：**机器人个性是否增强可区分性与互动参与？  
2. **RQ2：**长期记忆是否提升个性化与偏好意识？  
3. **RQ3：**协调机制是否提高交互质量？

### 3.2 实验条件与样本  
- **参与者**：共 105 名（男女比例约 6:3，年龄 18–77）。  
- **实验设计**：7 个条件 × 15 人 = 105 人（被试间 / 被试内设计混合）。  
  - 五种人格条件（O、C、E、A、N）；
  - 一种记忆对比条件（有 / 无长期记忆）；
  - 一种协调对比条件（有 / 无协调机制）。

### 3.3 实验形式  
- 使用**视频观测法**而非实时交互，确保对照公平；  
- 每段视频长约 10 分钟，对话脚本一致，仅变量差异。

### 3.4 任务设计  
- 各人格条件使用情境话题以突出特质：
  - 如 **开放性（Openness）**：讨论“非传统爱好”；  
  - **责任心（Conscientiousness）**：讨论“学习与聚会的抉择”；  
  - **神经质（Neuroticism）**：讨论“失去宠物的反应”；  
- 记忆条件：是否能回忆先前提及的偏好（如“最喜欢的颜色/食物”）；  
- 协调条件：讨论“AI 在教育中的应用”时控制机器人发言顺序。

### 3.5 评价指标  
- Likert 五点量表评估：可区分性、一致性、参与度；  
- 二项测试评估人格识别正确率；  
- t 检验与 Wilcoxon 检验评估显著性。

## 六、主要结论与发现

1. **人格可识别性（RQ1）**  
   - 用户可显著区分大部分人格特质（除责任心外均显著 p<0.001）。  
   - 人格表现稳定一致；  
   - 开放性与神经质最显著提升交互参与度。  

2. **长期记忆效应（RQ2）**  
   - 显著提升信息回忆准确度与用户偏好感知度（p<0.001）；  
   - 也提升自然度，但效果较中等；  
   - 体现记忆在个性化定制中的关键作用。  

3. **协调机制效应（RQ3）**  
   - 显著提升对话流畅度、应答合适性与无重叠性（p<0.001）；  
   - 尤以避免发言重叠为最显著改进；  
   - 证明集中协调对多机器人社交一致性至关重要。

4. **综合结论**  
   - 个体化（人格 + 记忆）与组织化（协调机制）是多智能体 HRI 的两大互补支柱。  
   - 缺一将导致交互碎片化或混乱。
