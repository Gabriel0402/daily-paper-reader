## 摘要
由多模态大型语言模型（Multimodal Large Language Models, MLLMs）驱动的自主图形用户界面（Graphical User Interface, GUI）智能体使得终端设备上的数字自动化成为可能。尽管参数和数据的规模化带来了显著性能提升，先进方法在资源受限设备上的部署成本仍然高昂。面对复杂的真实场景，轻量级 GUI 智能体在端到端的回合式学习中受限于容量不足和任务可扩展性差，难以适应多智能体系统（Multi-Agent Systems, MAS），而训练多个特定技能专家的成本亦居高不下。我们能否在成本与可扩展性的两难中取得有效平衡，使轻量级 MLLMs 能参与现实的 GUI 工作流？为解决这些挑战，我们提出了 LAMO 框架，使轻量级 MLLM 具备 GUI 专属知识与任务可扩展性，通过多角色协同（multi-role orchestration）拓展其在 GUI 自动化中的能力边界。LAMO 结合了面向角色的数据合成与两阶段训练方案：（i）使用困惑度加权交叉熵（Perplexity-Weighted Cross-Entropy）优化进行知识蒸馏和视觉感知增强的监督微调；（ii）面向角色的强化学习以实现协作式探索。基于 LAMO，我们开发了一个任务可扩展的原生 GUI 智能体 LAMO-3B，支持单体执行和 MAS 式协同。当与先进的规划器配对作为即插即用的策略执行器时，LAMO-3B 可持续受益于规划器的进步，从而获得更高的性能上限。大量静态与在线评估验证了我们设计的有效性。

## Abstract
Autonomous Graphical User Interface (GUI) agents powered by Multimodal Large Language Models (MLLMs) enable digital automation on end-user devices. While scaling both parameters and data has yielded substantial gains, advanced methods still suffer from prohibitive deployment costs on resource-constrained devices. When facing complex in-the-wild scenarios, lightweight GUI agents are bottlenecked by limited capacity and poor task scalability under end-to-end episodic learning, impeding adaptation to multi-agent systems (MAS), while training multiple skill-specific experts remains costly. Can we strike an effective trade-off in this cost-scalability dilemma, enabling lightweight MLLMs to participate in realistic GUI workflows? To address these challenges, we propose the LAMO framework, which endows a lightweight MLLM with GUI-specific knowledge and task scalability, allowing multi-role orchestration to expand its capability boundary for GUI automation. LAMO combines role-oriented data synthesis with a two-stage training recipe: (i) supervised fine-tuning with Perplexity-Weighted Cross-Entropy optimization for knowledge distillation and visual perception enhancement, and (ii) reinforcement learning for role-oriented cooperative exploration. With LAMO, we develop a task-scalable native GUI agent, LAMO-3B, supporting monolithic execution and MAS-style orchestration. When paired with advanced planners as a plug-and-play policy executor, LAMO-3B can continuously benefit from planner advances, enabling a higher performance ceiling. Extensive static and online evaluations validate the effectiveness of our design.

## 二、方法论：LAMO 框架

LAMO（**Lightweight Agent Multi-role Orchestration**）是一种为轻量级 MLLM 设计的**多角色协同训练与运行框架**  
核心思想：**将端到端的复杂 GUI 推理与执行过程分解为多个角色任务，通过参数共享实现高效多任务协同。**

### 1. 总体架构

LAMO 包含两大阶段：
1. **角色导向数据合成（Role-oriented Data Synthesis）**  
   构建五类 GUI 子任务数据，用于训练模型在不同角色中学习关键技能。  
   - **ATA（Action–Tool Alignment）**：从高层指令映射到原子动作；  
   - **LCC（Logic-Consistent Chain-of-Thought）**：保持跨步骤逻辑一致的推理链；  
   - **SU（Screen Understanding）**：整体界面语义描述；  
   - **GP（Goal Planning）**：任务分解与策略制定；  
   - **SG（Screen Grounding）**：视觉定位与坐标预测。  
   ➜ 数据由教师模型生成（Qwen2.5-VL-72B-Instruct、Gemini-2.5-Pro）。

2. **两阶段训练流程**
   - **阶段一：监督微调 (SFT)**  
     使用提出的 **困惑度加权交叉熵（Perplexity-Weighted Cross-Entropy, PWCE）** 优化目标，提高模型对高不确定性（如坐标）token 的关注，强化视觉-空间对齐能力。  
     - 核心思想：高困惑度区域（例如坐标或不确定感知token）权重更高，以提升低置信区域的学习效率。  
   - **阶段二：多任务强化学习 (RL)**  
     在五类角色任务上构建任务池与函数池，采用 GRPO 策略进行**协作式探索**，通过任务特定 reward 实现知识迁移与性能优化。  

### 3. 推理模式与多角色协同

LAMO-3B 智能体支持三种推理模式：
- **End-to-End Reasoning**：单模型执行感知、计划与动作；  
- **Multi-Agent System (MAS)**：在同一参数骨干上实现四角色协同（Observer、Planner、Allocator、Executor），基于上下文工程自组织协作；  
- **Policy Executor Mode**：作为执行器接入大型规划模型（如 Gemini-2.5-Pro、GPT-5），实现“规划器–执行器”混合智能体结构，获得更高性能上限。

**算法流程简述**  
1. 观察（Observer）：环境感知与界面描述；  
2. 规划（Planner）：目标分解与策略说明；  
3. 分配（Allocator）：基于上下文与历史分配可执行动作；  
4. 执行（Executor）：执行具体 PyAutoGUI 原子操作。
