## 摘要
视觉-语言-动作（Vision-Language-Action，VLA）模型已成为具身智能的主流。近期的 VLA 模型将输入模态从仅二维扩展到二维加三维范式，形成了多视觉模态 VLA（MVLA）模型。尽管空间感知能力有所提升，MVLA 因模态扩展导致输入 Token 数量增加而面临更高的加速需求。Token 剪枝是一种针对 MVLA 模型的有效优化方法。然而，现有的 Token 剪枝方案是为仅二维的 VLA 模型设计的，忽略了二维与三维模态显著性差异。本文沿着 MVLA 模型中多模态数据的应用流程，提出三阶段分析方法以捕捉二维/三维模态显著性的差异与动态。基于此，我们设计了相应的三阶段 Token 剪枝框架，以实现最优的二维/三维 Token 选择与高效剪枝。实验结果表明，我们的框架在精度几乎无损的情况下可实现最高达 2.55 倍的推理加速，仅增加 5.8% 的计算开销。我们的代码即将发布。

## Abstract
Vision-Language-Action (VLA) models have emerged as the mainstream of embodied intelligence. Recent VLA models have expanded their input modalities from 2D-only to 2D+3D paradigms, forming multi-visual-modal VLA (MVLA) models. Despite achieving improved spatial perception, MVLA faces a greater acceleration demand due to the increased number of input tokens caused by modal expansion. Token pruning is an effective optimization methods tailored to MVLA models. However, existing token pruning schemes are designed for 2D-only VLA models, ignoring 2D/3D modality salience differences. In this paper, we follow the application process of multi-modal data in MVLA models and develop a tri-stage analysis to capture the discrepancy and dynamics of 2D/3D modality salience. Based on these, we propose a corresponding tri-stage token pruning framework for MVLA models to achieve optimal 2D/3D token selection and efficient pruning. Experiments show that our framework achieves up to a 2.55x inference speedup with minimal accuracy loss, while only costing 5.8% overhead. Our Code is coming soon.

## 一、研究问题与背景

- **核心问题**：随着 Vision-Language-Action（VLA） 模型从单视觉模态（仅 2D 图像）扩展到多视觉模态（2D + 3D）的 MVLA 模型，模型的空间感知性能显著提升，但推理时计算量暴涨，导致实时性下降。
- **主要瓶颈**：3D 模态引入的额外 Token 数量使推理延迟升高，无法满足机器人控制实时要求（要求 20–30 Hz，而现有模型仅能达到 3–5 Hz）。
- **研究动机**：现有 Token 剪枝方法仅针对 2D 模型设计，忽略了 2D 和 3D 模态的重要性（salience）差异。本文旨在通过建立“模态显著性”度量机制，实现高效、精度保持的多模态 Token 剪枝。

## 二、方法论：三阶段模态显著性感知与剪枝框架

### 1. 总体思想

- **目标**：通过分阶段分析和动态预测，识别推理全过程中 2D 与 3D 模态的显著性差异与时序变化，从而实现最优剪枝策略。
- **结构**：整个推理分为三阶段：
  1. 数据预处理阶段（Data Preprocessing）  
  2. 语义合成阶段（Semantic Synthesis）  
  3. 动作迭代阶段（Action Iteration）
- 每阶段分别引入显著性分析、阈值判断与动态调整机制，形成完整的三阶段 Token 剪枝流程。

### 2. 技术细节

#### （1）阶段一：数据预处理显著性分析
- 利用模型输出特征（最后层的特征向量）计算 2D/3D 模态显著性。
- 以 **L1 范数** 衡量特征强度：
  \[
  MS1_{modal} = avg(\sum_p ||f_{modal,p}||_1 / (||f_{2D,p}||_1 + ||f_{3D,p}||_1))
  \]
- 双阈值机制：  
  - 当 3D 比例 < τ₂D → 保留 2D；  
  - 当 3D 比例 > τ₃D → 保留 3D；  
  - 介于两者之间 → 保留两者。

#### （2）阶段二：语义合成显著性分析
- 将场景划分为三个语义集：背景 (S_bg)、机器人主体 (S_rob)、目标物体 (S_obj)。
- 使用注意力分数矩阵进行聚类（1D K-Means, K=3）。
- 设计模态分解公式：
  \[
  \alpha_{Sem}^{3D} = \alpha_{Sem}^{3D,para} + \alpha_{Sem}^{3D,ortho}
  \]
  - 平行部分为与 2D 重叠的冗余信息；  
  - 正交部分为 3D 的独特信息。
- 根据分解得到的平均比例（MS2₂D、MS2₃D）动态决定保留模式。

#### （3）阶段三：动作迭代与时序动态
- 引入 **指数移动平均 (EMA)** 平滑机制，避免剪枝掩码随迭代出现“闪烁”：
  \[
  \hat{M}_{t,3D} = \beta \hat{M}_{t-1,3D} + (1-\beta) M_{t,3D}
  \]
- 动作迭代阶段依据平滑后的显著性动态调整剪枝预算，兼顾稳定与高效。

#### （4）综合融合与最终掩码生成
- 融合阶段将阶段一的模态候选集合（CS1）与阶段二的语义候选集合（CS2）进行交集融合：
  \[
  C_{final,p} = CS1_p \cap CS2_p
  \]
- 若交集为空则启用冲突回退机制，确保关键语义区保留必要 Token。
- 输出最终剪枝掩码 (Mask₂D, Mask₃D)，用于模型的编码器前端。
