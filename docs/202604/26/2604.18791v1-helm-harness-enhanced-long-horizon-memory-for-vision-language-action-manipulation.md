## 摘要
视觉-语言-动作（VLA）模型尽管在短程任务中表现强劲，但在长程操控任务中却存在系统性失效。我们表明，在当前的反应式执行设置下，仅靠延长上下文长度无法解决这一失效问题；相反，它源于执行循环中反复出现的三个缺陷：记忆鸿沟（memory gap）、验证鸿沟（verification gap）和恢复鸿沟（recovery gap）。我们提出了 HELM，这是一个与模型无关的框架，通过三个组件解决这些缺陷：一个通过 CLIP 索引关键帧检索关键任务历史的情景记忆模块（EMM）；一个学习得到的状态验证器（SV），它根据观测、动作、子目标和基于记忆的上下文在执行前预测动作失败；以及一个执行回滚和重规划的增强控制器（HC）。SV 是核心的学习贡献：它始终优于基于规则的可行性检查和集成不确定性基准，且其有效性关键取决于对情景记忆的访问。在 LIBERO-LONG 数据集上，HELM 将任务成功率相比 OpenVLA 提升了 23.1 个百分点（从 58.4% 提升至 81.5%），而将上下文窗口延长至 H=32 仅带来 5.4 个百分点的提升，且同等预算下的 LoRA 适配仍比 HELM 低 12.2 个百分点。HELM 还提升了在 CALVIN 上的长程表现，并在受控扰动下显著增强了恢复成功率。消融实验和机制分析分离了各组件的贡献，我们还发布了 LIBERO-Recovery，作为一种用于评估长程操控中失败恢复的扰动注入协议。

## Abstract
Vision-Language-Action (VLA) models fail systematically on long-horizon manipulation tasks despite strong short-horizon performance. We show that this failure is not resolved by extending context length alone in the current reactive execution setting; instead, it stems from three recurring execution-loop deficiencies: the memory gap, the verification gap, and the recovery gap. We present HELM, a model-agnostic framework that addresses these deficiencies with three components: an Episodic Memory Module (EMM) that retrieves key task history via CLIP-indexed keyframes, a learned State Verifier (SV) that predicts action failure before execution from observation, action, subgoal, and memory-conditioned context, and a Harness Controller (HC) that performs rollback and replanning. The SV is the core learning contribution: it consistently outperforms rule-based feasibility checks and ensemble uncertainty baselines, and its effectiveness depends critically on access to episodic memory. On LIBERO-LONG, HELM improves task success rate by 23.1 percentage points over OpenVLA (58.4% to 81.5%), while extending the context window to H=32 yields only a 5.4-point gain and same-budget LoRA adaptation remains 12.2 points below HELM. HELM also improves long-horizon performance on CALVIN and substantially boosts recovery success under controlled perturbations. Ablations and mechanism analyses isolate the contribution of each component, and we release LIBERO-Recovery as a perturbation-injection protocol for evaluating failure recovery in long-horizon manipulation.

### 1. 核心问题与整体含义（研究动机和背景）
现有的视觉-语言-动作（VLA）模型（如 OpenVLA, Octo）在短程任务中表现出色，但在长程操控任务（Long-horizon tasks）中性能大幅下降。作者发现，这种失效并非单纯因为上下文窗口（Context Window）不够长，而是源于当前“反应式执行”模式下的三个结构性缺陷：
*   **记忆鸿沟（Memory Gap, $F_M$）：** 固定长度的上下文窗口会丢弃已完成子目标的证据，导致机器人重复执行已完成的动作。
*   **验证鸿沟（Verification Gap, $F_V$）：** 动作在执行前缺乏可行性检查，导致错误动作静默执行并累积误差。
*   **恢复鸿沟（Recovery Gap, $F_R$）：** 一旦发生错误，模型无法检测并回滚，导致级联式失败。

### 2. 方法论：核心思想与关键技术
HELM 提出了一个模型无关的框架，通过三个核心组件增强现有的 VLA 模型：
*   **情景记忆模块 (EMM)：** 维护一个基于 CLIP 索引的关键帧存储库。它在子目标完成、检测到失败或每隔 20 步时写入关键帧。检索时，通过 CLIP 嵌入的余弦相似度提取最相关的历史信息，并将其序列化为文本注入 VLA 的语言输入中。
*   **状态验证器 (SV) —— 核心贡献：** 这是一个轻量级的 MLP 模型，用于在动作执行前预测失败概率 $p_{fail}$。它不仅考虑当前观测和动作，还结合了从 EMM 检索到的记忆上下文。这种“记忆增强型”验证是其区别于传统规则检查的关键。
*   **增强控制器 (HC)：** 负责协调整个循环。如果 SV 预测失败概率超过阈值，HC 会触发回滚（Rollback）或重规划。它维护一个子目标栈，并根据记忆中的检查点引导机器人恢复到之前的正确状态。

### 3. 实验设计
*   **数据集/场景：**
    *   **LIBERO-LONG：** 包含 10 个长程任务，平均 5.8 个子目标。
    *   **CALVIN (ABC → D)：** 评估在未见环境中的长程链式操作能力。
    *   **LIBERO-Recovery (本文提出)：** 一个新的评估协议，通过在随机子目标边界注入扰动（如物体位移、夹爪状态翻转）来测试模型的恢复能力。
*   **对比方法：** 包含 9 个基准，如 OpenVLA (H=8/32/64)、Oracle 记忆、基于规则的验证器、集成模型（Ensemble）、同等数据预算下的 LoRA 微调、以及 Reflexion（自我反思）等。

### 4. 资源与算力
*   **硬件：** 实验主要在单台 **NVIDIA A100 (80GB)** GPU 上运行。
*   **训练时长：** 状态验证器（SV）的训练需要约 5 万个 rollout 步骤的数据，训练时间约为 **2 小时**。
*   **推理开销：** 整个框架每步增加约 **12 毫秒** 的延迟，比单纯运行 VLA 骨干网络增加了约 15% 的计算开销。

### 5. 实验数量与充分性
*   **实验规模：** 进行了大规模测试，LIBERO-LONG 包含 500 个评估回合，CALVIN 包含 1000 个链式测试，所有实验均运行了 3 个随机种子并报告了标准差。
*   **充分性：** 实验设计非常全面。作者不仅对比了 SOTA 模型，还通过消融实验拆解了 EMM、SV 和回滚机制的各自贡献。此外，通过对比 H=64 的长上下文模型和同等预算的 LoRA 微调，有力地证明了 HELM 的改进并非来自简单的参数增加或数据拟合，而是结构性的优化。

### 6. 主要结论与发现
*   **长上下文并非万能：** 将上下文窗口从 8 增加到 32 仅提升了 5.4% 的成功率，而 HELM 提升了 23.1%。
*   **HELM 性能卓越：** 在 LIBERO-LONG 上，将 OpenVLA 的成功率从 58.4% 提升至 **81.5%**。
*   **验证器的关键性：** 学习得到的状态验证器（SV）优于基于规则的检查和集成不确定性方法，且其性能高度依赖于记忆模块提供的上下文。
*   **恢复能力：** 在注入扰动的测试中，HELM 的恢复成功率（RSR）远高于基准模型。

### 7. 优点与亮点
*   **模型无关性：** 作为一个“外挂”框架，HELM 可以适配 OpenVLA、Octo 等多种不同的 VLA 骨干网络。
*   **闭环验证：** 引入了“执行前验证”的概念，这比事后反思（Post-hoc reflection）更高效，能有效防止错误传播。
*   **低成本：** 相比于重新训练大型 VLA 模型，训练轻量级 SV 的成本极低，且推理延迟在可接受范围内。

### 8. 不足与局限
*   **数据依赖：** SV 的训练仍需要约 5 万步的环境特定数据，这限制了其在完全零样本（Zero-shot）场景下的部署。
*   **回滚假设：** 默认的回滚机制假设动作是可逆的。虽然提出了 HELM-Fwd（前向恢复）变体来应对不可逆场景，但其性能（76.3%）略低于回滚模式。
*   **仿真与现实差距：** 实验全部在仿真环境（LIBERO, CALVIN）中完成，真实机器人的视觉噪声和物理不确定性可能会对 SV 的准确性提出更高挑战。
