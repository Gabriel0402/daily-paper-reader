## 摘要
近期的导航系统在基准测试中取得了优异成绩，但在实际部署中往往表现出明显的“走走停停”现象。这一瓶颈源于“感知-推理-执行”循环仍然是阻塞式的：在每次获得新观测后，控制器必须等待感知、传输和推理完成后才能继续运动。因此，仅降低动作生成成本并不能消除冗余的等待。为解决这一问题，我们提出了 LiveVLN，这是一个无需训练的框架，通过为预训练的视觉语言模型（VLM）导航器增强多步动作延续能力，实现更连续的具身导航。LiveVLN 不再在每个完整的感知与推理周期中停顿，而是将执行过程与新观测的处理重叠，使得更新后的未来动作能在当前可执行前缀耗尽之前完成交付。这种设计使动作在运动过程中保持持续可用，减少了空闲等待，并实现了更平滑的在线执行。该框架在运行时操作，并可与兼容的预训练 VLM 导航器集成。在 R2R 和 RxR 数据集上，LiveVLN 在保持基准性能的同时，减少了等待时间并提高了动作可用性。在实际部署中，它将平均回合等待时间缩短了高达 77.7%，并将 StreamVLN 和 NaVIDA 的实际运行时间分别缩短了 12.6% 和 19.6%，从而在部署过程中实现了更连贯的执行。代码可在 https://github.com/NIneeeeeem/LiveVLN 获取。

## Abstract
Recent navigation systems achieve strong benchmark results, yet real-world deployment often remains visibly stop-and-go. This bottleneck arises because the sense-inference-execution loop is still blocking: after each new observation, the controller must wait for sensing, transmission, and inference before motion can continue. Reducing action-generation cost alone therefore does not remove redundant waiting. To address this issue, we present LiveVLN, a training-free framework for more continuous embodied navigation by augmenting pretrained VLM navigators with multi-step action continuation. Instead of pausing for each full sense-and-inference round, LiveVLN overlaps execution with the processing of newly arrived observations, allowing refreshed future actions to be handed off before the current executable prefix is exhausted. This design keeps actions continuously available during motion, reducing idle waiting and enabling smoother online execution. The framework operates at runtime and can be integrated with compatible pretrained VLM navigators. Across R2R and RxR, LiveVLN preserves benchmark performance while reducing waiting time and improving action availability. In real-world deployments, it cuts average episode waiting time by up to $77.7\%$ and shortens wall-clock episode time by $12.6\%$ on StreamVLN and $19.6\%$ on NaVIDA, yielding more coherent execution during deployment. Code is available at https://github.com/NIneeeeeem/LiveVLN.

*   **核心瓶颈**：传统的“感知-推理-执行”循环是**阻塞式**的。机器人必须在执行完当前动作后，停下来等待下一轮的感知数据传输和模型推理完成。即使推理速度很快，这种串行结构也会导致显著的空闲等待时间（实验显示原生 NaVIDA 部署中等待时间占比高达 30.5%）。
*   **研究目标**：提出一种无需重新训练模型的运行时框架，通过重叠执行与推理过程，实现连续、平滑的具身导航。

### 2. 方法论：LiveVLN 框架
*   **核心思想**：将执行线程与推理线程解耦。在机器人执行当前动作序列的同时，后台线程利用最新的观测值异步计算下一组动作，确保在当前动作耗尽前完成“接力”。
*   **关键技术细节**：
    *   **三段式动作状态管理**：将动作序列划分为：
        1.  **已执行动作 (Executed Actions)**：当前周期内已完成的部分。
        2.  **保护缓冲区 (Guard Buffer)**：已提交给控制器执行的动作前缀，其时长需覆盖下一轮推理的延迟。
        3.  **可修正尾部 (Revisable Tail)**：尚未提交给控制器的动作，可根据最新的感知结果进行更新或替换。
    *   **受保护的移交 (Guarded Handoff)**：当后台推理完成时，如果保护缓冲区尚未耗尽，则进行平滑切换，将新生成的动作与当前执行状态对齐。
    *   **实时自适应 (Real-time Adaptation)**：动态估计感知和推理的延迟（使用指数移动平均），并据此调整保护缓冲区的长度，以最小化动作延迟并最大化动作的新鲜度。
*   **算法流程**：线程 A 持续执行保护缓冲区中的动作；线程 B 获取新观测并进行异步推理。推理完成后，根据实测延迟更新下一轮的保护预算，并更新动作序列。

*   **即插即用**：无需重新训练模型，可直接适配任何输出多步动作序列的 VLM 导航器。
*   **显著改善用户体验**：解决了具身智能中长期存在的“思考时必须停下”的尴尬，使机器人动作更接近人类的自然反应。
*   **鲁棒性**：通过实时自适应机制，能够应对网络抖动和计算负载波动带来的延迟变化。

### 8. 不足与局限
*   **动作类型限制**：该框架假设导航器能够生成多步动作序列（Action Continuation），对于仅能输出单步动作的传统模型适配性较差。
*   **物理执行上限**：虽然消除了计算等待，但总任务时间的缩短受限于机器人的物理运动速度，效率提升存在物理天花板。
*   **环境复杂度风险**：在极高动态或极窄空间内，较长的保护缓冲区可能导致机器人无法及时根据最新观测做出避障反应（尽管论文通过“可修正尾部”缓解了此问题）。
