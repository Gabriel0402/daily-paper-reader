## 摘要
鸟瞰图（BEV）表示是自动驾驶中三维感知的主流范式，它提供了一个统一的空间画布，使检测和分割特征在相同的物理坐标系下实现几何对齐。然而，现有的雷达-相机融合方法通常将这些任务分开处理，从而错失了在任务间共享互补信息的机会：检测特征编码了物体层面的几何信息，可增强分割边界的精度，而分割特征则提供了密集的语义上下文，可用于强化检测。我们提出了一个双向模块——\textbf{CTAB}（Cross-Task Attention Bridge，跨任务注意力桥），通过共享 BEV 空间中的多尺度可变形注意力机制，在检测和分割分支之间交换特征。CTAB 被集成到一个多任务框架中，结合基于 Instance Normalization 的分割解码器与可学习的 BEV 上采样，以提供更精细的 BEV 表示。在 nuScenes 数据集上，CTAB 相较于联合多任务基线在 7 个类别上的分割性能有所提升，同时检测性能基本保持不变。在一个包含 4 个类别（可行驶区域、行人横道、人行道、车辆）的子集上，我们的联合多任务模型在保持相当 mIoU 水平的同时，能够同时实现三维检测。

## Abstract
Bird's-eye-view (BEV) representations are the dominant paradigm for 3D perception in autonomous driving, providing a unified spatial canvas where detection and segmentation features are geometrically registered to the same physical coordinate system. However, existing radar-camera fusion methods treat these tasks in isolation, missing the opportunity to share complementary information between them: detection features encode object-level geometry that can sharpen segmentation boundaries, while segmentation features provide dense semantic context that can anchor detection. We propose \textbf{CTAB} (Cross-Task Attention Bridge), a bidirectional module that exchanges features between detection and segmentation branches via multi-scale deformable attention in shared BEV space. CTAB is integrated into a multi-task framework with an Instance Normalization-based segmentation decoder and learnable BEV upsampling to provide a more detailed BEV representation. On nuScenes, CTAB improves segmentation on 7 classes over the joint multi-task baseline at essentially neutral detection. On a 4-class subset (drivable area, pedestrian crossing, walkway, vehicle), our joint multi-task model reaches comparable mIoU on 4 classes while simultaneously providing 3D detection.

## 一、研究动机与核心问题

- **背景**：  
  自动驾驶感知系统通常依赖鸟瞰视角（BEV）表示以统一三维理解，但“检测（Detection）”和“分割（Segmentation）”仍多被独立建模。  
  雷达传感器具有深度与速度稳健性，结合多视角相机的密集视觉信息，可带来高鲁棒性与低成本的 3D 感知方案。

- **核心问题**：  
  现有雷达-相机方法大多在 **检测或分割单一任务** 上设计，任务间缺乏显式的信息交流，导致互补语义未被利用。例如：  
  - 检测特征包含物体几何中心、尺度与朝向等信息，可帮助分割边界刻画；  
  - 分割特征具备语义上下文，可为检测提供道路与语义先验。  

- **研究目标**：  
  构建一个共享 BEV 表示的多任务框架，支持三维检测与语义分割的协同学习，探索 **跨任务特征交互** 对整体 BEV 感知性能的提升效果。

## 二、方法论与技术方案

### 1. 整体框架

- 基于 **RCBEVDet** 检测模型扩展出多任务结构。  
- 输入为多视角相机影像与多帧雷达点云。  
- 输出包括：
  1. 3D 目标检测结果（通过 CenterHead）；
  2. BEV 语义分割图（7 类与 4 类设置）。

### 2. 模型关键组件

#### （1）共享 BEV Backbone

- 图像分支：ResNet-50 + BEVDepth4D 深度估计与体素投影。  
- 雷达分支：RadarBEVNet，多帧编码 + 融合注入模块。  
- 融合策略：**多模态交叉注意力**（Cross-Attention）实现雷达与相机特征融合，输出 `F_bev ∈ R^(256×128×128)`。

#### （2）分割解码器（Seg Decoder）

- 三个残差块 + Instance Normalization（IN）结构，用于从 BEV 特征恢复语义信息。  
- IN 替代 BN，适应多视角、单样本批次的训练环境。  
- 输出中间特征 `F_seg` 提供给跨任务模块。

#### （3）BEV 上采样模块（Upsampling）

- 将分辨率由 128×128 (0.8 m/像素) 提升至 200×200 (0.5 m/像素)。  
- 使用双线性插值 + 可学习残差块实现细节恢复，仅增加约 0.30M 参数。  
- 目标：更好区分行人过道、车道线、停车区域等细小结构。

#### （4）跨任务注意力桥（CTAB）

- **核心创新模块**，通过双向多尺度可变形注意力（MSDA）交换检测与分割特征。  
- 机制如下：
  - 检测与分割特征投射至共享维度 \( d = 128 \) 后展平；
  - 双向注意力：
    - Det→Seg：分割特征查询检测特征，引入几何形状；
    - Seg→Det：检测特征查询分割特征，注入语义上下文；
  - 注意力参数：\( h=8 \) 个头、\( K=4 \) 个偏移点；
  - 输出经 GroupNorm、3×3 卷积及 **可学习门控参数 σ(g)** 控制信息量；
  - 门控初始化 \( g=-2.0 \Rightarrow σ≈0.12 \)，训练中自适应调节跨任务权重。

- 参数量：仅 0.58M，约占整体网络的 1.4%，高效轻量。

#### （5）损失函数与优化

- **多任务损失**：采用 *Homoscedastic Uncertainty Weighting*  
  \[
  L = \frac{1}{2σ^2_{det}}L_{det} + \logσ_{det} + \frac{1}{2σ^2_{seg}}L_{seg} + \logσ_{seg}
  \]
  自动平衡检测与分割的权重。
- 检测：CenterHead 热力图 + L1 回归 + 深度损失。  
- 分割：Sigmoid Focal（α=0.25, γ=3） + Dice Loss。  
- 优化器：AdamW + 余弦退火 + EMA。

## 六、主要结论与发现

- CTAB 模块在多任务学习中显著改善了分割性能，保持检测精度基本不变：
  - mIoU-7：+0.6 提升 (42.9 → 43.5)，mIoU-4：+0.6；
  - 检测 NDS 几乎不变 (55.7 → 55.6)，mAOE（方向误差）略优 (0.408 → 0.400)。
- 性能提升主要集中在**细窄类别**：
  - 人行横道 +1.8、停止线 +1.8、车道线划分 +0.7；
- 门控分析显示：
  - 分割分支门控提升更快（0.17→0.26），表示从检测特征获益更多；
  - 检测分支变化较小（0.13→0.18），证明 CTAB 作用为“温和的特征精炼”。
- 参数效率高，增加仅 5.8% 参数，实现单次前向即可联合推理两任务。
