## 摘要
行为树（Behavior Trees, BTs）为设计者提供了一种直观的图形界面，用于构建自主系统的长时规划。为了确保其正确性与安全性，严格的形式化模型和验证技术至关重要。时序行为树（Temporal BTs, TBTs）通过利用现有的时序逻辑形式体系来指定和验证行为树的执行，提供了一种前景良好的方法。然而，目前此类分析主要局限于事后离线分析和轨迹修复。本文使用三值信号时序逻辑（Signal Temporal Logic, STL）重新表述了 TBTs，使其适用于控制综合。三值逻辑引入了第三个真值——“未知”（Unknown），以形式化地刻画轨迹既未完全满足也未完全违背规范的情况。我们提出了在三值逻辑下针对部分轨迹 STL 和 TBTs 的混合整数线性编码，使得通过混合整数优化可为线性动力系统构造出正确性保证的控制策略。我们通过求解最优控制问题展示了该框架的有效性。

## Abstract
Behavior Trees (BTs) provide designers an intuitive graphical interface to construct long-horizon plans for autonomous systems. To ensure their correctness and safety, rigorous formal models and verification techniques are essential. Temporal BTs (TBTs) offer a promising approach by leveraging existing temporal logic formalisms to specify and verify the executions of BTs. However, this analysis is currently limited to offline post hoc analysis and trace repair. In this paper, we reformulate TBTs using a ternary-valued Signal Temporal Logic (STL) amenable for control synthesis. Ternary logic introduces a third truth value \textit{Unknown}, formally capturing cases where a trajectory has neither fully satisfied or dissatisfied a specification. We propose mixed-integer linear encodings for partial trajectory STL and TBTs over ternary logic allowing for correct-by-construction control strategies for linear dynamical systems via mixed-integer optimization. We demonstrate the utility of our framework by solving optimal control problems.

## 一、研究背景与核心问题

- **研究动机**：  
  行为树（Behavior Trees, BTs）作为一种模块化、层次化的控制结构，被广泛用于自主系统（如机器人）任务规划中。然而，传统 BT 的设计与验证多依赖人工经验，缺乏形式化保证。  
  现有的 *时序行为树（Temporal Behavior Trees, TBTs）* 将 BT 语义嵌入到时序逻辑（如 STL）中，实现了对任务执行的形式化描述，但目前主要应用于**离线验证或轨迹修复**，难以实现在线控制策略的自动合成。

- **核心问题**：  
  如何将 TBTs 转化为一种**可用于控制合成（control synthesis）**的形式体系，使得生成的控制策略既满足复杂的时序行为约束，又能按构造保证正确性与安全性。

## 二、方法与技术路线

### 1. 核心思想

- 作者提出使用 **三值信号时序逻辑（Ternary Signal Temporal Logic, 𝐾₃-STL）** 重新表述 TBT：
  - 三值逻辑包含 {True, False, Unknown} 三种真值；
  - “Unknown” 用于描述在有限或不确定信息下，尚未能确定公式是否满足的情况；
  - 该扩展允许对**部分轨迹（partial trajectory）**进行逻辑推理。

- 在此基础上，作者设计**混合整数线性编码（Mixed-Integer Linear Encoding, MILP）** 模型，
  实现从形式规范到控制策略的“正确性可证”求解。

### 2. 关键要素与公式

- **系统模型**：  
  研究线性离散时间系统  
  \[
  x_{t+1} = A_t x_t + B_t u_t,\quad y_t = C_t x_t + D_t u_t
  \]

- **优化问题定义**（Problem 1）：  
  给定系统动态方程、二次型控制代价 \( u^T R u \)，以及 TBT 规范公式 \( \varphi \)，求解满足约束的最优控制序列：
  \[
  u^* = \arg\min_{u_t} \sum_{t=0}^{T-1} u_t^T R u_t
  \quad \text{s.t.} \quad x_{t+1}=A_t x_t+B_t u_t,\; x,t \models \varphi
  \]

- **三值 STL 编码要点**：
  - 信号谓词以容差 δ 定义三值区间：
    - \( f(x_t) ≥ δ \Rightarrow True \)
    - \( -δ < f(x_t) < δ \Rightarrow Unknown \)
    - \( f(x_t) ≤ -δ \Rightarrow False \)
  - 时间算子（Always, Eventually）针对部分轨迹建立双时间索引编码；
  - 各逻辑算子（¬, ∧, ∨）映射为整数变量组合。

- **行为树控制节点的逻辑编码**：
  - Sequence 节点 → 连续满足的存在约束（存在 τ 使得前子任务完成后下一任务开始）；
  - Selector 节点 → 逻辑或（任一子任务成功即可）；
  - 序列运算满足**结合律**，支持递归结构的嵌套实现；
  - 整体可转化为 MIQP 问题以求解。

- **复杂度分析**：
  - STL 编码复杂度 ~ \( O(T^2) \)
  - Sequence/Selector 编码复杂度 ~ \( O(T^{k+1}) \)（k 为子任务数）
  - 复杂度劣化明显，但可在有限时域下求得全局最优解。
