## 摘要
世界模型评估通常通过与目标状态或观测的重叠来评分预测的未来。在稀疏变化世界中，这会将复制的持久状态转化为表面上的准确性。我们提出ScratchWorld，一个离线诊断基准，将Scratch项目视为可执行世界，并使用固定Scratch虚拟机生成重放验证的转换、隐藏变量、因果轨迹和反事实结果。ScratchWorld评估下一状态预测、长程跟踪、因果事件归因和反事实预测；每个重放验证的目标可以以原始程序、结构化状态、自然语言或渲染输入模态呈现，我们的实验使用结构化状态条件。其主要状态度量是值感知的变更字段F1，仅对变更字段及其执行值给予分数。在659个示例的发布中，七个提示语言/推理模型在仅状态的部分观察压力测试中最多达到13.8%的值感知变更字段F1。同实例复制诊断使重叠混淆具体化：复制输入状态达到98.0%的隐含全状态字段准确度和0.0%的变更字段F1，在真实项目上膨胀最大。辅助诊断区分隐藏状态展开漂移、干预敏感性、因果归因和扰动鲁棒性。在这些设置中，模型通常对动作或干预做出反应，但不遵循决定变更值的可执行规则。

## Abstract
World-model evaluations often score a predicted future by overlap with a target state or observation. In sparse-change worlds, this can turn copied persistent state into apparent accuracy. We introduce ScratchWorld, an offline diagnostic benchmark that treats Scratch projects as executable worlds and uses a pinned Scratch VM to produce replay-verified transitions, hidden variables, causal traces, and counterfactual outcomes. ScratchWorld evaluates next-state prediction, long-horizon tracking, causal event attribution, and counterfactual prediction; each replay-verified target can be presented under raw-program, structured-state, natural-language, or rendered input modalities, and our experiments use the structured-state condition. Its primary state metric is value-aware changed-field $F_1$, which gives credit only for the changed field and its executed value. In a 659-example release, seven prompted language/reasoning models reach at most 13.8% value-aware changed-field $F_1$ in a state-only partial-observation stress test. A same-instance copy diagnostic makes the overlap confound concrete: copying the input state reaches 98.0% implied full-state field accuracy and 0.0% changed-field $F_1$, with the largest inflation on real projects. Auxiliary diagnostics separate hidden-state rollout drift, intervention sensitivity, causal attribution, and perturbation robustness. Across these settings, models often react to actions or interventions without following the executable rule that determines the changed value.

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **核心问题**：当前世界模型（world models）评估通常使用未来状态与目标状态的重叠度（overlap）作为评分标准。在稀疏变化的世界中，模型可以通过复制大部分持久状态来获得高评分，而忽略了由动作引发的关键变化。这种评估方法无法区分“记住静态背景”和“正确计算动态变化”的能力，导致虚假的准确性。
- **研究动机**：针对规划、控制和交互决策，世界模型需要识别哪些状态变量发生改变、由哪条可执行规则产生该变化、以及干预下结果如何不同。现有视频基准缺乏动作和隐藏状态，机器人基准成本高，游戏基准常依赖奖励/任务成功，无法揭示错误来源。
- **整体含义**：提出一个可执行、可重放、可检查的诊断基准，严格测试模型是否真正“计算”了动作的可执行后果，而非仅仅预测一个看似合理的未来状态。

## 2. 论文提出的方法论

### 核心思想
将 Scratch 项目作为可执行世界，利用固定版本的 Scratch VM（scratch-vm@5.0.300）对每个动作-状态-结果进行重放验证，从而获得精确的 ground truth。所有评估实例均来自重放，而非表面观测匹配。

### 关键技术细节

- **状态表示**：结构化状态包含 sprite 位置、方向、隐藏变量、列表、克隆、广播、活动脚本等字段。平均每个输入有 235.8 个字段，但每次动作平均只改变 8.5 个字段。
- **评估指标**：
  - **值感知变更字段 F1（Value-aware Changed-field F1, F1^VA）**：仅当模型正确识别出发生变更的字段且预测的值与执行值一致（误差容限内）时，才给予分数。这是主要指标。
  - **存在性变更字段 F1（Presence-only Changed-field F1, F1^pres）**：仅奖励识别出哪个字段变了，不考虑值是否正确。
  - **全状态字段准确率（Full-state field accuracy）**：作为复制敏感性诊断，不用于评估变预测能力。
- **实例构建**：通过记录执行轨迹，提取一步转移、多步展开、因果事件窗口和反事实对。反事实干预（如改变初始变量、阈值、动作）后重放得到新的 ground truth。
- **模态封装**：每个实例支持五种输入模态：原始 JSON、结构化状态表、自然语言摘要、渲染截图、短执行视频。实验中主要使用结构化状态模态。

### 算法流程（文字说明）
1. **项目过滤**：从本地 Scratch 项目库中选择具有有意义状态改变行为的项目（游戏、模拟、动画等），去除无法执行、依赖未支持扩展、不稳定复现或近重复的项目。
2. **运行时插桩与重放**：对 Scratch 运行时插桩以记录结构化轨迹（状态、动作、观测、事件）。通过固定 VM 重放验证每个实例，仅保留重放结果与录制目标一致的实例。
3. **实例提取**：从轨迹中构建一步转移、长期展开、因果事件窗口、反事实对。保留无变化结果（空 delta）作为有效标签。
4. **语义扰动**：修改名称、坐标、阈值、造型等表面属性，构建扰动版本进行记忆化审计。
5. **评分**：使用上述指标对模型输出评分，未解析的输出视为错误。

## 3. 实验设计

### 数据集/场景
- **统一测试集**：共 659 个重放验证示例，来自 153 个匿名化项目（31 个真实项目，122 个合成项目）。分为：
  - 285 个 next-state 实例（带有状态 delta 目标）
  - 374 个 long-horizon 实例（带有完整最终状态目标）
  - 53 个 next-state 标签为空 delta（无字段变化）
- **合成机制家族**：9 个受控机制家族，涵盖运动、背景/造型变化、广播、克隆、碰撞、延迟、列表操作、多变化转移、隐藏变量。
- **任务**：
  1. **Next-state prediction**（下一状态预测）：给定当前状态和动作，预测下一状态。
  2. **Causal event attribution**（因果事件归因）：给定动作序列和目标事件，预测事件是否发生并识别因果链。
  3. **Counterfactual prediction**（反事实预测）：给定轨迹和干预，预测干预后的结果。
  4. **Long-horizon state tracking**（长程状态跟踪）：给定初始状态和长时间动作序列，预测最终状态（H=2,5,10,15,20）。

### Benchmark 与对比方法
- **Benchmark**：ScratchWorld 本身作为评估基准。
- **对比模型**：7 个当前主流语言/推理模型（作为提示预测器）：
  - GPT-5.5, GPT-5.4, DeepSeek-V4-Pro, Qwen3.7-Max, MiMo-V2.5-Pro, MiMo-V2.5, MiMo-V2-Omni
  - 使用最强推理设置（若有），标准 API 配置，大输出预算。
- **基线**：同实例复制先验（copy prior）——直接将输入状态作为预测输出。

### 实验充分性与公平性
- 所有实验在统一 265 个 next-state 实例（7 个模型都输出）上比较主结果。
- 额外诊断实验（扰动、因果、反事实、长程）使用不同子集，但均为 replay-verified。
- 提示中不含目标或标签，无法解析的输出视为错误，评估公平。
- 项目级划分防止数据泄露。语义扰动用于审计表面记忆。
- 但仅使用状态部分观察模式（隐藏程序规则），可能低估模型在完整程序信息下的能力。另外，缺乏训练后的控制导向世界模型（如 Dreamer）的对比，公平性有限。

## 6. 论文的主要结论与发现

1. **可执行状态变化预测对当前模型极为困难**：在状态部分观察条件下，7 个模型在 265 个 next-state 实例上的值感知 F1 仅 7.5%~13.8%；存在性 F1 约为其两倍（平均 22.1% vs 10.4%），表明模型更容易识别哪些字段变了，但无法正确计算新值。
2. **真实项目 vs 合成项目差距显著**：真实项目上值感知 F1 仅 3.8%~10.1%，而合成家族上为 20.8%~27.1%，因为真实项目包含大量持久状态、复杂交互。
3. **全状态重叠指标严重误导**：复制输入状态在 next-state 上达到 98.0% 全状态字段准确率，但变更字段 F1 为 0%。在真实项目中膨胀最大（98.5%）。
4. **模型对干预敏感但无法正确计算结果**：反事实敏感性 65.7%~85.1%，但精确 delta 匹配仅 16.0%~26.0%。
5. **事件检测相对容易，因果链归因极差**：事件检测准确率 62.7%~79.1%，但精确规则链匹配最高仅 10.0%。
6. **长程展开中隐藏状态快速漂移**：H=2 时隐藏状态准确率 40.3%~44.4%，H=20 时降至 2.8%~18.1%。
7. **表面扰动不是低性能的主因**：6/7 模型扰动后性能非正提升，GPT-5.4 仅 +2.9%，说明瓶颈在于深层计算而非提示敏感性。

## 7. 优点

- **严格的执行验证**：所有标签来自固定 VM 重放，确保 ground truth 的精确性和可重复性。
- **切中要害的指标设计**：值感知变更字段 F1 直接针对“是否计算了正确的值”，避免了全状态重叠的混淆。
- **多维诊断能力**：不仅评估下一状态，还涵盖因果、反事实、长程隐藏状态、扰动鲁棒性，提供错误模式分解。
- **可分离输入模态**：同一 ground truth 可用五种模态呈现，允许隔离不同感知渠道的影响。
- **开放可复用**：Scratch 项目易于获取和扩展，基准构建流水线完整，便于后续研究加入训练模型、代码解析器等。

## 8. 不足与局限

- **领域限制**：仅适用于 Scratch 这类块编程世界，不涵盖物理交互、机器人控制、开放视频预测等。可执行程序有限制，不能完全代表通用世界模型。
- **规模较小**：当前发布仅 659 个示例、153 个项目，真实项目子集仅 31 个。大模型可能遇到的多样性不足。
- **因果链可靠性**：Scratch 的并发、嵌套广播可能使因果链模糊，且 200 个人类标注校准虽然做了，但有限。论文排除了无法可靠恢复的实例，限制了覆盖。
- **缺乏训练模型基线**：仅评估了提示式推理模型，未包括如 Dreamer、基于潜变量的强化学习世界模型等，不公平于那些未在语言数据上预训练的模型。
- **部分观察设置**：大多数实验隐藏了完整程序规则（仅给当前状态和动作），可能低估了模型在获得完整程序信息时的能力。
- **资源披露不足**：未提供计算资源细节，增加了复现难度。
- **偏差风险**：项目来自 Scratch 社区，可能偏向教育性和简单游戏，真实世界应用的外推性需要更多验证。
