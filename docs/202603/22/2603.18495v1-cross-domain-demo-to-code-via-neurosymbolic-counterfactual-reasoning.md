## 摘要
视觉语言模型（VLM）的最新进展使得视频指令机器人编程成为可能，允许智能体解释视频演示并生成可执行的控制代码。我们将视频指令机器人编程公式化为一个跨域适配问题，其中演示与部署之间的感知和物理差异会导致程序不匹配。然而，当前的 VLM 缺乏重新表述因果依赖关系并在这种域偏移下实现任务兼容行为所需的程序理解能力。我们提出了 NeSyCR，这是一个神经符号反事实推理框架，能够实现任务程序的可验证适配，从而提供可靠的代码策略合成。NeSyCR 将视频演示抽象为捕捉底层任务程序的符号轨迹。给定部署观测，它推导出揭示跨域不兼容性的反事实状态。通过带有可验证检查的符号状态空间探索，NeSyCR 提出了程序修订方案，以恢复与演示程序的一致性。NeSyCR 在任务成功率上比最强基准模型 Statler 提高了 31.14%，在模拟和真实世界的操纵任务中均展现出鲁棒的跨域适配能力。

## Abstract
Recent advances in Vision-Language Models (VLMs) have enabled video-instructed robotic programming, allowing agents to interpret video demonstrations and generate executable control code. We formulate video-instructed robotic programming as a cross-domain adaptation problem, where perceptual and physical differences between demonstration and deployment induce procedural mismatches. However, current VLMs lack the procedural understanding needed to reformulate causal dependencies and achieve task-compatible behavior under such domain shifts. We introduce NeSyCR, a neurosymbolic counterfactual reasoning framework that enables verifiable adaptation of task procedures, providing a reliable synthesis of code policies. NeSyCR abstracts video demonstrations into symbolic trajectories that capture the underlying task procedure. Given deployment observations, it derives counterfactual states that reveal cross-domain incompatibilities. By exploring the symbolic state space with verifiable checks, NeSyCR proposes procedural revisions that restore compatibility with the demonstrated procedure. NeSyCR achieves a 31.14% improvement in task success over the strongest baseline Statler, showing robust cross-domain adaptation across both simulated and real-world manipulation tasks.

### 2. 方法论：NeSyCR 框架
NeSyCR 结合了神经模型（VLM）的常识推理能力和符号工具（Symbolic Tool）的逻辑严谨性，分为两个阶段：

*   **阶段一：符号世界模型构建（Symbolic World Model Construction）**
    1.  **符号状态转换**：利用 VLM 将演示视频的每一帧抽象为符号化的场景图（Scene Graphs），包含物体及其空间关系（如 `OnTopOf`, `InsideOf`）。
    2.  **动力学重建**：VLM 根据状态变化推断动作算子（Action Operators），定义每个动作的前置条件（Preconditions）和效果（Effects）。
    3.  **一致性验证**：使用符号工具（如 VAL）进行前向模拟，确保推断出的符号轨迹在逻辑上是自洽的。

*   **阶段二：神经符号反事实推理（Neurosymbolic Counterfactual Adaptation）**
    1.  **反事实识别**：将部署环境的初始观测作为“反事实状态”，符号工具模拟演示程序，识别出哪些步骤因前置条件不满足（如抽屉被挡住、工具位置改变）而导致失败。
    2.  **反事实探索**：针对不兼容的步骤，VLM 提出替代动作（增加或删除步骤），符号工具迭代验证这些修改的因果有效性。
    3.  **代码合成**：将修正后的符号规程翻译为最终的可执行 Python 代码策略。

### 6. 主要结论与发现
*   **性能飞跃**：NeSyCR 在任务成功率（SR）上比最强基准 Statler 平均提高了 **31.14%**。
*   **鲁棒性**：在环境障碍增加和机器人硬件改变（如从手指夹具换成吸盘）的情况下，NeSyCR 表现出极强的适配能力。
*   **因果修正**：实验证明，单纯依靠 VLM 的常识（如 Critic-V）容易产生逻辑错误，而 NeSyCR 的符号验证循环能有效拦截非法动作。
*   **局限性发现**：当部署任务与演示任务的复杂度差异过大时，性能会显著下降，表明反事实推理仍依赖于演示提供的基础因果链。

### 7. 优点
*   **可验证性**：引入符号工具对 VLM 生成的步骤进行硬约束验证，解决了大模型“幻觉”导致的机器人碰撞或非法操作问题。
*   **反事实框架**：将跨域问题转化为“如果...会怎样”的推理，比传统的特征对齐或行为克隆更具解释性。
*   **长程任务处理**：通过符号抽象，能够处理超过 100 步 API 调用的复杂任务，而不会像纯神经方法那样在长序列中迷失。

### 8. 不足与局限
*   **谓词依赖**：系统依赖于预定义的符号谓词集（如 `InsideOf`, `Open`），如果遇到谓词集未覆盖的新物理属性，系统可能失效。
*   **感知精度**：虽然实验证明了对场景图扰动有一定鲁棒性，但第一步“视频到符号”的转换精度仍是整个系统的瓶颈。
*   **实时性**：由于涉及多次 VLM 调用和迭代探索，目前的推理延迟（约 2 分钟）尚不支持高频动态环境下的实时重规划。
*   **应用限制**：主要适用于结构化程度较高的操纵任务，对于非刚体或极度不规则物体的处理能力尚未验证。
