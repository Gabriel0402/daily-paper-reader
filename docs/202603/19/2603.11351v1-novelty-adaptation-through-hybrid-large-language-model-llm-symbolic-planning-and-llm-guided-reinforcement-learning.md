## 摘要
在动态开放世界环境中，自主智能体经常遇到新颖性，这阻碍了它们寻找实现目标规划的能力。具体而言，当机器人的规划领域缺乏使其能够与环境中新颖物体进行适当交互的算子（operators）时，传统的符号规划器无法生成规划。我们提出了一种神经符号架构，该架构集成了符号规划、强化学习和大语言模型（LLM），以学习如何处理新颖物体。特别地，我们利用 LLM 的常识推理能力来识别缺失的算子，使用符号 AI 规划器生成规划，并编写奖励函数以引导强化学习智能体学习新识别算子的控制策略。我们的方法在连续机器人领域的算子发现和算子学习方面均优于现有最先进的方法。

## Abstract
In dynamic open-world environments, autonomous agents often encounter novelties that hinder their ability to find plans to achieve their goals. Specifically, traditional symbolic planners fail to generate plans when the robot's planning domain lacks the operators that enable it to interact appropriately with novel objects in the environment. We propose a neuro-symbolic architecture that integrates symbolic planning, reinforcement learning, and a large language model (LLM) to learn how to handle novel objects. In particular, we leverage the common sense reasoning capability of the LLM to identify missing operators, generate plans with the symbolic AI planner, and write reward functions to guide the reinforcement learning agent in learning control policies for newly identified operators. Our method outperforms the state-of-the-art methods in operator discovery as well as operator learning in continuous robotic domains.

在开放世界（Open-world）环境中，自主机器人经常会遇到预定义知识库中未包含的新颖物体（如新出现的抽屉、盖子等）。传统的**任务与运动规划（TAMP）**框架依赖于预定义的符号算子（Operators，如 PDDL 定义），一旦环境出现新颖性，规划器会因缺乏相应的操作逻辑而失效。
本文的核心问题是：**如何让机器人在发现规划失败时，自主识别缺失的算子，并学习到操作这些新物体的底层控制策略？** 研究动机在于结合 LLM 的常识推理能力与符号规划的严谨性，解决机器人对未知环境的自适应问题。

#### 2. 方法论：核心思想与关键技术
论文提出了一种名为 **Plan-Learn-Execute** 的神经符号循环架构，包含以下核心组件：

*   **混合 LLM-符号规划（Hybrid LLM-Symbolic Planning）：**
    *   当传统规划器报错时，系统调用 LLM（GPT-o3）分析当前状态、目标及新物体。
    *   LLM 负责定义缺失算子的 PDDL 结构（前提条件、参数、效果）。
    *   采用**自我一致性提示（Self-consistency Prompting）**采样多个候选算子，并利用符号规划器进行“前瞻搜索”验证，确保新增算子能打通通往目标的路径。
*   **LLM 引导的子目标强化学习（LLM-guided RL）：**
    *   将 LLM 定义的算子效果分解为有序的**子目标课程（Sub-goal Curriculum）**。
    *   利用 LLM（GPT-4o-mini）为每个子目标编写密集的**奖励塑造函数（Reward Shaping Functions）**代码。
    *   **并行筛选机制：** 为每个子目标启动多个 RL 智能体（每个对应不同的奖励函数候选），定期剔除表现最差的候选者（类似遗传算法），最终保留最优策略。
*   **算法流程：** 搜索与提示（识别算子） -> 符号验证（确认可行性） -> 奖励生成（代码编写） -> 强化学习（策略训练） -> 执行。

*   **实验环境：** 使用 `mimicgen` 机器人仿真环境，涉及连续动作空间的操作任务。
*   **测试场景（按难度递增）：**

    4.  **Coffee-Drawer (Hard)：** 打开新抽屉并取出咖啡荚（随机探索极难成功）。
*   **对比基准（Benchmarks）：**

    *   **LEAGUE-Sparse (LS)：** 仅在完成所有效果后给予稀疏奖励的基线。
    *   **Reward Machine (RM)：** 仅提供子目标奖金（Bonus）但不含 LLM 塑造的密集奖励。

*   **硬件配置：** 实验在一台 Ubuntu 20.04 服务器上运行，配置为 24 核 Intel i9-12900K CPU 和 **NVIDIA GeForce RTX 4090 GPU**。
*   **模型使用：** 规划阶段使用 GPT-o3，奖励函数生成阶段使用 GPT-4o-mini。
*   **训练时长：** 算子发现实验中，传统 OD 方法在复杂任务中耗时超过 7 小时仍失败，而本文方法在几百秒内即可完成算子识别。RL 训练部分，每个算子学习设定了步数上限（如 100 万步）。

*   **实验规模：** 每个任务均运行了 **10 个随机种子**，并进行了统计显著性分析（Wilcoxon 符号秩检验）。

*   **客观性：** 通过引入多个基准方法（LS, RM, OD）并展示奖励函数候选者的筛选曲线，客观证明了 LLM 密集奖励对收敛速度的贡献。

#### 6. 主要结论与发现
*   **算子识别：** LLM 能够准确识别缺失算子并生成符合 PDDL 规范的定义，成功率远高于依赖随机探索的传统方法（OD 在中高难度任务中成功率为 0）。
*   **学习效率：** LLM 引导的密集奖励显著加速了 RL 的收敛。在最难的“抽屉”任务中，本文方法达到了近 100% 的成功率，而基线方法几乎无法完成。
*   **鲁棒性：** 并行生成多个奖励函数并动态剔除差分项的策略，有效解决了 LLM 生成代码可能存在的逻辑错误或低效问题。

#### 7. 优点
*   **神经符号结合：** 巧妙地将 LLM 的常识（解决“做什么”）与符号规划的逻辑（解决“怎么组合”）以及 RL 的控制（解决“怎么做”）结合。
*   **自动化程度高：** 实现了从发现问题到编写奖励代码再到训练策略的全流程自动化，减少了人工干预。
*   **解决冷启动：** 通过 LLM 引导，解决了连续空间中 RL 智能体在复杂长程任务中“盲目探索”的难题。

*   **感知假设：** 论文假设机器人已经拥有能够检测新物体并将其映射到现有谓词（Predicates）的感知模块，但在现实复杂场景中，**谓词发现（Predicate Invention）**本身也是一个挑战。

*   **成本与延迟：** 依赖高性能 LLM（如 GPT-o3）会产生 API 调用成本和推理延迟，可能影响实时响应性。
