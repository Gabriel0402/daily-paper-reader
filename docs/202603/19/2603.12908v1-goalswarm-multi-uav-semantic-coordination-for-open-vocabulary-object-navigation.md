## 摘要
协作式视觉语义导航是空中机器人团队在未知环境中运行的基础能力。然而，由于在机载设备上部署重型感知模型的计算限制以及去中心化多智能体协作的复杂性，实现鲁棒的开放词汇目标导航仍然具有挑战性。我们提出了 GoalSwarm，这是一个用于零样本语义目标导航的全去中心化多无人机框架。每架无人机通过投影来自空中视角的深度观测，协作构建一个共享的轻量级 2D 自顶向下语义占据地图，在保留基本几何和语义结构的同时，消除了完整 3D 表示的计算负担。GoalSwarm 的核心贡献有三点：(1) 集成了零样本基础模型 SAM3，用于开放词汇检测和像素级分割，无需特定任务训练即可实现开放词汇目标识别；(2) 提出了一种贝叶斯价值图，将多视角检测置信度融合为每个像素的目标相关性分布，通过上置信界（UCB）探索实现知情的边界评分；(3) 提出了一种去中心化协作策略，结合了语义边界提取、基于测地线路径代价的成本-效用竞价以及空间分离惩罚，以最大限度地减少集群中的冗余探索。

## Abstract
Cooperative visual semantic navigation is a foundational capability for aerial robot teams operating in unknown environments. However, achieving robust open-vocabulary object-goal navigation remains challenging due to the computational constraints of deploying heavy perception models onboard and the complexity of decentralized multi-agent coordination. We present GoalSwarm, a fully decentralized multi-UAV framework for zero-shot semantic object-goal navigation. Each UAV collaboratively constructs a shared, lightweight 2D top-down semantic occupancy map by projecting depth observations from aerial vantage points, eliminating the computational burden of full 3D representations while preserving essential geometric and semantic structure. The core contributions of GoalSwarm are threefold: (1) integration of zero-shot foundation model -- SAM3 for open vocabulary detection and pixel-level segmentation, enabling open-vocabulary target identification without task-specific training; (2) a Bayesian Value Map that fuses multi-viewpoint detection confidences into a per-pixel goal-relevance distribution, enabling informed frontier scoring via Upper Confidence Bound (UCB) exploration; and (3) a decentralized coordination strategy combining semantic frontier extraction, cost-utility bidding with geodesic path costs, and spatial separation penalties to minimize redundant exploration across the swarm.

#### 1. 核心问题与整体含义（研究动机和背景）
论文针对在未知、复杂的室内环境中，多无人机（UAV）团队如何高效协作以寻找并导航至特定目标（ObjectNav）的问题展开研究。
*   **研究动机**：传统的语义导航方法通常依赖重型 3D 重建模型，计算开销巨大，难以在资源受限的无人机机载平台上运行。
*   **核心挑战**：如何在不进行特定任务训练的情况下实现“开放词汇”（即能识别任意名称的目标）识别，以及如何在去中心化架构下减少多机之间的重复搜索，提高探索效率。

#### 2. 方法论：核心思想与关键技术
GoalSwarm 提出了一个完全去中心化的框架，其核心流程包括：
*   **轻量级 2D 语义协作建图**：无人机将空中视角的 RGB-D 数据投影为 2D 俯视占据地图。通过高度过滤（Obstacle Height Band）保留障碍物信息，并利用空间变换器进行全局融合。这种 2D 表示极大地降低了通信和计算负担。
*   **零样本目标识别（SAM3）**：集成基础模型 SAM3，直接利用文本提示进行目标的检测、分割和跟踪，无需针对特定类别进行预训练。
*   **贝叶斯价值图（BVM）**：
    *   每个像素存储目标相关性的均值 $\mu$ 和方差 $\sigma^2$。
    *   利用深度相关的置信度锥（Confidence Cone）模型，通过贝叶斯公式动态融合多视角观测，处理感知不确定性。
*   **去中心化协调策略**：
    *   **语义边界提取与 UCB 评分**：利用上置信界（UCB）算法对地图边界进行评分，平衡“探索未知区域”与“利用高置信度目标区域”。
    *   **成本-效用竞标（Cost-Utility Bidding）**：结合测地线路径代价（FMM 算法）、边界大小和空间分离惩罚（防止无人机靠得太近），通过异步广播实现任务分配。

#### 3. 实验设计
*   **数据集与场景**：使用 **GOAT-Bench** 基准测试，运行在 **Habitat** 模拟器中，采用 **HM3D** 真实室内场景。
*   **物理模拟**：利用 **VisFly** 框架模拟真实的四旋翼动力学、碰撞检测和飞行控制。
*   **Benchmark 与对比方法**：
    *   **Single Agent**：单无人机基准。
    *   **Random Exploration**：无语义引导的随机边界选择。
    *   **No Shared Map**：多机独立运行，不共享地图。
    *   **Greedy Allocation**：贪婪分配，不使用 UCB 评分。
*   **评估指标**：成功率（SR）、路径长度加权成功率（SPL）、平均步数。

#### 4. 资源与算力
*   **硬件平台**：实验在一台配备 **NVIDIA RTX 4090 GPU (24 GB)** 的工作站上运行。
*   **运行模式**：SAM3 作为零样本检测器运行在外部 GPU 服务器上，通过 ZMQ 每 3 个仿真步查询一次。

#### 5. 实验数量与充分性
*   **实验规模**：在 HM3D 的 `val unseen` 拆分集中进行了 20 个 Episode 的详细评估，并在扩展测试中完成了 100 个 Episode 的大规模验证。
*   **消融实验**：设计了包括“无共享地图”、“随机探索”在内的多组消融实验，验证了协作机制和贝叶斯价值图的必要性。
*   **客观性**：实验在从未见过的场景中进行，且对比了单机和无协作的多机方案，结果显示 GoalSwarm 在 SR 和 SPL 上均有显著提升，实验设计较为客观、公平。

#### 6. 主要结论与发现
*   **协作优势**：多机协作显著优于单机，GoalSwarm 的成功率（45%）远高于单机（10%）。
*   **地图共享的重要性**：不共享地图会导致严重的冗余探索，虽然成功率可能尚可（40%），但路径效率（SPL）极低。
*   **不确定性感知**：基于 BVM 和 UCB 的评分机制能有效引导无人机优先访问检测不确定性高的区域，从而更快锁定目标。
*   **效率提升**：在 100 个 Episode 的大规模测试中，GoalSwarm 达到了 53.8% 的成功率，证明了其在大规模任务中的一致性。

#### 7. 优点（亮点）
*   **零样本能力**：通过集成 SAM3，框架具备极强的通用性，能够处理各种生僻的目标类别。
*   **轻量化设计**：将 3D 空间压缩为 2D 语义地图，既保留了导航所需的几何信息，又适应了无人机的计算和通信限制。
*   **鲁棒的协调机制**：引入空间分离惩罚和测地线代价，有效解决了多机任务冲突和路径规划效率问题。

#### 8. 不足与局限
*   **感知偏差（幽灵检测）**：早期帧的误报（False Positives）会在语义地图中留下残余，误导无人机飞向空地。
*   **2D 投影限制**：将所有障碍物投影到 2D 平面会丢失垂直结构信息（如桌子下方的空间或多层货架），可能导致路径规划在复杂 3D 环境中失效。
*   **理想化假设**：实验假设了完美的里程计（Odometry）和理想的通信环境，在现实部署中需考虑漂移、丢包和带宽限制。
*   **小物体识别**：对于镜子、雕像等特征不明显或易被遮挡的小物体，检测置信度往往低于阈值，导致任务超时失败。
