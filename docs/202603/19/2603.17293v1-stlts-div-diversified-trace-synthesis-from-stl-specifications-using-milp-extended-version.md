## 摘要
现代信息物理系统非常复杂，其需求通常使用信号时序逻辑（STL）编写。在实践中，编写正确的 STL 规范具有挑战性；工程师可以从说明规范实际允许行为的具体执行中获益。轨迹合成满足了这一需求，但单个示例往往不足以理解意图或探索边缘情况——多样化的满足行为更具信息量。我们引入了多样化轨迹合成：自动生成满足给定 STL 公式的一组行为多样的轨迹。基于 STL 和系统模型的 MILP 编码，我们形式化了三种互补的多样化目标——布尔距离、随机布尔距离和数值距离——这些目标均由一个目标函数捕获并迭代求解。我们在 STLts-Div 中实现了这些想法，这是一个集成了 Gurobi 的轻量级 Python 工具。

## Abstract
Modern cyber-physical systems are complex, and requirements are often written in Signal Temporal Logic (STL). Writing the right STL is difficult in practice; engineers benefit from concrete executions that illustrate what a specification actually admits. Trace synthesis addresses this need, but a single witness rarely suffices to understand intent or explore edge cases - diverse satisfying behaviors are far more informative. We introduce diversified trace synthesis: the automatic generation of sets of behaviorally diverse traces that satisfy a given STL formula. Building on a MILP encoding of STL and system model, we formalize three complementary diversification objectives - Boolean distance, random Boolean distance, and value distance - all captured by an objective function and solved iteratively. We implement these ideas in STLts-Div, a lightweight Python tool that integrates with Gurobi.

### 2. 方法论：核心思想与技术细节
该方法建立在将 STL 约束和系统动力学编码为混合整数线性规划（MILP）的基础之上，核心在于引入了三种多样化目标函数：
*   **布尔距离（Boolean Distance, BD）**：
    *   **核心思想**：通过改变 STL 公式中各子公式在不同时间间隔上的真值分配来增加差异。
    *   **实现**：计算当前候选轨迹与已生成轨迹集在子公式真值向量上的汉明距离，并在迭代中最大化该距离。
*   **随机布尔距离（Randomized Boolean Distance, RBD）**：
    *   **核心思想**：为了解耦当前解与历史解的直接依赖，每轮随机采样一个参考布尔向量。
    *   **实现**：最大化候选轨迹与随机参考向量之间的布尔距离，从而在搜索空间中引入随机性。
*   **数值距离（Value Distance, VD）**：
    *   **核心思想**：直接在信号的数值层面拉开差距。
    *   **实现**：在保持时间分区一致的前提下，最大化候选轨迹与已有轨迹在采样点上的 $L_1$ 范数距离。利用 MILP 的绝对值线性化技术进行求解。
*   **迭代流程**：系统采用迭代求解模式，每生成一个新轨迹，就更新目标函数以远离已有的轨迹点。

*   **充分性评价**：实验设计较为充分。作者不仅对比了多样化程度（APED），还分析了不同超时设置下的鲁棒性，并展示了每轮迭代的耗时变化曲线。

### 6. 主要结论与发现
*   **多样性提升**：BD、RBD 和 VD 方法生成的轨迹多样性（APED）通常比 Baseline（SP）高出一个数量级。
*   **方法特性**：
    *   **VD** 在数值差异上表现最强，且在严苛的时间限制下最鲁棒。
    *   **BD/RBD** 在逻辑层面的多样性较好，但依赖于子公式的数量。
*   **效率权衡**：VD 方法随着生成轨迹数量的增加，MILP 规模线性增长，导致后期计算压力增大；而 BD/RBD 的单次求解压力相对平稳。
*   **基准表现**：对于复杂的导航任务（NAV），在低变异性限制（N=10）下难以找到可行解，说明轨迹合成受限于预设的时间分区复杂度。

### 7. 优点
*   **首创性**：首次将多样化目标引入基于 MILP 的 STL 轨迹合成领域。
*   **多维度度量**：提供了从逻辑（布尔）到物理（数值）的不同维度多样化手段，适应不同工程需求。
*   **易用性**：开发了轻量级 Python 工具 `STLts-Div`，并支持双积分动力学和混合自动机等多种模型。

### 8. 不足与局限
*   **可扩展性风险**：VD 方法在生成大量轨迹（如数十条以上）时，由于引入大量辅助变量和绝对值约束，MILP 求解效率会显著下降。
*   **参数依赖**：合成效果高度依赖于预设的变异性边界 $N$。如果 $N$ 设置过小，可能找不到解；设置过大，则 MILP 求解极其缓慢。
*   **语义限制**：目前尚未完全支持 STL 的鲁棒语义（Robust Semantics）编码，这意味着无法直接优化轨迹的“安全裕度”。
*   **应用偏差**：实验主要集中在自动驾驶和简单导航，对于更高维度的复杂工业系统，MILP 的求解可行性尚待验证。
