## 摘要
多智能体强化学习的最新进展，特别是策略空间响应预言机（PSRO），使得在日益复杂的领域中计算近似博弈论均衡成为可能。然而，这些方法依赖于深度强化学习预言机，其生成的“黑盒”神经网络策略难以解释、信任或调试。我们引入了代码空间响应预言机（CSRO），这是一个通过将强化学习（RL）预言机替换为大语言模型（LLM）来应对这一挑战的新型框架。CSRO 将最优响应计算重新定义为代码生成任务，提示 LLM 直接生成人类可读的代码作为策略。这种方法不仅产生了本质上可解释的策略，还利用 LLM 的预训练知识来发现复杂的、类人的策略。我们探索了构建和增强基于 LLM 的预言机的多种方式：零样本提示、迭代细化以及 AlphaEvolve（一种基于 LLM 的分布式进化系统）。我们证明了 CSRO 在产生多样化可解释策略的同时，实现了与基准方法相当的性能。我们的工作为多智能体学习提供了一个新视角，将关注点从优化不透明的策略参数转向合成可解释的算法行为。

## Abstract
Recent advances in multi-agent reinforcement learning, particularly Policy-Space Response Oracles (PSRO), have enabled the computation of approximate game-theoretic equilibria in increasingly complex domains. However, these methods rely on deep reinforcement learning oracles that produce `black-box' neural network policies, making them difficult to interpret, trust or debug. We introduce Code-Space Response Oracles (CSRO), a novel framework that addresses this challenge by replacing RL oracles with Large Language Models (LLMs). CSRO reframes the best response computation as a code generation task, prompting an LLM to generate policies directly as human-readable code. This approach not only yields inherently interpretable policies but also leverages the LLM's pretrained knowledge to discover complex, human-like strategies. We explore multiple ways to construct and enhance an LLM-based oracle: zero-shot prompting, iterative refinement and \emph{AlphaEvolve}, a distributed LLM-based evolutionary system. We demonstrate that CSRO achieves performance competitive with baselines while producing a diverse set of explainable policies. Our work presents a new perspective on multi-agent learning, shifting the focus from optimizing opaque policy parameters to synthesizing interpretable algorithmic behavior.

### 论文总结：代码空间响应预言机 (CSRO)

#### 1. 核心问题与研究动机
*   **核心问题**：传统的多智能体强化学习（MARL）算法（如 PSRO）依赖深度强化学习作为“预言机”来生成最优响应策略。然而，这些策略通常是**不透明的“黑盒”神经网络**，难以被人类理解、验证或调试，且训练过程往往需要数百万次模拟，采样效率低下。
*   **研究背景**：在自动驾驶、网络安全等高风险领域，策略的可解释性和可信度至关重要。本文旨在探索是否可以利用大语言模型（LLM）的推理和编程能力，直接生成**人类可读的代码策略**，从而在保持竞争力的同时解决透明度问题。

#### 2. 方法论
*   **核心思想**：将 PSRO 框架中的深度强化学习预言机替换为 LLM。CSRO 不再优化神经网络参数，而是通过**程序合成（Program Synthesis）**生成可执行的 Python 代码作为策略。
*   **关键技术细节**：
    *   **提示词构建（Prompting）**：向 LLM 提供游戏规则、API 规范以及当前对手元策略的描述（包括对手的代码或自然语言总结）。
    *   **上下文抽象（Context Abstraction）**：为了应对长上下文限制，对对手策略进行总结或筛选（如仅保留权重最高的 Top-k 个对手）。
    *   **三种预言机精炼机制**：
        1.  **ZeroShot**：直接生成代码，无反馈循环。
        2.  **LinearRefinement**：根据评估结果（如胜率）进行单线程迭代修改，直到达到性能阈值或预算上限。
        3.  **AlphaEvolve**：利用分布式进化算法，通过 LLM 对代码进行多线程变异和筛选，寻找最优算法。
*   **算法流程**：在每一轮迭代中，计算当前元博弈的均衡，提示 LLM 生成针对该均衡的最优响应代码，评估后将其加入策略池，循环往复。

#### 3. 实验设计
*   **实验场景**：
    1.  **重复岩石纸剪刀 (RRPS)**：1000 轮对局，测试智能体识别对手模式和反制的能力。
    2.  **重复 Leduc 德州扑克**：100 轮对局，涉及私有信息、博弈推理及诈唬（Bluffing）策略。
*   **Benchmark 与对比方法**：
    *   **RRPS 基准**：包含 43 个来自国际竞赛的启发式机器人（如 iocainebot, greenberg）。
    *   **对比方法**：PSRO-IMPALA（深度 RL 基准）、Gemma 3 (27B) 直接动作预测、Tabular Q-Learning、CFR+（扑克的最优解）。

*   **模型使用**：主要使用 **Gemini 2.5 Pro** 作为代码生成预言机。

#### 5. 实验数量与充分性
*   **实验规模**：RRPS 实验采用了 5 个随机种子，Leduc 扑克采用了 3 个随机种子。
*   **消融实验**：进行了丰富的消融研究，包括输入格式（代码 vs. 自然语言描述）、对手筛选策略（Top-5 vs. 最小支持度阈值）以及不同精炼机制的对比。
*   **充分性评价**：实验设计较为**充分且客观**。作者不仅进行了内部自博弈评估，还引入了外部标准机器人库进行测试，确保了评价的公平性，避免了 LLM 仅在简单自博弈中获胜的局限。

#### 6 主要结论与发现
*   **性能竞争力**：CSRO 生成的代码策略在 RRPS 和 Leduc 扑克中均达到了与成熟基准（如 ContRM 或大型 LLM 直接推理）相当甚至更优的水平。
*   **进化优势**：AlphaEvolve 机制在降低策略“可被剥削性”（Exploitability）方面表现最强，能发现更稳健的博弈均衡。
*   **超纳什均衡剥削**：在扑克实验中，CSRO 发现的策略能针对特定对手（如 AlwaysCall）实现远超 CFR+（理论最优策略）的收益，展现了强大的对手建模能力。
*   **高度可解释性**：生成的代码包含复杂的逻辑，如 32 种专家集成、二阶“心理理论”模型、期望价值（EV）计算等，且附带详细注释。

#### 7. 优点
*   **本质可解释性**：策略以 Python 代码形式存在，人类可以直接阅读、审计和修改，解决了“黑盒”难题。
*   **采样效率高**：利用 LLM 的先验知识，无需从零开始通过数亿次试错学习基础逻辑。
*   **灵活性**：通过简单的提示词调整即可改变智能体的行为风格或约束条件。

#### 8. 不足与局限
*   **LLM 依赖性**：性能高度依赖于底层 LLM 的编程和逻辑推理能力，模型弱化会导致生成的代码出现语法错误或逻辑漏洞。
*   **扩展性挑战**：对于状态空间极其复杂（如《星际争霸》或《圣战》）的游戏，如何将复杂的观察空间和对手行为压缩进 LLM 的上下文窗口仍是难题。
*   **推理成本**：虽然训练采样少了，但高质量 LLM 的 API 调用成本在进行大规模进化搜索时仍然较高。
