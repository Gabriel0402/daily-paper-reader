## 摘要
我们提出了一种新的言语强化学习（VRL）框架，用于在执行不确定性下运行的移动机器人系统中进行可解释的任务级规划。该框架采用闭环架构，通过与物理环境的交互实现策略的迭代改进。在我们的框架中，可执行的行为树由大语言模型（LLM）执行器（actor）进行反复优化，该执行器使用由视觉语言模型（VLM）评论器（critic）生成的结构化自然语言反馈，而该评论器负责观察物理机器人和执行轨迹。与传统的强化学习不同，VRL 中的策略更新直接发生在符号规划层面，无需基于梯度的优化。这实现了透明的推理、显式的因果反馈以及人类可解释的策略演化。我们在执行不确定性下执行多阶段操作和导航任务的真实移动机器人上验证了所提出的框架。实验结果表明，该框架支持可解释的策略改进、对执行失败的闭环自适应以及在物理机器人系统上的可靠部署。

## Abstract
We propose a new Verbal Reinforcement Learning (VRL) framework for interpretable task-level planning in mobile robotic systems operating under execution uncertainty. The framework follows a closed-loop architecture that enables iterative policy improvement through interaction with the physical environment. In our framework, executable Behavior Trees are repeatedly refined by a Large Language Model actor using structured natural-language feedback produced by a Vision-Language Model critic that observes the physical robot and execution traces. Unlike conventional reinforcement learning, policy updates in VRL occur directly at the symbolic planning level, without gradient-based optimization. This enables transparent reasoning, explicit causal feedback, and human-interpretable policy evolution. We validate the proposed framework on a real mobile robot performing a multi-stage manipulation and navigation task under execution uncertainty. Experimental results show that the framework supports explainable policy improvements, closed-loop adaptation to execution failures, and reliable deployment on physical robotic systems.

在现实世界中，移动机器人面临显著的**执行不确定性**（如定位误差、控制失效、环境变化）。传统的任务规划方法（如行为树 BT）虽具可解释性，但需大量人工调优；而传统的强化学习（RL）虽能自主改进，但其策略通常是“黑盒”式的（亚符号空间），难以调试和解释。
本文提出了**言语强化学习（Verbal Reinforcement Learning, VRL）**框架，旨在结合大语言模型（LLM）的推理能力与视觉语言模型（VLM）的感知能力，通过**闭环、无梯度、符号化**的方式，实现机器人任务级策略的自动迭代与优化。

### 2. 方法论：核心思想与关键技术
VRL 框架采用“执行器-评论家”（Actor-Critic）架构，其核心流程如下：
*   **策略表示（Behavior Trees, BTs）：** 任务策略被表示为可执行的行为树。BT 具有模块化和透明性，LLM 直接修改其结构和参数，而非生成底层控制指令。
*   **LLM 执行器（Actor）：** 接收任务定义、环境规范、BT 节点库及来自评论家的反馈。它不直接看图像，而是根据反馈对当前的 BT 进行逻辑修改（如增加重试逻辑、调整顺序）。
*   **VLM 评论家（Critic）：** 观察机器人执行过程中的视觉快照和 BT 执行轨迹。它不输出标量奖励，而是生成**结构化的自然语言反馈**，并给出“警报分数”（Alarm Score）和“置信度”。
*   **闭环迭代（Algorithm 1）：** 
    1. 执行当前 BT。
    2. VLM 观察并诊断失败原因（如“方块放反了”）。
    3. LLM 根据诊断结果更新 BT。
    4. 重复此过程，直到策略趋于完善。

### 6. 主要结论与发现
*   **领域对齐胜过模型规模：** 微调后的 3B 模型在反馈质量和策略改进上优于未微调的 7B 模型，说明评论家对任务逻辑的理解比通用视觉能力更重要。
*   **结构化反馈的必要性：** 仅靠数值奖励（Scalar Rewards）无法区分硬件错误与规划错误，导致学习不稳定；而言语反馈能提供明确的因果诊断。
*   **感知是瓶颈：** 引入额外的符号化感知信息（BlockInfo）显著提升了学习速度和稳定性，表明 VRL 的上限受限于 VLM 的视觉接地（Grounding）能力。
*   **策略共进化：** LLM 执行器会根据评论家的局限性自动调整策略（例如，如果评论家看不清颜色，执行器会选择更保守的动作）。

### 7. 优点
*   **高可解释性：** 所有的策略更新都是人类可读的行为树修改，且附带 LLM 的推理逻辑。
*   **无梯度优化：** 无需大规模数据集或复杂的反向传播，直接在符号层面通过“对话”改进机器人行为。
*   **闭环自适应：** 能够有效处理真实世界中的随机失败（如抓取滑落），通过重试或逻辑分支增强鲁棒性。

### 8. 不足与局限
*   **时间窗口限制：** 评论家基于快照观察，难以处理具有长程依赖的潜在故障（即错误在很久之后才显现）。
*   **感知偏差风险：** 如果 VLM 产生幻觉或误判，LLM 可能会做出错误的策略调整（虽然实验显示 LLM 有一定的容错能力）。
*   **应用限制：** 目前主要针对任务级规划，对于需要极高频率反馈的底层运动控制（如精细避障），该框架的延迟可能过高。
