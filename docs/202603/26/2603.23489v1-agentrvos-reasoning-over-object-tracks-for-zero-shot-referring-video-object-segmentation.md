## 摘要
指代视频对象分割 (RVOS) 旨在根据自然语言查询在整个视频中分割目标对象。该任务的免训练方法通常遵循一个通用流程：多模态大语言模型 (MLLM) 选择关键帧，在这些帧中定位被指代的对象，然后由视频分割模型传播结果。虽然这种设计很直观，但它要求 MLLM 在获得任何对象级证据之前做出时间决策，从而限制了推理质量和时空覆盖范围。为了克服这一问题，我们提出了 AgentRVOS，这是一种基于 SAM3 和 MLLM 互补优势构建的免训练智能体流程。给定从查询中提取的概念，SAM3 通过生成的掩码轨迹在整个时空范围内提供可靠的感知。随后，MLLM 通过对这些对象级证据进行基于查询的推理来识别目标，并在 SAM3 的时间存在信息的指导下进行迭代剪枝。大量实验表明，AgentRVOS 在多个基准测试中达到了免训练方法中的先进水平，并且在不同的 MLLM 骨干网络上表现出一致的结果。我们的项目页面位于：https://cvlab-kaist.github.io/AgentRVOS/。

## Abstract
Referring Video Object Segmentation (RVOS) aims to segment a target object throughout a video given a natural language query. Training-free methods for this task follow a common pipeline: a MLLM selects keyframes, grounds the referred object within those frames, and a video segmentation model propagates the results. While intuitive, this design asks the MLLM to make temporal decisions before any object-level evidence is available, limiting both reasoning quality and spatio-temporal coverage. To overcome this, we propose AgentRVOS, a training-free agentic pipeline built on the complementary strengths of SAM3 and a MLLM. Given a concept derived from the query, SAM3 provides reliable perception over the full spatio-temporal extent through generated mask tracks. The MLLM then identifies the target through query-grounded reasoning over this object-level evidence, iteratively pruning guided by SAM3's temporal existence information. Extensive experiments show that AgentRVOS achieves state-of-the-art performance among training-free methods across multiple benchmarks, with consistent results across diverse MLLM backbones. Our project page is available at: https://cvlab-kaist.github.io/AgentRVOS/.

#### 1. 论文的核心问题与整体含义（研究动机和背景）
指代视频对象分割（RVOS）要求模型根据自然语言描述，在视频中精准分割出目标对象。现有的免训练（Training-free）方法通常采用“先选帧、后定位、再传播”的模式：即先让多模态大模型（MLLM）选择关键帧，在帧内定位对象，最后利用视频分割模型（如 SAM2）进行时空传播。
**核心问题：** 这种设计存在“先验决策”缺陷。MLLM 在尚未获得任何对象级证据（Object-level evidence）的情况下，必须预先做出时间维度的决策（选帧）。由于 MLLM 输入 token 限制导致采样稀疏，容易漏掉短时出现、非显著或被遮挡的对象，限制了复杂推理的质量和时空覆盖范围。

#### 2. 论文提出的方法论
AgentRVOS 提出了一种**代理化（Agentic）流水线**，将感知任务交给 SAM3，将推理任务交给 MLLM，实现两者的互补。

*   **核心思想：** 先利用感知工具（SAM3）在全视频范围内生成所有可能的候选对象轨迹，再由 MLLM 扮演“代理人”，基于对象级证据进行后验推理和筛选。
*   **关键技术流程：**
    1.  **候选掩码轨迹生成（Candidate Mask Track Generation）：** MLLM 提取查询中的“核心概念”和“宽泛概念”（如“猫”和“动物”）。SAM3 根据这些概念在全视频帧中生成一致的掩码轨迹。若未检测到对象，则通过 MLLM 修正概念并重试。
    2.  **迭代时空剪枝（Iterative Spatio-temporal Pruning）：** 
        *   **候选剪枝：** MLLM 观察带有掩码标记的视频，将候选对象分类为 `Accepted`（接受）、`Rejected`（拒绝）或 `Uncertain`（不确定）。
        *   **时间范围剪枝：** 针对 `Uncertain` 对象，系统利用 SAM3 提供的对象存在时间信息，自动缩小下一轮推理的时间窗口，使 MLLM 能专注于对象出现的特定片段。
    3.  **外观工具（可选）：** 当查询涉及颜色等细节时，调用工具生成对象的局部裁剪图，辅助 MLLM 克服掩码覆盖带来的视觉干扰。

*   **性能领先：** AgentRVOS 在免训练方法中取得了 SOTA 性能，在 MeViS 上相比前人提升了高达 40.0%，在 ReVOS 和 ReasonVOS 上也有显著增长。

*   **感知与推理的解耦：** 证明了将“时空定位”从 MLLM 任务中剥离，转而利用 SAM3 提供结构化证据，能极大释放 MLLM 的逻辑推理潜力。

#### 7. 优点
*   **免训练且通用：** 无需针对特定任务微调，可直接适配各种 MLLM，具有极强的通用性。
*   **高召回率：** 通过 SAM3 的全视频扫描和概念重试机制，有效解决了短时出现对象的漏检问题。
*   **高效推理：** 迭代剪枝机制允许模型在后续轮次中只关注少数模糊对象和特定时间段，提高了推理的针对性。

#### 8. 不足与局限
*   **依赖感知底座：** 系统的性能上限受限于 SAM3 的初始召回率。如果 SAM3 在所有概念下都无法检测到目标，后续推理将无法挽回。
*   **推理延迟：** 迭代式的代理流程（最多 3 轮）虽然提高了准确度，但相比单次推理的方法，在实时性要求极高的场景下可能存在延迟挑战。
*   **计算成本：** 虽然免训练，但多轮调用大型 MLLM（如 GPT-5）会产生较高的 API 调用成本或推理算力消耗。
