## 摘要
尽管机器人对机器人（R2R）通信提升了室内场景理解能力，超越了单机器人的局限，但若不增加大量的探索开销或扩大团队规模，仅靠 R2R 无法克服部分可观测性问题。相比之下，许多室内环境已经部署了低成本的物联网（IoT）传感器（如摄像头），它们能提供超越机载感知的、覆盖全建筑的持久上下文信息。因此，我们推出了 IndoorR2X，这是首个针对室内环境下具有机器人与万物（R2X）感知与通信能力的、由大语言模型（LLM）驱动的多机器人任务规划基准和仿真框架。IndoorR2X 整合了移动机器人和静态 IoT 设备的观测结果，以构建全局语义状态，支持可扩展的场景理解，减少冗余探索，并通过基于 LLM 的规划实现高层协同。IndoorR2X 提供了可配置的仿真环境、传感器布局、机器人团队和任务套件，用于系统地评估高层语义协同策略。在多种设置下的广泛实验表明，IoT 增强的世界建模提高了多机器人的效率和可靠性，我们还总结了推进机器人团队与室内 IoT 传感器之间基于 LLM 协作的关键见解和失败模式。

## Abstract
Although robot-to-robot (R2R) communication improves indoor scene understanding beyond what a single robot can achieve, R2R alone cannot overcome partial observability without substantial exploration overhead or scaling team size. In contrast, many indoor environments already include low-cost Internet of Things (IoT) sensors (e.g., cameras) that provide persistent, building-wide context beyond onboard perception. We therefore introduce IndoorR2X, the first benchmark and simulation framework for Large Language Model (LLM)-driven multi-robot task planning with Robot-to-Everything (R2X) perception and communication in indoor environments. IndoorR2X integrates observations from mobile robots and static IoT devices to construct a global semantic state that supports scalable scene understanding, reduces redundant exploration, and enables high-level coordination through LLM-based planning. IndoorR2X provides configurable simulation environments, sensor layouts, robot teams, and task suites to systematically evaluate high-level semantic coordination strategies. Extensive experiments across diverse settings demonstrate that IoT-augmented world modeling improves multi-robot efficiency and reliability, and we highlight key insights and failure modes for advancing LLM-based collaboration between robot teams and indoor IoT sensors.

*   **研究背景**：现代室内环境（如智能家居、办公室）已部署大量低成本物联网（IoT）传感器（如监控摄像头、智能家电）。
*   **整体含义**：论文提出了 **IndoorR2X**，这是首个将“机器人与万物协同”（Robot-to-Everything, R2X）理念引入室内多机器人任务规划的基准和框架。其核心动机是利用环境中的静态 IoT 资源作为“上帝视角”的补充，减少机器人的盲目探索，实现更高效的全局协同。

### 2. 方法论：核心思想与技术细节
*   **核心思想**：建立一个中心化的**协调枢纽（Coordination Hub）**，将移动机器人的机载感知与静态 IoT 设备的观测数据进行语义融合，构建全局语义状态，并驱动 LLM 进行在线规划。
*   **关键技术细节**：
    *   **全局语义状态（Global Semantic State）**：维护一个包含机器人状态（位置、技能、载荷）、对象状态（类别、坐标、容器关系、动态属性如开关状态）和区域信息的元组。
    *   **R2X 数据融合**：利用视觉语言模型（VLM，如 Qwen-VL）将 IoT 摄像头视频流转化为带时间戳的文本日志，并与机器人的观测结果同步更新到世界模型中。
    *   **LLM 驱动规划**：LLM 充当在线规划器，根据全局状态生成任务依赖图（DAG）。每个节点代表一个可并行执行的动作指令。
    *   **并行执行编排**：系统调度器根据机器人的空闲状态、空间距离和技能匹配度，动态分配 DAG 中的任务。
    *   **异步监控与重规划**：实时监控执行失败或环境变化（如 IoT 传回新信息），触发即时重规划。

*   **Benchmark 设定**：严格限制机器人的视野，仅允许其获知已访问区域的信息。IoT 摄像头覆盖约 50% 的区域。
*   **对比方法**：

    *   **R2R (Robot-to-Robot)**：机器人间共享地图和观测，无 IoT 支持。

*   **真实世界验证**：使用 Stretch 移动机器人和外部网络摄像头在三室环境中进行了实地部署。

### 5. 实验数量与充分性
*   **实验规模**：在 85 个场景下进行大规模测试，每种配置均有多次迭代，实验数据具有统计学意义。

### 6. 主要结论与发现
*   **效率显著提升**：相比 R2R，R2X 在保持高成功率（92%）的同时，将平均路径长度缩短了约 11%，动作步数减少了 7%，LLM Token 成本降低了 11% 以上。
*   **模型能力决定上限**：GPT-4.1 表现最优；中型模型（Gemma-27b）具有性价比；小型模型（Llama-8b）在复杂协同任务中基本失效。
*   **非对称鲁棒性**：系统对 **IoT 信息缺失（Omission）** 极具韧性（成功率不变，仅效率下降）；但对 **语义错误（Corruption）**（如错误报告设备状态）非常敏感，会导致规划崩溃。
*   **协同开销**：随着机器人团队规模增加（超过 5 个），虽然任务完成时间可能缩短，但由于空间冲突和分配复杂度增加，总路径长度和动作步数会上升。

### 7. 优点（亮点）
*   **范式创新**：首次系统性地将 V2X（车联网）理念引入室内机器人领域，填补了利用环境基础设施增强 LLM 规划的空白。
*   **现实约束**：不同于以往假设“上帝视角”的研究，该框架严格遵守局部可观测性，更符合实际应用。
*   **软硬件结合**：不仅有大规模仿真，还通过物理机器人验证了框架的可迁移性。

### 8. 不足与局限
*   **中心化风险**：依赖中心化的协调枢纽，若通信中断或枢纽故障，整个团队的协同能力将受限。
*   **语义验证缺失**：实验发现系统易受错误 IoT 信息误导，但目前框架缺乏对 IoT 信号进行交叉验证或置信度评估的机制。
*   **动态环境挑战**：虽然考虑了 IoT 延迟，但对于高度动态（如人员频繁走动）的环境，语义状态的实时同步压力和冲突解决机制仍需进一步探索。
*   **算力细节不透明**：缺乏对运行该框架所需的实际计算资源（尤其是 VLM 处理多路摄像头）的量化分析。
