## 摘要
多智能体辩论（MAD）系统日益依赖共享内存来支持长程推理，但这种便利性也带来了一个关键漏洞：单个损坏的条目就可能污染下游的内存增强推理，而仅靠辩论本身无法过滤此类错误。现有的保护措施通过启发式方法或基于大语言模型（LLM）的验证来过滤条目，但它们依赖于具有相同失败模式的 AI 判断，并忽视了 MAD 中的跨智能体动态。我们通过将 MAD 中的内存更新建模为一个零信任内存博弈来填补这一空白，在该博弈中，不假设任何智能体是诚实的，博弈的均衡状态作为最优内存信任的指标。在这一均衡的指导下，我们提出了 EquiMem，这是一种推理时校准机制，它利用智能体现有的检索查询和遍历路径作为证据，通过算法量化每次更新相对于共享内存状态的影响，而不是征求任何 LLM 的判断。EquiMem 实现了针对嵌入式和基于图的内存的校准。在不同的基准测试、MAD 框架和内存架构中，它始终优于现有的保护措施，在对抗性智能体环境下保持稳健，且推理开销微乎其微。

## Abstract
Multi-agent debate (MAD) systems increasingly rely on shared memory to support long-horizon reasoning, but this convenience opens a critical vulnerability: a single corrupted entry can contaminate the downstream memory-augmented reasoning, and debate alone fails to filter such errors. Existing safeguards filter entries via heuristics or LLM-based validation, yet they rely on AI judgments that share the same failure modes and overlook the cross-agent dynamics of MAD. We address this gap by formulating memory updating in MAD as a zero-trust memory game, in which no agent is assumed honest and the game's equilibrium serves as an indicator of optimal memory trust. Guided by this equilibrium, we propose EquiMem, an inference-time calibration mechanism that quantifies each update algorithmically against the shared memory state, using agents' existing retrieval queries and traversal paths as evidence rather than soliciting any LLM judgment. EquiMem instantiates calibration for both embedding- and graph-based memory, and across diverse benchmarks, MAD frameworks, and memory architectures, it consistently outperforms existing safeguards, remains robust under adversarial agents, and incurs negligible inference overhead.

### 2. 论文提出的方法论
*   **核心思想**：将内存更新建模为**零信任内存博弈（Zero-Trust Memory Game）**。在这个博弈中，提议更新的智能体是“贡献者”，其余智能体是“审计者”。系统不听取智能体的投票或评分，而是观察它们的行为证据。
*   **关键技术细节**：
    *   **效用函数定义**：为贡献者定义了包含“影响力”与“检测惩罚”的效用函数；为审计者定义了包含“一致性信号”与“努力成本”的效用函数。
    *   **EquiMem 校准机制**：
        *   **嵌入式内存校准**：通过检测新条目是否异常扰动了向量空间的局部密度（检测信号），以及是否显著改变了其他智能体已有查询的检索排名（对齐信号）来评分。
        *   **图内存校准**：检查新边是否符合局部关系邻域（局部检查）以及是否与现有的多步推理路径一致（路径检查）。
    *   **可信度权重（Credibility Weighting）**：根据智能体提交的证据（如检索查询或遍历路径）是否结构化有效，动态调整其在校准中的权重，从而抑制对抗性智能体。
    *   **信任折扣检索**：将校准得分作为权重融入内存索引，低分条目在检索时会被自动降权。

*   **充分性评价**：实验设计非常全面且客观。它不仅验证了性能提升，还通过对抗性攻击证明了“零信任”假设的有效性，并对比了不同主干模型（DeepSeek-R1, Llama-3.1）的泛化性。

### 6. 论文的主要结论与发现
*   **全局状态感知胜过孤立检查**：EquiMem 在所有配置下均排名第一，证明了将更新置于全局内存状态下评估比单独审计条目更有效。
*   **无需信任的鲁棒性**：即使在 50% 的智能体是敌对的情况下，EquiMem 仍能保持接近正常水平的性能，而传统方法会崩溃。
*   **高性价比**：由于校准过程不调用 LLM，EquiMem 实现了“零 Token 额外开销”和极低的推理延迟。
*   **架构普适性**：该机制能无缝适配向量内存和图内存，且能放大不同内存架构在特定任务上的原生优势。

### 7. 优点
*   **理论严密**：将 MAD 内存问题提升到博弈论高度，提供了坚实的数学证明和均衡分析。
*   **设计巧妙**：通过“拦截”智能体本就要进行的检索行为作为证据，实现了无感、高效的校准。
*   **强鲁棒性**：解决了 MAD 系统中常见的谄媚和合谋攻击问题，真正实现了零信任。

### 8. 不足与局限
*   **冷启动问题**：在内存极度稀疏的初始阶段，由于缺乏对比基准，校准信号可能不够准确。

*   **语义歧义**：在图内存中，如果关系提取本身存在严重的同义词歧义，可能会导致误判（False Positives）。
