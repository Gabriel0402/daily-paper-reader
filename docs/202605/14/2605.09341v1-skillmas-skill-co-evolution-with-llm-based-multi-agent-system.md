## 摘要
大语言模型（LLM）智能体系统在部署后被越来越多地期望能够持续改进，但现有工作通常将两个适应目标解耦：技能演化与多智能体系统（MAS）重构。这种分离可能导致组织瓶颈、上下文压力以及错误专业化。我们提出了 SkillMAS，这是一个用于多智能体系统自适应专业化的非参数化框架，它将技能演化与 MAS 重构相结合。SkillMAS 使用效用学习（Utility Learning）从经过验证的执行轨迹中分配信用，通过有界技能演化在不导致库无序增长的情况下优化可重用程序，并在保留的失败案例和执行器效用（Executor Utility）表明结构失配时，进行基于证据门控的 MAS 重构。在具身操纵、命令行执行和零售工作流中，SkillMAS 在报告的测试基准下具有竞争力，同时阐明了部署后的专业化是如何归因、更新和应用的。

## Abstract
Large language model (LLM) agent systems are increasingly expected to improve after deployment, but existing work often decouples two adaptation targets: skill evolution and multi-agent system (MAS) restructuring. This separation can create organization bottlenecks, context pressure, and mis-specialization. We present SkillMAS, a non-parametric framework for adaptive specialization in multi-agent systems that couples skill evolution with MAS restructuring. SkillMAS uses Utility Learning to assign credit from verified execution traces, bounded skill evolution to refine reusable procedures without unfiltered library growth, and evidence-gated MAS restructuring when retained failures and Executor Utility indicate a structural mismatch. Across embodied manipulation, command-line execution, and retail workflows, SkillMAS is competitive under the reported harnesses while clarifying how post-deployment specialization is attributed, updated, and applied.

### 论文总结：SkillMAS —— 基于大语言模型的多智能体系统技能协同演化

#### 1. 论文的核心问题与整体含义
论文关注的是大语言模型（LLM）智能体系统在部署后的**持续进化**问题。研究动机源于现有方法的“适应性解耦”缺陷：目前的系统通常将“技能演化”（学习新技能）与“系统重构”（调整智能体组织结构）分开处理。这种分离会导致三个问题：
*   **组织瓶颈**：技能库的增长可能超出固定组织结构的路由能力。
*   **上下文压力**：过多的技能检索会增加单个智能体的认知负荷。
*   **专业化偏差**：在没有结构调整的情况下，技能可能无法被正确分配给最合适的执行者。
**SkillMAS** 的核心含义在于提出了一种**耦合演化**框架，让技能的更新与系统结构的调整基于同一套验证轨迹证据进行协同优化。

#### 2. 方法论：核心思想与关键技术
SkillMAS 是一个**非参数化**（无需重新训练模型权重）的框架，其核心流程包含以下四个关键环节：
*   **效用学习（Utility Learning）**：系统维护“技能效用表”和“执行器效用表”。信用分配仅基于**经过验证的执行轨迹**，即只有被实际调用并导致成功的技能/执行器才会获得正向评分，避免了仅凭检索相关性就分配信用的误区。
*   **保留证据集（Retained Evidence Set）**：系统并不处理所有轨迹，而是筛选出重复失败、险些成功、可重用成功以及检索/执行不匹配的案例作为后续演化的输入。
*   **有界技能演化（Bounded Skill Evolution）**：基于保留证据对技能库进行编辑（创建、精炼、修剪）。“有界”体现在它通过诊断算子识别失败的根本原因，仅在原因明确时才进行局部修复，防止技能库无序膨胀。
*   **证据门控 MAS 重构（Evidence-Gated MAS Restructuring）**：这是该框架的独特之处。只有当执行器效用显示出明显的组织瓶颈（如某个执行器负责的技能过多导致干扰）时，才会触发结构调整（增加专家、合并冗余、修改职责边界）。

论文在三个具有代表性的领域进行了评估：
*   **具身操纵（ALFWorld）**：在“未见领域”（Unseen domain）进行测试，考察智能体在家庭环境中的任务完成能力。

*   **模型使用**：实验主要使用了 **GPT-4o-mini** 作为执行器模型，在 $\tau$-Bench 中使用了 GPT-4.1-mini。

#### 6. 主要结论与发现
*   **性能领先**：SkillMAS 在三个基准测试中均取得了最高成功率（ALFWorld 94.0%, OS Task 76.7%, Retail 70.2%）。

*   **证据驱动**：所有的更新都严格基于“验证过的轨迹”，保证了演化的可靠性和可解释性。
