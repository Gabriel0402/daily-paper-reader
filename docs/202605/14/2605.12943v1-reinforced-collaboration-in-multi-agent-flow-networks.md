## 摘要
多智能体系统通过将复杂任务分解为由不同智能体处理的专门子任务，为扩展大语言模型（LLMs）提供了一种强大的方式。然而，它们的性能往往受到错误传播的阻碍，这种错误源于次优的工作流设计或不准确的智能体输出，并可能在智能体协作过程中传播，从而降低最终结果。为了应对这些挑战，我们提出了 MANGO（多智能体网络梯度优化），这是一个数据驱动的框架，通过从过去成功的工作流构建的流网络来组织和优化智能体协作。MANGO 结合了强化学习和文本梯度，以共同优化工作流路径和智能体行为，同时一种跳过机制防止了对已优化良好的智能体进行冗余更新，从而提高了效率。在七个基准测试上的广泛实验表明，MANGO 相比最先进的基准模型实现了高达 12.8% 的性能提升，效率提高了 47.4%，并能有效地泛化到未知领域。我们的代码和数据集可在 https://github.com/openJiuwen-ai/agent-store/tree/main/community/mango 公开获取。

## Abstract
Multi-agent systems provide a powerful way to extend large language models (LLMs) by decomposing a complex task into specialized subtasks handled by different agents. However, their performance is often hindered by error propagation, arising from suboptimal workflow design or inaccurate agent outputs, which can propagate through the agent collaboration process and degrade final results. To address the challenges, we present MANGO (Multi-Agent Network Gradient Optimization), a data-driven framework that organizes and refines agent collaboration via a flow network constructed from past successful workflows. MANGO integrates reinforcement learning and textual gradients to jointly optimize workflow paths and agent behaviors, while a skipping mechanism prevents redundant updates to well-optimized agents for improving efficiency. Extensive experiments on seven benchmarks show that MANGO achieves up to 12.8% performance improvement over state-of-the-art baselines, enhances efficiency by 47.4%, and generalizes effectively to unseen domains. Our code and datasets are publicly available at https://github.com/openJiuwen-ai/agent-store/tree/main/community/mango.

虽然将复杂任务分解为子任务由多个智能体处理能提升大语言模型（LLM）的性能，但存在两大挑战：
*   **工作流生成错误 (C1)**：系统为任务规划了错误的执行路径（例如该看视频时却去搜索网页）。
*   **单智能体执行错误 (C2)**：某个智能体产生幻觉或错误输出，导致后续环节“步步错”。
现有的启发式搜索（如MCTS、进化算法）或简单的文本梯度方法往往效率低下，且在长链协作中面临梯度消失或冗余计算问题。

### 2. 方法论：MANGO 框架
MANGO（Multi-Agent Network Gradient Optimization）提出了一种数据驱动的流网络模型，将协作问题转化为**路径（边）与智能体行为（节点）的联合优化**。

*   **流网络构建**：从过去成功的任务工作流中提取操作，通过语义相似度聚类形成节点（智能体），并保留节点间的转移关系（边）。
*   **路径优化（强化学习）**：将工作流生成建模为马尔可夫决策过程（MDP）。使用 **REINFORCE 算法** 训练策略网络，根据当前任务状态和节点角色描述，动态选择最优的下一个执行节点。奖励函数结合了过程正确性（是否符合历史成功路径）和最终结果准确性。
*   **节点优化（文本梯度）**：利用 **Textual Gradients (TG)** 优化智能体的 System Prompt。为了解决长链条下的梯度消失，MANGO 引入了**全局信号**（最终结果反馈）与**局部信号**（中间步骤反馈）相结合的更新机制。
*   **跳过机制（Skipping Mechanism）**：为了提升效率，允许策略网络在路径中跳过那些已经优化良好的节点，减少冗余的 LLM 调用和 Prompt 更新。

*   **数据驱动而非启发式**：摆脱了对人工设计启发式规则（如 MCTS 搜索策略）的依赖，通过历史经验自动学习。

*   **冷启动依赖**：系统依赖于“过去成功的历史工作流”来构建初始网络。如果对于某个全新领域完全没有成功案例，构建流网络的难度会增加。

*   **系统复杂度**：结合了 RL 和 TG 的联合优化机制增加了系统的实现难度和训练时的不稳定性。
