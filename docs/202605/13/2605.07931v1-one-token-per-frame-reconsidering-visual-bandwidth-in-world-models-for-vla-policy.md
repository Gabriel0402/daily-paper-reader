## 摘要
视觉-语言-动作（VLA）模型日益依赖辅助世界模块进行长时程规划，然而如何在预训练 VLA 之上对这些模块进行参数化仍是一个开放的设计问题。现有的增强了世界模型的 VLA 通常以高视觉带宽将逐帧视觉流输入世界模块，并将其实例化推演视为动作预测的副产品；在冻结骨干网络且适应预算受限的情况下，这使得逐帧表示和潜在动作耦合都未得到充分研究。我们推出了 OneWM-VLA，它通过自适应注意力池化（Adaptive Attention Pooling）将每个视角压缩为每帧单个语义 token，并在单一流匹配（flow-matching）目标下生成结果潜在流和动作轨迹，而不是通过单独的解码器将它们连接起来。实验表明，在我们的设置下，逐帧视觉带宽可以减少到单个 token，而不会损害长时程性能。OneWM-VLA 在 π₀ (2B) 骨干网络上使用 14.71M LoRA 参数进行训练，在 MetaWorld MT50 上的平均成功率从 47.9% 提高到 61.3%，在 LIBERO-Long 上达到 95.6%（π₀ 为 85.2%），并在真实 Piper 机械臂上的长时程可变形任务“折叠衣服”中达到 60.0%（π₀ 为 20.0%）。

## Abstract
Vision-language-action (VLA) models increasingly rely on auxiliary world modules to plan over long horizons, yet how such modules should be parameterized on top of a pretrained VLA remains an open design question. Existing world-model-augmented VLAs typically pass the per-frame visual stream into the world module at high visual bandwidth and treat its rollout as a side product of action prediction; under a constrained adaptation budget on a frozen backbone, this leaves both the per-frame representation and the latent action coupling under-examined. We introduce OneWM-VLA, which compresses each view into a single semantic token per frame through an Adaptive Attention Pooling, and produces the resulting latent stream and the action trajectory under a single flow-matching objective rather than connecting them through a separate decoder. Empirically, we find that per-frame visual bandwidth can be reduced to a single token without compromising long-horizon performance under our setup. Trained with 14.71M LoRA parameters on a $π_0$ (2B) backbone, OneWM-VLA improves the average success rate from 47.9% to 61.3% on MetaWorld~MT50, reaches 95.6% on LIBERO-Long (vs.85.2% for $π_0$), and reaches 60.0% on the long-horizon deformable task Fold Cloth on a real Piper arm (vs.20.0% for $π_0$).

视觉-语言-动作（VLA）模型在短程模仿任务中表现出色，但在长程规划（Long-horizon planning）中常因缺乏对未来环境变化的预测而导致误差累积。现有的“世界模型增强型 VLA”通常面临两个挑战：
*   **视觉带宽过高：** 逐帧输入大量视觉 Token 导致计算量随规划时程呈线性爆炸，难以在有限的微调预算下扩展。
*   **耦合方式松散：** 许多模型将未来帧预测视为动作预测的副产品，或使用沉重的像素级解码器，导致模型过度关注光度细节而非控制相关的动力学。
**核心问题：** 在预训练 VLA 基础上，世界模块究竟需要多少视觉带宽才能支持长程控制？如何更高效地耦合潜在状态流与动作生成？

### 2. 方法论：OneWM-VLA
论文提出了 **OneWM-VLA**，其核心思想是“瓶颈-推演耦合”（Bottleneck-Rollout Coupling）：
*   **自适应注意力池化（Adaptive Attention Pooling）：** 这是一个轻量级的瓶颈设计。它将每个视角的视觉特征（如 256 个 Token）压缩为**每帧仅 1 个语义 Token**。
    *   **多策略池化：** 同时使用最大值（MAX）、求和（SUM）和可学习（LEARN）三种评分函数来捕捉不同的显著性特征。
    *   **自适应视角融合：** 通过可学习的权重将上述三种策略生成的 Token 融合，确保压缩后的 Token 包含最关键的控制信息。
*   **联合流匹配（Joint Flow Matching）：** 放弃了单独的解码器，将未来潜在状态流（Latent Stream）和动作轨迹（Action Trajectory）放在同一个 Transformer 专家模型中，使用统一的流匹配目标进行训练。
    *   在推理时，潜在 Token 和动作 Token 从高斯噪声中共同去噪生成，潜在流作为动作生成的结构化先验。

### 3. 实验设计

论文进行了非常详尽的实验验证：
*   **主实验：** 覆盖了仿真（MetaWorld, LIBERO）和真实机器人，跨越了不同的规划时程（H=5 到 30）。

### 6. 主要结论与发现
*   **带宽并非越多越好：** 令人惊讶的是，在受限的微调预算下，增加每帧 Token 数（从 1 增加到 12）反而会导致成功率单调下降。
*   **长程优势显著：** 随着规划时程 H 的增加，OneWM-VLA 相比基准模型的领先优势扩大。在 MetaWorld H=30 时，成功率比 $\pi_0$ 高出 15.15%。
*   **语义压缩优于像素：** 在潜在空间进行预测比在像素空间预测更具抗噪性，且计算开销更低。
*   **联合生成是关键：** 潜在流预测不仅是辅助任务，它通过联合流匹配为动作序列提供了必要的动力学约束。

### 7. 优点
*   **极高的效率：** 将视觉带宽压缩到极致（1 Token/Frame），解决了长程规划中的计算瓶颈。
*   **即插即用：** 作为预训练 VLA 的轻量级插件，仅需极少量 LoRA 参数即可实现显著提升。
*   **泛化性强：** 在可变形物体操纵（折叠衣服）这种极具挑战性的长程任务上表现优异。

### 8. 不足与局限
*   **骨干网络单一：** 实验主要基于 $\pi_0$ 框架，尚未验证该结论在其他非流匹配架构（如纯自回归 VLA）上的普适性。
*   **感知复杂度受限：** 目前测试的环境相对洁净，在极度复杂的视觉背景或需要精细视觉特征（如阅读微小文字）的任务中，单 Token 可能会丢失关键信息。
*   **训练时长：** 论文提到单 Token 的优势可能与当前训练步数（30k）有关，若大幅增加训练量，更高带宽的表现是否会反超尚不可知。
