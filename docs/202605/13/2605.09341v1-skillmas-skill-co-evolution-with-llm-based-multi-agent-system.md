## 摘要
大语言模型（LLM）智能体系统在部署后被越来越多地期望能够持续改进，但现有工作通常将两个适应目标解耦：技能演化与多智能体系统（MAS）重构。这种分离可能导致组织瓶颈、上下文压力和错误专业化。我们提出了 SkillMAS，这是一个用于多智能体系统自适应专业化的非参数化框架，它将技能演化与 MAS 重构相结合。SkillMAS 利用效用学习从经过验证的执行轨迹中分配信用，通过有界技能演化在不导致库无序增长的情况下优化可重用程序，并在持续失败和执行器效用表明结构失配时，采用证据门控的 MAS 重构。在具身操纵、命令行执行和零售工作流中，SkillMAS 在报告的测试基准下具有竞争力，同时阐明了部署后专业化是如何归因、更新和应用的。

## Abstract
Large language model (LLM) agent systems are increasingly expected to improve after deployment, but existing work often decouples two adaptation targets: skill evolution and multi-agent system (MAS) restructuring. This separation can create organization bottlenecks, context pressure, and mis-specialization. We present SkillMAS, a non-parametric framework for adaptive specialization in multi-agent systems that couples skill evolution with MAS restructuring. SkillMAS uses Utility Learning to assign credit from verified execution traces, bounded skill evolution to refine reusable procedures without unfiltered library growth, and evidence-gated MAS restructuring when retained failures and Executor Utility indicate a structural mismatch. Across embodied manipulation, command-line execution, and retail workflows, SkillMAS is competitive under the reported harnesses while clarifying how post-deployment specialization is attributed, updated, and applied.

### 2. 方法论：核心思想与关键技术
SkillMAS 的核心是一个**非参数化的自适应循环**，主要包含三个关键模块：

*   **效用学习 (Utility Learning)**：
    *   系统维护两个效用表：**技能效用 (Skill Utility)** 和 **执行器效用 (Executor Utility)**。
    *   信用分配仅基于“经过验证的执行轨迹”。只有被实际调用且导致成功的技能/执行器才会获得正向信用更新。
    *   使用基于计数的动态学习率，使新证据的影响力高于重复证据。
*   **受限技能演化 (Bounded Skill Evolution)**：
    *   从保留的成功轨迹中提取“可重用基元”，从失败轨迹中识别“可诊断原因”（如缺少前提条件、动作顺序错误）。
    *   通过“策略卡片索引”提供先验知识，对技能库进行增、删、改、精炼。
    *   新技能会进入验证池，只有经过充分验证后才会正式启用，防止技能库无序膨胀。
*   **证据门控 MAS 重构 (Evidence-Gated MAS Restructuring)**：
    *   这是该框架的独特之处。只有当执行器效用显示出明显的组织瓶颈（如某个执行器负责的技能过多导致干扰，或特定任务族持续失败）时，才会触发架构调整。
    *   重构操作包括：增加专家智能体、合并/删除冗余智能体、修改职责边界。

### 6. 主要结论与发现
*   **性能领先**：SkillMAS 在三个基准上均取得了最高成功率（ALFWorld 94.0%, OS Task 76.7%, $\tau$-Bench 70.2%）。
*   **耦合的重要性**：实验证明，如果只增加技能而不调整架构，系统会因选择干扰而性能下降；反之，只调整架构而无技能支撑则会导致系统脆弱。
*   **非均匀增益**：在 ALFWorld 中，大部分性能提升来自于对特定弱项（如“检查”类任务）的专业化重构。
*   **自适应选择**：在某些场景（如零售服务）中，SkillMAS 能够识别出“保持单智能体架构”比“盲目扩张”更有效，体现了其证据门控机制的灵活性。

### 7. 优点（亮点）

*   **可解释性**：通过效用表和演化轨迹，可以清晰地观察到系统是如何一步步变得专业化的。
