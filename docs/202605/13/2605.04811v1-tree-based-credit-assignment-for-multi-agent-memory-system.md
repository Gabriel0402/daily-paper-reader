## 摘要
记忆系统被广泛用于增强大语言模型（LLMs）处理长程任务的能力，通常被组织为包含记忆构建、摘要和检索智能体的多智能体流水线。为了增强该系统，现有的基于强化学习（RL）的方法要么对所有智能体统一应用最终的下游任务奖励（如问答准确率），这种方式较为粗糙且模糊；要么为不同子任务上的智能体设计特定任务的奖励，这需要昂贵的标注（如关键证据）且难以可靠地定义。为了解决这些局限性，我们提出了面向多智能体记忆系统的基于树的信用分配（TreeMem），它在无需特定任务标注的情况下，从最终奖励中推导出特定于智能体的信用。具体而言，TreeMem 将多智能体流水线（构建者-摘要者-检索者）扩展为树状结构，其中每个智能体的输出被扩展为多个后续分支。每个智能体的贡献通过对其后续分支进行蒙特卡洛平均来估计，从而捕捉中间智能体动作如何影响最终奖励。这将粗糙的最终奖励转化为特定于智能体的优化信号。这些信号随后被用于同时更新所有智能体的策略，帮助异构智能体有效地实现专业化。在长程基准测试上的实验表明，TreeMem 相比强基准线提升了记忆系统的性能，验证了树状结构信用分配在多智能体记忆系统中的有效性。

## Abstract
Memory systems are widely adopted to enhance LLMs for long-horizon tasks, and are commonly organized as multi-agent pipelines with memory building, summarizing, and retrieval agents. To empower this system, existing RL-based methods either apply final downstream task rewards (e.g., QA accuracy) for all agents uniformly, which are coarse and ambiguous, or design task-specific rewards for agents on different subtasks, which require costly annotations (e.g., key evidence) and are difficult to define reliably. To address these limitations, we propose Tree-based Credit Assignment for Multi-Agent Memory Systems (TreeMem), which derives agent-specific credit from the final reward without task-specific annotations. Specifically, TreeMem extends the multi-agent pipeline (builder--summarizer--retrieval) into a tree structure, where each agent's outputs are expanded into multiple subsequent branches. The contribution of each agent is estimated via Monte Carlo averaging over its subsequent branches, capturing how intermediate agent actions may influence the final reward. This converts the coarse final reward into agent-specific optimization signals. These signals are then used to update all agent policies simultaneously, helping heterogeneous agents specialize effectively. Experiments on long-horizon benchmarks show that TreeMem improves memory system performance over strong baselines, validating the effectiveness of tree-structured credit assignment for the multi-agent memory system.

*   **研究动机**：在处理超长文本或长程任务时，大语言模型（LLM）通常依赖由多个智能体（构建者 Builder、摘要者 Summarizer、检索者 Retrieval）协作的记忆系统。
*   **核心痛点**：现有的强化学习（RL）优化方法存在两难困境：
    1.  **奖励模糊**：若仅使用最终问答结果作为奖励，信号过于粗糙，难以确定是哪个智能体表现好或坏（信用分配难题）。
    2.  **标注昂贵**：若为每个智能体设计特定子任务奖励（如证据覆盖率），则需要大量人工标注，且容易诱发“奖励作弊”（Reward Hacking）。
*   **研究目标**：在无需额外人工标注的情况下，实现对多智能体记忆系统中各异构智能体的精准信用分配。

### 2. 方法论
*   **核心思想**：将线性的多智能体流水线扩展为**树状采样结构**，通过蒙特卡洛平均（Monte Carlo Averaging）估算中间动作对最终奖励的贡献。
*   **关键技术流程**：
    1.  **树状 Rollout 采样**：
        *   **Builder**：针对输入历史采样 $G$ 个候选记忆。
        *   **Summarizer**：针对每个候选记忆采样 $J$ 个摘要分支。
        *   **Responder**：针对每个摘要采样 $K$ 个最终答案。
    2.  **信用分配（Credit Assignment）**：
        *   **答案层**：直接获得最终任务奖励。
        *   **摘要层**：其信用值为其下方 $K$ 个答案奖励的平均值。
        *   **构建层**：其信用值为其下方 $J \times K$ 个路径奖励的平均值，并引入**长度惩罚项**（鼓励简洁，防止冗余）。
    3.  **策略优化**：采用组相对策略优化（类似 GRPO 算法），利用计算出的特定信用（Credit）作为优势函数（Advantage），同时更新三个智能体的策略。

### 6. 主要结论与发现
*   **性能领先**：TreeMem 在所有长程任务基准上均优于强基准线，相比现有 RL 方法有显著提升（如在 PersonaMem 上提升约 5.5%~7.5%）。
*   **智能体专业化**：树状信用分配能引导智能体各司其职。Builder 学会了过滤噪声，Summarizer 学会了高层抽象，而非简单的信息堆砌。
*   **效率优势**：通过长度惩罚，TreeMem 构建的记忆更加精炼，Token 压缩率更高，且训练收敛速度快于传统 RL 方法。

### 7. 优点
*   **端到端自动化**：完全摆脱了对中间步骤人工标注的依赖，降低了系统构建成本。
*   **解决信用分配难题**：通过数学上的分支平均，将模糊的全局奖励转化为清晰的局部优化信号。
*   **推理零成本**：树状结构仅用于训练阶段，推理时依然保持线性流水线，不增加额外延迟。

### 8. 不足与局限
*   **训练计算开销**：虽然推理无额外开销，但训练时的树状采样（$G \times J \times K$）会显著增加 Rollout 阶段的计算量和显存占用。
*   **采样稀疏性风险**：如果任务极其复杂，较小的采样分支数（如 $J=2, K=2$）可能无法准确估计信用，存在方差较大的风险。
*   **任务类型局限**：目前主要验证了问答（QA）类任务，对于需要多步推理或实时交互的复杂 Agent 场景，其信用分配的有效性尚需进一步验证。
