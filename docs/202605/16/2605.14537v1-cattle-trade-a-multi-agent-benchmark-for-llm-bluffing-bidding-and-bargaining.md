## 摘要
我们推出了 Cattle Trade，这是一个多智能体基准测试，用于评估大语言模型（LLM）作为智能体在不完全信息、对抗性交互和资源约束下的策略推理能力。该基准测试在长达 50-60 轮的长时程博弈中，融合了拍卖、隐藏报价交易挑战（TCs）、讨价还价、虚张声势、对手建模和资源分配。与以往孤立测试这些能力的智能体基准不同，Cattle Trade 评估智能体是否能在具有利益冲突的竞争性多智能体经济博弈中整合这些能力。该基准记录了每一次出价、TC 报价、反向报价和卡片选择，从而实现了超越最终得分或胜率的行为分析。我们在 242 场博弈中评估了七个高性价比的语言模型和三个确定性代码智能体。研究发现，策略连贯性（特别是支出效率、资源纪律和阶段自适应竞价）与排名的相关性比支出总量或任何单一子技能更强。两个启发式代码智能体的表现优于大多数受测 LLM，而行为轨迹揭示了 LLM 反复出现的失败模式，包括过度竞价、自我竞价、破产式 TC 发起以及对对手状态的适应性较弱。评估智能体能力需要能够在具有利益冲突、不确定性和经济动态的多智能体环境中测试多种能力协同部署的基准。

## Abstract
We introduce \textsc{Cattle Trade, a multi-agent benchmark for evaluating large language models (LLMs) as agents in strategic reasoning under imperfect information, adversarial interaction, and resource constraints. The benchmark combines auctions, hidden-offer trade challenges (TCs), bargaining, bluffing, opponent modeling, and resource allocation within a single long-horizon game lasting 50--60 turns. Unlike prior agent benchmarks that test these abilities in isolation, \textsc{Cattle Trade} evaluates whether agents integrate them across a competitive, multi-agent economic game with conflicting incentives. The benchmark logs every bid, TC offer, counteroffer, and card selection, enabling behavioural analysis beyond final scores or win rates. We evaluate seven cost-efficient language models and three deterministic code agents across 242 games. Strategic coherence, in particular spending efficiency, resource discipline, and phase-adaptive bidding, is associated with rank more strongly than spending volume or any single subskill. Two heuristic code agents outperform most tested LLMs, and behavioural traces surface recurring LLM failure modes including overbidding, self-bidding, bankrupt TC initiation, and weak opponent-state adaptation. Evaluating agentic competence requires benchmarks that test the joint deployment of multiple capabilities in multi-agent environments with conflicting incentives, uncertainty, and economic dynamics.

*   **研究动机**：现有的 LLM 基准测试（如 MMLU、HumanEval）侧重于知识和代码生成，而游戏类基准（如 Avalon、Werewolf）往往孤立地测试欺骗或推理能力。
*   **核心问题**：在现实的经济交互（如谈判、采购、投资组合管理）中，智能体需要同时具备竞价、讨价还价、虚张声势和资源管理的能力。论文旨在测试 LLM 是否能在具有**不完全信息、对抗性激励和长期资源约束**的复杂环境中，将这些孤立的技能整合为连贯的战略行为。

### 2. 方法论：Cattle Trade 基准设计
*   **核心思想**：改编自经典桌游《Kuhhandel》（You're Bluffing!），构建一个 3-5 人的多智能体博弈环境，模拟复杂的经济动态。
*   **关键机制**：
    *   **拍卖机制**：玩家竞拍动物卡，拍卖者拥有“优先购买权”（Buy-right），即可以按最高出价自行买下卡片。
    *   **隐藏报价交易（TC）**：核心博弈点。两名玩家交换动物卡，双方提交面朝下的货币卡。对手可以选择直接接受（盲收）或反击（Counter）。若反击，则双方交换货币，出价高者赢得动物。
    *   **虚张声势（Bluffing）**：货币中包含 0 值卡，允许玩家在 TC 中以极低成本伪装高额报价。
    *   **资源约束**：采用离散货币且不设找零（例如用 100 元付 60 元的账，40 元会损失），强制智能体进行精确的流动性管理。
    *   **乘法评分规则**：最终得分 = (完成的动物组价值总和) × (完成的组数)。这种规则奖励多样化收集，增加了决策维度。

### 3. 实验设计
*   **场景设置**：标准配置为 4 名玩家，10 种动物类型，每种 4 张，游戏持续 50-60 轮。
*   **对比方法**：
    *   **LLM 智能体**：评估了 7 种模型，包括 Gemini 3 Flash/3.1 Flash Lite/2.5 Flash Lite、Claude Sonnet 4.5/Haiku 4.5、GPT-5.4 Nano 和 DeepSeek v3.2。
    *   **代码智能体（基准线）**：
        1.  **TrackerAgent**：完美记录所有公开信息，基于对手预算和持卡进行贪婪决策。
        2.  **SetRaceAgent**：激进追求成组，忽略成本。
        3.  **EconomyAgent**：保守的资源管理者，限制每轮支出。
*   **实验格式**：包括“纯 LLM 锦标赛”和“LLM 对战 3 名代码智能体”的混合赛。

### 4. 资源与算力

### 5. 实验数量与充分性
*   **实验规模**：共进行了 **242 场完整博弈**。其中 98 场为标准模式（70 场纯 LLM，28 场混合赛），另外 144 场用于不同配置下的鲁棒性测试。
*   **充分性评价**：实验设计较为充分。通过 TrueSkill 评分系统进行排名，并利用 7 种不同的代码智能体组合（C1-C7）来测试 LLM 的鲁棒性。行为分析深入到了出价激进程度、虚张声势频率、自我竞价率等微观指标，具有较高的客观性和说服力。

### 6. 主要结论与发现
*   **战略连贯性胜过单一技能**：排名靠前的模型（如 Gemini 3 Flash）在支出效率、资源纪律和阶段性自适应出价上表现更好，而非仅仅靠出价高。
*   **代码智能体依然强劲**：TrackerAgent 和 SetRaceAgent 击败了大多数 LLM，说明 LLM 在利用已知信息（如算牌）方面仍不如简单的确定性算法。
*   **LLM 的典型失败模式**：
    *   **过度竞价（Overbidding）**：出价超过自身总资产导致罚款。
    *   **自我竞价螺旋**：在没有竞争对手的情况下不断抬高自己的价格。
    *   **破产式发起**：在没有现金的情况下发起 TC 交易。
*   **模型排名**：Gemini 3 系列表现最佳，Claude Sonnet 4.5 居中，DeepSeek 和 GPT Nano 表现较弱。

### 7. 优点与亮点
*   **综合性强**：首次在一个基准中整合了拍卖、谈判、欺骗和资源管理，比单一任务基准更接近真实世界。
*   **诊断性强**：不仅看胜率，还通过结构化日志分析“为什么输”，识别出如“自我竞价”等具体的认知缺陷。
*   **开源与可扩展**：提供了完整的 Python 引擎和多种拍卖模式，支持从微型对决到马拉松赛程的参数配置。

### 8. 不足与局限
*   **模型覆盖范围**：主要测试的是“高性价比”模型（Flash/Nano/Lite 等），缺乏对顶级旗舰模型（如 GPT-4o 全量版或 Claude 3.5 Opus）在全量推理预算下的评估。
*   **缺乏交流层**：目前的基准仅限于结构化动作（JSON），排除了人类博弈中至关重要的自然语言说服、结盟和外交手段。
*   **提示词敏感性**：所有模型使用相同的提示词，未针对特定模型进行 Prompt Engineering，可能无法完全发挥某些模型的潜力。
*   **样本量限制**：部分模型（如 Sonnet 4.5）的实验场次较少（n=14），导致置信区间较宽。
