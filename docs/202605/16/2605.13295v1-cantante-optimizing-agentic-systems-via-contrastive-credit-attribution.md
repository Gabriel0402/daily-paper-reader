## 摘要
基于大语言模型（LLM）的多智能体系统在软件工程、预测建模和检索增强生成等复杂的现实任务中表现出了强大的性能。然而，自动配置这些系统仍然是一个结构性挑战，因为评分仅在系统层面可用，而控制智能体行为的参数却是局部的。我们认为，优化这些系统本质上是一个信用分配问题。因此，我们提出了 CANTANTE 框架，该框架通过对比同一查询下多个联合配置的运行结果，将系统级奖励分解为每个智能体的更新信号。我们将其应用于提示词优化，将智能体提示词视为可学习的系统参数。我们在编程（MBPP）、数学推理（GSM8K）和多跳问答（HotpotQA）任务上，将 CANTANTE 与 GEPA 和 MIPROv2 进行了对比评估。在这些基准测试中，CANTANTE 在所有评估的优化器中获得了最佳平均排名，并始终优于未优化的提示词。在 MBPP 上，它比最强的基准线提高了 18.9 个百分点，在 GSM8K 上提高了 12.5 个百分点，同时推理成本更低。在 HotpotQA 上，它的表现与最强基准线的差距保持在一个标准差之内。至关重要的是，我们的信用相关性分析证实，归因器产生了有意义的单智能体信号，而不是简单地重复全局系统评分。

## Abstract
LLM-based multi-agent systems have demonstrated strong performance across complex real-world tasks, such as software engineering, predictive modeling, and retrieval-augmented generation. Yet automating their configuration remains a structural challenge, as scores are available only at the system level, whereas the parameters governing agent behavior are local. We argue that optimizing these systems is fundamentally a credit-assignment problem. We therefore introduce CANTANTE, a framework that decomposes system-level rewards into per-agent update signals by contrasting rollouts of multiple joint configurations on the same query. We instantiate it for prompt optimization, treating agent prompts as learnable system parameters. We evaluate CANTANTE against GEPA and MIPROv2 on programming (MBPP), mathematical reasoning (GSM8K), and multi-hop question answering (HotpotQA). Across these benchmarks, CANTANTE achieves the best average rank among all evaluated optimizers and consistently outperforms unoptimized prompts. It improves over the strongest baseline by +18.9 percentage points on MBPP and +12.5 percentage points on GSM8K, while incurring a lower inference cost. It remains within one standard deviation of the strongest baseline on HotpotQA. Crucially, our credit correlation analysis confirms that the attributer produces meaningful per-agent signals rather than echoing the global system score.

#### 1. 核心问题与整体含义（研究动机和背景）
随着大语言模型（LLM）的发展，多智能体系统（MAS）在处理复杂任务（如软件工程、多步推理）中表现卓越。然而，**自动化配置 MAS** 面临一个核心结构性矛盾：**全局反馈与局部参数的不匹配**。
*   **痛点**：系统评估通常只能给出一个全局标量评分（如任务是否成功），但决定系统行为的参数（如每个智能体的提示词）是局部的。
*   **研究动机**：作者认为 MAS 的优化本质上是一个**信用分配（Credit Assignment）**问题。现有的优化方法（如 MIPROv2 或 GEPA）要么将系统视为黑盒整体优化，要么简单地将全局分数传播给所有智能体，导致优化效率低下且难以精准定位“害群之马”或“功臣”。

#### 2. 方法论：核心思想与技术细节
论文提出了 **CANTANTE** 框架，其核心思想是**对比式组内归因（Contrastive In-Group Attribution）**。
*   **核心流程**：
    1.  **候选生成**：每个智能体的局部优化器（如 CAPO）提出 $K$ 个候选参数（提示词）。
    2.  **联合运行（Rollout）**：将这些候选参数组合成 $K$ 个联合配置，在相同查询（Query）下运行，记录执行轨迹（Trajectory）和全局评分。
    3.  **对比归因**：使用一个专门的“归因 LLM”作为裁判。它同时观察同一查询下的多个运行轨迹，通过对比不同配置导致的结果差异，为每个智能体在每次运行中分配一个 $[-1, +1]$ 的信用分。
    4.  **局部更新**：每个智能体的局部优化器接收到这些针对性的信用信号，而非模糊的全局分数，从而进行更精准的参数迭代。
*   **关键技术**：该方法独立于具体的局部优化器，通过对比机制解决了“所有配置得分相同”时的信号缺失问题（此时归因器可以根据中间推理质量给出反馈）。

#### 3. 实验设计
*   **数据集与场景**：
    *   **MBPP (编程)**：测试条件循环拓扑（规划者-执行者-验证者）。
    *   **GSM8K (数学推理)**：测试并行集成拓扑（三个执行者+一个共识智能体）。
    *   **HotpotQA (多跳问答)**：测试 RAG 检索增强拓扑（检索-阅读-总结-幻觉检测）。
*   **对比方法**：
    *   **GEPA**：基于反思的提示词进化方法。
    *   **MIPROv2**：基于贝叶斯优化的指令与示例优化器。
    *   **Initial**：未经优化的初始提示词。
*   **评估指标**：测试集准确率、平均排名、推理成本（Token 消耗）。

#### 4. 资源与算力
*   **硬件**：使用 **NVIDIA L40 GPU**。
*   **算力消耗**：主实验耗时约 **100 GPU-hours**，消融实验耗时约 **30 GPU-hours**，总计约 **130 GPU-hours**。
*   **模型使用**：下游任务智能体使用 Qwen3-30B，优化器和归因器使用 GPT-OSS-120B 或 Qwen3-80B。

#### 5. 实验数量与充分性
*   **实验规模**：在 3 个不同性质的基准测试上进行了评估，每个实验运行 3 个随机种子以确保统计显著性。
*   **消融实验**：非常充分。作者测试了归因器提示词、归因模型选择、局部优化器类型、对比组大小（Group Size）以及开发集大小对结果的影响。
*   **客观性**：实验设置了统一的 Token 预算（10M tokens），确保了不同优化器之间对比的公平性。

#### 6. 主要结论与发现
*   **性能领先**：CANTANTE 在所有优化器中获得最佳平均排名。在 MBPP 上比最强基准提升 **18.9%**，在 GSM8K 上提升 **12.5%**。
*   **推理效率**：在提升准确率的同时，CANTANTE 优化出的提示词在推理时的 Token 消耗往往更低（例如在 MBPP 和 GSM8K 上比初始提示词更省 Token）。
*   **归因有效性**：相关性分析证明，归因器产生的信用分与系统表现高度相关，能够识别出哪些智能体是性能瓶颈（如 MBPP 中的验证者智能体）。
*   **鲁棒性**：即使在小样本（开发集仅 75 个样本）下，CANTANTE 依然能保持较强的优化能力。

#### 7. 优点（亮点）
*   **解耦设计**：将信用分配与参数更新解耦，使其可以适配任何局部优化算法。
*   **对比机制**：通过“同题对比”显著降低了归因的难度，使 LLM 能够识别出细微的行为差异对最终结果的影响。
*   **诊断功能**：信用相关性分析可以作为 MAS 拓扑设计的诊断工具，帮助开发者发现哪些环节需要重新设计。

#### 8. 不足与局限
*   **拓扑固定**：目前仅优化提示词参数，不涉及工作流拓扑结构（如增减智能体或改变连接关系）的动态调整。
*   **归因器依赖**：优化的上限受限于归因 LLM 的能力。如果归因器模型弱于下游智能体，可能会产生错误的引导信号。
*   **可扩展性挑战**：随着智能体数量增加，执行轨迹会变长，可能超出归因器的上下文窗口或导致信息过载。
*   **特定任务局限**：在 HotpotQA 上的提升较小，说明该方法在某些受限于拓扑结构而非提示词质量的任务中潜力有限。
