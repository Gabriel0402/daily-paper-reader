## 摘要
多智能体辩论（MAD）系统日益依赖共享内存来支持长程推理，但这种便利性也带来了一个关键漏洞：单个损坏的条目就可能污染下游的内存增强推理，而仅靠辩论本身无法过滤此类错误。现有的保护措施通过启发式方法或基于大语言模型（LLM）的验证来过滤条目，但它们依赖于具有相同失败模式的 AI 判断，且忽视了 MAD 中的跨智能体动态。我们通过将 MAD 中的内存更新建模为一个零信任内存博弈来填补这一空白，在该博弈中不假设任何智能体是诚实的，博弈的均衡状态作为最优内存信任的指标。在这一均衡的指导下，我们提出了 EquiMem，这是一种推理时校准机制，它利用智能体现有的检索查询和遍历路径作为证据，而非征求任何 LLM 判断，从而根据共享内存状态对每次更新进行算法量化。EquiMem 实现了针对嵌入式和图结构内存的校准，在多种基准测试、MAD 框架和内存架构中，它始终优于现有的保护措施，在对抗性智能体环境下保持稳健，且推理开销微乎其微。

## Abstract
Multi-agent debate (MAD) systems increasingly rely on shared memory to support long-horizon reasoning, but this convenience opens a critical vulnerability: a single corrupted entry can contaminate the downstream memory-augmented reasoning, and debate alone fails to filter such errors. Existing safeguards filter entries via heuristics or LLM-based validation, yet they rely on AI judgments that share the same failure modes and overlook the cross-agent dynamics of MAD. We address this gap by formulating memory updating in MAD as a zero-trust memory game, in which no agent is assumed honest and the game's equilibrium serves as an indicator of optimal memory trust. Guided by this equilibrium, we propose EquiMem, an inference-time calibration mechanism that quantifies each update algorithmically against the shared memory state, using agents' existing retrieval queries and traversal paths as evidence rather than soliciting any LLM judgment. EquiMem instantiates calibration for both embedding- and graph-based memory, and across diverse benchmarks, MAD frameworks, and memory architectures, it consistently outperforms existing safeguards, remains robust under adversarial agents, and incurs negligible inference overhead.

*   **核心问题**：在多智能体辩论（MAD）系统中，共享内存虽然能增强长程推理能力，但也极易受到“内存污染”的影响。单个智能体的幻觉、谄媚（Sycophancy）或错误断言一旦进入共享内存，会误导所有后续推理。
*   **研究背景**：现有的内存保护机制（如启发式过滤或基于 LLM 的审计）存在两个缺陷：一是依赖的 AI 判断本身也可能产生幻觉；二是忽视了多智能体之间的动态博弈，无法处理智能体间的合谋或对抗行为。
*   **整体含义**：论文旨在建立一种“零信任”机制，不依赖任何单个智能体的诚实性，而是通过博弈论均衡来算法化地校准内存更新的可信度。

### 2. 方法论：核心思想与关键技术
*   **核心思想**：将内存更新建模为**零信任内存博弈（Zero-Trust Memory Game）**。在这个博弈中，提议更新的智能体是“贡献者”，其余智能体是“审计者”。系统不听取智能体的自我报告（如置信度分数），而是观察其行为证据。
*   **关键技术细节**：
    *   **效用函数定义**：为贡献者定义了包含“影响力”和“检测惩罚”的效用函数；为审计者定义了包含“一致性信号”和“努力成本”的效用函数。
    *   **EquiMem 校准机制**：一种无需 LLM 调用的算法化校准。它利用智能体在辩论中产生的现有检索查询（Query）和图遍历路径作为“探针”。
    *   **两种内存架构的实例化**：
        *   **基于嵌入的内存**：检查新条目是否扰乱了向量空间的局部密度（检测信号），以及是否改变了审计者查询的检索排名（一致性信号）。
        *   **基于图的内存**：检查新边是否符合局部关系邻域（局部检查）以及是否存在现有的多步推理路径支持（路径检查）。
    *   **可信度权重（Credibility Weight）**：根据审计者提供的证据质量动态调整其权重，通过乘法衰减惩罚提交无效证据的智能体。

### 6. 主要结论与发现
*   **全局状态感知胜过孤立检查**：EquiMem 在所有配置下均排名第一，证明了将更新置于全局内存状态下评估比单条目评分更有效。
*   **零信任下的鲁棒性**：即使在 50% 的智能体是对抗性的情况下，EquiMem 仍能保持接近正常水平的性能，而基准方法往往会崩溃至低于“无内存”水平。
*   **低成本高收益**：由于不调用 LLM 进行校准，EquiMem 在不增加 Token 成本的前提下显著提升了质量。

### 7. 优点（亮点）
*   **理论完备**：基于严谨的博弈论框架，将社会选择理论中的“零信任”引入多智能体协作。
*   **架构通用**：同时支持向量内存和图内存，具有很强的普适性。
*   **推理高效**：巧妙地复用了智能体辩论过程中的检索行为作为证据，实现了“零额外 Token”校准。

### 8. 不足与局限
*   **冷启动问题**：在内存极度稀疏的初始阶段，由于缺乏对比基准，校准信号可能不够准确。
*   **关联幻觉风险**：如果所有智能体（由于使用相同底座模型）对某个事实持有相同的预训练偏见（Correlated Hallucinations），该机制可能无法识别出这种共有的错误。
*   **语义歧义**：在图内存中，如果关系提取存在严重的语义歧义（如同义词未对齐），可能会导致误判。
