## 摘要
视觉-语言-动作（VLA）模型日益依赖辅助世界模块进行长时程规划，然而如何在预训练 VLA 之上对这些模块进行参数化仍是一个开放的设计问题。现有的增强了世界模型的 VLA 通常以高视觉带宽将逐帧视觉流输入世界模块，并将其展开（rollout）视为动作预测的副产品；在冻结主干网络且适应预算受限的情况下，这使得逐帧表示和潜在动作耦合都未得到充分研究。我们推出了 OneWM-VLA，它通过自适应注意力池化（Adaptive Attention Pooling）将每个视角压缩为每帧单个语义 token，并在单一流匹配（flow-matching）目标下生成结果潜在流和动作轨迹，而不是通过单独的解码器将它们连接起来。实验表明，在我们的设置下，逐帧视觉带宽可以减少到单个 token，而不会损害长时程性能。OneWM-VLA 在 π₀ (2B) 主干网络上使用 14.71M LoRA 参数进行训练，在 MetaWorld MT50 上的平均成功率从 47.9% 提高到 61.3%，在 LIBERO-Long 上达到 95.6%（π₀ 为 85.2%），并在真实 Piper 机械臂上的长时程可变形任务“折叠布料”中达到 60.0%（π₀ 为 20.0%）。

## Abstract
Vision-language-action (VLA) models increasingly rely on auxiliary world modules to plan over long horizons, yet how such modules should be parameterized on top of a pretrained VLA remains an open design question. Existing world-model-augmented VLAs typically pass the per-frame visual stream into the world module at high visual bandwidth and treat its rollout as a side product of action prediction; under a constrained adaptation budget on a frozen backbone, this leaves both the per-frame representation and the latent action coupling under-examined. We introduce OneWM-VLA, which compresses each view into a single semantic token per frame through an Adaptive Attention Pooling, and produces the resulting latent stream and the action trajectory under a single flow-matching objective rather than connecting them through a separate decoder. Empirically, we find that per-frame visual bandwidth can be reduced to a single token without compromising long-horizon performance under our setup. Trained with 14.71M LoRA parameters on a $π_0$ (2B) backbone, OneWM-VLA improves the average success rate from 47.9% to 61.3% on MetaWorld~MT50, reaches 95.6% on LIBERO-Long (vs.85.2% for $π_0$), and reaches 60.0% on the long-horizon deformable task Fold Cloth on a real Piper arm (vs.20.0% for $π_0$).

论文探讨了在视觉-语言-动作（VLA）模型中集成“世界模型”（World Model）时的**视觉带宽**问题。
*   **研究动机**：现有的世界模型增强型 VLA 通常将高分辨率、多 token 的视觉流输入世界模块，这导致在长时程（Long-horizon）规划时计算开销巨大。此外，现有的方法往往将未来状态预测视为动作预测的副产品，两者耦合不够紧密。
*   **核心问题**：在预训练 VLA 的微调阶段，世界模型究竟需要多少视觉带宽才能支持有效的长程控制？

### 2. 方法论：OneWM-VLA
论文提出了 **OneWM-VLA** 框架，其核心思想是“极度压缩”与“联合生成”：
*   **自适应注意力池化（Adaptive Attention Pooling）**：
    *   **多策略池化**：对每个视角的视觉特征（如 256 个 token），分别通过最大值（MAX）、总和（SUM）和可学习 MLP（LEARN）三种策略计算显著性得分，并进行加权聚合。
    *   **自适应视图融合**：将上述三种策略产生的 token 通过可学习的凸组合（Softmax 权重）融合为**每帧仅 1 个语义 token**。
*   **联合流匹配（Joint Flow Matching）**：
    *   不再将动作预测和潜在状态预测分开，而是将未来动作轨迹 $a$ 和未来潜在世界 token $z$ 放在同一个概率路径下。
    *   使用统一的**条件流匹配（CFM）**目标函数进行训练，使模型在推理时能通过自回归或 ODE 采样同时生成动作和环境演化的潜在表示。
    *   这种设计让潜在流成为动作生成的结构化先验，而非独立的辅助输出。

### 3. 实验设计
*   **数据集与场景**：
    *   **MetaWorld MT50**：包含 50 个操纵任务，分为 Easy 到 Very Hard 四个难度等级，测试长程规划能力（H=5 到 30）。

*   **Benchmark 与对比方法**：
    *   主要对比基准是 **$\pi_0$ (2B)** 和更大的 **$\pi_{0.5}$ (3B)**。

*   **实验规模**：论文进行了大规模的仿真测试（50+ 任务）和实机测试，实验设计非常详尽。

### 6. 主要结论与发现
*   **带宽并非越多越好**：在有限的微调预算下，将视觉带宽减少到每帧 1 个 token 反而比多个 token 效果更好（成功率从 20.5% 提升至 53.1%），小 token 具有隐式正则化作用。
*   **长程优势显著**：随着规划时程 $H$ 增加，基准模型性能大幅下降，而 OneWM-VLA 表现稳健。
*   **语义空间优于像素空间**：在语义潜在空间进行预测比在像素空间预测更高效且抗噪。
*   **联合训练是关键**：如果去掉潜在状态的预测损失，模型性能会大幅崩溃，证明了世界模型对动作生成的引导作用。

### 7. 优点
*   **极高的计算效率**：通过将视觉带宽压缩至极限，解决了长程规划中的计算瓶颈和显存溢出问题。
*   **架构简洁**：利用统一的流匹配框架实现了动作与环境动力学的深度耦合。
*   **实战表现强**：在极具挑战性的“折叠布料”任务中表现远超基准模型，证明了其在复杂动力学环境下的建模能力。

### 8. 不足与局限
*   **主干依赖**：实验主要基于 $\pi_0$ 这一种 VLA 主干，尚不确定该结论是否能完美推广到更大规模（如 7B+）或其他架构的 VLA 上。
*   **感知复杂度限制**：目前的测试场景虽然包含可变形物体，但背景相对干净。在极度复杂的开放世界视觉环境下，1 个 token 是否足以捕捉所有关键细节仍存疑。
*   **推理延迟**：虽然减少了 token 数，但流匹配本身需要多次 ODE 采样步骤，单步推理速度仍有优化空间。
