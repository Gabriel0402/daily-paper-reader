## 摘要
视觉-语言-动作（VLA）模型是有效的机器人动作执行器，但由于长时闭环规划和多样化物理操作的双重负担，它们在长程任务中仍受到限制。因此，我们提出了 VLAs-as-Tools，这是一种将该负担分配给负责时间推理的高层视觉语言模型（VLM）智能体和一系列负责多样化局部物理操作的专用 VLA 工具的策略。VLM 处理场景分析、全局规划和恢复，而每个 VLA 工具执行一个有界的子任务。为了在长程任务中将智能体规划与 VLA 工具执行紧密耦合，我们引入了一个 VLA 工具族接口，该接口暴露了显式的工具选择和执行中的进度反馈，从而实现了高效的事件触发式智能体重规划，而无需持续的智能体轮询。为了获得能够忠实遵循智能体调用的多样化专用 VLA 工具，我们进一步提出了工具对齐后训练（TAPT），该方法构建了用于指令遵循的调用对齐训练单元，并采用工具族残差适配器进行高效的工具专业化。实验表明，VLAs-as-Tools 在 LIBERO-Long 上将 $π_{0.5}$ 的成功率提高了 4.8 个百分点，在 RoboTwin 上提高了 23.1 个百分点，并进一步将通过非偏率（Non-biased Rate）衡量的调用保真度提高了 15.0 个百分点。代码将发布。

## Abstract
Vision-language-action (VLA) models are effective robot action executors, but they remain limited on long-horizon tasks due to the dual burden of extended closed-loop planning and diverse physical operations. We therefore propose VLAs-as-Tools, a strategy that distributes this burden across a high-level vision language model (VLM) agent for temporal reasoning and a family of specialized VLA tools for diverse local physical operations. The VLM handles scene analysis, global planning, and recovery, while each VLA tool executes a bounded subtask. To tightly couple agent planning with VLA tool execution in long-horizon tasks, we introduce a VLA tool-family interface that exposes explicit tool selection and in-execution progress feedback, enabling efficient event-triggered agent replanning without continuous agent polling. To obtain diverse specialized VLA tools that faithfully follow agent invocations, we further propose Tool-Aligned Post-Training (TAPT), which constructs invocation-aligned training units for instruction following and adopts tool-family residual adapters for efficient tool specialization. Experiments show that VLAs-as-Tools improves the success rate of $π_{0.5}$ by 4.8 points on LIBERO-Long and 23.1 points on RoboTwin, and further enhances invocation fidelity by 15.0 points as measured by Non-biased Rate. Code will be released.

*   **研究动机**：端到端 VLA 模型往往难以兼顾规划与执行，容易受到训练数据偏差的影响，且在发生错误时缺乏恢复能力。而传统的“代码即策略”系统虽然擅长推理，但依赖于手动定义的受限机器人 API。本文旨在结合两者的优势，将 VLA 重新定义为可被高层智能体调用的“工具”。

### 2. 方法论
论文提出了 **VLAs-as-Tools** 框架，核心包含以下三个部分：
*   **分层架构**：使用一个高容量的 VLM（如 GPT-4o 或 Gemini）作为“智能体”负责目标分解、状态跟踪和重规划；使用一组专门化的 VLA 模型作为“工具族”负责具体的物理子任务。
*   **VLA 工具族接口 (Interface)**：
    *   **调用消息 (C)**：智能体发送包含“工具标签”（如抓取、打开）和“场景指令”的消息。

*   **工具对齐后训练 (TAPT)**：
    *   **调用对齐训练单元**：将原始演示数据分割成有界的子任务片段，并自动标注工具标签和进度。
    *   **工具族残差适配器 (Residual Adapters)**：在共享的 VLA 骨干网络上，为每个工具族（如 Push, Rotate）添加轻量级的 LoRA 适配器，实现参数高效的专业化，避免任务间的干扰。
    *   **优化目标**：结合了动作克隆（IL）和基于子任务奖励的强化学习（RL，使用 GRPO 算法）。

*   **实验规模**：涵盖了 3 个主流基准测试和 1 个反事实测试集，针对 3 种不同的 VLA 骨干网络进行了验证。

### 6. 主要结论与发现
*   **性能提升**：VLAs-as-Tools 显著优于端到端 VLA，在 $\pi_{0.5}$ 基础上将 LIBERO-Long 成功率提升 4.8%，RoboTwin 提升 23.1%。
*   **指令忠实度**：TAPT 显著增强了模型对指令的遵循能力，减少了因视觉先验导致的错误动作（非偏率提升 15% 以上）。
*   **规划效率**：引入进度反馈机制后，高层 VLM 的调用频率大幅降低（从每 5 步一次降至每任务不到 2 次），且保持了极高的重规划成功率。

### 7. 优点
*   **解耦设计**：成功将复杂的长程规划与局部的物理控制解耦，降低了单一模型的负担。
*   **参数高效**：通过残差适配器实现了“一机多用”，无需为每个技能训练独立的大模型。
*   **鲁棒性**：进度反馈机制为具身智能体提供了闭环的错误恢复能力，这在处理现实世界的不确定性时至关重要。

### 8. 不足与局限
*   **环境限制**：实验主要在模拟环境（Simulator）中完成，尚未在真实物理机器人（Real-robot）上进行大规模验证。
*   **数据依赖**：自动标注流水线依赖于高质量的多模态大模型，标注过程中的偏差可能会传播到 VLA 工具中。
*   **任务范围**：目前主要集中在单臂操纵任务，对于需要高度双臂协同或全身协调的任务，该工具化策略的适用性有待进一步探索。
