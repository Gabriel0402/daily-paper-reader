## 摘要
自主室内移动机器人可以使用 ROS 2 Navigation 2 等成熟框架可靠地导航到度量坐标，但它们缺乏解释表达意图而非位置的自然语言指令的能力。视觉语言模型（VLM）提供了弥合这一差距所需的语义推理，但其推理延迟（在消费级硬件上每次决策需 2-9 秒）和会话间的遗忘限制了实际部署。本文提出了语义自主栈（Semantic Autonomy Stack），这是一个用于语义自主室内导航的六层参考框架，并在配备现成边缘硬件的物理机器人上验证了一个包含混合确定性-VLM 推理和跨机器人自适应记忆的完整实例。一个七步参数化解析器在不调用语言模型、摄像头或 GPU 的情况下，在 0.1 毫秒内处理了 88% 的指令；只有真正模糊的指令才会升级到 VLM 推理。一个具有明确范围分类（全局环境知识、特定操作员偏好、特定机器人能力）的五类语义记忆框架实现了跨会话学习和跨机器人知识迁移：通过在一台机器人上进行 VLM 交互学习到的偏好被提升为确定性解析，并通过共享的编译摘要传输到第二台机器人，实现了实测 103,000 倍的延迟降低。在两台定制的差速驱动机器人上进行的 82 次场景级决策和三个会话的实验验证表明，语义迁移准确率达到 100%（33/33，95% 置信区间 [0.894, 1.000]），语义解析准确率达到 100%，并证明了并发多机器人运行的可行性——所有这些均在无板载 GPU 的树莓派 5（Raspberry Pi 5）平台上实现，且无需任何训练数据。

## Abstract
Autonomous indoor mobile robots can navigate reliably to metric coordinates using established frameworks such as ROS 2 Navigation 2, yet they lack the ability to interpret natural language instructions that express intent rather than positions. Vision-Language Models offer the semantic reasoning required to bridge this gap, but their inference latency (2-9 seconds per decision on consumer hardware) and session-by-session amnesia limit practical deployment. This paper presents the Semantic Autonomy Stack, a six-layer reference framework for semantically autonomous indoor navigation, and validates a complete instance featuring hybrid deterministic-VLM reasoning and cross-robot adaptive memory on physical robots with off-the-shelf edge hardware. A seven-step parametric resolver handles 88% of instructions in under 0.1 milliseconds without invoking a language model, camera, or GPU; only genuinely ambiguous instructions escalate to VLM reasoning. A five-category semantic memory framework with explicit scope taxonomy (global environment knowledge, per-operator preferences, per-robot capabilities) enables cross-session learning and cross-robot knowledge transfer: preferences learned through VLM interactions on one robot are promoted to deterministic resolution and transferred to a second robot via a shared compiled digest, achieving a measured latency reduction of 103,000-fold. Experimental validation on two custom-built differential-drive robots across 82 scenario-level decisions and three sessions demonstrates 100% semantic transfer accuracy (33/33, 95% CI [0.894, 1.000]), 100% semantic resolution accuracy, and concurrent multi-robot operation feasibility - all on Raspberry Pi 5 platforms with no onboard GPU, requiring zero training data.

*   **研究动机**：视觉语言模型（VLM）虽能提供语义推理，但在边缘设备上部署面临三大挑战：
    1.  **高延迟**：消费级硬件推理需 2-9 秒，难以满足实时连续运行需求。
    2.  **会话遗忘**：VLM 通常不具备跨会话记忆，每次重启需重新学习用户偏好。
    3.  **缺乏跨机器人知识迁移**：在一台机器人上学到的语义关联难以直接应用到同环境的其他机器人上。

#### 2. 方法论：核心思想与关键技术
论文提出了**语义自主栈（Semantic Autonomy Stack, SAS）**，这是一个六层参考框架（L0-L5），核心创新在于：
*   **混合双过程推理（Hybrid Reasoning）**：
    *   **L3a（确定性快速解析）**：采用一个**七步参数化级联解析器**。按顺序执行：偏好匹配（M3）、显式节点 ID、节点名称匹配、对象 ID 匹配、属性评分、单类匹配、邻近度+类别匹配。该过程不调用 VLM 和摄像头，延迟低于 0.1ms。
    *   **L3b（VLM 视觉语义推理）**：仅当 L3a 无法解析时，才激活 VLM（如 Qwen 3.5:4b）进行深度推理。
*   **五类语义记忆框架（M1-M5）**：
    *   **M1/M2**：环境知识与时间模式（全局共享）。
    *   **M3**：操作员偏好（通过 VLM 交互学习并提升为确定性规则，全局共享）。
    *   **M4/M5**：机器人能力与任务历史（特定于机器人）。
*   **执行契约 $\langle A, O, V, L \rangle$**：定义了动作能力清单（Affordance）、观测归一化（Observation）、动作验证（Validation）和审计日志（Logging），确保 VLM 输出的安全性和可追溯性。

#### 3. 实验设计
实验在物理环境（大学走廊，包含 24 个节点和 18 个静态兴趣点）中进行，使用两台定制机器人（Xplorer-B 和 Xplorer-C）。
*   **实验场景**：
    *   **Session A（学习周期）**：在机器人 C 上通过 VLM 学习新指令（如“找个能坐下休息的地方”）。
    *   **Session B（跨机器人迁移）**：将机器人 C 学到的偏好通过“记忆摘要”迁移至机器人 B，验证其能否通过 L3a 快速解析。
    *   **Session C（并发运行）**：验证多机器人共享记忆下的协同可行性。
*   **对比基准**：主要对比了纯 VLM 推理与所提混合框架在延迟、准确率和计算资源消耗上的差异。

#### 4. 资源与算力
*   **边缘端**：机器人板载 **Raspberry Pi 5**（8GB/16GB RAM），仅使用 CPU 进行导航、感知（YOLO26n）和确定性推理。**无板载 GPU**。
*   **服务端**：使用一台配备 **NVIDIA RTX 5050 GPU** 的工作站，通过 Ollama 运行 **Qwen 3.5:4b** 模型处理 L3b 推理请求。
*   **通信**：通过 FastAPI 构建 HTTP 桥接和 ROS 2 DDS 进行数据交换。

#### 5. 实验数量与充分性
*   **实验规模**：共收集了 **82 组场景级决策数据**，涵盖了从 VLM 学习到确定性重用的完整闭环。
*   **充分性**：实验设计包含了统计学验证（如 Shapiro-Wilk 正态性检验、Mann-Whitney U 检验、Fisher 精确检验）。虽然场景集中在单一走廊环境，但涵盖了跨机器人、跨会话和并发运行等多个维度，对于验证框架的逻辑可行性较为充分。

#### 6. 主要结论与发现
*   **高效性**：混合解析器在 **88%** 的情况下绕过了 VLM，直接通过确定性路径解析指令。
*   **延迟大幅降低**：通过记忆提升（M3），指令解析延迟从 VLM 的 6.7 秒降低至 0.06 毫秒，**加速比达 10.3 万倍**。
*   **高准确率**：跨机器人语义迁移准确率达到 **100% (33/33)**。
*   **解耦验证**：实验证明语义解析准确率（100%）与导航执行成功率（88%）是独立的，导航失败主要受限于 AMCL 定位精度而非语义理解。

#### 7. 优点
*   **架构优雅**：SAS 六层框架实现了感知、推理、执行的深度解耦，具有很强的可扩展性。
*   **实用性强**：针对边缘设备算力不足的现状，通过“System 1（快）/ System 2（慢）”的认知理论显著提升了部署可行性。
*   **零训练数据**：无需针对特定环境重新训练模型，依靠预训练 VLM 和在线学习实现自适应。

#### 8. 不足与局限
*   **环境单一**：实验仅在单一走廊验证，尚未在复杂多房间或动态大场景中测试扩展性。
*   **词法匹配限制**：目前的 M3 匹配依赖 Jaccard 相似度（词法匹配），对于语义相同但措辞完全不同的指令（如“我急需如厕” vs “带我去洗手间”）可能失效，未来需引入轻量级语义嵌入（Embedding）。
*   **硬件限制**：低位安装的摄像头无法识别高处的物体（如墙上的消火栓），导致视觉确认环节在某些 POI 上失效。
*   **VLM 随机性**：VLM 推理仍存在不确定性（实验中一致性为 86%），虽然有阈值过滤，但仍存在错误学习的风险。
