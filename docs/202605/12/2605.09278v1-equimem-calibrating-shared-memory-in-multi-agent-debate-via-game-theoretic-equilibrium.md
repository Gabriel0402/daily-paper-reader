## 摘要
多智能体辩论（MAD）系统日益依赖共享内存来支持长程推理，但这种便利性也带来了一个关键漏洞：单个损坏的条目就可能污染下游的内存增强推理，且仅靠辩论本身无法过滤此类错误。现有的保护机制通过启发式方法或基于大语言模型（LLM）的验证来过滤条目，但它们依赖于具有相同失败模式的 AI 判定，并忽略了 MAD 中的跨智能体动态。我们通过将 MAD 中的内存更新建模为一个零信任内存博弈来填补这一空白。在该博弈中，不假设任何智能体是诚实的，博弈的均衡状态作为最优内存信任的指标。在这一均衡的指导下，我们提出了 EquiMem，这是一种推理时校准机制。它利用智能体现有的检索查询和遍历路径作为证据，而非征求任何 LLM 判定，从而根据共享内存状态对每次更新进行算法量化。EquiMem 实现了针对嵌入式和基于图的内存的校准。在不同的基准测试、MAD 框架和内存架构中，它始终优于现有的保护机制，在对抗性智能体环境下保持稳健，且推理开销微乎其微。

## Abstract
Multi-agent debate (MAD) systems increasingly rely on shared memory to support long-horizon reasoning, but this convenience opens a critical vulnerability: a single corrupted entry can contaminate the downstream memory-augmented reasoning, and debate alone fails to filter such errors. Existing safeguards filter entries via heuristics or LLM-based validation, yet they rely on AI judgments that share the same failure modes and overlook the cross-agent dynamics of MAD. We address this gap by formulating memory updating in MAD as a zero-trust memory game, in which no agent is assumed honest and the game's equilibrium serves as an indicator of optimal memory trust. Guided by this equilibrium, we propose EquiMem, an inference-time calibration mechanism that quantifies each update algorithmically against the shared memory state, using agents' existing retrieval queries and traversal paths as evidence rather than soliciting any LLM judgment. EquiMem instantiates calibration for both embedding- and graph-based memory, and across diverse benchmarks, MAD frameworks, and memory architectures, it consistently outperforms existing safeguards, remains robust under adversarial agents, and incurs negligible inference overhead.

*   **研究背景**：在多智能体辩论（Multi-Agent Debate, MAD）系统中，共享内存被广泛用于存储中间推理结果，以支持长程任务。
*   **核心痛点**：现有的共享内存机制存在“单点崩溃”风险。一旦某个智能体写入错误或虚假信息（内存污染），这些错误会迅速传播并误导后续所有智能体的推理。
*   **研究动机**：现有的防御手段（如 LLM 审计或启发式过滤）要么成本高昂（需要额外调用 LLM），要么容易受到 AI 幻觉和共谋攻击的影响。本文旨在寻找一种无需额外 LLM 判定、且能客观量化内存条目可信度的校准机制。

### 2. 方法论
EquiMem 的核心思想是将内存更新过程建模为一个**零信任博弈（Zero-Trust Memory Game）**，通过博弈论均衡来确定最优的内存信任度。

*   **核心思想**：不假设任何智能体是诚实的。系统通过分析智能体在检索过程中的“客观行为证据”（如查询语句、在知识图谱中的遍历路径），而非其“主观陈述”，来评估新信息的质量。
*   **关键技术细节**：
    *   **证据提取**：利用智能体检索时的查询（Queries）和路径（Paths）作为证据。如果多个智能体在检索时避开了某个条目，或者该条目与大多数检索路径冲突，则其可信度降低。
    *   **三大校准指标**：
        1.  **冲突检测 ($\rho_{detect}$)**：衡量新条目与现有内存中已验证事实的逻辑矛盾程度。
        2.  **对齐度 ($\rho_{align}$)**：计算新条目与智能体检索意图及上下文的相关性。
        3.  **效用评估 ($\rho_{utility}$)**：基于博弈论中的纳什均衡（Nash Equilibrium），计算该条目对提升整体系统奖励的边际贡献。
*   **算法流程**：在每一轮辩论中，系统收集所有智能体的内存更新建议，计算上述指标，通过求解博弈均衡得到每个条目的权重，最后根据权重决定是否写入或如何更新共享内存。

### 6. 主要结论与发现
*   **性能领先**：EquiMem 在所有基准测试中均显著优于 Vanilla 和现有的过滤方法。
*   **高性价比**：在提升准确率的同时，EquiMem 的 Token 消耗远低于 LLM Audit，因为它避免了昂贵的自我审计调用。
*   **对抗鲁棒性**：在存在恶意智能体的情况下，传统方法性能会断崖式下跌，而 EquiMem 能够通过博弈均衡有效识别并隔离错误信息，保持性能稳定。
*   **架构普适性**：证明了博弈论校准在图内存（Graph Memory）上比在向量内存上更有效，因为图结构提供了更丰富的路径证据。

### 7. 优点
*   **创新性强**：首次将内存更新建模为零信任博弈，摆脱了对“诚实智能体”的假设。
*   **低延迟/低成本**：巧妙利用检索路径作为证据，实现了“免费”的校准，非常适合实时交互系统。
*   **数学支撑**：引入博弈论均衡，为原本感性的“信任”提供了理性的数学量化手段。

### 8. 不足与局限
*   **冷启动限制**：在任务开始的最初几轮，由于内存中缺乏足够的历史证据，校准效果可能受限。

*   **实现复杂度**：相比于简单的启发式规则，博弈论均衡的数学实现和参数调节对开发者有一定的门槛要求。
