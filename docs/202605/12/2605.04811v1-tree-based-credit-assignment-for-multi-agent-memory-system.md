## 摘要
记忆系统被广泛用于增强大语言模型（LLM）处理长程任务的能力，通常被组织为包含记忆构建、摘要和检索智能体的多智能体流水线。为了增强该系统，现有的基于强化学习（RL）的方法要么对所有智能体统一应用最终的下游任务奖励（如问答准确率），这种方式较为粗糙且模糊；要么为不同子任务上的智能体设计特定任务的奖励，这需要昂贵的标注（如关键证据）且难以进行可靠定义。为了解决这些局限性，我们提出了面向多智能体记忆系统的基于树的信用分配（TreeMem），它在无需特定任务标注的情况下，从最终奖励中推导出针对特定智能体的信用。具体而言，TreeMem 将多智能体流水线（构建者-摘要者-检索者）扩展为树状结构，其中每个智能体的输出都会扩展为多个后续分支。每个智能体的贡献通过对其后续分支进行蒙特卡洛平均来估算，从而捕捉中间智能体动作如何影响最终奖励。这将粗糙的最终奖励转化为针对特定智能体的优化信号。随后，这些信号被用于同时更新所有智能体的策略，帮助异构智能体有效地实现专业化。在长程基准测试上的实验表明，TreeMem 相比强基准模型提升了记忆系统的性能，验证了树状结构信用分配在多智能体记忆系统中的有效性。

## Abstract
Memory systems are widely adopted to enhance LLMs for long-horizon tasks, and are commonly organized as multi-agent pipelines with memory building, summarizing, and retrieval agents. To empower this system, existing RL-based methods either apply final downstream task rewards (e.g., QA accuracy) for all agents uniformly, which are coarse and ambiguous, or design task-specific rewards for agents on different subtasks, which require costly annotations (e.g., key evidence) and are difficult to define reliably. To address these limitations, we propose Tree-based Credit Assignment for Multi-Agent Memory Systems (TreeMem), which derives agent-specific credit from the final reward without task-specific annotations. Specifically, TreeMem extends the multi-agent pipeline (builder--summarizer--retrieval) into a tree structure, where each agent's outputs are expanded into multiple subsequent branches. The contribution of each agent is estimated via Monte Carlo averaging over its subsequent branches, capturing how intermediate agent actions may influence the final reward. This converts the coarse final reward into agent-specific optimization signals. These signals are then used to update all agent policies simultaneously, helping heterogeneous agents specialize effectively. Experiments on long-horizon benchmarks show that TreeMem improves memory system performance over strong baselines, validating the effectiveness of tree-structured credit assignment for the multi-agent memory system.

### 1. 核心问题与整体含义（研究动机和背景）
随着大语言模型（LLM）处理的任务日益复杂，**多智能体记忆系统**（通常包含构建、摘要、检索三个角色）成为解决长文本、长程任务的关键。然而，如何优化这些异构智能体面临两大挑战：
*   **奖励信号模糊**：仅使用最终任务结果（如QA准确率）作为奖励，对中间环节的智能体来说信号过于粗糙，难以指导特定角色的专业化。
*   **标注成本高且易偏离**：为每个智能体设计特定任务奖励（如证据覆盖率）需要大量人工标注，且容易导致“奖励黑客（Reward Hacking）”行为，即智能体优化了中间指标却损害了最终表现。
**研究动机**：在无需额外人工标注的情况下，如何从最终奖励中精准地为每个智能体分配信用（Credit Assignment）。

### 2. 方法论：TreeMem 核心思想与技术细节
TreeMem 提出了一种基于树状结构的强化学习训练框架，核心流程如下：
*   **树状结构采样（Tree-structured Rollout）**：将原本线性的流水线扩展为树。构建者（Builder）采样 $G$ 个动作；对于每个构建动作，摘要者（Summarizer）采样 $J$ 个分支；对于每个摘要，检索者（Responder）采样 $K$ 个最终答案。
*   **蒙特卡洛信用估算（Monte Carlo Credit Assignment）**：
    *   **检索者信用**：直接等于最终奖励 $R$。
    *   **摘要者信用**：对其后续 $K$ 个答案分支的奖励取平均。
    *   **构建者信用**：对其后续所有 $J \times K$ 个分支的奖励取平均，并加入长度惩罚项（防止冗余）。
*   **策略优化**：采用类似 GRPO 的组相对优化原则。从树中随机选出一组完整轨迹，将上述估算的信用归一化为优势函数（Advantage），同时更新三个智能体的策略。
*   **推理阶段**：训练完成后，推理时不再需要树状扩展，直接按线性流程调用优化后的策略，保持高效。

### 6. 主要结论与发现
*   **性能领先**：TreeMem 在所有长程任务基准上均显著优于现有方法，在 PersonaMem 上相比最强基准提升了约 3.2% - 8.8%。

*   **效率提升**：TreeMem 训练出的模型在推理时具有更高的 Token 效率（更简洁的记忆表示），且训练收敛速度优于统一奖励方法。

### 7. 优点（亮点）
*   **无需标注的细粒度优化**：巧妙地利用树状采样解决了多智能体协作中的“信用分配”难题，无需昂贵的中间过程标注。
*   **端到端对齐**：信用信号直接源自最终任务目标，避免了手动设计中间奖励可能导致的奖励黑客问题。
*   **训练与推理解耦**：复杂的树结构仅存在于训练阶段，不增加推理负担。

### 8. 不足与局限
*   **训练开销**：虽然论文提到通过采样缓解了压力，但树状扩展（$G \times J \times K$ 次采样）在训练初期的计算量和显存占用仍显著高于线性流水线。
*   **采样偏差**：信用估算的准确性高度依赖于分支采样数（$J, K$），若采样不足，信号可能带有噪声；若采样过多，计算成本激增。
*   **应用限制**：目前主要针对“构建-摘要-检索”这种固定角色的流水线，对于角色动态变化或更复杂拓扑的多智能体系统，其扩展性有待验证。
