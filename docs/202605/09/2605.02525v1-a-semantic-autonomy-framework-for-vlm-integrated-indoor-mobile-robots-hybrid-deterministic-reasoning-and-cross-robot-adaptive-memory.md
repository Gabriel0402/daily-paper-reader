## 摘要
自主室内移动机器人可以使用 ROS 2 Navigation 2 等成熟框架可靠地导航到度量坐标，但它们缺乏解释表达意图而非位置的自然语言指令的能力。视觉语言模型（VLM）提供了弥合这一差距所需的语义推理，但其推理延迟（在消费级硬件上每次决策需 2-9 秒）和会话间的遗忘限制了实际部署。本文提出了语义自主栈（Semantic Autonomy Stack），这是一个用于语义自主室内导航的六层参考框架，并在配备现成边缘硬件的物理机器人上验证了一个包含混合确定性-VLM 推理和跨机器人自适应记忆的完整实例。一个七步参数解析器在不调用语言模型、摄像头或 GPU 的情况下，在 0.1 毫秒内处理了 88% 的指令；只有真正模糊的指令才会升级到 VLM 推理。一个具有显式范围分类（全局环境知识、特定操作员偏好、特定机器人能力）的五类语义记忆框架实现了跨会话学习和跨机器人知识迁移：通过一台机器人上的 VLM 交互学习到的偏好被提升为确定性解析，并通过共享的编译摘要迁移到第二台机器人，实现了实测 103,000 倍的延迟降低。在两台定制的差速驱动机器人上进行的 82 次场景级决策和三个会话的实验验证表明，语义迁移准确率达到 100%（33/33，95% 置信区间 [0.894, 1.000]），语义解析准确率达到 100%，且具备并发多机器人运行的可行性——所有这些均在无板载 GPU 的 Raspberry Pi 5 平台上实现，且无需任何训练数据。

## Abstract
Autonomous indoor mobile robots can navigate reliably to metric coordinates using established frameworks such as ROS 2 Navigation 2, yet they lack the ability to interpret natural language instructions that express intent rather than positions. Vision-Language Models offer the semantic reasoning required to bridge this gap, but their inference latency (2-9 seconds per decision on consumer hardware) and session-by-session amnesia limit practical deployment. This paper presents the Semantic Autonomy Stack, a six-layer reference framework for semantically autonomous indoor navigation, and validates a complete instance featuring hybrid deterministic-VLM reasoning and cross-robot adaptive memory on physical robots with off-the-shelf edge hardware. A seven-step parametric resolver handles 88% of instructions in under 0.1 milliseconds without invoking a language model, camera, or GPU; only genuinely ambiguous instructions escalate to VLM reasoning. A five-category semantic memory framework with explicit scope taxonomy (global environment knowledge, per-operator preferences, per-robot capabilities) enables cross-session learning and cross-robot knowledge transfer: preferences learned through VLM interactions on one robot are promoted to deterministic resolution and transferred to a second robot via a shared compiled digest, achieving a measured latency reduction of 103,000-fold. Experimental validation on two custom-built differential-drive robots across 82 scenario-level decisions and three sessions demonstrates 100% semantic transfer accuracy (33/33, 95% CI [0.894, 1.000]), 100% semantic resolution accuracy, and concurrent multi-robot operation feasibility - all on Raspberry Pi 5 platforms with no onboard GPU, requiring zero training data.

### 论文总结：集成 VLM 的室内移动机器人语义自主框架

#### 1. 核心问题与整体含义（研究动机和背景）
传统的室内移动机器人（如基于 ROS 2 Nav2 栈的机器人）在度量导航（坐标到坐标）方面已趋于成熟，但难以理解人类的**语义意图**（如“带我去可以休息的地方”）。虽然视觉语言模型（VLM）能提供强大的语义推理，但在实际部署中面临三大挑战：
*   **高推理延迟**：在消费级边缘硬件上，VLM 每次决策需 2-9 秒，难以满足实时性需求。
*   **会话遗忘（Amnesia）**：机器人无法保留过往交互经验，每次重启都需重新推理。
*   **跨机器人知识共享缺失**：在一台机器人上学到的环境偏好无法直接传递给同环境下的其他机器人。

#### 2. 方法论：核心思想与关键技术
论文提出了 **语义自主栈（Semantic Autonomy Stack, SAS）**，这是一个六层参考架构（L0-L5），核心思想是**双过程推理（Dual-Process Reasoning）**：

*   **混合推理架构（L3）**：
    *   **L3a（确定性快速路径）**：采用一个**七步参数化解析器**。它按顺序检查：M3 记忆偏好匹配、显式节点 ID、节点名称匹配、对象 ID 匹配、属性评分、单类匹配、邻近度+类别匹配。该路径不调用 VLM，不使用 GPU，延迟极低。
    *   **L3b（VLM 慢速路径）**：仅当 L3a 无法解析时，才调用 VLM（如 Qwen 3.5:4b）。它结合当前图像、导航图、语义对象和记忆摘要生成 Prompt。
*   **五类语义记忆框架（L5, M1-M5）**：
    *   将记忆分为：环境统计（M1）、时间模式（M2）、操作员偏好（M3）、平台能力（M4）和任务历史（M5）。
    *   **知识提升机制**：当某个 VLM 推理结果在频率和一致性上达到阈值时，会被“提升”为 M3 偏好，存入共享的编译摘要（Digest），从而在后续任务中进入 L3a 快速路径。
*   **执行契约 $\langle A, O, V, L \rangle$**：定义了可供性清单、观测归一化、动作验证和审计日志，确保 VLM 输出的安全性和可追溯性。

#### 3. 实验设计
*   **场景**：在大学走廊环境（24 个节点，60 条边，18 个静态 POI）中进行。
*   **机器人**：两台定制的差速驱动机器人（Xplorer-B 和 Xplorer-C），硬件配置略有差异（单/双树莓派 5）。
*   **实验流程**：
    *   **会话 A**：在机器人 C 上进行学习，通过 VLM 处理模糊指令（如“找个能坐下放松的地方”）。
    *   **会话 B**：验证跨机器人迁移。将机器人 C 学到的偏好通过共享摘要传递给机器人 B，观察其是否能通过 L3a 快速解析。
    *   **会话 C**：验证多机器人并发运行的可行性。
*   **对比基准**：主要对比了 VLM 推理路径（L3b）与确定性路径（L3a）在延迟和准确率上的差异。

#### 4. 资源与算力
*   **边缘端（机器人）**：使用 **Raspberry Pi 5 (16GB RAM)**，无板载 GPU。负责导航（Nav2）、感知（YOLO26n CPU 推理）和本地控制。
*   **推理端（工作站）**：使用配备 **NVIDIA RTX 5050 GPU** 的笔记本电脑，通过 Ollama 运行 **Qwen 3.5:4b** 模型。
*   **通信**：机器人与工作站之间通过 HTTP（获取上下文）和 ROS 2 DDS（发送动作指令）连接。

#### 5. 实验数量与充分性
*   **实验规模**：共收集了 **82 组场景级决策数据**。
*   **充分性**：实验涵盖了从 VLM 推理到记忆提升，再到跨机器人迁移的完整闭环。通过统计学检验（如 Mann-Whitney U 检验、Fisher 精确检验）验证了延迟降低的显著性和语义迁移的准确性。
*   **客观性**：使用了两台硬件不同的机器人，证明了框架的平台无关性；同时区分了“语义解析成功”与“导航物理到达”，客观分析了失败原因（如 AMCL 定位漂移）。

#### 6. 主要结论与发现
*   **延迟大幅降低**：通过记忆提升，指令解析延迟从 VLM 的 6.7 秒降低至 0.065 毫秒，**缩减了 10.3 万倍**。
*   **高准确率**：跨机器人语义迁移准确率达到 **100% (33/33)**。
*   **VLM 负载优化**：在实际运行中，**88% 的指令**由确定性解析器处理，极大地节省了 GPU 资源。
*   **分层解耦有效性**：实验中发现的感知层问题（YOLO 误报）可以在不改动推理层的情况下独立修复。

#### 7. 优点
*   **实用性强**：成功在低功耗边缘设备（树莓派）上实现了复杂的语义导航，不依赖昂贵的板载 GPU。
*   **架构优雅**：SAS 六层模型为语义导航提供了清晰的标准，双过程推理平衡了灵活性与效率。
*   **闭环学习**：不仅是使用 VLM，还实现了从“慢思考”到“快反应”的知识进化。

#### 8. 不足与局限
*   **环境单一**：实验仅在单一走廊环境进行，尚未在多房间、多楼层或动态剧烈变化的环境中验证。
*   **语义匹配简单**：目前的 M3 偏好匹配基于 Jaccard 词频相似度，而非语义向量（Embedding），对同义词或复杂改写的鲁棒性有限。
*   **视觉确认受限**：由于摄像头安装高度固定（16cm），无法确认高处的物体（如壁挂式消火栓）。
*   **并发瓶颈**：虽然支持多机器人，但若多个机器人同时触发 VLM 推理，共享工作站会出现排队延迟。
