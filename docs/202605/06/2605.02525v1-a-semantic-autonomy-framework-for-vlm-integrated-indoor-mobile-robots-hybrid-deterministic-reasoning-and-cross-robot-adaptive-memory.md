## 摘要
自主室内移动机器人可以使用 ROS 2 Navigation 2 等成熟框架可靠地导航到度量坐标，但它们缺乏解释表达意图而非位置的自然语言指令的能力。视觉语言模型（VLM）提供了弥合这一差距所需的语义推理能力，但其推理延迟（在消费级硬件上每次决策需 2-9 秒）和逐会话遗忘限制了实际部署。本文提出了语义自主栈（Semantic Autonomy Stack），这是一个用于语义自主室内导航的六层参考框架，并在配备现成边缘硬件的物理机器人上验证了一个包含混合确定性-VLM 推理和跨机器人自适应记忆的完整实例。一个七步参数化解析器在不调用语言模型、摄像头或 GPU 的情况下，在 0.1 毫秒内处理了 88% 的指令；只有真正模糊的指令才会升级到 VLM 推理。一个具有明确范围分类（全局环境知识、各操作员偏好、各机器人能力）的五类语义记忆框架实现了跨会话学习和跨机器人知识迁移：通过在一台机器人上进行 VLM 交互学习到的偏好被提升为确定性解析，并通过共享的编译摘要传输到第二台机器人，实现了实测 103,000 倍的延迟降低。在两台定制的差速驱动机器人上进行的 82 次场景级决策和三个会话的实验验证表明，语义迁移准确率达到 100%（33/33，95% 置信区间 [0.894, 1.000]），语义解析准确率达到 100%，且具备并发多机器人运行的可行性——所有这些均在无板载 GPU 的树莓派 5 平台上实现，且无需任何训练数据。

## Abstract
Autonomous indoor mobile robots can navigate reliably to metric coordinates using established frameworks such as ROS 2 Navigation 2, yet they lack the ability to interpret natural language instructions that express intent rather than positions. Vision-Language Models offer the semantic reasoning required to bridge this gap, but their inference latency (2-9 seconds per decision on consumer hardware) and session-by-session amnesia limit practical deployment. This paper presents the Semantic Autonomy Stack, a six-layer reference framework for semantically autonomous indoor navigation, and validates a complete instance featuring hybrid deterministic-VLM reasoning and cross-robot adaptive memory on physical robots with off-the-shelf edge hardware. A seven-step parametric resolver handles 88% of instructions in under 0.1 milliseconds without invoking a language model, camera, or GPU; only genuinely ambiguous instructions escalate to VLM reasoning. A five-category semantic memory framework with explicit scope taxonomy (global environment knowledge, per-operator preferences, per-robot capabilities) enables cross-session learning and cross-robot knowledge transfer: preferences learned through VLM interactions on one robot are promoted to deterministic resolution and transferred to a second robot via a shared compiled digest, achieving a measured latency reduction of 103,000-fold. Experimental validation on two custom-built differential-drive robots across 82 scenario-level decisions and three sessions demonstrates 100% semantic transfer accuracy (33/33, 95% CI [0.894, 1.000]), 100% semantic resolution accuracy, and concurrent multi-robot operation feasibility - all on Raspberry Pi 5 platforms with no onboard GPU, requiring zero training data.

### 论文总结：集成 VLM 的室内移动机器人语义自主框架 (SAS)

#### 1. 核心问题与整体含义（研究动机和背景）
传统的室内移动机器人导航（如 ROS 2 Nav2）主要依赖度量坐标（坐标点），无法理解人类基于意图的自然语言指令（如“带我去可以休息的地方”）。虽然视觉语言模型（VLM）能提供语义推理，但面临三大挑战：
*   **高推理延迟**：在边缘硬件上每次决策需 2-9 秒，难以满足实时性。
*   **会话失忆**：机器人无法保留过往经验，每次重启都需重新推理。
*   **跨机器人协作难**：缺乏在多机器人间共享语义知识和操作员偏好的机制。

#### 2. 方法论：核心思想与关键技术
论文提出了 **语义自主栈 (Semantic Autonomy Stack, SAS)**，这是一个六层参考框架（L0-L5），核心思想是**双过程推理**（类似人类的快慢思考系统）：

*   **混合推理架构 (L3)**：
    *   **L3a（确定性快速解析）**：一个七步参数化级联解析器。它首先检查是否有已学习的偏好（M3 存储），然后依次匹配节点 ID、名称、对象属性、类别及距离。该过程不调用 VLM，延迟低于 0.1ms。
    *   **L3b（VLM 深度推理）**：仅当 L3a 无法解析时，才调用 VLM（如 Qwen 3.5:4b）。它结合当前图像、导航图、语义对象和记忆摘要进行推理。
*   **五类语义记忆框架 (L5)**：
    *   **M1-M5 分类**：涵盖环境统计、时间模式、操作员偏好、平台能力和任务历史。
    *   **知识提升与迁移**：通过 VLM 解决的指令，若达到一定频率和一致性，会被“提升”为确定性规则，并编译进一个轻量级摘要（<500 tokens），供所有机器人共享。
*   **执行合同 ⟨A, O, V, L⟩**：规范了可供性清单、观测归一化、动作验证和审计日志，确保模型无关性和安全性。

#### 3. 实验设计
*   **场景**：在大学走廊环境（24 个节点，18 个静态语义对象）中进行。
*   **机器人**：两台定制的差速驱动机器人（Xplorer-B 和 Xplorer-C），硬件配置略有差异以验证通用性。
*   **实验流程**：
    *   **会话 A**：学习周期验证。让机器人 C 通过 VLM 学习新指令（如“找个能坐下放松的地方”）。
    *   **会话 B**：跨机器人迁移验证。将机器人 C 学习到的偏好通过记忆摘要迁移给机器人 B。
    *   **会话 C**：并发运行验证。两台机器人同时运行，共享记忆。
*   **对比基准**：主要对比了 VLM 直接推理与经过记忆提升后的确定性推理在延迟和准确率上的差异。

#### 4. 资源与算力
*   **边缘端（机器人）**：使用 **Raspberry Pi 5 (16GB RAM)**，无板载 GPU。负责导航、感知（YOLO 目标检测）和本地控制。
*   **推理端（工作站）**：使用配备 **NVIDIA RTX 5050 GPU** 的笔记本电脑，通过 Ollama 运行 **Qwen 3.5:4b** 模型。
*   **训练时长**：该方法为 **Zero-shot（零样本）**，无需针对特定任务进行模型训练或微调。

#### 5. 实验数量与充分性
*   **实验规模**：共收集了 **82 次场景级决策数据**。
*   **充分性**：实验涵盖了学习、迁移、并发三个关键维度。通过统计学检验（如 Mann-Whitney U 检验、Fisher 精确检验）验证了结果的显著性。
*   **客观性**：实验在物理机器人而非仿真环境中完成，考虑了传感器噪声和定位漂移。虽然环境单一，但对于验证架构逻辑而言是充分且公平的。

#### 6. 主要结论与发现
*   **延迟大幅降低**：通过确定性解析器绕过了 88% 的 VLM 调用，延迟从秒级降至亚毫秒级，**缩减了 103,000 倍**。
*   **高准确率**：跨机器人语义迁移准确率达到 **100% (33/33)**。
*   **解耦成功**：证明了语义解析准确率（100%）与底层导航成功率（88%）是解耦的，导航失败主要是由于定位漂移而非理解错误。
*   **边缘可行性**：证明了在不具备板载 GPU 的低功耗设备（树莓派）上实现复杂语义导航的可行性。

#### 7. 优点
*   **架构优雅**：六层 SAS 框架实现了感知、推理、执行的深度解耦，易于组件替换。
*   **实用性强**：通过“快慢系统”解决了 VLM 算力消耗大的痛点，适合工业和室内服务场景。
*   **知识共享**：创新的 M1-M5 记忆分类法为多机器人协同提供了清晰的知识共享逻辑。

#### 8. 不足与局限
*   **环境单一**：仅在单一走廊环境验证，尚未在多房间、多楼层或动态剧烈变化的环境中测试。
*   **匹配算法简单**：目前的记忆匹配依赖 Jaccard 词汇相似度，对于语义相同但措辞完全不同的指令（如“我内急” vs “去洗手间”）可能失效。
*   **VLM 串行化**：在当前架构下，若多台机器人同时请求 VLM 推理，工作站会产生排队延迟。
*   **视野限制**：低安装位的摄像头无法确认高处的物体（如壁挂式消火栓），导致视觉确认环节受限。
