## 摘要
基于大语言模型（LLM）的多智能体系统在生产环境中的失败率在 41% 到 87% 之间，这主要归因于协同缺陷而非基础模型的能力。现有的应对方案分为两类：一类是根据经验对失败模式进行分类，另一类是提供声明式编排框架作为工程工具；但两者都未能提供从协同配置到可预测失败模式特征的原则性映射。我们认为，协同应被视为一个可配置的架构层，与智能体逻辑和信息访问相分离，从而实现架构推理，而不仅仅是提高工程生产力。我们通过预测市场上的信息控制设计来实例化这一观点：在五种参考协同配置中，使用单一 LLM、固定工具、固定的单次调用输出上限和固定的提示词模板，并将每个问题的总计算量视为内生架构输出。Brier 分数的 Murphy 分解将校准度与判别力分开，因此即使总分一致，不同的配置也会留下可区分的特征。在模型训练截止日期（claude-opus-4-6）之后结算的 100 个 Polymarket 二元市场中，我们报告了 Murphy 特征、成本-质量帕累托前沿、类别条件分析以及自助法（bootstrap）效能预测。五个预设预测中有三个在方向上得到了验证；在此方案下，有两种配置在帕累托前沿占据主导地位；探索性自助法区间将共识对齐与其他配置区分开来，尽管在 n=100 时两两测试未能通过 Bonferroni 校正。我们还在启用网络搜索的条件下，将相同的配置作为实时智能体部署在 Foresight Arena 上，作为一个并行积累的链上复制通道。测试框架、追踪数据集和生产智能体均已发布。我们将其定位为验证方法论的首次实例化，而非通用的跨模型结论。

## Abstract
Multi-agent LLM systems fail in production at rates between 41% and 87%, mostly due to coordination defects rather than base-model capability. Existing responses split between cataloguing failure modes empirically and shipping declarative orchestration frameworks as engineering tools; neither delivers a principled mapping from coordination configuration to predictable failure-mode signature. We argue that coordination should be treated as a configurable architectural layer, separable from agent logic and from information access, enabling architectural reasoning rather than only engineering productivity.   We instantiate this with an information-controlled design on prediction markets: a single LLM, fixed tools, fixed per-call output cap, and fixed prompt template across five reference coordination configurations, with total compute per question treated as an endogenous architectural output. The Murphy decomposition of the Brier score separates calibration from discriminative power, so configurations leave distinguishable signatures even when aggregate scores coincide.   On 100 Polymarket binary markets resolved after the model's training cutoff (claude-opus-4-6) we report Murphy signatures, a cost-quality Pareto frontier, category-conditioned analysis, and a bootstrap power-projection. Three of five pre-specified predictions are upheld in direction; two configurations dominate the Pareto frontier within this regime; exploratory bootstrap intervals separate consensus alignment from others, though pairwise tests do not survive Bonferroni correction at n=100. We also deploy the same configurations as live agents on Foresight Arena under web-search-enabled conditions, as an on-chain replication channel accumulating in parallel. Harness, trace dataset, and production agents are released. We position this as a methodology-validating first instantiation, not a general cross-model claim.

*   **核心问题**：多智能体大语言模型（LLM）系统在生产环境中的失败率极高（41%-87%），且大部分失败源于**协同缺陷**（如指令歧义、智能体间失调、验证缺失），而非基础模型的能力不足。
*   **研究动机**：现有的研究要么只是罗列失败模式，要么只是将“编排”视为一种工程上的便利（如 YAML 定义工作流），缺乏一种将**协同配置**与**系统行为/失败特征**进行关联的原则性架构理论。
*   **整体含义**：论文主张将“协同”视为一个独立的**架构层**，通过科学的方法论（信息控制实验）来预测和解释不同协同结构对系统性能（准确度、校准度、成本）的影响。

### 2. 方法论：核心思想与关键技术
*   **三层架构分解**：将系统分为**信息层**（工具、上下文）、**协同层**（拓扑、权限、同步、聚合规则）和**智能体层**（具体 LLM 调用）。
*   **信息控制原则**：为了消除干扰，实验严格固定了底层模型（Claude 3 Opus）、工具集、提示词模板和单次调用 Token 上限，仅改变协同拓扑结构。
*   **Murphy 分解法**：不只看总分（Brier Score），而是将其分解为：
    *   **可靠性（REL）**：预测概率与实际发生频率的匹配程度（校准度）。
    *   **判别力（RES）**：区分不同结果的能力。
*   **五种参考协同配置**：
    1.  **独立集成（IE）**：N 个智能体独立回答，取平均值。
    2.  **同行评议辩论（PC）**：智能体互相观察并修正。
    3.  **编排者-专家（OS）**：由一个规划者分解任务并分发给专家。
    4.  **顺序流水线（SP）**：研究、分析、预测按序进行。
    5.  **共识对齐（CA）**：迭代直到智能体达成一致。

### 3. 实验设计与 Benchmark
*   **场景与数据集**：使用 **Polymarket 预测市场**。选择了 100 个在模型训练截止日期（2025年8月）之后结算的二元预测市场，确保模型无法通过记忆获知结果。
*   **Benchmark**：以 **Polymarket 市场共识价格**（人类群体的集体预测）作为基准。
*   **对比维度**：对比了上述五种协同架构在 Brier 分数、Alpha 分数（相对于市场的超额表现）、Murphy 分解特征以及成本（Token 消耗）上的差异。

### 4. 资源与算力
*   **算力消耗**：论文未提及 GPU 训练算力，因为实验基于 API 调用。
*   **实验开销**：总计消耗约 **1710 万个 Token**，API 成本为 **109.65 美元**。
*   **运行时间**：在 100 个市场上的完整实验运行耗时 **3 小时 9 分钟**。

### 5. 实验数量与充分性
*   **实验规模**：共 100 个市场，涵盖加密货币、政治、体育、经济、地缘政治和娱乐六大类。
*   **充分性与客观性**：
    *   **充分性**：对于初步验证方法论是充分的，但论文坦承 n=100 的样本量在进行 Bonferroni 校正后的统计显著性不足，需要更大规模（n≈500-2000）的后续实验。
    *   **客观性**：采用了“信息控制”设计，确保了不同架构之间的对比是公平的（没有因为某个架构多看了搜索结果而占便宜）。

### 6. 主要结论与发现
*   **架构决定特征**：不同的协同配置确实产生了可预测的 Murphy 特征。例如，“共识对齐”会导致判别力（RES）大幅下降，容易陷入“平庸的中间值”。
*   **帕累托前沿**：**独立集成（IE）**和**顺序流水线（SP）**构成了成本-质量的最优边界。而“编排者-专家”和“同行辩论”在当前设置下被证明是“劣质”的（成本更高但效果更差）。
*   **负 Alpha 现象**：在禁用网络搜索的情况下，所有配置的表现均未超过市场共识，证明了信息获取（而非单纯的协同）是预测任务的核心瓶颈。
*   **预测验证**：预设的 5 个架构行为预测中有 3 个得到了证实。

### 7. 优点与亮点
*   **方法论严谨**：首次在多智能体研究中严格区分了“架构效应”和“信息访问效应”，解决了现有研究中的混淆问题。
*   **可解释性强**：引入 Murphy 分解，让研究者能看清系统是“盲目自信”还是“缺乏见解”，而不仅仅是一个准确率数字。
*   **开源贡献**：发布了完整的实验框架、包含 1700 万 Token 的推理追踪数据集以及生产级智能体代码。

### 8. 不足与局限
*   **样本量限制**：n=100 导致部分两两对比的统计效能不足，无法得出绝对的统计显著结论。
*   **模型单一**：实验仅基于 Claude 3 Opus，尚未验证这些架构特征在 GPT 或其他模型上是否具有通用性。
*   **信息受限**：为了控制变量禁用了网络搜索，这使得实验结果更偏向于测试模型的“内部推理协同”，而非实际生产中“搜索+协同”的综合能力。
*   **提示词敏感性**：虽然固定了模板，但未对角色描述的微小差异进行消融实验。
