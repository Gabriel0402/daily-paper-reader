## 摘要
我们推出了 Cattle Trade，这是一个多智能体基准，用于评估大语言模型（LLM）作为智能体在不完全信息、对抗性交互和资源约束下的策略推理能力。该基准在长达 50-60 轮的长时程博弈中，融合了拍卖、隐藏报价交易挑战（TCs）、讨价还价、虚张声势、对手建模和资源分配。与以往孤立测试这些能力的智能体基准不同，Cattle Trade 评估智能体是否能在具有利益冲突的竞争性多智能体经济博弈中整合这些能力。该基准记录了每一次出价、TC 报价、反向报价和卡牌选择，从而实现了超越最终得分或胜率的行为分析。我们在 242 场博弈中评估了七个高性价比的语言模型和三个确定性代码智能体。研究发现，策略连贯性（特别是支出效率、资源纪律和阶段自适应竞价）与排名的相关性比支出总量或任何单一子技能更强。两个启发式代码智能体的表现优于大多数受测 LLM，而行为轨迹揭示了 LLM 反复出现的失败模式，包括过度竞价、自我竞价、破产式 TC 发起以及对对手状态的适应性较弱。评估智能体能力需要能够在具有利益冲突、不确定性和经济动态的多智能体环境中测试多种能力协同部署的基准。

## Abstract
We introduce \textsc{Cattle Trade, a multi-agent benchmark for evaluating large language models (LLMs) as agents in strategic reasoning under imperfect information, adversarial interaction, and resource constraints. The benchmark combines auctions, hidden-offer trade challenges (TCs), bargaining, bluffing, opponent modeling, and resource allocation within a single long-horizon game lasting 50--60 turns. Unlike prior agent benchmarks that test these abilities in isolation, \textsc{Cattle Trade} evaluates whether agents integrate them across a competitive, multi-agent economic game with conflicting incentives. The benchmark logs every bid, TC offer, counteroffer, and card selection, enabling behavioural analysis beyond final scores or win rates. We evaluate seven cost-efficient language models and three deterministic code agents across 242 games. Strategic coherence, in particular spending efficiency, resource discipline, and phase-adaptive bidding, is associated with rank more strongly than spending volume or any single subskill. Two heuristic code agents outperform most tested LLMs, and behavioural traces surface recurring LLM failure modes including overbidding, self-bidding, bankrupt TC initiation, and weak opponent-state adaptation. Evaluating agentic competence requires benchmarks that test the joint deployment of multiple capabilities in multi-agent environments with conflicting incentives, uncertainty, and economic dynamics.

#### 1. 论文的核心问题与整体含义
论文的核心问题在于：现有的 LLM 基准测试（如 MMLU、HumanEval）主要侧重于知识和代码生成，而游戏类基准（如 Avalon、Werewolf）往往孤立地测试单一能力（如欺骗或合作）。在现实的经济交互（如谈判、采购、投资组合管理）中，智能体需要**在不确定性、对抗性激励和资源约束下，整合多种策略能力**。
**研究动机**：评估 LLM 是否能在一个长时程（50-60 轮）、多能力的经济博弈中，将竞价、讨价还价、虚张声势和资源管理有机结合。

#### 2. 论文提出的方法论
论文提出了 **Cattle Trade** 基准，改编自经典桌游《Kuhhandel》（又名《你是在吹牛吧！》）。
*   **核心思想**：通过一个包含 40 张动物卡和 55 张货币卡的博弈环境，迫使智能体在信息不对称的情况下进行决策。
*   **关键机制**：
    *   **拍卖与优先购买权（Buy-right）**：拍卖者可以选择接受最高出价，或支付同等金额自己留下卡牌。
    *   **隐藏报价贸易挑战（TC）**：双方进行背对背报价，金额隐藏。这引入了**虚张声势（Bluffing）**机制（使用 0 面值货币卡）。
    *   **离散货币与不设找零**：支付必须使用固定面额卡片，多付不退，增加了资源管理的难度。
    *   **乘法评分规则**：总分 = (完成的动物组价值之和) × (完成的组数)。这鼓励智能体追求多样性而非单一高价值组。
*   **算法流程**：智能体接收自然语言观察（手牌、公开信息、历史记录），通过结构化 JSON 输出行动指令，并维护一个用于长期记忆的“草稿本（Scratchpad）”。

#### 3. 实验设计
*   **场景设置**：标准配置为 4 名玩家，10 种动物类型，游戏持续 50-60 轮。
*   **对比方法**：
    *   **LLM 智能体**：评估了 7 个模型，包括 Gemini 3 Flash (G3-F)、Gemini 3.1 Flash Lite (G3.1-FL)、Claude Sonnet 4.5、DeepSeek v3.2、GPT-5.4 Nano、Claude Haiku 4.5、Gemini 2.5 Flash Lite。
    *   **代码智能体（基准线）**：三个确定性启发式智能体：**TrackerAgent**（完美信息跟踪）、**SetRaceAgent**（贪婪追求成组）、**EconomyAgent**（保守资源管理）。
*   **实验格式**：(1) 纯 LLM 锦标赛；(2) 混合模式（1 个 LLM 对阵 3 个代码智能体）。

#### 4. 资源与算力
*   论文**未明确说明**具体的 GPU 型号、数量或训练时长。
*   由于该基准主要针对推理阶段的评估而非模型训练，其成本主要体现在 API 调用上。论文提到不同模型的 Token 消耗差异巨大（G3-F 约为 G3.1-FL 的 20 倍）。

#### 5. 实验数量与充分性
*   **实验规模**：共进行了 **242 场完整博弈**（包含 228 场核心实验和 14 场探索性实验）。
*   **充分性与客观性**：
    *   实验涵盖了多种模型组合和代码智能体配置，使用了 TrueSkill 评分系统进行排名，具有较好的统计学意义。
    *   通过记录每一轮的详细行为（如出价激进程度、虚张声势频率），分析不仅停留在胜率，还深入到了行为模式。
    *   **局限性**：Sonnet 4.5 的样本量较小（n=14），且所有实验均在低推理努力（low reasoning effort）设置下进行。

#### 6. 论文的主要结论与发现
*   **策略连贯性胜过单一技能**：排名靠前的模型（如 G3-F）在支出效率、资源纪律和阶段性自适应竞价方面表现更好。
*   **代码智能体表现强劲**：TrackerAgent 和 SetRaceAgent 击败了大多数受测 LLM，说明 LLM 在信息利用和决策一致性上仍有欠缺。
*   **LLM 典型失败模式**：
    *   **过度出价（Overbidding）**：出价超过自身持有现金。
    *   **自我竞价（Self-bidding）**：在没有竞争对手加价的情况下不断抬高自己的价格。
    *   **破产式挑战**：在没有现金的情况下发起贸易挑战。
*   **模型排名**：Gemini 3 Flash 表现最强，显著优于其他模型和代码智能体。

#### 7. 优点
*   **综合性强**：首次在一个环境中整合了拍卖、博弈、欺骗和资源管理等多种复杂经济行为。
*   **诊断价值高**：通过结构化日志，能够精准定位模型失败的具体原因（如是数学计算错误还是策略失误）。
*   **设计精巧**：引入“不设找零”和“0 面值虚张声势”等机制，极大地增加了策略深度。

#### 8. 不足与局限
*   **模型覆盖**：主要测试了高性价比模型，未全面测试顶级旗舰模型在全量推理预算下的表现。
*   **提示词敏感性**：所有模型使用相同提示词，未探讨提示词工程对策略提升的潜力。
*   **缺乏人类对比**：未引入人类玩家数据，难以判断 LLM 的失败模式是模型特有还是人类也会犯的策略错误。
*   **交互限制**：目前仅支持结构化行动，不支持智能体之间的自然语言对话（如结盟、劝说）。
