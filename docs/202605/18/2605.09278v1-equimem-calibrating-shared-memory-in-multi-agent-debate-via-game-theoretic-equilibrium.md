## 摘要
多智能体辩论（MAD）系统日益依赖共享内存来支持长程推理，但这种便利性也带来了一个关键漏洞：单个损坏的条目就可能污染下游的内存增强推理，而仅靠辩论本身无法过滤此类错误。现有的保护措施通过启发式方法或基于大语言模型（LLM）的验证来过滤条目，但它们依赖于具有相同失败模式的 AI 判断，并忽视了 MAD 中的跨智能体动态。我们通过将 MAD 中的内存更新建模为一个零信任内存博弈来填补这一空白，在该博弈中，不假设任何智能体是诚实的，博弈的均衡状态作为最优内存信任的指标。在这一均衡的指导下，我们提出了 EquiMem，这是一种推理时校准机制，它利用智能体现有的检索查询和遍历路径作为证据，通过算法量化每次更新相对于共享内存状态的影响，而不是征求任何 LLM 的判断。EquiMem 实现了针对嵌入式和基于图的内存的校准。在不同的基准测试、MAD 框架和内存架构中，它始终优于现有的保护措施，在对抗性智能体环境下保持稳健，且推理开销微乎其微。

## Abstract
Multi-agent debate (MAD) systems increasingly rely on shared memory to support long-horizon reasoning, but this convenience opens a critical vulnerability: a single corrupted entry can contaminate the downstream memory-augmented reasoning, and debate alone fails to filter such errors. Existing safeguards filter entries via heuristics or LLM-based validation, yet they rely on AI judgments that share the same failure modes and overlook the cross-agent dynamics of MAD. We address this gap by formulating memory updating in MAD as a zero-trust memory game, in which no agent is assumed honest and the game's equilibrium serves as an indicator of optimal memory trust. Guided by this equilibrium, we propose EquiMem, an inference-time calibration mechanism that quantifies each update algorithmically against the shared memory state, using agents' existing retrieval queries and traversal paths as evidence rather than soliciting any LLM judgment. EquiMem instantiates calibration for both embedding- and graph-based memory, and across diverse benchmarks, MAD frameworks, and memory architectures, it consistently outperforms existing safeguards, remains robust under adversarial agents, and incurs negligible inference overhead.

# 论文总结：《EquiMem: Calibrating Shared Memory in Multi-Agent Debate via Game-Theoretic Equilibrium》

## 1. 核心问题与研究动机
*   **核心问题**：在多智能体辩论（Multi-Agent Debate, MAD）系统中，共享内存虽然能增强长程推理能力，但也极易受到“内存污染”的影响。单个智能体的幻觉、谄媚（Sycophancy）或错误自信会导致错误信息进入内存，进而误导所有下游推理。
*   **研究动机**：
    *   **现有防御的缺陷**：目前的保护机制（如 LLM 审计或困惑度过滤）要么依赖于同样会产生幻觉的 AI 判断，要么忽视了多智能体之间复杂的互动动态。
    *   **辩论失效**：仅靠辩论本身无法过滤错误，因为智能体往往会屈从于高置信度的错误主张，导致内存被错误信息占据。
    *   **目标**：建立一种不依赖于单一智能体诚实性假设、且无需额外 LLM 调用开销的内存校准机制。

## 2. 方法论
*   **核心思想**：将内存更新过程建模为**零信任内存博弈（Zero-Trust Memory Game）**。在该博弈中，不假设任何智能体是诚实的。系统通过博弈论中的均衡状态（Equilibrium）来识别最优的内存信任度。
*   **关键技术细节（EquiMem 机制）**：
    *   **角色分配**：更新提案者为“贡献者”，其余智能体为“审计者”。
    *   **校准指标 ($\rho$)**：由检测信号 ($\rho_{detect}$) 和对齐信号 ($\rho_{align}$) 的几何平均值组成。
    *   **检测信号 ($\rho_{detect}$)**：
        *   **嵌入式内存**：检查新条目是否在向量空间中过于密集（可能试图主导检索）。
        *   **图内存**：检查新边是否符合局部关系邻域和现有的多跳推理路径。
    *   **对齐信号 ($\rho_{align}$)**：利用审计者在辩论中产生的**现有检索查询或遍历路径**作为“探针”。如果新条目导致检索结果发生异常偏移或与现有路径冲突，则降低其得分。
    *   **信誉权重 ($w_i$)**：根据智能体提供的证据是否结构化有效，动态调整其在校准中的权重，实现跨回合的信誉追踪。
*   **算法流程**：EquiMem 拦截智能体的原生检索行为，通过算法量化更新条目与全局内存状态的一致性，产生校准分数。分数高于动态阈值的条目被允许存入，并带有与其分数成正比的“信任权重”，在后续检索中进行加权。

## 3. 实验设计
*   **数据集与场景**：
    *   **推理密集型**：HotpotQA（多跳问答）、LoCoMo（长程对话内存）。
    *   **动作密集型**：ALFWorld（具身家庭任务）、WebShop（网络购物决策）。
*   **对比基准（Baselines）**：
    *   **Vanilla**：无保护的共享内存。
    *   **LLM-Audit**：使用辅助 LLM 对更新进行评分。
    *   **PPL-Filter**：基于困惑度（Perplexity）的过滤器。
    *   **A-MemGuard**：基于推理路径共识的防御方法。
*   **覆盖范围**：实验跨越了 3 种 MAD 框架（AutoGen, MacNet, DyLAN）和 4 种内存架构（Voyager, MemoryBank, AriGraph, G-Memory）。

## 4. 资源与算力
*   **硬件设施**：使用单张 **NVIDIA RTX 6000 Ada Generation (49 GB VRAM)** GPU。
*   **模型后端**：主要使用 **Qwen3-VL-8B-Instruct**。此外还测试了 DeepSeek-R1-Distill-7B 和 Llama-3.1-8B 以验证通用性。
*   **部署方式**：通过 Ollama 进行本地部署。

## 5. 实验数量与充分性
*   **实验规模**：论文进行了极其详尽的实验，包括：
    *   在 4 个数据集、3 个框架、4 种内存架构上的全交叉评估。
    *   **消融实验**：分别移除检测信号、对齐信号和信誉权重，验证各组件贡献。
    *   **鲁棒性测试**：模拟了 1 到 3 个（最高 50%）对抗性智能体进行“内存投毒”和“审计合谋”的极端场景。
    *   **效率分析**：对比了 Token 消耗和推理延迟。
*   **充分性评价**：实验设计非常全面且客观。通过引入对抗性攻击场景，充分证明了该方法在“零信任”环境下的有效性，而非仅在理想环境下工作。

## 6. 主要结论与发现
*   **性能领先**：EquiMem 在所有基准测试、框架和内存架构组合中均排名第一。
*   **鲁棒性极强**：即使在 50% 的智能体是恶意对抗者的情况下，EquiMem 仍能保持接近良性环境的性能，而基准方法往往会崩溃至低于“无内存”水平。
*   **极低开销**：由于校准过程是算法化的（LLM-Free），它增加了 **0 个额外的 Token 消耗**，且推理延迟增加**低于 5%**。
*   **状态感知的重要性**：实验证明，将更新放在全局内存状态中校准（State-aware），比孤立地检查单个条目（Isolated check）能更有效地捕获逻辑冲突。

## 7. 优点
*   **理论完备**：将内存安全问题提升到博弈论高度，提供了严谨的数学证明和均衡分析。
*   **架构无关**：设计精巧，能够同时适配向量数据库和知识图谱两种主流内存架构。
*   **实用性高**：不依赖昂贵的 LLM 审计调用，直接复用智能体原有的检索行为作为证据，极具工程落地价值。

## 8. 不足与局限
*   **冷启动限制**：在内存运行初期（条目极少时），由于缺乏足够的全局状态参考，校准信号的准确性可能会受限。
*   **共性偏见风险**：如果所有参与辩论的智能体（由于预训练数据相似）对某个事实持有相同的错误偏见，EquiMem 可能无法通过“对齐信号”识别出该错误。
*   **图语义依赖**：在图内存中，校准质量高度依赖于关系提取的准确性，若提取的关系标签存在歧义，可能导致误判。
