## 摘要
具身智能体（Embodied AI agents）日益依赖大语言模型（LLMs）进行规划，但每步调用 LLM 会带来严重的延迟和成本。在本文中，我们展示了具身任务表现出极强的规划局部性（plan locality），即下一个规划在很大程度上可以根据当前规划进行预测。基于此，我们引入了 AgenticCache，这是一个通过重用缓存规划来避免每步调用 LLM 的规划框架。在 AgenticCache 中，每个智能体查询频繁规划转换的运行时缓存，而后台的缓存更新器（Cache Updater）则异步调用 LLM 以验证和优化缓存条目。在四个多智能体具身基准测试中，AgenticCache 在 12 种配置（4 个基准测试 x 3 个模型）下平均将任务成功率提高了 22%，模拟延迟降低了 65%，Token 使用量减少了 50%。因此，基于缓存的规划重用为实现低延迟、低成本的具身智能体提供了一条切实可行的路径。代码可在 https://github.com/hojoonleokim/MLSys26_AgenticCache 获取。

## Abstract
Embodied AI agents increasingly rely on large language models (LLMs) for planning, yet per-step LLM calls impose severe latency and cost. In this paper, we show that embodied tasks exhibit strong plan locality, where the next plan is largely predictable from the current one. Building on this, we introduce AgenticCache, a planning framework that reuses cached plans to avoid per-step LLM calls. In AgenticCache, each agent queries a runtime cache of frequent plan transitions, while a background Cache Updater asynchronously calls the LLM to validate and refine cached entries. Across four multi-agent embodied benchmarks, AgenticCache improves task success rate by 22% on average across 12 configurations (4 benchmarks x 3 models), reduces simulation latency by 65%, and lowers token usage by 50%. Cache-based plan reuse thus offers a practical path to low-latency, low-cost embodied agents. Code is available at https://github.com/hojoonleokim/MLSys26_AgenticCache.

### 1. 核心问题与整体含义（研究动机和背景）
*   **研究背景**：具身智能体（Embodied AI）目前越来越多地采用大语言模型（LLM）作为规划核心。传统的“感知-规划-行动”循环通常是同步的，即智能体必须等待 LLM 生成下一步规划后才能行动。
*   **核心痛点**：
    1.  **高延迟**：LLM 推理速度慢，导致智能体在执行过程中频繁停顿，无法满足实时性需求。
    2.  **高成本**：每一小步动作都调用 LLM API，导致 Token 消耗巨大。
*   **核心发现**：具身任务具有显著的**“规划局部性”（Plan Locality）**。例如，一旦智能体决定“去抓取苹果”，接下来的动作序列（抓取、转身、运输、放置）在很大程度上是可预测的重复模式。
*   **研究目的**：利用这种局部性，通过缓存机制减少对 LLM 的频繁调用，实现低延迟、低成本的具身规划。

### 2. 方法论：AgenticCache
*   **核心思想**：将规划过程类比为计算机体系结构中的“分支预测”。通过维护一个存储“规划转换模式”的运行时缓存，让智能体优先从缓存中获取下一步动作，而 LLM 则在后台异步地验证和修正这些预测。
*   **关键技术细节**：
    *   **2-Gram 缓存结构**：缓存条目表示为 `⟨Pi → Pj⟩`（当前规划到下一规划的转换），并附带环境元数据（如步数、持有物品数、完成进度等）作为过滤条件。
    *   **评分机制**：采用公式 $S(P_i \to P_j) = C(P_i \to P_j) \cdot I(P_j)$。其中 $C$ 是局部转换计数，$I$ 是全局重要性因子（由 LLM 确认的频率决定）。
    *   **异步更新器 (Cache Updater)**：在后台运行，不阻塞智能体行动。它定期调用 LLM，并将 LLM 的建议与缓存执行的轨迹进行对比。
    *   **确认与修正 (Confirmation & Correction)**：若 LLM 建议与缓存一致，则强化该缓存条目；若不一致，则立即替换（Replace）当前正在执行的错误动作，并更新缓存。
    *   **预填充 (Warm-Start)**：可选步骤，通过极少量（1-4 个）离线成功轨迹初始化缓存，解决冷启动问题。

### 6. 主要结论与发现

*   **可靠性**：通过异步修正机制，AgenticCache 能够有效处理动态环境变化，避免了纯模式匹配导致的僵化行为。

*   **系统级创新**：将传统的缓存思想巧妙地应用于具身智能体规划，解决了 LLM 推理速度与物理环境实时性不匹配的根本矛盾。
*   **极高实用性**：显著降低了使用昂贵 LLM API 的成本，且对硬件资源要求极低。
*   **即插即用**：作为一种规划框架，它可以兼容现有的各种 LLM 驱动的智能体系统。

### 8. 不足与局限
*   **多智能体协调风险**：在需要极度精确同步的时刻（如两个智能体同时抢夺一个资源），异步修正的延迟可能导致短时间的冲突或死锁。
*   **规划局部性的依赖**：在完全开放、无序或高度随机的环境中，如果动作序列没有明显的重复模式，缓存命中率会下降，性能增益将受限。
*   **修正滞后**：由于 LLM 调用是异步的，当环境发生突变时，智能体可能会在错误路径上执行几步后才收到修正信号。
