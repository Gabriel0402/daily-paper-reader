## 摘要
多智能体辩论（MAD）系统日益依赖共享内存来支持长程推理，但这种便利性也带来了一个关键漏洞：单个损坏的条目就可能污染下游的内存增强推理，而仅靠辩论本身无法过滤此类错误。现有的保护措施通过启发式方法或基于大语言模型（LLM）的验证来过滤条目，但它们依赖于具有相同失败模式的 AI 判断，且忽视了 MAD 中的跨智能体动态。我们通过将 MAD 中的内存更新建模为一个零信任内存博弈来填补这一空白，在该博弈中不假设任何智能体是诚实的，博弈的均衡状态作为最优内存信任的指标。在这一均衡的指导下，我们提出了 EquiMem，这是一种推理时校准机制，它利用智能体现有的检索查询和遍历路径作为证据，而非征求任何 LLM 判断，从而根据共享内存状态对每次更新进行算法量化。EquiMem 实现了针对嵌入式和基于图的内存的校准，在不同的基准测试、MAD 框架和内存架构中，它始终优于现有的保护措施，在对抗性智能体环境下保持稳健，且推理开销微乎其微。

## Abstract
Multi-agent debate (MAD) systems increasingly rely on shared memory to support long-horizon reasoning, but this convenience opens a critical vulnerability: a single corrupted entry can contaminate the downstream memory-augmented reasoning, and debate alone fails to filter such errors. Existing safeguards filter entries via heuristics or LLM-based validation, yet they rely on AI judgments that share the same failure modes and overlook the cross-agent dynamics of MAD. We address this gap by formulating memory updating in MAD as a zero-trust memory game, in which no agent is assumed honest and the game's equilibrium serves as an indicator of optimal memory trust. Guided by this equilibrium, we propose EquiMem, an inference-time calibration mechanism that quantifies each update algorithmically against the shared memory state, using agents' existing retrieval queries and traversal paths as evidence rather than soliciting any LLM judgment. EquiMem instantiates calibration for both embedding- and graph-based memory, and across diverse benchmarks, MAD frameworks, and memory architectures, it consistently outperforms existing safeguards, remains robust under adversarial agents, and incurs negligible inference overhead.

*   **核心问题**：在多智能体辩论（MAD）系统中，共享内存虽然增强了长程推理能力，但也引入了“内存污染”风险。单个智能体的幻觉、谄媚（Sycophancy）或错误断言一旦写入共享内存，会污染后续所有推理。
*   **研究动机**：现有的内存保护机制（如基于LLM的审计或启发式过滤）存在两个缺陷：一是审计者本身也是LLM，具有相同的失效模式；二是忽视了多智能体之间复杂的互动博弈（如合谋或过度自信导致的错误共识）。

### 2. 方法论：零信任内存博弈与 EquiMem
*   **核心思想**：将内存更新建模为一个**零信任内存博弈（Zero-Trust Memory Game）**。不假设任何智能体是诚实的，通过博弈论中的**纳什均衡（Equilibrium）**状态来衡量内存更新的可信度。
*   **关键技术细节**：
    *   **角色定义**：提出更新的智能体为“贡献者”，其余为“审计者”。
    *   **效用函数**：贡献者追求影响力的最大化同时降低被检测风险；审计者在最小化努力成本的同时追求内存的一致性。
    *   **EquiMem 机制**：一种无需LLM调用的算法校准机制。它拦截智能体在辩论中产生的原生行为（如检索查询、图遍历路径）作为证据。
    *   **双重校准指标**：
        1.  **检测信号 ($\rho_{detect}$)**：检查更新是否破坏了内存的几何分布（嵌入空间）或拓扑结构（知识图谱）。
        2.  **对齐信号 ($\rho_{align}$)**：通过 Rank-Biased Overlap (RBO) 算法测量新条目对其他智能体检索结果的扰动程度。
    *   **信誉权重 ($w_i$)**：根据智能体提供证据的结构有效性，动态调整其在校准中的权重，实现对恶意智能体的自动“禁言”。

### 6. 主要结论与发现
*   **性能领先**：EquiMem 在所有配置下均排名第一，显著优于现有的基于 LLM 的防御手段。
*   **零信任有效性**：即使在半数智能体反水的情况下，系统性能依然接近正常水平，证明了结构化校准优于信任个体判断。

*   **架构普适性**：无论是向量数据库还是知识图谱内存，EquiMem 都能通过相应的几何/拓扑检查实现有效校准。

### 7. 优点与亮点
*   **理论扎实**：将复杂的社会心理学现象（辩论崩溃）转化为严谨的博弈论数学模型。
*   **设计巧妙**：不要求智能体额外打分，而是“废物利用”，将智能体为了完成任务而必须进行的检索行为转化为校准证据。
*   **防御鲁棒**：引入动态信誉衰减机制，能够有效识别并抑制恶意智能体的合谋。

### 8. 不足与局限
*   **冷启动问题**：在内存极度稀疏的初始阶段（前 1-2 轮），由于缺乏对比基准，校准信号可能不够准确。
*   **关联幻觉风险**：如果所有参与辩论的智能体都基于相同的预训练偏差产生完全一致的幻觉，EquiMem 的对齐检查可能会失效（因为大家都“错得一致”）。
*   **语义理解局限**：在图内存中，如果智能体提取的关系标签存在同义词歧义（如 inside vs located_in），可能会导致误判。
