## 摘要
我们介绍了 Cattle Trade，这是一个多智能体基准，用于评估大语言模型（LLM）作为智能体在不完全信息、对抗性交互和资源约束下的策略推理能力。该基准在持续 50-60 轮的长周期博弈中，融合了拍卖、隐藏报价交易挑战（TCs）、讨价还价、虚张声势、对手建模和资源分配。与以往孤立测试这些能力的智能体基准不同，Cattle Trade 评估智能体是否能在具有利益冲突的竞争性多智能体经济博弈中整合这些能力。该基准记录了每一次出价、TC 报价、反向报价和卡片选择，从而实现了超越最终得分或胜率的行为分析。我们在 242 场博弈中评估了七个高性价比的语言模型和三个确定性代码智能体。策略连贯性，特别是支出效率、资源纪律和阶段自适应竞价，与排名的相关性比支出总量或任何单一子技能更强。两个启发式代码智能体的表现优于大多数受测 LLM，而行为轨迹揭示了 LLM 反复出现的失败模式，包括过度竞价、自我竞价、破产性 TC 发起以及对对手状态的适应性较弱。评估智能体能力需要能够在具有利益冲突、不确定性和经济动态的多智能体环境中测试多种能力协同部署的基准。

## Abstract
We introduce \textsc{Cattle Trade, a multi-agent benchmark for evaluating large language models (LLMs) as agents in strategic reasoning under imperfect information, adversarial interaction, and resource constraints. The benchmark combines auctions, hidden-offer trade challenges (TCs), bargaining, bluffing, opponent modeling, and resource allocation within a single long-horizon game lasting 50--60 turns. Unlike prior agent benchmarks that test these abilities in isolation, \textsc{Cattle Trade} evaluates whether agents integrate them across a competitive, multi-agent economic game with conflicting incentives. The benchmark logs every bid, TC offer, counteroffer, and card selection, enabling behavioural analysis beyond final scores or win rates. We evaluate seven cost-efficient language models and three deterministic code agents across 242 games. Strategic coherence, in particular spending efficiency, resource discipline, and phase-adaptive bidding, is associated with rank more strongly than spending volume or any single subskill. Two heuristic code agents outperform most tested LLMs, and behavioural traces surface recurring LLM failure modes including overbidding, self-bidding, bankrupt TC initiation, and weak opponent-state adaptation. Evaluating agentic competence requires benchmarks that test the joint deployment of multiple capabilities in multi-agent environments with conflicting incentives, uncertainty, and economic dynamics.

### 论文总结：Cattle Trade - LLM 策略推理的多智能体基准测试

#### 1. 论文的核心问题与整体含义
论文的研究动机源于现有大语言模型（LLM）基准测试的局限性。虽然 MMLU 或 HumanEval 等基准可以衡量知识和代码能力，但它们无法评估智能体在复杂、对抗性且具有不完全信息的环境下的**策略推理能力**。现有的游戏类基准（如《阿瓦隆》或《狼人杀》）通常孤立地测试欺骗或协作，而缺乏一个能够要求智能体在长周期（50-60 轮）内整合**拍卖、讨价还价、虚张声势和资源管理**的综合性经济博弈环境。Cattle Trade 旨在填补这一空白，测试模型是否能展现出“策略连贯性”。

#### 2. 论文提出的方法论
该基准基于经典桌游 *Kuhhandel*（中译《大金刚》或《你是在吹牛吧！》）改编，核心思想是构建一个多维度的经济博弈模型：
*   **核心机制**：
    *   **拍卖（Auctions）**：采用“优先购买权（Buy-right）”机制，拍卖者可以支付最高出价来自己保留卡片，这要求智能体平衡流动性与资产获取。
    *   **隐藏报价贸易挑战（TCs）**：双方提交面朝下的货币卡。由于存在面值为 0 的货币卡，智能体可以进行虚张声势（Bluffing）。对手必须在未见报价的情况下决定接受或反击。
    *   **离散货币约束**：支付时不设找零（例如用 100 元付 60 元的账，40 元会作废），这强加了严苛的资源管理压力。
    *   **乘法评分规则**：最终得分 = (完成套组的总价值) × (完成套组的数量)。这种规则鼓励资产的多样性而非单一集中。
*   **技术实现**：使用 Python 开发博弈引擎，通过结构化 JSON 与 LLM 交互，并利用 **TrueSkill**（一种贝叶斯评分系统）对智能体进行排名。

#### 3. 实验设计
*   **实验场景**：
    *   **纯 LLM 锦标赛**：不同 LLM 之间进行对战，以观察头对头的动态博弈。
    *   **混合博弈**：单个 LLM 对阵三个确定性的代码智能体（基准线）。
*   **对比方法（智能体）**：
    *   **LLM 智能体**：包括 Gemini 3 Flash (G3-F)、Gemini 3.1 Flash Lite、Claude Sonnet 4.5、DeepSeek v3.2、GPT-5.4 Nano、Claude Haiku 4.5、Gemini 2.5 Flash Lite。
    *   **代码智能体（Baselines）**：
        1.  **TrackerAgent**：完美记录所有公开信息，基于对手预算进行贪婪决策。
        2.  **SetRaceAgent**：激进地追求套组完成，不计成本。
        3.  **EconomyAgent**：保守的资源管理者，专注于现金流。

#### 4. 资源与算力
论文**未明确说明**具体的硬件算力（如 GPU 型号或数量）。由于该研究侧重于对现成 API 模型的评估而非模型训练，其主要资源消耗体现在 **Token 消耗**上。文中提到不同模型的 Token 使用量差异巨大（最高达 20 倍），例如 Gemini 3 Flash 每次调用产生约 1500 个 Token（包含长链推理），而 Gemini 3.1 Flash Lite 仅约 80 个。

#### 5. 实验数量与充分性
*   **实验规模**：共进行了 **242 场完整博弈**（每场 50-60 轮），其中 60 场为纯 LLM 锦标赛，168 场为混合配置博弈，14 场为针对 Sonnet 4.5 的探索性博弈。
*   **充分性评价**：实验设计较为充分。通过 TrueSkill 评分系统提供了统计学上的技能估计（μ 和 σ），并分析了 98 场标准模式下的详细行为轨迹。不过，Sonnet 4.5 的样本量（n=14）相对较小，导致其置信区间较宽。整体而言，实验覆盖了多种对手组合，具有较好的客观性和公平性。

#### 6. 论文的主要结论与发现
*   **模型排名**：Gemini 3 Flash 表现最强，显著优于所有代码智能体。然而，**TrackerAgent 和 SetRaceAgent 击败了除 Gemini 3 系列之外的所有 LLM**。
*   **LLM 失效模式**：
    *   **过度竞价（Overbidding）**：弱模型经常出价超过自身总资产，导致罚款。
    *   **自我竞价螺旋**：在没有竞争对手加价的情况下，模型仍不断提高自己的出价。
    *   **破产性发起**：在没有现金的情况下发起贸易挑战，导致白送资产。
*   **成功因素**：成功的关键不在于出价的绝对频率，而在于**支出效率**（每单位货币换取的得分）和**阶段自适应性**（前期积累资金，后期激进抢购）。

#### 7. 优点
*   **综合性强**：将多种博弈论机制（拍卖、博弈、资源管理）整合在一个长程任务中，比单一任务基准更接近真实世界的经济决策。
*   **诊断性深度**：不仅提供胜率排名，还通过结构化日志分析“为什么失败”（如支出效率、虚张声势校准度等指标）。
*   **代码智能体基准**：引入了基于规则的强力基准，揭示了当前大多数 LLM 在基础逻辑推理和状态跟踪上的不足。

#### 8. 不足与局限
*   **模型覆盖范围**：主要测试了“成本效益型”模型（Flash/Nano 等），缺乏对顶级全量推理模型（如 GPT-4o 原型或 o1 系列）的全面评估。
*   **提示词敏感性**：所有模型使用相同的提示词，未针对不同模型的指令遵循偏好进行优化，可能存在提示词偏差。
*   **缺乏人类对比**：目前仅有 LLM 与代码智能体的对比，缺乏与人类玩家的基准对比，难以判断 LLM 的策略是否达到了人类专家水平。
*   **静态策略**：代码智能体虽然强大但策略固定，无法模拟人类在博弈中可能出现的心理变化或社会性协作。
