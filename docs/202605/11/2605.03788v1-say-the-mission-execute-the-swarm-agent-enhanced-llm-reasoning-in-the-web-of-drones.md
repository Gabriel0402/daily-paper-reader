## 摘要
大语言模型（LLMs）正越来越多地被探索作为信息物理系统的高级推理引擎，但由于异构接口、有限的具身关联（grounding）以及对长期闭环执行的需求，其在实时无人机（UAV）集群管理中的应用仍面临挑战。本文提出了一种任务无关的、智能体增强的 LLM 框架，用于无人机集群控制。用户通过自然语言表达任务目标，系统通过具身的实时交互自主执行任务。所提出的架构将基于 LLM 的智能体核心（Agent Core）与模型上下文协议（MCP）网关以及基于 W3C 物联网（WoT）标准的无人机网（Web-of-Drones）抽象相结合。通过将无人机、传感器和服务公开为标准化的 WoT 物（Things），该框架实现了结构化的基于工具的交互、持续的状态观察和安全的动作执行，而无需依赖代码生成。我们使用基于 ArduPilot 的仿真，在四个集群任务和六个最先进的 LLM 上对该框架进行了评估。结果表明，尽管具有强大的推理能力，但在没有明确的具身关联和执行支持的情况下，当前的通用 LLM 即使在处理简单的集群任务时也难以实现可靠执行。特定任务的规划工具和运行时护栏显著提高了鲁棒性，而 Token 消耗量本身并不能指示执行质量或可靠性。

## Abstract
Large Language Models (LLMs) are increasingly explored as high-level reasoning engines for cyber-physical systems, yet their application to real-time UAV swarm management remains challenging due to heterogeneous interfaces, limited grounding, and the need for long-running closed-loop execution. This paper presents a mission-agnostic, agent-enhanced LLM framework for UAV swarm control, where users express mission objectives in natural language and the system autonomously executes them through grounded, real-time interactions. The proposed architecture combines an LLM-based Agent Core with a Model Context Protocol (MCP) gateway and a Web-of-Drones abstraction based on W3C Web of Things (WoT) standards. By exposing drones, sensors, and services as standardized WoT Things, the framework enables structured tool-based interaction, continuous state observation, and safe actuation without relying on code generation. We evaluate the framework using ArduPilot-based simulation across four swarm missions and six state-of-the-art LLMs. Results show that, despite strong reasoning abilities, current general-purpose LLMs still struggle to achieve reliable execution - even for simple swarm tasks - when operating without explicit grounding and execution support. Task-specific planning tools and runtime guardrails substantially improve robustness, while token consumption alone is not indicative of execution quality or reliability.

*   **核心问题**：如何构建一个通用的架构，让用户仅通过自然语言描述任务，就能让 LLM 自主、实时、安全地指挥异构无人机集群完成任务？

### 2. 方法论：核心思想与关键技术
该论文提出了一种**任务无关的智能体增强架构**，核心思想是“不生成代码，而是直接调用工具”。
*   **Web-of-Drones (WoT) 抽象**：基于 W3C 物联网标准，将每架无人机、传感器和服务抽象为“物（Thing）”。通过“物描述（TD）”定义标准化的属性（位置、电量）、动作（起飞、前往）和事件。
*   **MCP  gateway（模型上下文协议网关）**：作为 LLM 与物理世界之间的唯一桥梁，提供资源发现和交互接口，确保 LLM 只能通过标准化的工具进行操作。
*   **Agent Core（智能体核心）**：
    *   **推理-执行循环**：LLM 不一次性生成所有指令，而是观察状态、推理下一步、调用工具、获取反馈，循环往复。
    *   **运行时护栏（Guardrails）**：动态注入提示词，当检测到执行停滞、未安全着陆或任务未完成时，强制引导 LLM 修正行为。
    *   **辅助工具**：提供如“等待起飞完成”等同步工具，减轻 LLM 的逻辑负担。

### 3. 实验设计
*   **实验场景**：在 ArduPilot SITL 仿真环境中模拟了 10 架多旋翼无人机。
*   **任务类型**：
    1.  **区域覆盖（有/无规划工具）**：测试 LLM 的空间规划能力。
    2.  **编队控制（星形阵）**：测试多机协同与避障能力（人为设置了远距离交叉路径以增加碰撞风险）。
    3.  **智能灌溉**：测试长链路任务，包括传感器数据采集、逻辑判断（温湿度阈值）和动作触发。
*   **对比方法（Benchmark）**：对比了 6 种主流 LLM：GPT-5.2、DeepSeek V3.2、GLM 4.7、Grok 4.1 Fast、Claude Haiku 4.5 和 Qwen3 8B。

### 5. 实验数量与充分性
*   **实验规模**：每种模型在每个任务上重复进行 **10 次独立实验**，以应对 LLM 的非确定性。

### 6. 主要结论与发现
*   **具身关联是关键**：通用 LLM 在没有显式工具支持时，即使是简单的覆盖任务也难以完成。
*   **规划工具的必要性**：特定任务工具（如路径规划器）能显著提升成功率，但不同模型对工具的依赖程度不同（如 GPT 在无工具时表现反而更好，而 Qwen 必须依赖工具）。
*   **Token 消耗不等于质量**：Token 消耗量与任务成功率之间没有正相关关系。高消耗可能意味着模型在“胡言乱语”或陷入逻辑死循环。
*   **模型差异**：GLM 4.7 在综合任务中表现最稳健；GPT 具备极强的推理能力但有时会忽略安全约束（如未着陆就结束任务）；小模型（如 Qwen 8B）在长链路任务中表现较差。

### 7. 优点与亮点
*   **标准化架构**：引入 W3C WoT 和 MCP 协议，解决了机器人领域长期存在的接口碎片化问题，具有很强的通用性。
*   **闭环控制**：摒弃了不可控的“代码生成”模式，采用“推理-执行-监控”的闭环模式，更符合工业控制逻辑。
*   **动态护栏**：通过 Prompt 工程实现的运行时安全检查，在不硬编码逻辑的前提下提升了安全性。

### 8. 不足与局限
*   **仿真与现实的差距**：实验完全在仿真环境中进行，未考虑现实中的通信延迟、丢包及复杂的风力干扰。
*   **实时性限制**：依赖云端 LLM API 的推理速度可能无法满足极高动态下的避障需求（目前避障主要靠预设逻辑或低频调整）。
*   **成本问题**：长时任务（如灌溉）消耗了大量 Token，对于大规模集群的商业化应用，成本控制仍是挑战。
*   **偏差风险**：实验中发现某些模型存在决策偏差（如 GPT 倾向于过度灌溉），这可能源于预训练数据的偏见。
