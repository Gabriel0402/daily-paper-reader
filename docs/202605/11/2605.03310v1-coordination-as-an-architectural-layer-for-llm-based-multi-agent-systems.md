## 摘要
基于大语言模型（LLM）的多智能体系统在生产环境中的失败率在 41% 到 87% 之间，这主要归因于协同缺陷而非基础模型的能力。现有的应对方案要么是经验性地对失败模式进行分类，要么是提供声明式编排框架作为工程工具；两者都未能提供从协同配置到可预测失败模式特征的原则性映射。我们认为，协同应该被视为一个可配置的架构层，与智能体逻辑和信息访问相分离，从而实现架构推理，而不仅仅是提高工程生产力。我们通过预测市场上的信息控制设计来实例化这一理念：在五种参考协同配置中，使用单一 LLM、固定工具、固定单次调用输出上限和固定提示词模板，并将每个问题的总计算量视为内生架构输出。Brier 分数的 Murphy 分解将校准度与判别力分开，因此即使在总分一致的情况下，不同配置也会留下可区分的特征。在模型训练截止日期（claude-opus-4-6）之后结算的 100 个 Polymarket 二元市场中，我们报告了 Murphy 特征、成本-质量帕累托前沿、类别条件分析以及自助法效能预测。五个预设预测中有三个在方向上得到了验证；在此机制下，有两种配置在帕累托前沿中占据主导地位；探索性自助法区间将共识对齐与其他配置区分开来，尽管在 n=100 时成对检验未能通过 Bonferroni 校正。我们还在启用网络搜索的条件下，将相同的配置作为实时智能体部署在 Foresight Arena 上，作为一个并行积累的链上复制通道。测试框架、追踪数据集和生产智能体均已发布。我们将其定位为验证方法论的首次实例化，而非通用的跨模型主张。

## Abstract
Multi-agent LLM systems fail in production at rates between 41% and 87%, mostly due to coordination defects rather than base-model capability. Existing responses split between cataloguing failure modes empirically and shipping declarative orchestration frameworks as engineering tools; neither delivers a principled mapping from coordination configuration to predictable failure-mode signature. We argue that coordination should be treated as a configurable architectural layer, separable from agent logic and from information access, enabling architectural reasoning rather than only engineering productivity.   We instantiate this with an information-controlled design on prediction markets: a single LLM, fixed tools, fixed per-call output cap, and fixed prompt template across five reference coordination configurations, with total compute per question treated as an endogenous architectural output. The Murphy decomposition of the Brier score separates calibration from discriminative power, so configurations leave distinguishable signatures even when aggregate scores coincide.   On 100 Polymarket binary markets resolved after the model's training cutoff (claude-opus-4-6) we report Murphy signatures, a cost-quality Pareto frontier, category-conditioned analysis, and a bootstrap power-projection. Three of five pre-specified predictions are upheld in direction; two configurations dominate the Pareto frontier within this regime; exploratory bootstrap intervals separate consensus alignment from others, though pairwise tests do not survive Bonferroni correction at n=100. We also deploy the same configurations as live agents on Foresight Arena under web-search-enabled conditions, as an on-chain replication channel accumulating in parallel. Harness, trace dataset, and production agents are released. We position this as a methodology-validating first instantiation, not a general cross-model claim.

*   **研究动机**：现有的研究要么只是罗列失败模式，要么只是提供工程上的编排工具，缺乏一种将“协同配置”与“预测性失败特征”联系起来的原则性架构方法。
*   **整体含义**：论文提出将**协同（Coordination）**视为一个独立的架构层，与智能体逻辑和信息获取分离。通过这种分离，开发者可以像分析软件架构一样，预测特定协同模式（如辩论、流水线等）会导致什么样的系统性偏差或错误。

### 2. 方法论：核心思想与关键技术
*   **三层架构分解**：
    1.  **信息层**：固定的工具、数据源和上下文。
    2.  **协同层**：定义智能体拓扑、权力分配、同步机制、聚合规则和终止条件。
    3.  **智能体层**：具体的 LLM 调用和角色提示词。
*   **信息控制实验设计**：为了消除“信息增量”带来的干扰（即性能提升是因为协同好，还是因为多智能体读了更多资料？），论文严格固定了模型、工具、提示词模板和单次调用 token 上限，仅改变协同拓扑。
*   **Murphy 分解评估法**：不只看总分（Brier Score），而是将其分解为：
    *   **可靠性（Reliability, REL）**：预测概率与实际发生频率的吻合度（校准度）。
    *   **判别力（Resolution, RES）**：区分不同结果的能力。
    这使得研究者能识别出某些架构虽然总分高，但可能存在“过度自信”或“盲目从众”的特征。

### 3. 实验设计：场景、Benchmark 与对比方法
*   **实验场景**：**预测市场（Polymarket）**。选择在模型训练截止日期（2025年8月）之后结算的 100 个真实二元预测市场，确保模型无法通过记忆获知答案。
*   **对比方法（五种协同配置）**：
    1.  **独立并行（Independent Ensemble）**：智能体互不干扰，结果取平均。
    2.  **同行评议辩论（Peer-critique Debate）**：多轮互相观察与修正。
    3.  **编排者-专家（Orchestrator-specialist）**：由一个规划者分配任务并整合。
    4.  **顺序流水线（Sequential Pipeline）**：研究、分析、预测按序进行。
    5.  **共识对齐（Consensus Alignment）**：迭代直到智能体达成一致。
*   **Benchmark**：以 Polymarket 的市场共识价格作为基准。

### 4. 资源与算力
*   **模型**：统一使用 `claude-opus-4-6` (Anthropic API)。
*   **算力消耗**：论文未提及 GPU 训练，因为是基于 API 的推理实验。
*   **实验成本**：总计消耗约 1710 万个 token，API 费用为 109.65 美元。
*   **时长**：100 个市场的 5 种配置运行总耗时 3 小时 9 分钟。

### 5. 实验数量与充分性
*   **实验规模**：共 100 个市场，涵盖加密货币、政治、体育、经济、地缘政治、娱乐六大类。
*   **充分性分析**：
    *   **样本量**：n=100 对于识别显著的架构差异是足够的，但对于区分细微的性能排名（如两个表现接近的架构谁更好）尚显不足。作者通过自助法（Bootstrap）效能预测指出，要完全区分所有配置，样本量需达到 500-2000。
    *   **客观性**：实验设计非常严谨，通过“信息控制”确保了对比的公平性，避免了多智能体研究中常见的“计算量换性能”的偏见。

### 6. 主要结论与发现
*   **架构决定特征**：不同的协同配置确实留下了独特的 Murphy 特征。例如，“共识对齐”配置表现出最低的判别力（RES），容易陷入平庸的中间值。
*   **帕累托前沿**：在成本与质量的权衡中，“独立并行”和“顺序流水线”构成了最优边界。而“编排者-专家”和“同行辩论”在当前设置下被证明是“被支配的”（即成本更高且效果更差）。
*   **预测验证**：预设的 5 个架构预测中有 3 个得到了验证。
*   **负 Alpha 现象**：在禁用网络搜索的情况下，所有配置都未能跑赢市场共识，证明了信息获取对预测任务的决定性作用。

### 7. 优点：亮点与创新
*   **方法论严谨**：首次在多智能体评估中严格区分了“架构效应”和“信息效应”。
*   **评估维度深**：引入 Murphy 分解，超越了单一的准确率指标，能深入分析智能体的心理特征（如校准度）。
*   **开源贡献**：发布了完整的实验框架、包含 1700 万 token 的推理追踪数据集，以及在链上实时运行的生产级智能体。

### 8. 不足与局限
*   **样本量限制**：n=100 导致部分成对比较在统计学上不够显著（未通过 Bonferroni 校正）。
*   **单一模型与任务**：实验仅基于 Claude 系列模型和预测市场任务，其结论（如哪些架构更好）可能无法直接推广到代码生成或创意写作等领域。
*   **提示词敏感性**：虽然固定了模板，但未对角色提示词的微小差异进行消融实验，可能存在提示词敏感性带来的偏差。
*   **信息受限**：为了控制变量禁用了网络搜索，这限制了系统发挥多智能体协同搜索的潜力。
