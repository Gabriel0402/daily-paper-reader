## 摘要
大语言模型（LLMs）越来越多地以团队形式部署，但现有的协作方法往往处于两个极端。高度结构化的方法依赖于预先分配的固定角色、流水线或任务分解。相比之下，完全非结构化的团队虽然具备适应性和探索能力，但存在效率低下的问题，如错误传播、智能体间冲突以及资源浪费（以时间、Token 或文件操作衡量）。我们提出了用于任务演化的语言智能体团队（LATTE），这是一个受分布式系统启发的 LLM 团队协作框架，其中处理器必须在部分可观测性和通信限制下运行。在 LATTE 中，智能体团队协作构建并维护一个共享的、不断演化的协作图，该图编码了子任务依赖关系、个体智能体分配以及子任务进度的当前状态。该协议在保持一致性的同时，赋予智能体动态分配工作、调整协作并发现新任务的能力。在多项协作任务和多种基础模型中，我们证明了 LATTE 如何在减少 Token 使用量、实际运行时间、通信和协作失败（如文件冲突和冗余输出）的同时，达到或超过包括 MetaGPT、去中心化团队、自顶向下的领导者-执行者层级结构以及静态分解在内的标准设计的准确率。

## Abstract
Large language models (LLMs) are increasingly deployed in teams, yet existing coordination approaches often occupy two extremes. Highly structured methods rely on fixed roles, pipelines, or task decompositions assigned a priori. In contrast, fully unstructured teams enable adaptability and exploration but suffer from inefficiencies such as error propagation, inter-agent conflicts, and wasted resources (measured in time, tokens, or file operations). We introduce Language Agent Teams for Task Evolution (LATTE), a framework for coordinating LLM teams inspired by distributed systems, where processors must operate under partial observability and communication constraints. In LATTE, a team of agents collaboratively construct and maintain a shared, evolving coordination graph which encodes sub-task dependencies, individual agent assignment, and the current state of sub-task progress. This protocol maintains consistency while empowering agents to dynamically allocate work, adapt coordination, and discover new tasks. Across multiple collaborative tasks and a variety of base models, we demonstrate how LATTE reduces token usage, wall-clock time, communication, and coordination failures (e.g. file conflicts and redundant outputs) while matching or exceeding the accuracy of standard designs including MetaGPT, decentralized teams, top-down Leader-Worker hierarchies, and static decompositions.

#### 1. 核心问题与整体含义
论文针对大语言模型（LLM）多智能体团队协作中的**“结构化悖论”**提出解决方案。现有的协作模式通常处于两个极端：
*   **高度结构化（如 MetaGPT）：** 依赖预设的固定角色和流水线，虽有一致性但缺乏灵活性，难以应对执行过程中新出现的依赖关系或错误。
*   **完全非结构化（去中心化）：** 智能体自主性强但容易产生冲突（如重复修改文件）、通信冗余和错误传播，导致资源（Token 和时间）严重浪费。

#### 2. 方法论：LATTE 框架
LATTE（Language Agent Teams for Task Evolution）的核心思想是**动态协作任务图**。

*   **核心组件：**
    *   **动态协作图 (Gt)：** 一个有向无环图（DAG），节点代表子任务，边代表依赖关系。每个节点包含状态（待处理、进行中、已完成等）和分配的智能体。
    *   **角色分工：** 
        *   **领导者 (Lead)：** 负责全局一致性，拥有修改图结构的特权操作。
        *   **执行者 (Workers)：** 负责具体任务执行，并根据局部观察向 Lead 提议新任务。
*   **关键技术细节：**
    *   **图变动操作符：** 定义了 7 种显式操作（DISCOVER, ASSIGN, CLAIM, COMPLETE, RELEASE, CLOSE, VERIFY），每种都有明确的前置和后置条件，确保图的 DAG 特性。
    *   **混合协作模型：** 采用“提议-验证”机制。Worker 发现新子任务时发起 `DISCOVER` 提议，Lead 进行全局评估后合并。这在数学上被建模为一种**马尔可夫链蒙特卡罗（MCMC）采样过程**的近似，旨在寻找最优的任务分解。
    *   **自调度与心跳监测：** 允许空闲 Worker 主动 `CLAIM`（认领）任务以减少 Lead 负担；通过心跳机制检测“掉队者”（Stragglers），Lead 可执行 `RELEASE` 重新分配任务。
    *   **上下文范围限制：** Worker 仅接收与其子任务相关的上下文，避免长文本导致的推理疲劳。

#### 7. 优点
*   **动态适应性：** 能够根据执行过程中的反馈（如测试失败）实时发现并插入新任务。
*   **可解释性与审计性：** 任务图将智能体原本“黑盒”的决策过程显性化，便于人类开发者监控和调试。
*   **资源利用率高：** 仅在有任务处于“前沿（Frontier）”状态时才激活智能体，避免了无效的轮询和通信。

*   **规划开销：** 对于非常简单或短小的任务，构建和维护任务图的初始开销可能超过其带来的收益。
*   **任务边界依赖：** 目前主要适用于有清晰子任务边界的领域（如编程、数据分析），在极度模糊或非结构化的创意任务中表现尚待验证。
*   **团队规模固定：** 实验中固定了 5 个智能体的规模，尚未探讨在超大规模（如数百个智能体）团队中的图维护压力和通信瓶颈。
