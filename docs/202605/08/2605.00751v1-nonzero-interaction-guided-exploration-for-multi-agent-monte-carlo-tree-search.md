## 摘要
蒙特卡洛树搜索（MCTS）在协作式多智能体领域的可扩展性较差，因为节点扩展必须考虑呈指数级增长的联合动作集，这在现实的搜索预算下严重限制了探索。我们提出了 NonZero，它通过在低维非线性表示上运行代理引导的选择，并使用交互引导的提议规则，使多智能体 MCTS 保持可处理性，而不是直接探索完整的联合动作空间。我们的探索使用了一种交互评分：单智能体偏差按预测增益排序，而双智能体偏差则通过一种混合差分度量进行评分，该度量即使在单个智能体无法独立改进的情况下也能揭示协作收益。我们将候选提议形式化为局部偏差上的多臂老虎机问题，并推导出一种提议规则 NonZero，该规则具有次线性的局部悔恨度（local-regret）保证，可以在不枚举联合动作空间的情况下达到近似图局部最优。实验表明，在相同的搜索预算下，NonZero 在 MatGame、SMAC 和 SMACv2 上的样本效率和最终性能相较于强大的基于模型和无模型的基准方法均有所提升。

## Abstract
Monte Carlo Tree Search (MCTS) scales poorly in cooperative multi-agent domains because expansion must consider an exponentially large set of joint actions, severely limiting exploration under realistic search budgets. We propose NonZero, which keeps multi-agent MCTS tractable by running surrogate-guided selection over a low-dimensional nonlinear representation using an interaction-guided proposal rule, instead of directly exploring the full joint-action space. Our exploration uses an interaction score: single-agent deviations are ranked by predicted gain, while two-agent deviations are scored by a mixed-difference measure that reveals coordination benefits even when no single agent can improve alone. We formalize candidate proposal as a bandit problem over local deviations and derive a proposal rule, NonZero, with a sublinear local-regret guarantee for reaching approximate graph-local optima without enumerating the joint-action space. Empirically, NonZero improves sample efficiency and final performance on MatGame, SMAC, and SMACv2 relative to strong model-based and model-free baselines under matched search budgets.

### 1. 核心问题与整体含义（研究动机和背景）
在协作式多智能体强化学习（MARL）中，蒙特卡洛树搜索（MCTS）面临严重的**组合爆炸问题**。对于拥有 $n$ 个智能体、每个智能体有 $d$ 个动作的系统，联合动作空间大小为 $d^n$。传统的 MCTS 节点扩展需要枚举所有联合动作，这在搜索预算有限的情况下导致探索效率极低。此外，多智能体任务中存在“协调陷阱”，即单个智能体的改变可能降低奖励，只有多个智能体同时改变动作才能获得更高收益。现有的方法要么假设奖励是线性可分的，要么依赖随机采样，难以有效捕捉这种非线性的协调效应。

### 2. 方法论：核心思想与关键技术
**NonZero** 的核心思想是：**不直接在 $d^n$ 空间搜索，而是在低维非线性表示上通过“交互引导”来提议候选动作。**

*   **低维非线性表示**：使用 `asinh-GLM`（反双曲正弦广义线性模型）作为奖励代理模型。它能捕捉非线性特征，且其数学性质（无限可微、梯度不消失）有利于优化。
*   **交互引导提议规则 (NonUCT)**：
    *   **一阶偏差（单智能体）**：计算改变单个智能体动作带来的预测增益。
    *   **二阶混合差分（双智能体交互）**：计算两个智能体同时改变动作带来的额外收益（协同效应）。这能识别出单人改进无效但双人协作有效的“协调陷阱”。
*   **算法流程**：
    1.  **选择与扩展**：在每个节点维护一个固定大小的候选集 $C(s)$，使用 NonUCT 规则根据一阶和二阶增益提议新动作加入候选集。
    2.  **超网络初始化**：使用 Hypernetwork 根据状态预测代理模型的初始参数 $\theta$，加速收敛。
    3.  **在线更新**：在回溯阶段，利用模拟得到的奖励通过梯度下降更新节点特定的参数 $\theta$，使代理模型更贴合局部奖励景观。
*   **理论保证**：论文证明了 NonUCT 具有**次线性局部悔恨度**（$O(T^{3/4})$），且其样本复杂度与动作空间维度无关（Action-dimension-free），能收敛至图局部最优。
