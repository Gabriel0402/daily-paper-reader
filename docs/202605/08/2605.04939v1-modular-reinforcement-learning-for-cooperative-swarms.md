## 摘要
协作机器人集群是由共享共同目标的计算受限机器人组成的集体。每个机器人只能与其同伴的一小部分子集进行交互，而不知道这种交互如何影响集体效用。分布式多智能体强化学习的最新进展表明，尽管每个机器人都是独立学习的，但机器人仍有可能学习如何以符合共同目标的方式与其他机器人进行有效交互。然而，这要求每个机器人表示潜在的组合数量的交互状态，这对机器人的内存能力提出了挑战。本文提出了一种在集群多机器人强化学习中表示空间交互状态的替代方法。该方法采用模块化（分解）表示，其中状态的每个特征由单独的学习过程处理，并对结果进行聚合。我们在模拟机器人集群执行觅食任务的大量实验中证明了该方法的有效性。

## Abstract
A cooperative robot swarm is a collective of computationally-limited robots that share a common goal. Each robot can only interact with a small subset of its peers, without knowing how this affects the collective utility. Recent advances in distributed multi-agent reinforcement learning have demonstrated that it is possible for robots to learn how to interact effectively with others, in a manner that is aligned with the common goal, despite each robot learning independently of others. However, this requires each robot to represent a potentially combinatorial number of interaction states, challenging the memory capabilities of the robots. This paper proposes an alternative approach for representing spatial interaction states for multi-robot reinforcement learning in swarms. A modular (decomposed) representation is used, where each feature of the state is handled by a separate learning procedure, and the results aggregated. We demonstrate the efficacy of the approach in numerous experiments with simulated robot swarms carrying out foraging.

### 1. 核心问题与研究动机
*   **核心问题**：在协作机器人集群中，多智能体强化学习（MARL）面临**“状态爆炸”**问题。为了实现有效的协作（如避障、觅食），机器人需要表征复杂的空间交互状态，但传统的全状态表征会导致状态数量随特征增加呈指数级增长。
*   **研究背景**：典型的集群机器人（如 e-puck2, Kilobots）计算资源极其匮乏（内存通常小于 512KB，主频仅百兆赫兹），无法运行深度神经网络或存储庞大的 Q 表。因此，如何在极低内存占用下实现有效的分布式强化学习是当前的主要瓶颈。

### 2. 核心方法论
论文提出了一种**模块化（Modular/Decomposed）状态表征**方法，核心思想是将全局空间状态分解为多个独立的局部特征：
*   **状态分解**：将机器人的 8 个距离传感器视为 8 个独立的特征。每个传感器方向分配一个独立的学习进程（Learner），该进程仅关注该方向的传感器读数（如：是否有障碍物的二值状态）。
*   **独立学习器**：每个模块使用简单的多臂老虎机算法（如 Continuous UCB-1）进行学习。对于 $k$ 个特征，状态空间从 $O(2^k)$ 降低到 $O(k)$。
*   **议会机制（The Council）**：这是一个动作融合过程。每个独立学习器根据其负责的特征推荐一个动作（8 个方向向量之一）。“议会”通过高斯分布对这些推荐进行加权聚合，生成一个概率分布，并从中采样最终执行的动作。
*   **奖励对齐**：采用**差异奖励（Difference Rewards, $\Delta$）**机制。通过将碰撞开销转化为内部时间度量，使个体机器人的奖励函数与集群的整体目标（觅食效率）保持一致，解决分布式学习中的信度分配问题。

### 6. 主要结论与发现
*   **性能持平**：模块化 RL 在性能上与存储需求更高的全状态 R-Learning 基本持平，甚至在某些高密度场景下表现更优。
*   **内存优势**：模块化方法将状态数从 256 个（全状态）减少到 16 个（2状态×8模块），极大地节省了内存。
*   **鲁棒性强**：模块化表征对奖励函数的选择不敏感，即使在非对齐的奖励下也能保持稳定，而全状态学习算法则会崩溃。
*   **动作空间**：学习直接控制运动向量（Vectorial Actions）比学习调用预设的避障宏动作（Algorithmic Actions）效果更好。

### 7. 优点
*   **极低资源占用**：非常适合部署在真实的廉价微型机器人上。
*   **可扩展性**：状态空间随传感器数量线性增长，解决了特征增加时的组合爆炸问题。
*   **去中心化**：无需机器人间通信或全局信号，完全依赖局部感知。

### 8. 不足与局限
*   **任务单一**：实验主要集中在觅食任务，尚未在更复杂的协作任务（如集体搬运、编队控制）中验证。
*   **信息损失**：由于将状态强行分解，模块之间可能丢失了某些几何关联信息（虽然实验证明在避障中这种损失影响较小）。
*   **环境依赖**：在 Arena 3（双基地）中，模块化方法的表现略逊于固定算法，说明在多目标决策时，简单的动作融合机制可能存在冲突。
