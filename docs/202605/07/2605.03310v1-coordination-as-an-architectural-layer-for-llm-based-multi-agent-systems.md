## 摘要
基于大语言模型（LLM）的多智能体系统在生产环境中的失败率在 41% 到 87% 之间，这主要归因于协同缺陷而非基础模型的能力。现有的应对方案要么是对失败模式进行经验性分类，要么是提供声明式编排框架作为工程工具；两者都未能提供从协同配置到可预测失败模式特征的原则性映射。我们认为，协同应被视为一个可配置的架构层，可与智能体逻辑和信息访问分离，从而实现架构推理，而不仅仅是提高工程生产力。我们通过预测市场上的信息控制设计来实例化这一观点：在五种参考协同配置中，使用单一 LLM、固定工具、固定的单次调用输出上限和固定的提示词模板，并将每个问题的总计算量视为内生的架构输出。Brier 分数的 Murphy 分解将校准度与判别力分开，因此即使在总分一致的情况下，不同配置也会留下可区分的特征。在模型训练截止日期（claude-opus-4-6）之后结算的 100 个 Polymarket 二元市场中，我们报告了 Murphy 特征、成本-质量帕累托前沿、类别条件分析以及自助法能力预测。五个预设预测中有三个在方向上得到了验证；在此方案中，有两种配置在帕累托前沿占据主导地位；探索性自助法区间将共识对齐与其他配置区分开来，尽管在 n=100 时成对检验未能通过 Bonferroni 校正。我们还在启用网络搜索的条件下，将相同的配置作为实时智能体部署在 Foresight Arena 上，作为一个并行积累的链上复制通道。测试框架、追踪数据集和生产智能体均已发布。我们将其定位为验证方法论的首次实例化，而非通用的跨模型主张。

## Abstract
Multi-agent LLM systems fail in production at rates between 41% and 87%, mostly due to coordination defects rather than base-model capability. Existing responses split between cataloguing failure modes empirically and shipping declarative orchestration frameworks as engineering tools; neither delivers a principled mapping from coordination configuration to predictable failure-mode signature. We argue that coordination should be treated as a configurable architectural layer, separable from agent logic and from information access, enabling architectural reasoning rather than only engineering productivity.   We instantiate this with an information-controlled design on prediction markets: a single LLM, fixed tools, fixed per-call output cap, and fixed prompt template across five reference coordination configurations, with total compute per question treated as an endogenous architectural output. The Murphy decomposition of the Brier score separates calibration from discriminative power, so configurations leave distinguishable signatures even when aggregate scores coincide.   On 100 Polymarket binary markets resolved after the model's training cutoff (claude-opus-4-6) we report Murphy signatures, a cost-quality Pareto frontier, category-conditioned analysis, and a bootstrap power-projection. Three of five pre-specified predictions are upheld in direction; two configurations dominate the Pareto frontier within this regime; exploratory bootstrap intervals separate consensus alignment from others, though pairwise tests do not survive Bonferroni correction at n=100. We also deploy the same configurations as live agents on Foresight Arena under web-search-enabled conditions, as an on-chain replication channel accumulating in parallel. Harness, trace dataset, and production agents are released. We position this as a methodology-validating first instantiation, not a general cross-model claim.

*   **研究动机**：目前的工程实践（如声明式编排框架）侧重于开发便捷性，而学术研究侧重于失败模式的分类，两者之间缺乏一个**原则性的映射**，即：什么样的协调架构会导致什么样的失败特征？
*   **整体含义**：论文主张将“协调”视为一个独立的、可配置的**架构层**，通过将其与智能体逻辑和信息访问分离，实现对系统行为的预测性推理。

### 2. 论文提出的方法论
*   **三层架构模型**：
    1.  **信息层**：固定的工具、上下文和数据源。
    2.  **协调层 (C)**：定义智能体拓扑、权限分配、同步机制、聚合规则等。
    3.  **智能体层**：具体的 LLM 调用和角色提示词。
*   **协调层七要素规范**：包括智能体端点、通信拓扑（有向图）、权限分布（谁决定什么）、同步机制（事件驱动或轮次制）、聚合规则（如均值、中位数、层级选择）、终止条件和错误处理策略。
*   **评估指标（Murphy 分解）**：论文引入气象预测中的 Murphy 分解，将衡量预测准确性的 **Brier 分数** 拆解为：
    *   **可靠性 (REL)**：校准误差，即预测概率与实际发生频率的差距。
    *   **判别力 (RES)**：区分不同结果的能力。
    *   **不确定性 (UNC)**：环境固有的随机性。
    *   这种分解允许研究者识别出即使总分相同、但行为特征（如过度自信或过于保守）迥异的架构。

### 3. 实验设计
*   **场景与数据集**：使用 **Polymarket**（去中心化预测市场）的二元预测任务。选择了 100 个在模型训练截止日期（2025年8月）之后结算的市场，确保模型无法通过记忆获知结果。
*   **Benchmark**：以**市场共识价格**（Market Consensus）作为基准。
*   **对比的五种协调架构**：
    1.  **独立集成 (IE)**：N个智能体独立响应，结果取平均。
    2.  **同伴评论辩论 (PC)**：智能体互相观察并修正，迭代固定轮次。
    3.  **编排者-专家 (OS)**：由一个规划者分解任务并分发给专家。
    4.  **顺序流水线 (SP)**：研究、分析、预测按阶段线性执行。
    5.  **共识对齐 (CA)**：智能体迭代直至达成一致。
*   **信息控制原则**：严格固定 LLM（Claude 3 Opus）、工具集、提示词模板和单次调用 Token 上限，仅改变协调架构。

### 4. 资源与算力
*   **算力消耗**：实验基于 **Anthropic API (claude-opus-4-6)** 进行，而非本地训练。
*   **实验规模**：总计处理了 100 个市场，每个市场运行 5 种配置。
*   **成本与时长**：总 API 成本为 **$109.65**，总运行耗时 **3 小时 9 分钟**。
*   **数据量**：生成并开源了约 **17.1M tokens** 的多智能体推理追踪数据集。

### 5. 实验数量与充分性
*   **实验组数**：共 500 次完整的系统运行（100 市场 × 5 架构）。
*   **充分性**：
    *   **样本量**：n=100 对于初步验证架构分离是足够的，但论文通过功效分析（Power Analysis）指出，要达到 Bonferroni 校正后的统计显著性，某些细微差异的对比可能需要 n=350 甚至更多样本。
    *   **客观性**：采用了“信息控制”设计，解决了以往研究中“架构增益可能来自更多信息获取”的混淆变量问题，实验设计非常严谨。
    *   **公平性**：所有架构共享相同的底层模型和工具，计算量（Token 消耗）作为架构的内生结果被记录而非人为抹平。

### 6. 论文的主要结论与发现
*   **帕累托前沿**：**独立集成 (IE)** 和 **顺序流水线 (SP)** 构成了成本-质量的帕累托前沿。IE 成本最低且表现稳健，SP 准确率最高但成本也最高（约为 IE 的 3.6 倍）。
*   **架构特征验证**：
    *   **共识对齐 (CA)** 表现最差，容易陷入“中点锚定”陷阱，判别力最低。
    *   **同伴评论 (PC)** 确实能改善校准度，但会因“对齐压力”抑制少数派正确意见，导致判别力下降。
    *   **编排者-专家 (OS)** 容易产生单点故障，编排者的错误会向下游级联。
*   **负 Alpha 现象**：在禁用 Web 搜索的情况下，所有配置均未能超越市场共识基准，证明了在缺乏实时信息时，复杂的协调架构无法凭空创造预测优势。

### 7. 优点（亮点）
*   **方法论创新**：首次将协调提升为独立的“架构层”，并提供了形式化的定义规范。
*   **严谨的实验控制**：通过固定信息访问，消除了 MAS 研究中长期存在的“信息-架构”混淆。
*   **高质量资源贡献**：开源了详尽的推理追踪数据集（17.1M tokens），为后续研究多智能体语义漂移提供了宝贵素材。
*   **跨学科工具引入**：引入 Murphy 分解，为 LLM 评估提供了比单一准确率更细致的诊断工具。

### 8. 不足与局限
*   **样本规模限制**：n=100 的样本量虽然能看到趋势，但在成对显著性检验上略显不足。
*   **模型单一性**：实验仅基于 Claude 3 Opus，尚未验证这些架构特征在其他模型（如 GPT-4o 或 Llama 3）上的迁移性。
*   **信息环境受限**：为了控制变量禁用了 Web 搜索，这虽然在方法论上是正确的，但限制了系统在现实预测任务中的绝对性能表现。
*   **提示词敏感性**：尽管固定了模板，但不同架构内部的角色指令可能仍存在微小的提示词敏感性偏差。
